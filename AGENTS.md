# 数据产品 AI 协作工作区

## 项目说明

这是一个长期AI协作工作区，用于数据产品相关的分析、开发、研究与知识管理。

## Agent 化目标

本工作区的目标是成为一个「数据产品工作 Agent」，围绕四类能力运转：

1. **记录工作**：自动/半自动记录每日工作、周总结、沟通记录、绩效过程素材
2. **帮助工作**：复用知识库、模板、Skill、脚本推进数据分析、产品方案、项目执行
3. **辅助规划**：持续维护 P0/P1/P2/明确不做清单，辅助周计划、季度 OKR、资源取舍
4. **辅助决策**：基于历史决策、当前重点、知识资产和数据证据给出建议，并沉淀决策记录

## Agent 启动入口

每次开始任务时，默认由 **Coordinator（协调者）** 作为入口，按以下顺序读取上下文：

1. `AGENTS.md`：工作区规则、目录结构、归档要求、业务踩坑硬规则
2. `USER.md`：用户角色、偏好、工具链、决策风格
3. `CURRENT.md`：当前重点、本周建议、明确不做
4. `WORK-PLAN.md`：P0/P1/P2、阻塞、近期完成项
5. `MEMORY.md`：长期记忆（五段结构：当前重点/关键决策/我的偏好/近期记忆/业务术语）
6. `DECISIONS.md`：重要决策与回看事项
7. `WORKFLOWS.md`：记录、规划、决策、归档的标准流程
8. `.agents/README.md`：多Agent协作体系与角色分工
9. `00-知识库/README.md`：知识资产导航；按任务再读对应知识域 README

Coordinator 理解需求后拆解任务，派单给对应子Agent执行：
- 数据查询/分析 → DataAnalyst（读取 `.agents/data-analyst/AGENT.md` 角色定义）
- 文档/归档/飞书同步 → DocWriter（读取 `.agents/doc-writer/AGENT.md` 角色定义）

入口文件职责：

| 文件 | 作用 | 更新频率 |
|------|------|----------|
| `USER.md` | 个人上下文：角色、工具链、偏好、决策风格 | 角色变化时 |
| `CURRENT.md` | 当前状态：本周重点、P0、明确不做、下一步 | 每周或重大变化时 |
| `DECISIONS.md` | 决策记录：路线选择、依据、风险、回看时间 | 重要决策后 |
| `WORKFLOWS.md` | 工作流手册：记录、帮助、规划、决策、归档流程 | 流程变化时 |

### 知识库使用规则
1. **启动时优先读取知识库索引**：做任务前先检查`00-知识库/README.md`和对应知识域README，复用已有沉淀
2. **完成任务后及时沉淀**：可复用的方法论、脚本、模板、案例必须沉淀到对应知识域
3. **遵循五维度组织**：每个知识域下内容按`concepts/`(概念)、`methods/`(方法)、`pitfalls/`(陷阱)、`cases/`(案例)、`templates/`(模板)分类存放
4. **建立关联链接**：相关知识在README中互相引用，形成知识网络
5. **临时工作区内容定期归档**：`03-数据分析/`是临时工作区，完成后可复用内容必须移动到知识库对应分类

## AI 协作规则

### 工作原则
1. **先理解，后行动**：接收到任务后，先确认需求和输出位置
2. **文件输出确认路径**：创建任何文件前，先确认保存位置
3. **遵循现有结构**：新文件按照上述分类放到对应目录
4. **版本化管理**：同一项目的迭代，保留历史版本
5. **沉淀知识**：完成任务后，可复用的方法论/脚本/模板及时沉淀到对应目录
6. **先结论后细节**：输出优先给出总结论，再展开细节
7. **数据标注来源**：引用数据必须标注来源和时间；无数据支撑的判断标注「推断」
8. **表格优先**：竞品分析、对比类内容优先用表格，避免大段文字
9. **飞书文档规范**：涉及飞书文档操作，先确认文档是否存在；有则追加更新，无则新建；禁止非重建场景覆盖写入
10. **不猜测不确定内容**：不确定的事情主动提问，不主观臆断
11. **文档强制归档**：所有生成的文档（飞书文档链接、Markdown文件、HTML文件、PDF/PPT等）必须归档到`00-知识库/文档归档/`对应目录，并维护归档索引，禁止文档生成后散落在临时目录

### 文件命名规范
- 业务项目目录：`项目名/版本号/` 或 `项目名/YYYYMMDD-描述/`
- 报告命名：`YYYY-MM-DD-主题-报告.md`
- SQL文件：`业务主题-指标描述.sql`
- 会议纪要：`MM-DD 会议主题.txt/pdf`

### 任务执行流程
1. 先读入口文件：`USER.md`、`CURRENT.md`、`WORK-PLAN.md`、`MEMORY.md`、`DECISIONS.md`、`WORKFLOWS.md`
2. 分析用户需求，确定属于记录工作、帮助工作、辅助规划、辅助决策中的哪一类
3. 确认输出路径（默认按分类存放，正式文档必须归档）
4. 执行任务，生成内容或修改文件
5. 内容验证，确保完整性与可追溯
6. 根据结果判断是否需要回写 `CURRENT.md`、`WORK-PLAN.md`、`MEMORY.md`、`DECISIONS.md`
7. 提示用户结果位置，询问是否需要进一步调整

### 决策闭环规则
1. 重要取舍、路线选择、优先级调整必须写入 `DECISIONS.md`
2. 会影响长期偏好或项目方向的决策同步写入 `MEMORY.md`
3. 会影响执行节奏的决策同步更新 `CURRENT.md` 或 `WORK-PLAN.md`
4. 没有数据支撑的判断必须标注「推断」
5. 决策建议必须包含：结论、依据、风险、下一步
