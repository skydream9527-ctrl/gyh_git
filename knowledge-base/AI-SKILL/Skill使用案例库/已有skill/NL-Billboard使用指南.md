# NL-Billboard 使用指南

## 1、简介

### 什么是 NL-Billboard？

NL-Billboard 是一个**数据看板生成器**技能，帮助用户通过自然语言描述快速创建专业、美观的数据可视化看板，无需手写 HTML 和配置图表。

### 核心能力

| 能力 | 说明 |
|-|-|
| 🎨 自然语言输入 | 用日常语言描述需求，自动生成看板 |
| 📊 多业务线支持 | 浏览器主端、浏览器信息流、内容中心、搜索、小说 |
| 📈 多图表类型 | 折线图、柱状图、饼图、热力图、漏斗图等 |
| 🔗 数据源集成 | 飞书文档/表格、数据工厂、SQL 数据库 |
| ✅ 自动校验 | 数据完整性、合理性、格式自动校验 |

## 2、安装前置依赖

### 2.1 安装 Mi Code CLI

MacOS / Linux / WSL / Matrix 实例，在终端运行：

```Bash
bash -c "$(curl -fsSL https://cnbj1-fds.api.xiaomi.net/mi-code-public/install.sh)"
```

### 2.2 安装依赖 Skill

```Bash
micode skills add user_gongyunhe/nl-billboard -i  # 数据看板生成器
micode skills add user_gongyunhe/nl-sql -i        # 自然语言生成SQL（可选）
micode skills add ai-team/feishu -i               # 写飞书文档（可选）
```

### 2.3 获取数据工坊 Token（如需查询数据）

1. 登录数据工厂：https://data.mioffice.cn/workspace/?wid=11329#/workspace/11329/config
2. 生成新 Token，复制保存
3. 配置环境变量：

```Bash
export DATAWORKS_TOKEN_ID="your_token_here"
```

### 2.4 申请表权限

常用数据表（需提前申请权限）：

- `iceberg_zjyprc_hadoop.browser.dwm_browser_event_aggregation_label_di`
- `iceberg_zjyprc_hadoop.dw.dwd_ot_event_di_31000000442`

## 3、使用方法

### 3.1 启动 Mi Code

```Bash
mkdir -p ~/micode && cd ~/micode
micode
```

### 3.2 执行 NL-Billboard Skill

在 Mi Code 对话框中输入：

```Plain Text
执行 nl-billboard
```

或直接描述需求：

```Plain Text
帮我创建一个浏览器主端DAU趋势看板
```

### 3.3 交互流程

**Step 1：确认数据来源**

```Plain Text
请问您是否已有数据？
1. 是，我有数据（飞书文档/表格链接）
2. 否，我需要查询数据
```

**Step 2A：有数据（飞书文档/表格）**

1. 提供飞书文档/表格链接
2. 选择图表类型（趋势图、对比图、占比图、数据表格）
3. 选择筛选维度
4. 生成看板

**Step 2B：无数据（需要查询）**

1. 调用 nl-sql skill 查询数据
2. 选择业务线（浏览器主端、浏览器信息流、内容中心、搜索、小说）
3. 选择数据类型（核心指标、埋点数据）
4. 描述查询需求
5. 系统自动校验数据准确性
6. 生成看板

**Step 3：生成看板**

系统自动生成：

- HTML 看板文件（包含筛选器、指标卡片、图表、数据表格）
- 飞书在线文档（汇总看板信息）

**Step 4：用户确认**

```Plain Text
看板已生成完成！

📊 看板信息：
- 看板名称：xxx
- 看板链接：xxx
- 包含图表：趋势图、对比图、占比图
- 包含指标：DAU、PV、UV
- 数据条数：xxx 条

📄 飞书文档已创建：
- 飞书文档链接：xxx

您是否满意？
```

## 4、使用示例

### 示例 1：从数据工厂创建看板

**用户输入**：

```Plain Text
执行 nl-billboard
→ 选择：2. 否，我需要查询数据
→ 选择：1. 浏览器主端
→ 选择：1. 核心指标
→ 输入：近7天的浏览器主启DAU，按天分组
```

**生成看板**：

- 指标卡片：DAU 总数、环比变化
- 趋势图：DAU 随时间变化趋势
- 数据表格：按日期展示详细数据
- 筛选器：日期范围选择

### 示例 2：从飞书表格创建看板

**用户输入**：

```Plain Text
执行 nl-billboard
→ 选择：1. 是，我有数据
→ 提供链接：https://mi.feishu.cn/sheets/xxx
→ 选择图表：趋势图、数据表格
→ 选择维度：日期、产品类型
```

**生成看板**：

- 多维度筛选器
- 趋势图展示
- 完整数据表格

### 示例 3：创建运营监控看板

**用户输入**：

```Plain Text
帮我创建一个浏览器信息流消费数据看板，包含消费UV、消费VV、人均消费时长
```

**生成看板**：

- 核心指标卡片（消费UV、消费VV、人均消费时长）
- 趋势图（指标随时间变化）
- 对比图（不同维度对比）
- 数据表格（详细数据）

## 5、支持的图表类型

### 5.1 趋势类图表

| 图表类型 | 说明 | 适用场景 |
|-|-|-|
| 折线图 | 展示数据随时间的变化趋势 | DAU趋势、收入趋势 |
| 面积图 | 强调数量随时间的变化 | 用户增长、流量变化 |
| 柱状图 | 比较不同类别的数值 | 各产品线对比、各渠道对比 |

  
\### 5.2 占比类图表  

| 图表类型 | 说明 | 适用场景 |
|-|-|-|
| 饼图 | 展示各部分占整体的比例 | 用户来源分布、产品占比 |
| 环形图 | 更现代的占比展示方式 | 市场份额、用户类型分布 |
| 堆叠柱状图 | 展示分类占比和总量 | 各渠道贡献、各产品线占比 |

  
\### 5.3 关系类图表  

| 图表类型 | 说明 | 适用场景 |
|-|-|-|
| 散点图 | 展示两个变量的关系 | 用户行为分析、相关性分析 |
| 气泡图 | 三维数据关系展示 | 多维度对比 |
| 热力图 | 矩阵数据可视化 | 用户活跃度分析、时段分布 |

### 5.4 流程类图表

| 图表类型 | 说明 | 适用场景 |
|-|-|-|
| 漏斗图 | 展示转化流程 | 用户转化漏斗、销售漏斗 |
| 桑基图 | 展示流向和转化 | 用户路径分析 |

### 5.5 指标类组件

| 组件类型 | 说明 | 适用场景 |
|-|-|-|
| 数字卡片 | 展示关键指标 | DAU、GMV、转化率 |
| 仪表盘 | 展示指标完成度 | 目标达成率 |
| 子弹图 | 对比目标值和实际值 | KPI达成情况 |

## 6、看板功能特性

### 6.1 筛选器功能

**日期筛选器**：

- 日期范围选择（开始日期 - 结束日期）
- 分天筛选（支持按天筛选数据）
- 快捷选项（今天、昨天、最近7天、最近30天、本月、上月等）

**维度筛选器**：

- 根据数据中的维度字段自动生成
- 支持多选或单选
- 筛选器变化时同时更新图表和数据表格

### 6.2 数据表格功能

- 显示完整数据
- 支持排序（点击表头可按列排序）
- 支持分页（每页显示 20 条数据）
- 支持导出（CSV 或 Excel 文件）
- 筛选联动（筛选器变化时，表格数据同步更新）

### 6.3 性能优化

- 使用 CDN 加载第三方库（ECharts、XLSX）
- 分页加载表格数据（每页 20 条）
- 懒加载图表（进入视口时才渲染）
- 数据缓存（避免重复查询）

## 7、常见问题

### Q1：提示"数据查询失败"怎么办？

**原因**：数据工厂连接失败或权限不足。

**解决方案**：

1. 检查数据工坊 Token 是否配置正确
2. 确认已申请对应表的访问权限
3. 检查网络连接是否正常

### Q2：如何查看支持的指标列表？

索引文件位置：

- 浏览器主端：`reference/browser-main/metric-name-index.md`
- 浏览器信息流：`reference/browser-feed/metric-name-index.md`
- 内容中心：`reference/content-center/metric-name-index.md`

### Q3：看板生成后如何修改？

**解决方案**：

1. 告知 AI 具体修改需求（调整指标、调整样式等）
2. AI 会重新生成看板并更新

### Q4：如何导出看板数据？

**解决方案**：

1. 在看板的数据表格区域点击"导出"按钮
2. 选择导出格式（CSV 或 Excel）
3. 下载文件
