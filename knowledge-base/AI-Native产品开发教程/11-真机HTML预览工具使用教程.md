## 使用条件

要正常使用，这个 APK 只有一个核心前提：

**你填进去的网址，必须是手机真的能访问到的网址。**

常见情况分两种：

### 情况 A：内网 / 局域网使用

如果你填的是内网地址，那么使用者必须满足对应网络条件。

例如：

- 如果是公司内网地址，手机需要连公司内网或 VPN。
- 如果是你电脑在同一 Wi‑Fi 下开的局域网地址，手机也必须连同一个局域网。

这类地址通常类似：

```Plain Text
http://192.168.x.x:端口
```

```Plain Text
http://10.x.x.x:端口
```

### 情况 B：对外分享使用

如果你是发给别人安装，最好使用**公网 HTTPS 地址**。

> AI老师项目这里将要使用gitlab pages网址，暂时还没支持

这类地址通常类似：

```Plain Text
https://你的正式网址
```

这样别人不需要和你在同一个网络下，也不需要额外配置。

## 怎么生成网址

推荐做法很简单，在 IDE 里打开你的 HTML 项目，然后跟他说：

```Markdown
启动本地 HTTPS 服务，并给我Vx.x的html可直接在浏览器打开的本地服务https://地址
```

## 你最少要记住什么

1. APK 只是网页壳，不会自动把网页内容打包进去
2. 真正决定能不能用的，是你填进去的网址能不能被手机访问
3. 内网地址只能给在同样网络条件下的人使用
4. `localhost` 和 `127.0.0.1` 是本地地址，不能直接发给别人体验
5. 最稳妥的方案是：通过 IDE 跑出页面后，拿到一个真正可访问的网址，再填进 APK
