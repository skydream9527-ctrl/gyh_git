⑥ 手机端边界与确认

✏ 会改的（跟桌面共用）

  • frontend/src/styles/tokens.css · global.css · index.html 字体
  • 手机端样式自动随 token 一起变 —— global.css 里的 @media (max-width:800px) 规则不动，仅其内部用到的颜色变量被替换

🚫 不会动的

  • 手机端的布局规则（topnav 折叠、user-pill 隐藏 uname、cm-card 全宽等 media query）
  • 底部 Tab Bar / FAB 这类新交互（v3 没有就不引入，避免越界）
  • 手机端的页面跳转 / 路由 / 抽屉逻辑

💬 等你拍板的（手机端补充）

  • ① 移动端整体方向 OK 吗？跟桌面提案保持一致即可
  • ② 用户气泡：v3 是蜜橘渐变气泡，v4 提案是纯色克制蓝。要不要保留 v3 的渐变手感（只换色相蜜橘 → 蓝渐变）？
  • ③ Composer 输入框：v4 改成胶囊形（圆角 99px）。手机端 textarea 多行输入时仍维持胶囊有点怪，是否多行后自动转 14px 圆角？
  • ④ 移动端要不要保留你之前看过的「顶部三段切换」（数据源 / 对话 / Studio），还是说工作区维持 v3 现有的单页结构？注意：这是 IA 改动，不在"换皮"范围内。要做需要单独立项。
