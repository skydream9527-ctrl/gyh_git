# ICE Workbench v6 - 全场景功能融合设计思路

既然 1、2、3、4 个方向都需要，这意味着 v6 将面临一次**系统架构和用户交互体验的双重跨越**。它不再只是一个 "聊天的对话框"，而是一个真正的 "数据工作台 (Workspace)"。我们需要用统一的页面设计将这四个能力无缝融合。

## 架构与状态设计的融合点

要让这四个功能跑通，我们需要一个**统一的任务流车载体 (Task Runner)**。

1. **(方向 1 & 4 的融合) 异步 Agent 执行流**：
   任务 (Task) 的运行状态必须和用户的 WebSocket 连接解绑。用户在 Chat 发送指令后，后端生成一个包含完整「执行计划 (Plan)」的运行实例（Run）。即使用户关闭页面，Agent 依然根据 Plan 派发给对应的子 Agent 执行。
   
2. **(方向 2 的融合) 中断与恢复机制**：
   当执行流遇到配置为“需人工确认”的节点（方向 2），或者调用了飞书写操作（方向 3）前，执行流状态变为 `Paused`（挂起）。系统产生一条 Notification（方向 4），通知用户回来审批。

## v6 核心页面设计思路

为了承载上述能力，我们需要对现有的核心工作区（Workspace / 任务详情页）进行重新设计。

### 1. 任务工作台 (Workspace) 的 "三栏 / 多区" 布局设想

当前 v5 是左右两栏（左侧大对话，右侧属性/设置）。v6 建议采用**渐进式展开的多区面板布局**：

*   **左区：数据与知识挂载 (Data & Context) —— 对应方向 3**
    *   **职能**：管理本任务的上下文。
    *   **展示**：绑定的飞书文档列表（作为 RAG 知识）、挂载的飞书多维表格（Bitable）、本地上传的 CSV/Excel。
    *   **交互**：支持直接预览这些数据源。Agent 会随时读取这里的数据。
*   **中区：流式对话与协作 (Conversation) —— 对应方向 2 & 4**
    *   **职能**：人机指令交互和消息时间线。
    *   **展示**：不仅有常规气泡，更要支持 **Rich Cards（富交互卡片）**。
        *   当任务挂起时，中间会弹出一张 **"等待审批 (Pending Approval)"** 的卡片，展示 Agent 接下来要执行的 SQL 或要修改的数据对比，用户可点击 `Approve` 或 `Reject & Edit`。
        *   当任务在后台离线跑完后，时间线插入一条带时间戳的系统消息。
*   **右区 / 侧边抽屉：执行计划与子代理 (Execution Plan) —— 对应方向 1**
    *   **职能**：让“黑盒”变成“白盒”。
    *   **展示**：树状的 Todo List 或执行图。当主 Agent 下发指令时，这里会动态生成节点（如 `[✓] 分析需求` -> `[ ] 数据查询 (Data Agent 处理中...)` -> `[ ] 生成图表`）。
    *   **交互**：用户可以直接看到当前卡在哪一步，哪个 Sub-agent 正在工作。

### 2. 状态提示与全局通知系统 (Global Notification) —— 对应方向 4

*   **TopNav 通知铃铛**：需要强化。后台长耗时任务跑完，或有挂起等待人工干预的任务时，这里要有醒目的红点。
*   **离线任务仪表盘 (Dashboard 增强)**：仪表盘不仅要显示“我有哪些任务”，更要有一块专门的区域展示“**正在后台运行的任务 (Running)**”和“**等待我处理的任务 (Action Required)**”。
