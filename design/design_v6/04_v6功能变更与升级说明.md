# ICE Workbench v6 功能变更与升级说明

本文档详细记录了 ICE Workbench 从 v5 演进至 v6 过程中的核心功能架构调整、交互理念升级以及前端 UI 页面的全局重构。

## 1. 升级背景与核心理念

随着内部数据业务复杂度的提升，单纯的“一问一答”式聊天交互（v5 模式）已无法满足需要长链路思考、后台跑批、多系统串联的数据工作流需求。

因此，v6 版本的核心理念从 **「AI 问答窗口」** 正式转向 **「基于多智能体协同的数据驾驶舱 (Data Workspace)」**。

*   **打破黑盒**：将 Agent 思考和调度的过程显性化，用户可以看到每一步是由哪个子节点（Sub-agent）完成的。
*   **人机协同 (HITL)**：承认 AI 不是万能的，在关键节点允许系统主动挂起，请求人类介入、审批或修改中间态数据。
*   **解绑实时连接**：弱化 WebSocket 强绑定，支持纯异步执行（关掉网页依然能跑批，跑完推送通知）。

## 2. 全局架构与底层变更

为了支撑上述理念，在页面和交互流上发生了如下变更：

*   **三栏式异步工作台**：原先简单的左右两栏被扩展。新增了“左侧 Context 持久化挂载区”和“右侧执行流拓扑区”，中间区域除了聊天气泡，大量引入了 **Rich Cards（富交互审批表单）**。
*   **定时与任务的无缝转化**：任何在工作台内手动调试跑通的任务（Task），都可以通过标题栏下拉菜单，直接挂载 Cron 表达式，一键转化为 Scheduled Task。
*   **全局通知体系**：无论用户身处哪个页面（甚至在 Admin 或首页 Dashboard），后台一旦有任务报错或需要“人工审批”，顶部导航栏都会出现提醒，一键直达对应挂起的工作台节点。

## 3. 前端页面升级明细

我们在 `design_v6` 目录下完成了全量核心页面的视觉与交互原型重构（全面汉化版）：

### 3.1 核心工作流页面
*   **`workspace_v6.html` (工作台)**：
    *   这是改版的最核心成果。左侧清晰列出“数据源”与“RAG知识库”。
    *   中间演示了 **“人工介入卡片”**：Agent 发现异常数据时停止，抛出包含 `input` 输入框的表格，等待用户核对修改后，点击“确认修正并继续”。
    *   右侧显示了垂直的树状 **“执行计划 (Execution Plan)”**，可清晰看到主调度器分配给“数据分析节点”和“报告生成节点”的流转状态。
    *   引入了基于标题的下拉操作菜单（重命名、配置定时等）。

### 3.2 导航与大盘
*   **`dashboard_v6.html` (仪表盘)**：
    *   剥离出高优的 **“待人工处理 (Action Required)”** 和 **“后台运行中 (Running Async)”** 专区。
    *   正在运行的任务会有带百分比的进度条显示。
*   **`scheduled_v6.html` (定时调度)**：
    *   统一了表格化管理视图，快速开关定时大盘。

### 3.3 后台管理与能力配置
*   **`agent_detail_v6.html` (智能体编排)**：
    *   面向 AI 开发者的后台。提供直观的上下游 **拓扑关联图**，显示 Planner Agent 可以调用哪些具体的执行节点（如 DA 或 Research Agent）。
    *   自带右侧测试沙盒。
*   **`admin_v6.html` (系统管理后台)**：
    *   极简左侧抽屉导航，右侧包含核心 Tokens 消耗和用户注册看板。

### 3.4 登录与门户
*   **`introduce_v6.html` (产品落地页)**：
    *   强化“多智能体”、“HITL”和“离线飞书整合”的全新产品主张，大字号排版引导用户。
*   **`login_v6.html` (登录页)**：
    *   聚焦飞书一键登录与账密登录的极简卡片。

## 4. 交互设计语言 (Design Language)

v6 在视觉上延续并在 v5 基础上进一步克制化：

*   **色彩体系**：保留了深受好评的 **Claude 暖纸白**（`--bg: #faf4ed`）和 **铜橙色**（`--primary: #cc785c`）高亮点缀。
*   **字体**：彻底抛弃了衬线体，全面拥抱 **Inter** 和 **JetBrains Mono**，以最大化提升密集数据表格和代码块的可读性。
*   **形态与质感**：摒弃大面积渐变，采用更平整的色块和极柔和的 1px/2px 阴影（`shadow-card`, `shadow-btn`）。大圆角（12px+），带来现代 SaaS 工具的冷静感。

## 5. 后续演进计划

1.  **富交互协议定义**：前后端需协商一套新的 JSON 协议，以便后端动态向中间聊天流下发包含“表格+输入框+按钮”的定制化审批卡片。
2.  **后端 Runtime 改造**：目前后端仍受限于 5 轮超时机制，需全面转向后台进程驻留、中间态持久化存储的新 Runtime 架构。
