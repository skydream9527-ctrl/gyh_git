# ICE 推广与消费

> 目的：把 ICE Data Workbench 从「建好了」推进到「别人不找我就能自助用」，并留下可被扫描/面谈引用的消费证据。
> 对应 PM AI Native L0-L4 模型中 **D型·效能平台的 L4 达标标准**：消费方（其他 PM）能不能不找你就能用你的工具。
> 建立：2026-07-13

## 为什么建这个目录

| 判断 | 说明 |
|------|------|
| ICE 的 dim2 已达标 | 多用户 + 飞书 SSO + 自托管 + 24 路由 + 10 Agent + 17 Skill，AI Native 产出已经很扎实 |
| 但缺「消费方自助」证据 | D 型 L4 看的不是"你做了工具"，而是"别人不找你就能用"。这需要：接入指南 + 真实消费记录 + 闭环案例 |
| 不与主线冲突（护栏 G1）| ICE + nl-sql Skill 正是数据夜校三阶段里「8-9 月 AI 工具赋能全员」的载体，服务 Q3 主线 |

## 自助消费闭环

```
自助接入指南（让人能用）
   → 消费登记（记录谁在自助用、自助率多少）
   → badcase 回流 nl-sql 范式库（工具越用越准）
   → 反哺 ICE / Skill 迭代
```

## L4 达标判断（D 型）

- [ ] 有 ≥3 个消费方不找龚云荷就完成了取数（自助率可量化）
- [ ] 消费记录可追溯（登记表持续更新）
- [ ] 至少 1 条完整消费闭环（消费 → 反馈 → 工具迭代），与 `00-知识库/data-analysis/案例库/` 的 nl-sql 闭环案例互链
- 目标窗口：随数据夜校 8-9 月 AI 工具赋能全员推进（推断）

> 说明：ICE 产品代码在 `ICE-Data-Workbench-v3/`（独立 git 仓库）；本目录只放推广运营与消费证据，归属 data-product 工作区，便于被工作区扫描看到。
