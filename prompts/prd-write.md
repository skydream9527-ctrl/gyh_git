# PRD 撰写 Prompt 模板

## 使用方式

复制以下 Prompt，替换 `{ }` 中的内容，发给 AI 即可。

---

## Prompt

> 执行前先读 USER.md 和 MEMORY.md，已有信息（业务、用户群、核心指标、当前项目）直接引用，不要重复问用户。只问缺失的信息。

我是一名产品经理，负责 {产品/功能名称}。请帮我撰写一份产品需求文档（PRD）。

### 业务背景

{一句话描述为什么要做这个功能}

### 目标用户

{目标用户画像}

### 核心功能

1. {功能点 1}
2. {功能点 2}
3. {功能点 3}

### 约束条件

- 技术栈：{技术栈}
- 时间要求：{时间要求}
- 依赖方：{依赖方}

### 输出要求

1. 按标准 PRD 结构输出：背景 → 目标 → 用户故事 → 功能详述 → 非功能需求 → 验收标准 → 风险
2. 每个用户故事必须包含验收标准（AC）
3. 必须覆盖异常流程和边界条件
4. 标注每个功能的优先级（P0 / P1 / P2）
5. 如果有技术风险或依赖风险，请明确指出
