# 数据异常诊断 Prompt 模板

## 使用方式

当指标出现突变（暴涨/暴跌）时使用。复制以下 Prompt，替换 `{ }` 中的内容，发给 AI 即可。

---

## Prompt

> 执行前先读 USER.md 和 MEMORY.md，已有信息直接引用，不要重复问用户。

某个核心指标出现了异常波动，请帮我系统性地排查原因。

### 异常描述

- 指标名称：{例如：浏览器 DAU}
- 异常表现：{例如：昨日环比下降 15%}
- 发现时间：{例如：2026-07-08 早报}
- 影响范围：{例如：全量用户 / 仅 Android / 仅某渠道}

### 已知背景

{已排除的原因、近期的版本发布、活动变更、技术变更等}

### 排查要求

1. 按以下维度逐层下钻排查：
   - 时间维度：是突变还是渐变？拐点在哪？
   - 用户维度：新/老用户、渠道、地域、设备
   - 功能维度：哪个模块/页面贡献了变化？
   - 技术维度：是否有埋点异常、数据延迟、口径变更？
2. 每个维度给出数据事实（标注来源）和归因判断（标注「推断」）
3. 区分直接原因和根本原因
4. 给出后续行动建议（需要拉谁、做什么实验、看什么数据）
5. 输出用表格汇总排查结果
