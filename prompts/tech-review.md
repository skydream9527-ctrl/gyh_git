# 技术方案评审 Prompt 模板

## 使用方式

评审研发同学提交的技术方案时使用。复制以下 Prompt，替换 `{ }` 中的内容。

---

## Prompt

> 执行前先读 USER.md 和 MEMORY.md，了解我的产品背景和技术栈。

我是产品经理，需要评审以下技术方案。请帮我从产品视角提出问题和建议。

### 方案信息

- 方案名称：{例如：nl-sql 多轮对话支持}
- 方案文档：{粘贴方案内容或提供链接}
- 对应需求：{PRD 链接或需求描述}

### 评审维度

请从以下维度逐一评审：

1. **需求覆盖**：方案是否覆盖了 PRD 中的所有需求点？有无遗漏？
2. **异常处理**：边界条件和异常流程是否考虑充分？
3. **用户体验影响**：对前端交互、响应速度、可用性的影响？
4. **数据影响**：对数据埋点、指标统计、数据迁移的影响？
5. **兼容性**：对现有功能、历史数据、其他模块的影响？
6. **可扩展性**：方案是否支持后续迭代？是否过度设计？
7. **风险**：主要技术风险和回退方案？

### 输出要求

1. 用表格列出问题清单：问题描述 / 严重程度（阻塞/建议/可选）/ 建议修改方向
2. 给出总体评审结论（通过/有条件通过/不通过）
3. 标注哪些问题必须在开发前解决，哪些可以并行
