📐 这张稿的布局重设计

  • 原版：居中卡片（左 360 brand + 右 420 form），卡片外有 grid 网格 + 双 orb 模糊球背景。
  • v5：全屏 split（左 56% 品牌叙事 / 右 44% 表单），砍掉 orb 模糊球（与暖米白纸张感冲突），grid 也去掉。左侧暖米白底，右侧白色 surface 形成层级。
  • 左侧叙事区：顶 brand → 中部 hero（衬线大标题 "从问题到结论，只走一段对话" + 副本）→ 下部三步循环演示（USER → TOOL → AGENT，立体卡 + 范式色 ico）→ 底部链接 footer
  • 右侧表单区：表单宽度 max 380，顶部 h2 "登录"（衬线）→ pill 切换（登录 / 注册，激活态铜橙渐变，width:fit-content 只占内容宽度，不撑满父）→ 输入框 44px 高 / 8px 圆角 → CTA 按钮 44px / 铜橙渐变带"→"
  • 米盾入口隐藏：当前先隐藏米盾 tab（设置层关），登录页只剩账号 / 邮箱。tabs 容器整个移除，副标题改成 "使用账号 / 邮箱登录"
  • 密码规则清单：保留功能（6 条），但用 surface-2 卡 + Mono 字 + ✓/✗ 着色（success/error token）
  • 移动端：左侧叙事整个隐藏，单栏布局：brand → 标题 → mode → form → 底部链接。input height 48 / font-size 16（防 iOS zoom）。mode pill 用 inline style 撑满 100%

两个关键状态展示

  • 注册 + 密码规则：pw-rules 清单 ✓/✗ 实时反馈，与后端 _validate_password_strength 完全对齐
  • 待审批：warning 色 hint card + 申请信息 + 双按钮（回登录 / 再申请）

和 v4 / 当前实现的差异

  • 背景：grid 网格 + 双 orb 模糊球 → 纯纸张暖米白
  • 容器：居中卡片（780×?） → 全屏 split（左右 panel）
  • 左侧叙事：仅 brand + 简单 loop → 大衬线 hero + 立体三步循环卡
  • 登录方式入口：双 tab（米盾 / 账号密码） → 米盾先隐藏，整个 tabs 移除
  • 登录/注册切换：pill → pill（保留）但激活态用铜橙渐变 + width:fit-content 修复未居中
  • CTA 按钮：实色蓝 → 铜橙渐变 + 阴影 + 内 24% 白高光 + 末尾 →
