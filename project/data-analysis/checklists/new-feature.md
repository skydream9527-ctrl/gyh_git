# 新功能上线效果 Checklist

> 新功能灰度 / 全量后，按此评估效果，决定放量、迭代还是回滚。

## 核心评估

- [ ] **核心指标有没有提升？** 提升幅度多少？统计显著吗？（避免把波动当效果）
- [ ] **对其他指标有没有负向影响？** 护栏指标（留存、崩溃率、性能、投诉）有没有跌
- [ ] **影响范围**：覆盖多少用户？全量后预计影响

## 用户分层效果

- [ ] 对哪类用户效果最好？哪类最差？（新老 / 渠道 / 版本 / 行为分群）
- [ ] 用户行为有什么符合 / 不符合预期的变化？（路径、时长、深度）
- [ ] 有没有意料之外的副作用？（其他功能使用变化）

## 决策

- [ ] **ROI 怎么样？** 收益 vs 开发 / 维护成本
- [ ] **要不要全量？** 还是迭代优化？还是回滚？
- [ ] **放量节奏**：灰度比例 → 全量时间表
- [ ] **监控告警**：全量后持续监控哪些指标，异常阈值多少

## 评估方法

- **AB 测试（金标准）**：实验组 vs 对照组，统计显著性
- **前后对比（次选）**：上线前后同口径对比，需剔除季节性
- **同期群对比**：新老版本用户同期群留存对比
