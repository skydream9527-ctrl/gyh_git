# 指标下跌排查 Checklist

> 核心指标下跌时按顺序逐项排查，90% 的问题能快速定位。

## 排查步骤

- [ ] **数据本身有没有问题？** 埋点丢失、ETL 延迟、口径变更、技术故障
  > 一半以上的"异常"最后都是数据问题，必须先确认
- [ ] **是整体下跌还是局部？** 哪个维度贡献最大？（算贡献度，优先拆贡献最大的维度）
- [ ] **什么时间点开始跌的？** 那个时间点前后有什么事件？（发版、活动、故障、外部变化）
- [ ] **维度下钻定位**：
  - 拆平台（iOS / Android）
  - 拆版本（新版本有没有 bug）
  - 拆渠道（某个渠道质量变化）
  - 拆用户群（新老用户）
- [ ] **匹配可能原因**：数据问题 / 技术问题 / 产品问题 / 运营问题 / 外部因素
- [ ] **影响多大？** 影响多少用户 / 收入 / DAU
- [ ] **怎么修复？** 短期修复 + 长期预防机制
- [ ] **怎么避免下次？** 监控告警、灰度发布、回滚机制

## 贡献度公式

`某部分贡献度 = (该部分变化量) / 整体变化量`

优先看贡献度最大的部分，不要盲目拆所有维度。

## 常见异动原因

| 类别 | 典型原因 |
|------|----------|
| 数据问题 | 埋点丢失、ETL 延迟、口径变更 |
| 技术问题 | 服务故障、崩溃、性能下降、CDN 问题 |
| 产品问题 | 新版本 bug、功能改动负向、策略调整 |
| 运营问题 | 活动上线 / 结束、push、渠道投放变化 |
| 外部因素 | 节假日、周末、政策、竞对动作、社会热点 |
