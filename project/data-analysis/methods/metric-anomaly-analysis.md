# 指标异动分析

## 什么是指标异动
核心指标突然出现超出正常波动范围的涨跌，需要快速定位原因。

## 标准分析流程（5步法）
1. **确认真实性**：先排除数据问题（埋点、管道、统计口径、计算错误）
2. **确认幅度和范围**：涨跌多少？是全量还是局部？持续了多久？
3. **维度拆解下钻**：按平台、版本、渠道、地域、用户群、时间等维度拆分，找到贡献最大的异常点
4. **关联事件匹配**：异常时间点前后有没有：版本发布、活动上线/下线、运营动作、技术故障、外部因素（节假日/政策/竞对）
5. **验证因果，输出结论**：验证事件和异常的因果关系，给出影响范围和修复建议

## 常见异动原因分类
| 类别 | 典型原因 |
|------|----------|
| 数据问题 | 埋点丢失、ETL延迟、口径变更、统计错误 |
| 技术问题 | 服务故障、接口超时、崩溃、CDN问题、性能下降 |
| 产品问题 | 新版本bug、功能改动带来负向影响、策略调整 |
| 运营问题 | 活动上线/结束、push推送、渠道投放变化、运营活动到期 |
| 用户问题 | 渠道质量变化、刷量/作弊、用户自然波动 |
| 外部因素 | 节假日、季节性、周末效应、政策变化、社会热点、竞对动作 |

## 异动判断标准
基于历史数据计算正常波动区间（±3σ原则），超出区间即判定为异常：
- 小指标波动大：±20%波动可能是正常
- 核心大盘指标稳定：±3%就可能是异常
- 不要每个涨跌都分析，建立告警阈值，只分析超出置信区间的异常
