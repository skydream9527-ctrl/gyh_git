# RFM 用户价值分层分析

## 什么是RFM
RFM是经典的用户价值分层模型，通过三个核心维度衡量用户价值和活跃状态，广泛用于精准营销、用户运营、会员体系设计。

三个维度：
- **R (Recency)**：最近一次消费/活跃距今天数 → 用户最近有没有来？
- **F (Frequency)**：统计周期内的消费/活跃次数 → 用户来得勤不勤？
- **M (Monetary)**：统计周期内的累计消费金额/贡献价值 → 用户花了多少钱/创造多少价值？

> 💡 注：M是金额，适合交易类产品；如果是内容/工具类非付费产品，可以用"使用时长""核心行为次数"等价值指标替代。

## 适用场景
1. ✅ 用户价值分层，识别高价值用户/重要挽留用户等
2. ✅ 精准营销，不同层级用户用不同运营策略
3. ✅ 会员体系设计，不同等级对应不同权益
4. ✅ 用户健康度监控，各层级用户占比变化反映业务健康
5. ✅ 活动效果评估，活动前后各层级用户迁移
6. ✅ 流失预警，R值越来越大的用户高流失风险

不适用场景：
❌ 一次性消费/极低频次产品（比如买车、买房，F基本都是1）
❌ 0→1新产品用户量太少时

## 核心原理
RFM三个维度从不同角度刻画用户：
- R：反映用户留存概率，最近有行为的用户更容易再次活跃
- F：反映用户忠诚度，频次越高粘性越强
- M：反映用户直接价值贡献，是商业变现核心

三个维度组合可以把用户分成8个经典层级，每个层级特征和运营策略完全不同。

## 标准分析流程

### 第1步：确定业务口径
根据你的产品类型确定三个维度的定义：

| 产品类型 | R（最近一次） | F（频次） | M（价值） | 统计周期 |
|----------|--------------|-----------|-----------|----------|
| 电商/零售 | 最近一次下单距今天数 | 近90天下单次数 | 近90天累计消费金额 | 90天/180天 |
| 内容/社区 | 最近一次登录/浏览距今天数 | 近30天启动/发帖/互动次数 | 近30天内容贡献/广告价值 | 30天/90天 |
| SaaS/工具 | 最近一次使用距今天数 | 近30天核心功能使用次数 | 付费金额/ARPU | 30天/90天 |
| 游戏 | 最近一次登录距今天数 | 近7天登录/游戏局数 | 近30天充值金额 | 7天/30天 |

**口径确定原则**：
- 周期长度要符合用户正常使用频次，比如高频电商用90天，低频大额用180/365天
- M要选和商业价值直接相关的指标，不要用无关指标
- 必须是客观可统计的行为数据，不要用主观标签

### 第2步：数据准备与预处理
提取数据，做数据清洗：
1. 提取每个用户的R、F、M三个值
2. 异常值处理：比如M特别大的极端值（测试账号/刷单/企业用户）单独处理
3. 去重：确保一个用户一条记录
4. 处理缺失值：没有行为的用户标记为流失

### 第3步：维度打分（分箱）
三个维度分别打分，常用1-5分制（也可用3分制简化），打分有两种方式：

#### 方法A：等频分箱（推荐，最常用）
按用户占比平均分，每20%用户打1-5分：
- R：R越小（最近刚来过）得分越高 → 5分
- F：F越大（来的次数多）得分越高 → 5分
- M：M越大（价值越高）得分越高 → 5分

示例：
| R（天数） | 得分 | F（次数） | 得分 | M（金额） | 得分 |
|-----------|------|-----------|------|-----------|------|
| 0-7天 | 5分 | ≥20次 | 5分 | ≥1000元 | 5分 |
| 8-14天 | 4分 | 10-19次 | 4分 | 500-999元 | 4分 |
| 15-30天 | 3分 | 5-9次 | 3分 | 200-499元 | 3分 |
| 31-60天 | 2分 | 2-4次 | 2分 | 50-199元 | 2分 |
| ≥61天 | 1分 | 1次 | 1分 | <50元 | 1分 |

#### 方法B：业务规则打分
根据业务实际情况定分数线，比如：
- 7天内来算R5，因为超过7天召回成本大幅上升
- 付费超过100元算M4，超过500元算M5

> 两种方法选一种，核心是分完之后每个分数段有明显的行为差异。

### 第4步：用户分层（经典8分法）
把三个维度的得分和平均值比较，判断高低：
- 得分 ≥ 平均分 → 高（H）
- 得分 < 平均分 → 低（L）

三个维度高低组合，得到8类用户：

| 用户类型 | R | F | M | 占比（参考） | 用户特征 | 运营策略 |
|----------|---|---|---|-------------|----------|----------|
| **重要价值用户** | 高 | 高 | 高 | 5-10% | 最近刚来，频次高，花钱多 → 顶级核心用户 | VIP服务、专属权益、忠诚度计划、优先体验新功能，重点维护 |
| **重要发展用户** | 高 | 低 | 高 | 10-15% | 最近刚来，花钱多，但频次低 → 潜力股 | 提升复购/频次，引导养成使用习惯，交叉销售，推荐更多商品/功能 |
| **重要保持用户** | 低 | 高 | 高 | 5-10% | 很久没来，但以前频次高花钱多 → 曾经的核心用户要流失 | 主动触达召回，调研流失原因，定向发优惠券/专属福利唤回 |
| **重要挽留用户** | 低 | 低 | 高 | 5% | 很久没来，频次低，但曾经花过大钱 → 高价值召回 | 高优先级召回，人工回访/大额优惠券，了解流失原因 |
| **一般价值用户** | 高 | 高 | 低 | 10-15% | 最近刚来，频次高，但花钱少 → 活跃但低价值 | 提升客单价/ARPU，引导付费，推荐高价值商品/功能，升级转化 |
| **一般发展用户** | 高 | 低 | 低 | 15-20% | 新用户/最近刚来，频次低花钱少 → 新手期 | 新手引导，激活转化，培养使用习惯，首单优惠 |
| **一般保持用户** | 低 | 高 | 低 | 10% | 很久没来，以前频次高但花钱少 → 普通流失用户 | 低成本召回，推送感兴趣内容/活动 |
| **一般挽留用户** | 低 | 低 | 低 | 30-40% | 很久没来，频次低价值低 → 流失/低价值 | 减少运营投入，或低成本普发活动，没必要投入高成本召回 |

> 分层逻辑可以简化记忆：**M高的都是重要用户，优先运营；R高F高的是活跃用户，重点维护；R低F低M低的是流失用户，降低投入。**

### 第5步：结果分析
输出分析结论，重点看：
1. **各层级用户占比**：重要价值用户占比多少？如果低于5%说明用户结构不健康
2. **迁移变化**：和上月/上季度比，各层级用户怎么迁移？重要价值用户是变多还是变少？
3. **各层级特征画像**：每个层级用户在性别/年龄/渠道/品类偏好上有什么差异？
4. **投入产出**：运营资源要向高ROI的用户倾斜，不要在低价值用户身上浪费钱

### 第6步：落地运营策略
针对不同层级制定差异化策略，执行后再监控各层级迁移。

## 简化版：5层快速分层（适合快速落地）
如果觉得8层太细，可以简化成5层：
1. **核心用户**：R高F高M高 → VIP服务
2. **潜力用户**：R高F/M低 → 培养转化
3. **活跃用户**：R高F高M低 → 提升价值
4. **沉睡用户**：R低F高M高 → 召回
5. **流失用户**：R低F低M低 → 放弃/低成本触达

## 高级玩法

### 1. RFM动态监控
每个月/季度跑一次RFM，做**用户迁移矩阵**：
- 上月重要价值用户，本月还有多少留在这个层级？多少降级了？
- 本月新的重要价值用户从哪里来？是重要发展用户升级来的，还是新用户？
- 流失率最高的是哪个层级？
- → 用户资产变化是业务健康的先行指标

### 2. 加权RFM
根据业务重要性给三个维度不同权重：
- 交易类产品：M权重最高（比如R:0.2, F:0.3, M:0.5）
- 内容/社区：F权重最高（R:0.3, F:0.5, M:0.2）
- 加权后算总分排序，而不是简单高低分类

### 3. RFM结合用户生命周期
- 新用户：R高F低M低 → 对应一般发展用户，做激活
- 成长用户：R高F上升M上升 → 引导升级
- 成熟用户：R高F高M高 → 重点维护
- 衰退用户：R上升F下降M下降 → 预警
- 流失用户：R低F低M低 → 召回/放弃

### 4. 自定义RFM维度
根据业务替换维度：
- 社交产品：R(最近互动)、F(互动次数)、M(关系链数量)
- UGC产品：R(最近发帖)、F(发帖数)、M(内容曝光/获赞数)
- 本质是"最近-频次-价值"三个维度，可以灵活替换指标

## 常见坑和避坑指南

| 坑 | 后果 | 避坑 |
|----|------|------|
| 周期选得不合理 | 用户分层完全不准，比如高频产品用365天周期，所有人都是R5 | 根据产品用户行为频次定周期，高频短周期，低频长周期 |
| 打分用固定值不用分箱 | 大部分用户集中在某一个分数段，分层没有区分度 | 优先用等频分箱，确保每个分数段都有用户，有区分度 |
| 所有用户用一样的分箱线 | 不同品类/渠道用户行为差异大，混在一起打分不准 | 可以按品类/渠道分别打分，或者在整体分箱后看细分差异 |
| 分层完不做差异化策略 | 分析完就完了，没有业务价值 | 分层的目的是运营，必须给每个层级对应明确的运营动作和负责人 |
| 只分层不监控迁移 | 不知道用户结构是变好还是变差 | 定期（月/季）做迁移分析，监控用户资产健康度 |
| 把RFM当万能模型 | 忽略了用户生命周期、用户意图，分层太粗 | RFM是基础分层，可以结合其他标签和模型用，不要孤立用 |
| 对低价值用户过度投入 | 把预算花在本来就不会回来的用户身上，ROI极低 | 资源永远向高价值用户倾斜，重要挽留>重要保持>重要发展>重要价值，低价值用户降低投入 |

## 实战案例：电商用户RFM运营

**背景**：某电商平台月活100万，营销预算每月100万，之前是全用户普发优惠券，ROI只有1:2。

**分析过程**：
1. 按90天周期做RFM分层
2. 发现：重要价值用户占比6%，贡献了40%GMV；重要挽留用户占比5%，贡献了过去30%GMV但已经60天没来；低价值流失用户占比40%，只贡献了5%GMV
3. 策略调整：
   - 重要价值用户（6%）：发专属VIP权益，20%预算，ROI 1:8
   - 重要保持/挽留用户（10%）：定向大额优惠券+短信召回，50%预算，ROI 1:5
   - 潜力/一般用户（44%）：发中等面额优惠券引导复购，25%预算，ROI 1:2
   - 低价值流失用户（40%）：只在大促时普发，5%预算，ROI 1:0.8（投入小于产出即可）
4. 效果：整体营销ROI从1:2提升到1:4.5，重要用户留存提升20%，召回了15%的沉睡高价值用户。
