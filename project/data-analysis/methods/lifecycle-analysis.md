# 用户生命周期分析（Lifecycle Analysis）

## 什么是用户生命周期分析
按用户与产品的关系演进阶段（引入→成长→成熟→衰退→流失），识别用户当前所处阶段，匹配差异化的运营目标和动作，实现全生命周期的精细化用户管理。

核心价值：从"用户视角"而非"指标视角"看业务，把零散的运营动作串成体系，避免"只管拉新不管流失"。

## 适用场景
1. ✅ 全局用户结构诊断：各阶段用户占比是否健康，瓶颈在哪
2. ✅ 差异化运营策略：不同阶段用户配不同动作，提升资源效率
3. ✅ 流失预警和召回：衰退期用户提前干预，流失期用户分层召回
4. ✅ 新用户引导优化：新手期是流失高发期，决定后续留存曲线
5. ✅ 用户价值最大化：成熟期用户挖掘变现、传播价值

不适用场景：
❌ 用户生命周期极短（一次性工具类，用单次ROI）
❌ 用户阶段难以界定（B2B长期项目制，按项目阶段更合适）
❌ 用户量太小（<1万），分阶段后样本不足

## 核心指标

| 指标 | 定义 | 计算方式 |
|------|------|----------|
| 阶段用户规模 | 各生命周期阶段的用户数及占比 | 阶段用户数 / 总用户数 |
| 阶段迁移率 | 用户从N阶段迁移到N+1阶段的比例 | 迁移用户数 / N阶段用户数 |
| 阶段留存率 | 用户在当前阶段稳定停留的比例 | 次周仍在同阶段用户 / 本阶段用户 |
| 流失率 | 用户进入流失期的比例 | 流失用户数 / 活跃用户数 |
| 召回率 | 流失用户被召回重新活跃的比例 | 召回用户数 / 流失用户数 |
| 阶段ARPU | 各阶段用户的人均收入 | 阶段收入 / 阶段用户数 |
| 阶段时长 | 用户在某个阶段平均停留时间 | Σ停留天数 / 用户数 |

## 标准分析流程

### 第1步：定义生命周期阶段
1. **新手期（引入期）**：新用户刚注册/首次使用，目标是引导体验核心价值，提升次留
2. **成长期**：已体验核心价值，开始频繁使用，目标是提升活跃度和使用深度，养成习惯
3. **成熟期**：高频使用，是核心高价值用户，目标是提升留存、变现、推荐传播
4. **衰退期**：活跃度/频率开始下降，目标是预警、干预、防止流失
5. **流失期**：一段时间没来，目标是召回（高价值）或放弃（低价值）

### 第2步：定义阶段判定规则
1. **基于活跃度和时间**：用近N日活跃天数+注册时长判定
2. **阈值要有业务依据**：参考历史数据分布，结合产品节奏
3. **规则互斥且穷尽**：每个用户只能属于一个阶段

**示例：信息流用户生命周期判定规则**
```
新手期：注册≤7天
成长期：注册>7天，近7日活跃≥3天，近7日活跃<近30日均值的1.2倍
成熟期：注册>7天，近7日活跃≥5天，且近7日活跃≥近30日均值的1.2倍
衰退期：注册>30天，近7日活跃1-2天，近30日曾有活跃≥3天
流失期：近30日活跃=0天
```

### 第3步：计算阶段标签并落表
**示例：生命周期标签计算（Presto/Kyuubi）**
```sql
INSERT INTO TABLE dws_user_value
PARTITION (dt = '${dt}')
SELECT
    user_id,
    CASE
        WHEN days_since_register <= 7 THEN '新手期'
        WHEN active_days_7d >= 5 AND active_days_7d >= avg_active_30d * 1.2 THEN '成熟期'
        WHEN active_days_7d >= 3 THEN '成长期'
        WHEN active_days_7d BETWEEN 1 AND 2 AND active_days_30d >= 3 THEN '衰退期'
        WHEN active_days_30d = 0 THEN '流失期'
        ELSE '成长期'
    END AS lifecycle_stage
FROM (
    SELECT
        user_id,
        date_diff('day', register_dt, '${dt}') AS days_since_register,
        count_if(active_dt >= date_sub('${dt}', 7)) AS active_days_7d,
        count_if(active_dt >= date_sub('${dt}', 30)) AS active_days_30d,
        count_if(active_dt >= date_sub('${dt}', 30)) / 4.0 AS avg_active_30d
    FROM dwd_user_event
    WHERE dt = '${dt}'
    GROUP BY user_id, register_dt
) t;
```

### 第4步：诊断全局用户结构
输出阶段分布表，判断结构是否健康：

| 阶段 | 用户数 | 占比 | 健康范围 | 状态 |
|------|--------|------|----------|------|
| 新手期 | 800万 | 16% | 10-20% | 健康 |
| 成长期 | 1500万 | 30% | 25-35% | 健康 |
| 成熟期 | 1200万 | 24% | 20-30% | 健康 |
| 衰退期 | 600万 | 12% | 5-10% | 偏高 |
| 流失期 | 900万 | 18% | 10-15% | 偏高 |

**结构判断**：衰退期+流失期占比30%，高于健康线25%，用户老化严重。

### 第5步：分析阶段迁移路径
1. 看相邻周期用户阶段迁移矩阵
2. 识别"上升通道"（成长→成熟）和"下降通道"（成熟→衰退→流失）
3. 关注迁移率异常变化

**迁移矩阵示例**：
| 本周\上周 | 新手 | 成长 | 成熟 | 衰退 | 流失 |
|-----------|------|------|------|------|------|
| 新手 | 45% | 35% | 5% | 5% | 10% |
| 成长 | - | 60% | 25% | 10% | 5% |
| 成熟 | - | 5% | 80% | 12% | 3% |
| 衰退 | - | 8% | 5% | 50% | 37% |
| 流失 | - | 2% | 1% | 5% | 92% |

### 第6步：制定阶段化运营策略
| 阶段 | 核心目标 | 关键动作 |
|------|----------|----------|
| 新手期 | 体验核心价值，提升次留 | 新手引导、首推爆款内容、降低使用门槛 |
| 成长期 | 养成习惯，提升频次和深度 | 签到、跨品类推荐、push个性化 |
| 成熟期 | 价值最大化，防流失 | 专属权益、内测资格、变现提效 |
| 衰退期 | 预警干预，延缓流失 | 流失预警模型、个性化召回push、利益刺激 |
| 流失期 | 分层召回 | 高价值重点召回、低价值热点push |

### 第7步：监控效果并迭代
1. 监控各阶段规模、迁移率、核心指标变化
2. 评估运营动作对阶段迁移的影响（如召回push是否提升了流失→衰退的迁移率）
3. 季度复盘阶段判定规则，业务变化时调整阈值

## 高级玩法

### 1. 流失预警模型
- 用历史数据训练分类模型（XGBoost/LightGBM），预测用户未来7/14天流失概率
- 特征：近期活跃度、内容消费、互动行为、设备/渠道
- 对高流失概率用户提前1-2周干预，比事后召回成本低5-10倍

### 2. 阶段迁移预测
- 预测用户下一阶段走向（上升/稳定/下降）
- 识别"即将进入衰退期"的成熟用户，提前延长成熟期
- 识别"即将进入成长期"的新手，加速转化

### 3. 生命周期价值曲线
- 画出用户从注册到流失的价值贡献曲线
- 识别价值高峰、衰减拐点
- 指导最佳运营介入时机

## 常见坑和避坑指南

| 坑 | 后果 | 避坑 |
|----|------|------|
| 阶段定义拍脑袋 | 阈值不合理，阶段分布失衡 | 用数据分位+业务校准，定期复盘 |
| 只看静态分布 | 错过迁移趋势，预警滞后 | 重点看迁移率和迁移方向 |
| 重拉新轻流失 | 进水比漏水快，用户池不增反降 | 流失召回和拉新同等重要 |
| 阶段动作一刀切 | 高价值用户被当低价值运营，浪费 | 必须分阶段差异化策略 |
| 流失定义太松 | 30日未活跃才判流失，召回窗口太晚 | 按7/14日预警+30日流失分级处理 |
| 忽略阶段间过渡 | 成熟→衰退的过渡期没有干预动作 | 在衰退前期就介入，延长成熟期 |
| 阶段标签不更新 | 用户已流失但标签还显示成熟 | 标签按日/周更新，关键群体实时计算 |

## 实战案例：信息流用户生命周期运营体系搭建

**背景**：浏览器信息流DAU 5000万，但近6个月流失率从8%升到12%，新增用户留存率仅35%，整体用户池增长停滞。

**分析过程**：
1. 定义五阶段：用近7日/30日活跃天数+注册时长判定
2. 诊断结构：衰退期15%+流失期20%，合计35%，远超健康线25%
3. 迁移矩阵：成熟→衰退迁移率从8%升到15%，衰退→流失迁移率从30%升到40%
4. 定位瓶颈：成熟期用户加速流失是核心问题，主要集中在中高端机型用户
5. 调研归因：竞品A同期推出高补贴内容，中高端用户被分流
6. 策略制定：
   - 新手期：首推3个爆款内容，7日内push召回未回访用户
   - 成熟期：中高端用户专属权益（无广告、内测功能），流失预警模型提前7天干预
   - 衰退期：个性化召回push（基于历史偏好），利益刺激（签到红包）
   - 流失期：高价值用户人工召回，低价值用户热点push
7. 效果验证：3个月后衰退+流失占比从35%降到28%，成熟期留存提升8%，DAU恢复增长

**结论**：生命周期运营体系搭建后，核心瓶颈（成熟期加速流失）得到缓解。关键动作是流失预警模型+成熟期专属权益，使成熟→衰退迁移率从15%降回9%，整体用户池恢复健康增长。
