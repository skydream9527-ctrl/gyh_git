# 流失分析（Churn Analysis）

## 什么是用户流失
用户在一段时间内不再使用产品/服务即视为流失，不同产品流失定义不同：
- 高频产品（内容/社交）：7天/14天未登录算流失
- 中频产品（电商/工具）：30天/60天未登录算流失
- 低频产品（旅游/招聘）：90天/180天未使用算流失
- 订阅制产品：订阅到期未续费算流失

浏览器内容生态属于高频产品，常用口径：**14天未消费信息流/未完成1次核心行为**算流失。

## 适用场景
1. ✅ 大盘流失率监控：流失率有没有异常升高？趋势是改善还是恶化？
2. ✅ 新用户流失诊断：新用户首日/7日内流失集中在哪里？
3. ✅ 高价值用户挽留：识别高流失风险的高价值用户，提前干预
4. ✅ 召回效果评估：召回push/短信/弹窗的实际召回率和召回后留存
5. ✅ 渠道质量评估：不同渠道拉来的用户流失率差异，反推渠道质量
6. ✅ 版本/活动影响评估：改版/活动后流失率是否恶化？

不适用场景：
❌ 还没有稳定DAU的早期产品（用户基数小，流失率波动大没意义）
❌ 单次消费场景（用户买完就走是正常的，不算流失问题）
❌ 季节性产品（本身就是周期性使用，淡季不活跃不等于流失）

## 核心指标

| 指标 | 定义 | 计算方式 |
|------|------|----------|
| 流失率 | 周期内流失用户数 / 周期初活跃用户数 | 流失用户数 / 周期初活跃用户数 |
| 流失规模 | 周期内流失的绝对用户数 | 流失用户数 |
| 流失用户画像 | 流失用户的人口/行为/渠道特征 | 流失用户特征分布 vs 大盘分布 |
| 召回率 | 流失用户中被召回后重新活跃的比例 | 召回成功用户数 / 触达流失用户数 |
| 召回后留存 | 被召回用户在召回后的留存情况 | 召回后第N天留存率 |
| 流失预警命中率 | 预测流失且实际流失的比例 | 命中数 / 预测流失数 |

## 标准分析流程

### 第1步：定义流失口径
1. **多久不活跃算流失**：结合产品频次，高频用7-14天，中频30天，低频90天+
2. **活跃定义**：打开APP？消费内容？完成核心行为？建议用"核心行为"而非"启动"，避免误判
3. **观测窗口**：周流失率还是月流失率？大盘监控用月，运营动作评估用周
4. **去重口径**：按用户ID去重，跨设备合并

### 第2步：流失趋势监控
输出日/周/月流失率趋势曲线，标注关键事件点（版本/活动/故障）。
```sql
-- Presto/Kyuubi：信息流用户月流失率
WITH active_base AS (
  SELECT device_id
  FROM dws_user_active_d
  WHERE dt = date_add('day', -30, current_date)
),
churned AS (
  SELECT b.device_id
  FROM active_base b
  LEFT JOIN dws_user_active_d a
    ON a.device_id = b.device_id
   AND a.dt > date_add('day', -30, current_date)
   AND a.is_feed_active = 1
  WHERE a.device_id IS NULL
)
SELECT
  count(*) AS churn_count,
  count(*) * 1.0 / (SELECT count(*) FROM active_base) AS churn_rate
FROM churned;
```

### 第3步：流失用户画像
从多个维度刻画流失用户特征：
| 维度 | 看什么 |
|------|--------|
| 人口属性 | 年龄/性别/地域/机型，是否某类用户流失集中 |
| 来源渠道 | 哪个渠道来的用户流失率高 |
| 生命周期阶段 | 新用户流失还是老用户流失？比例各多少？ |
| 行为特征 | 流失前用了什么功能？活跃频次？消费深度？ |
| 价值分层 | 高价值用户流失多吗？流失的高价值用户特征 |

### 第4步：流失原因定位
1. **定量对比**：留存用户 vs 流失用户的行为差异（消费深度、功能使用、活跃频次）
2. **关键行为识别**：找到"做了就不流失"的核心行为（Aha moment）
3. **流失前先兆**：流失前1周/2周的行为有没有衰减迹象
4. **定性研究**：流失用户问卷/访谈，收集真实原因

### 第5步：流失预警建模
1. **特征工程**：用过去7/14/30天的行为特征预测未来流失概率
2. **模型选择**：逻辑回归/GBDT/随机森林，追求可解释性优先
3. **阈值设定**：高价值用户用低阈值（宁可误召回），低价值用户用高阈值
4. **离线评估**：AUC、命中率、召回率，按业务可接受的误打扰率定阈值

### 第6步：制定召回策略
按用户价值和流失原因分层召回：
| 用户价值 | 流失原因 | 召回策略 |
|----------|----------|----------|
| 高价值 | 自然衰减 | 个性化push + 内容偏好召回 |
| 高价值 | 体验问题 | 修复后定向触达 + 补偿 |
| 中价值 | 内容疲劳 | push + 新内容/新功能引导 |
| 低价值 | 任意 | 低成本push/短信，控制触达成本 |

### 第7步：效果评估
- 触达率 → 召回率 → 召回后7/30留存 → ROI
- 召回后留存差说明召回策略治标不治本，需优化召回内容和时机
- A/B测试：召回组 vs 不召回组，看真实增量

## 高级玩法

### 1. 生存分析（Survival Analysis）
- 用Kaplan-Meier曲线看用户"存活"概率随时间变化
- 比较不同Cohort/渠道的生存曲线，找流失加速点
- Cox比例风险模型量化各因素对流失风险的影响

### 2. 流失前先兆序列
- 用序列模式挖掘，找流失前N天的高频行为路径
- 例：流失用户前3天往往出现"消费深度断崖下跌 + 减少使用时长"

### 3. 增量召回评估
- 不要只看召回组留存，要和不召回组对比看真实增量
- 用Uplift Model区分"必流失""必留存""可挽回"三类用户，只对"可挽回"投入资源

### 4. 流失分级预警
- 红色预警：高流失风险 + 高价值用户 → 人工介入
- 黄色预警：高流失风险 + 中价值 → 自动化触达
- 绿色：低风险 → 不打扰

## 常见坑和避坑指南

| 坑 | 后果 | 避坑 |
|----|------|------|
| 流失定义太宽（90天） | 流失已成定局，召回率极低 | 高频产品用7-14天，越早干预召回率越高 |
| 活跃定义太浅（启动即算） | 把"打开就退出"的用户当活跃，低估流失 | 用核心行为定义活跃，如消费内容≥3条 |
| 只看流失率不看流失规模 | 流失率不高但流失的都是高价值用户，损失大 | 流失率 × 用户价值一起看 |
| 不区分新/老用户流失 | 新用户流失率高正常，和老用户流失性质不同 | 分生命周期阶段看，新用户看转化，老用户看流失 |
| 召回只看召回率不看留存 | 召回了但留不住，浪费触达成本 | 召回后7/30留存才是真指标 |
| 把相关性当因果 | "流失用户都看过广告" → 误以为广告导致流失 | 用对照组验证，可能是渠道本身质量差 |
| 预警阈值一刀切 | 高价值用户被漏掉，低价值用户被频繁打扰 | 按用户价值分层设阈值 |

## 常见流失原因
- **新用户流失**：产品没看懂、新手体验差、没找到核心价值、流量不精准
- **体验问题流失**：bug多、性能差、广告太多、付费坑
- **价值流失**：内容/商品不好、没有满足需求、找到更好的替代品
- **自然流失**：需求不存在了（比如买完房了就不看买房APP了）
- **竞争流失**：竞品补贴/体验更好，用户迁移

## 实战案例：浏览器信息流新用户流失诊断

**背景**：浏览器信息流新用户14日流失率从35%升到48%，产品经理需要定位原因。

**分析过程**：
1. 拆分生命周期：新用户流失占整体流失的62%，是主要矛盾，聚焦新用户
2. 渠道下钻：发现"信息流广告"渠道新用户14日流失率达61%，远高于自然量28%
3. 行为对比：广告渠道新用户首日平均消费内容3.2条，自然量12.5条；首日消费<5条的用户14日流失率高达72%
4. 先兆识别：流失用户在Day3-7出现"刷新次数下降、停留时长缩短"的衰减迹象
5. 定性调研：广告素材和落地页内容不符，用户觉得"被骗进来"，信任度低
6. 召回测试：对广告渠道流失用户做内容偏好召回push，召回率8%，但召回后7留存仅15%

**结论**：流失率升高的根因是Q1加大广告投放，低质量渠道稀释了新用户质量。短期建议：广告渠道降低出价、优化素材与落地页一致性、过滤首日消费<3条的低质用户；长期建议：新用户首刷做内容冷启动，把首刷深度从3条提升到8条以上，14日流失率目标降到40%以内。
