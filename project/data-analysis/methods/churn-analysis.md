# 流失分析（Churn Analysis）

## 什么是用户流失
用户在一段时间内不再使用产品/服务即视为流失，不同产品流失定义不同：
- 高频产品（内容/社交）：7天/14天未登录算流失
- 中频产品（电商/工具）：30天/60天未登录算流失
- 低频产品（旅游/招聘）：90天/180天未使用算流失
- 订阅制产品：订阅到期未续费算流失

## 核心指标
- 流失率 = 周期内流失用户数 / 周期初活跃用户数
- 流失用户画像：流失用户的共同特征
- 召回率：流失用户中被召回后重新活跃的比例

## 分析流程
1. **定义流失口径**：多久不活跃算流失，符合业务实际
2. **流失趋势监控**：日/周/月流失率趋势，有没有异常升高
3. **流失用户画像**：
   - 人口属性：年龄/性别/地域
   - 来源渠道：什么渠道来的用户流失率高
   - 行为特征：流失前用了什么功能？活跃度如何？有没有什么先兆行为？
   - 生命周期阶段：新用户流失？还是老用户流失？
4. **流失原因定位**：
   - 定量：行为数据对比（留存vs流失用户行为差异）
   - 定性：流失用户调研、访谈、反馈收集
5. **流失预警建模**：找到高流失风险用户特征，提前干预
6. **召回策略制定**：针对不同流失原因和用户价值，制定差异化召回策略
7. **效果评估**：召回率、召回后留存率、ROI

## 常见流失原因
- **新用户流失**：产品没看懂、新手体验差、没找到核心价值、流量不精准
- **体验问题流失**：bug多、性能差、广告太多、付费坑
- **价值流失**：内容/商品不好、没有满足需求、找到更好的替代品
- **自然流失**：需求不存在了（比如买完房了就不看买房APP了）
- **竞争流失**：竞品补贴/体验更好，用户迁移
