# 用户分群（User Segmentation）

## 什么是用户分群
根据用户的属性、行为、价值等特征，把用户分成不同的群体，对不同群体用差异化的产品/运营策略，比"千人一面"效率高很多。

核心价值：识别高价值用户、定位问题群体、支持精细化运营、为推荐/定价/资源分配提供决策依据。

## 适用场景
1. ✅ 差异化运营策略：高价值用户保权益，低价值用户提活跃，流失用户做召回
2. ✅ 推荐系统冷启动：新用户用属性分群匹配热门内容，老用户用行为分群做个性化
3. ✅ 增长投放优化：识别高LTV用户群特征，反推投放定向人群
4. ✅ 流失预警：对衰退期高价值用户提前干预
5. ✅ AB测试分层：实验前后用分群校验效应是否在不同群体上一致

不适用场景：
❌ 用户量太小（<1万），分群后样本不足，结论不稳定
❌ 强实时场景（秒级响应），分群标签更新有延迟，用规则实时判断更好
❌ 单一行为目标（只看是否下单），直接看转化漏斗更直接

## 核心指标

| 指标 | 定义 | 计算方式 |
|------|------|----------|
| 分群规模占比 | 各群体用户数占总用户比例 | 群体用户数 / 总用户数 |
| 分群价值贡献 | 各群体贡献的收入/时长占比 | 群体总收入 / 总收入 |
| 群体活跃度 | 群体内DAU/MAU、人均时长 | Σ活跃用户 / 群体规模 |
| 群体留存率 | 群体在N日后的留存比例 | N日后回访用户数 / 群体规模 |
| 群体迁移率 | 用户从一个群体迁移到另一个的比例 | 迁移用户数 / 原群体规模 |
| RFM分层 | 最近一次消费R、消费频率F、消费金额M综合分层 | 三维度分箱后组合编码 |

## 标准分析流程

### 第1步：明确分群目标
1. **业务问题是什么**：提升留存？提升变现？优化推荐？做召回？
2. **分群后要做什么动作**：决定分群粒度——能驱动不同动作才需要分开
3. **明确可用数据**：属性数据、行为日志、价值数据是否齐全

**示例**：浏览器信息流想提升整体留存 → 用活跃度+内容偏好分群，对低活用户做内容召回

### 第2步：选择分群维度和方法
1. **规则分群**：RFM、生命周期、活跃度分层，简单可解释，业务最常用
2. **算法分群**：K-Means/DBSCAN聚类，适合多维度复杂场景，需要数据团队支撑
3. **组合分群**：先用规则做大类（如生命周期），再用算法在子群体内细分

**常用分群维度**：
| 维度类型 | 具体维度 | 适用场景 |
|----------|----------|----------|
| 属性分群 | 年龄、性别、地域、设备、注册时间 | 新用户冷启动、投放定向 |
| 行为分群 | 活跃度、使用时长、功能偏好、内容品类 | 推荐策略、运营触达 |
| 价值分群 | RFM、LTV、付费金额 | 高价值用户运营、资源倾斜 |
| 生命周期分群 | 新手/成长/成熟/衰退/流失 | 全生命周期运营 |
| 需求分群 | 用户核心目的（资讯/视频/搜索） | 内容供给策略 |

### 第3步：定义分群规则和阈值
1. **阈值要有业务依据**：不能拍脑袋，参考历史数据分布和业务节奏
2. **分群互斥且穷尽**：每个用户只能属于一个群体，且所有用户都能被覆盖
3. **分群数量适中**：3-8个群体，太少没差异，太多运营成本高

**示例：信息流用户活跃度分群**
```
高活用户：近7日使用≥5天
中活用户：近7日使用3-4天
低活用户：近7日使用1-2天
沉默用户：近7日使用0天（近30日内有使用）
流失用户：近30日未使用
```

### 第4步：计算分群标签并落表
用SQL把分群规则落成标签表，按日/周更新。

**示例：信息流用户价值分群（Presto/Kyuubi）**
```sql
INSERT INTO TABLE dws_user_value
PARTITION (dt = '${dt}')
SELECT
    user_id,
    CASE
        WHEN r_score = 5 AND f_score >= 4 AND m_score >= 4 THEN '高价值'
        WHEN r_score >= 3 AND f_score >= 3 THEN '成长价值'
        WHEN r_score <= 2 AND f_score >= 3 THEN '衰退价值'
        WHEN r_score <= 2 AND f_score <= 2 THEN '低价值'
        ELSE '潜力'
    END AS user_segment,
    r_score, f_score, m_score
FROM (
    SELECT
        user_id,
        NTILE(5) OVER (ORDER BY last_active_dt DESC) AS r_score,
        NTILE(5) OVER (ORDER BY active_days_30d DESC) AS f_score,
        NTILE(5) OVER (ORDER BY total_consume_time_30d DESC) AS m_score
    FROM dwd_user_event
    WHERE dt = '${dt}'
      AND user_id IS NOT NULL
) t;
```

### 第5步：对比各群体差异
输出分群对比表，找差异点：

| 群体 | 规模占比 | 活跃度 | 留存率 | 价值贡献 | 核心特征 |
|------|----------|--------|--------|----------|----------|
| 高价值 | 8% | 6.2天/周 | 85% | 42% | 高频深度消费，偏好财经/科技 |
| 成长价值 | 22% | 4.1天/周 | 72% | 28% | 中频消费，多品类 |
| 潜力 | 35% | 2.3天/周 | 55% | 18% | 低频但有时长，娱乐内容为主 |
| 低价值 | 25% | 1.1天/周 | 32% | 8% | 浅消费，push召回为主 |
| 流失 | 10% | 0 | 5% | 4% | 30日以上未回访 |

### 第6步：制定差异化策略
针对每个群体的核心问题，设计差异化动作：
1. **高价值用户**：保权益、提深度，比如专属内容、内测资格
2. **成长价值用户**：提频次、扩品类，比如跨品类推荐、签到
3. **潜力用户**：养习惯、提留存，比如push召回、内容个性化
4. **低价值用户**：低成本触达，比如热点push、轻量互动
5. **流失用户**：高价值召回、低价值放弃

### 第7步：验证分群效果并迭代
1. 策略上线后，对比各群体核心指标变化
2. 看群体迁移率：是否有用户从低价值流向高价值
3. 分群规则定期复盘（季度），业务变化时及时调整阈值

## 高级玩法

### 1. RFM分层精细化
- R/F/M三维度各分5档，组合出125个细分群体
- 实际不用这么细，合并成8-10个可执行群体即可
- 关键是识别"高R低F"（待激活）、"低R高F"（待召回）等关键群体

### 2. 行为聚类+规则标签融合
- 用K-Means对用户行为向量（时长、品类、时段、频次）聚类
- 聚类结果用业务语言解释（如"早间资讯型"、"晚间娱乐型"）
- 比纯规则分群更能发现隐性群体

### 3. 分群迁移矩阵
- 看用户在相邻周期内的群体迁移路径
- 识别"上升通道"（低价值→高价值）和"流失通道"（高价值→流失）
- 重点优化流失通道的关键转化点

## 常见坑和避坑指南

| 坑 | 后果 | 避坑 |
|----|------|------|
| 分群维度太多 | 群体碎片化，运营无法落地 | 控制在3-5个核心维度，群体数量3-8个 |
| 阈值拍脑袋 | 分群失衡，80%用户挤在一个群体 | 用数据分位（NTILE）+业务校准 |
| 只分群不落地 | 报告好看但没有动作，分群成摆设 | 分群前先定动作，分群后跟进策略效果 |
| 用静态标签 | 用户变了但标签没更新，策略失效 | 标签按日/周更新，关键群体实时计算 |
| 忽略群体迁移 | 只看静态画像，错过流失预警 | 建迁移矩阵，关注高价值向下迁移 |
| 分群和业务目标脱节 | 分了一堆群但和留存/变现无关 | 分群目标必须对应核心业务指标 |

## 实战案例：信息流用户分群驱动内容运营

**背景**：浏览器信息流DAU 5000万，但人均时长3个月没增长，运营策略"千人一面"效率低。

**分析过程**：
1. 明确目标：用分群支持差异化内容推荐和push策略，提升人均时长
2. 选择维度：活跃度（R）+ 内容品类偏好（F）+ 消费深度（M，单次时长）
3. 定义规则：用NTILE(4)分档，组合成6个群体
4. 对比差异：发现"深度资讯型"（占12%）贡献了35%时长，但"浅浏览型"（占38%）人均时长只有1.2分钟
5. 制定策略：
   - 深度资讯型：财经/科技优质内容加权，提频次
   - 浅浏览型：首屏推高互动短视频，降阅读门槛
   - 流失型（10%）：热点push召回，标题党+轻量内容
6. AB测试：策略组人均时长提升11%，浅浏览型提升23%

**结论**：分群驱动差异化运营，整体人均时长从4.1分钟提升到4.6分钟，高价值用户留存提升5%。
