# 趋势分析（Trend Analysis）

## 什么是趋势分析
按时间维度观察指标的长期变化方向、周期性、拐点，识别业务是增长、平稳还是衰退，并预测未来走势。

核心价值：判断业务健康度、发现异常拐点、评估策略效果、为目标和资源规划提供依据。

## 适用场景
1. ✅ 核心指标监控：DAU、时长、收入、留存的日常趋势
2. ✅ 增长/衰退判断：业务是在上行还是下行，速度多快
3. ✅ 策略效果评估：改版/活动上线后指标趋势是否改变
4. ✅ 异常预警：指标突然偏离趋势，及时定位
5. ✅ 目标预测：基于历史趋势预测下季度/年度指标

不适用场景：
❌ 一次性事件分析（用归因/漏斗更直接）
❌ 用户行为路径类问题（用路径分析）
❌ 无时间维度的横向对比（用分群对比）

## 核心指标

| 指标 | 定义 | 计算方式 |
|------|------|----------|
| 绝对值 | 指标在某时刻的数值 | 直接统计（DAU、收入） |
| 同比增长率 | 与去年同期对比的增长率 | (本期-去年同期)/去年同期 |
| 环比增长率 | 与上一周期对比的增长率 | (本期-上期)/上期 |
| 移动平均 | 去噪后的平滑趋势 | 近N期值的平均（MA7、MA30） |
| 复合增长率(CAGR) | 周期内的年均增长速度 | (末期/初期)^(1/N) - 1 |
| 拐点 | 趋势方向发生反转的时间点 | 移动平均线斜率变号 |

## 标准分析流程

### 第1步：明确分析目标和指标
1. **业务问题**：看增长健康度？评估策略效果？预测目标？
2. **选对指标**：核心指标优先（DAU、时长、收入），不要看虚荣指标
3. **定义口径**：指标的统计口径必须一致，跨期可比
4. **确定时间粒度**：日（监控）、周（短期趋势）、月（长期趋势）

**示例**：评估浏览器信息流DAU下半年的增长趋势 → 用日粒度DAU，看7-12月

### 第2步：准备时间序列数据
1. **数据完整性**：检查是否有断点、缺失日期
2. **口径一致性**：指标定义、统计范围在周期内不能变
3. **剔除已知干扰**：版本发布、统计口径变更要在数据上标注

**示例：拉取信息流DAU序列（Presto/Kyuubi）**
```sql
SELECT
    dt,
    COUNT(DISTINCT user_id) AS dau,
    SUM(consume_time) / COUNT(DISTINCT user_id) AS avg_time
FROM dwd_user_event
WHERE dt BETWEEN '2025-07-01' AND '2025-12-31'
  AND event_type IN ('feed_impression', 'feed_click', 'feed_consume')
GROUP BY dt
ORDER BY dt;
```

### 第3步：可视化趋势
1. **折线图**：最直观，日粒度+移动平均叠加
2. **标注关键事件**：版本发布、活动、节假日、口径变更
3. **多指标叠加**：DAU和时长一起看，避免单一指标误判

**输出标准趋势表**：
| 日期 | DAU | 环比 | 同比 | MA7 | MA30 | 事件标注 |
|------|-----|------|------|-----|------|----------|
| 11-01 | 5200万 | +1.2% | +8.5% | 5180万 | 5100万 | - |
| 11-08 | 5350万 | +2.9% | +10.2% | 5290万 | 5150万 | 推荐算法v2上线 |

### 第4步：拆解趋势成分
1. **长期趋势**：用移动平均（MA30）看整体方向
2. **周期性/季节性**：
   - 周内周期：工作日vs周末（信息流周末通常降10-15%）
   - 月内周期：月初月末
   - 节假日：春节、国庆波动大
3. **随机波动**：去掉趋势和周期后的噪音
4. **拐点识别**：移动平均线斜率从正变负或负变正

### 第5步：同环比结合分析
1. **环比看短期动能**：近7日vs上一个7日，是否加速
2. **同比看长期健康度**：今年vs去年，是否跑赢大盘
3. **同环比背离**：
   - 环比涨同比跌 → 短期反弹但长期仍衰退
   - 环比跌同比涨 → 长期向上但短期回调
4. **季节性调整**：对比同期要剔除节假日影响

### 第6步：定位趋势变化原因
1. **拐点定位**：趋势反转的具体时间点
2. **多维度拆分**：按渠道、平台、用户群拆，看是全局还是局部
3. **关联事件**：版本、活动、竞品、市场环境
4. **归因验证**：用AB测试或控制变量验证假设

### 第7步：预测未来走势并给建议
1. 基于历史趋势+季节性，预测下月/下季度指标
2. 给出趋势是健康还是预警的判断
3. 针对趋势变化提出策略建议

## 高级玩法

### 1. 同环比结合看真实增长
- 单看环比会被季节性误导，单看同比会被基数效应误导
- 同环比一起看：环比看动能，同比看健康度
- 长周期看CAGR（复合增长率），避免被短期波动干扰

### 2. 季节性调整
- 用同期对比（今年周末vs去年周末）剔除周内周期
- 节假日单独建模（春节影响15-30天）
- 调整后的趋势更能反映真实增长

### 3. 异常检测
- 用移动平均±2倍标准差做预警线
- 异常值要区分"业务真实变化"和"数据问题"
- 配合自动告警，异常发生时及时通知

### 4. 趋势预测
- 短期（1-4周）：移动平均+季节性调整
- 中期（1-3月）： Holt-Winters/ARIMA等时间序列模型
- 长期（季度以上）：结合业务规划、市场环境做情景预测

## 常见坑和避坑指南

| 坑 | 后果 | 避坑 |
|----|------|------|
| 只看绝对值不看同环比 | 误判趋势方向 | 同环比结合，多维度看 |
| 忽略季节性 | 把周末下降当异常，过度反应 | 建立周期基准，同期对比 |
| 口径变更未标注 | 趋势断崖是数据问题不是业务问题 | 维护口径变更日志，数据上标注 |
| 用日粒度看长期 | 噪音太大看不出趋势 | 长期用周/月粒度或移动平均 |
| 把相关性当因果 | 某事件和趋势变化同时发生就归因 | 多维度验证、AB测试确认 |
| 只看单一指标 | DAU涨但时长跌，误判健康 | 核心指标组合看（DAU+时长+留存） |
| 预测忽略业务拐点 | 历史趋势外推失效 | 预测要结合业务规划和大事件 |

## 实战案例：信息流DAU下半年增长趋势评估

**背景**：浏览器信息流Q3做了推荐算法升级，Q4做了内容供给扩充，需要评估下半年DAU增长是否健康，为来年目标定基。

**分析过程**：
1. 拉取7-12月日粒度DAU+人均时长，叠加MA7、MA30
2. 拆解趋势成分：长期趋势向上，周内周期明显（周末降12%），无异常断点
3. 同环比：7月环比+3%，同比+15%；12月环比+1.5%，同比+22%——同比加速，长期健康
4. 拐点定位：9月15日（算法v2上线）环比斜率从1.2%提升到2.8%，11月内容扩充后斜率回落到1.5%
5. 多维拆分：安卓同比+25%，iOS同比+18%；新用户同比+30%，老用户同比+12%
6. 关联指标：DAU增长同时，人均时长同比+8%，7日留存同比+4%，无"以质换量"问题
7. 预测：基于Q4趋势+季节性，预测来年Q1 DAU可达6000万，建议目标定为5800-6200万

**结论**：下半年DAU增长健康，算法升级是主要驱动力（贡献约60%增量），内容扩充贡献约25%。来年Q1目标建议5800-6200万，重点继续优化算法和扩充优质内容供给。
