# 留存分析（Retention Analysis）

## 什么是留存分析
留存分析衡量用户在**首次使用产品后，后续N天/周/月再次回来使用**的比例，是衡量产品价值和用户粘性的核心指标。留存高说明产品真正解决了用户问题，留存低说明产品还没找到PMF（产品市场匹配）。

经典观点：**新用户留存是产品的生命线，是所有增长的前提。** 100个新用户进来，次留只有20%，花再多钱拉新也是漏水的桶。

## 适用场景
1. ✅ PMF验证：新用户留存是否达到行业基准，是否真正满足用户需求
2. ✅ 产品迭代效果评估：新版本/新功能上线后留存是否提升
3. ✅ 渠道质量评估：不同渠道新用户留存差异，判断渠道质量
4. ✅ 用户分群差异：哪类用户留存高，哪类低，针对性优化
5. ✅ 流失预警：留存率持续下降是产品出问题的先行信号
6. ✅ LTV测算：留存曲线是计算用户终身价值的基础

## 核心留存指标

### 常见留存类型
| 指标 | 定义 | 计算公式 | 适用场景 |
|------|------|----------|----------|
| **次日留存（Day1）** | 首次使用后第2天还来的用户比例 | 第2天活跃用户数 / 首日新增用户数 | 新手引导/首次体验评估，新用户质量 |
| **7日留存（Day7）** | 首次使用后第7天还来的用户比例 | 第7天活跃用户数 / 首日新增用户数 | 用户是否养成初步使用习惯 |
| **30日留存（Day30）** | 首次使用后第30天还来的用户比例 | 第30天活跃用户数 / 首日新增用户数 | 产品长期价值验证，PMF核心指标 |
| **周留存（WeekN）** | 首次使用后第N周活跃比例 | 第N周活跃用户数 / 首周新增用户数 | 中高频产品，按周看趋势 |
| **月留存（MonthN）** | 首次使用后第N月活跃比例 | 第N月活跃用户数 / 首月新增用户数 | 低频产品，长期粘性 |

### 留存曲线解读
- 看**下滑速度**：前几天快速下滑是正常的（用户筛选期），如果一直快速下滑说明产品没价值
- 看**拐点**：曲线什么时候开始平缓？一般好产品在7-14天左右进入平稳期
- 看**平台期高度**：曲线平稳后留存率是多少？这是产品长期留存水平：
  - 工具类：平台期30%+优秀，20%+合格
  - 内容/社区：平台期40%+优秀，25%+合格
  - 电商：平台期20%+优秀，10%+合格
  - SaaS：月留存90%+优秀，80%+合格

## 标准分析流程

### 第1步：明确留存口径
常见口径坑，必须提前定义清楚：
1. **"首次使用"定义**：注册？启动APP？还是完成核心行为？（建议：完成核心行为才算真正激活用户，比纯启动留存更有意义，叫"行为留存"）
2. **"活跃"定义**：启动就算？还是要完成某个核心行为？（根据业务定，建议用核心行为定义活跃，否则是"虚留存"）
3. **计算方式**：
   - 精确留存（N日后同一天活跃）：严格第N天活跃，标准算法
   - 区间留存（N日内任意一天活跃）：第1天到第N天只要来过就算，宽松口径
4. **去重口径**：按用户ID还是设备ID？

### 第2步：计算留存率，画留存曲线
标准留存表（Cohort表）格式：

| 新增日期 | 新增用户数 | Day1 | Day3 | Day7 | Day14 | Day30 |
|----------|------------|------|------|------|-------|-------|
| 6月1日 | 10,000 | 42% | 28% | 20% | 15% | 12% |
| 6月2日 | 12,000 | 40% | 26% | 18% | 13% | 11% |
| 6月3日 | 11,000 | 43% | 29% | 21% | - | - |

**看两个维度**：
1. **横向看**：同一批用户随时间留存下滑趋势
2. **纵向看**：不同日期新增用户的同期留存对比（看版本/策略变化影响）

### 第3步：留存问题诊断
如果留存低，按这个顺序诊断：

#### 第一阶段：新用户首访体验（影响次日留存）
次留低，问题基本都在新手期：
- 用户进来第一秒有没有看懂产品是干什么的？
- 新用户引导是不是太长太复杂？
- 有没有让用户快速体验到"Aha时刻"（核心价值）？
- 加载速度/崩溃/性能问题？
- 渠道流量是不是不精准，来的不是目标用户？

→ 看新用户首访路径分析、漏斗分析、崩溃/性能数据、渠道留存对比

#### 第二阶段：习惯养成期（影响7日留存）
用户过了新手期但留不下来：
- 产品有没有让用户形成使用习惯？（有没有触发点？有没有钩子？）
- 核心功能体验好不好用？
- 有没有给用户回来的理由？（消息通知？订阅？更新？）
- 有没有引导用户建立关系链/收藏/上传内容等"沉没成本"？

→ 看7日内用户行为，留存vs未留存用户行为差异

#### 第三阶段：长期留存（影响30日留存）
用户用了几周还是走了：
- 产品有没有持续提供价值？
- 有没有持续的内容/功能更新？
- 用户有没有遇到痛点解决不了？
- 有没有被竞品抢走？

→ 用户访谈、流失用户调研、竞品分析

### 第4步：区分"新用户留存"和"活跃用户留存"
- **新用户留存**：产品吸引力、新手体验、PMF验证 → 增长团队负责
- **老用户留存**：产品长期价值、运营策略、用户粘性 → 产品/运营团队负责
两者问题不一样，不要混在一起看。

### 第5步：留存vs未留存用户行为对比
这是找到留存驱动因素最有效的方法：
1. 把同一批新用户分成两组：7日留存的vs没留存的
2. 对比两组用户在首周的行为差异：
   - 做了什么行为的用户留存率高？
   - 来多少次？用了多长时间？
   - 用了哪些功能？
   - 有没有什么行为是留存用户90%都做了，而流失用户很少做的？
3. 找到**魔法数字（Magic Number）**：
   - 例：Facebook发现"10天内加7个好友"的用户留存率极高
   - 例：抖音发现"首周看100个视频"的用户留存高
   - → 引导新用户尽快完成这个行为，就能大幅提升留存

## 核心留存提升方法论

### Aha时刻优化（提升次留）
用户越快体验到核心价值，留存越高：
- 简化新手流程，减少注册门槛
- 新用户引导直奔核心价值，不要塞太多功能介绍
- 个性化推荐/内容，让用户第一次来就看到感兴趣的东西
- 首屏加载速度优化，性能问题是留存杀手

### 习惯养成（提升7日留存）
- 设置外部触发：push/短信/邮件通知，合适的时间唤回用户
- 打造使用钩子：每日签到、每日任务、个性化推荐
- 引导用户建立"投资"：收藏、关注、发布内容、加好友，用户投入越多越不容易走
- 周期性更新：每天有新内容/新任务，给用户回来的理由

### 长期价值（提升30日留存）
- 持续迭代功能/内容，保持新鲜感
- 构建社区/关系链，用户的朋友在这就不会走
- 建立会员/积分体系，锁定用户长期价值
- 流失前预警，主动召回高风险用户

## 高级玩法

### 1. 行为留存vs启动留存
只看启动留存会有"虚假繁荣"：用户误触/误点进来马上走也算启动。建议看**核心行为留存**，比如内容产品看"浏览N个内容"留存，电商看"有过浏览商品行为"留存，更能反映真实产品价值。

### 2. 留存细分
- 分渠道留存：判断渠道质量，淘汰低留存渠道
- 分版本留存：新版本上线后留存有没有变化
- 分用户分群留存：不同特征用户留存差异，找到高价值用户群体
- 分功能留存：用了A功能的用户vs没用过的留存差多少，判断功能价值

### 3. 留存预测
用机器学习模型预测新用户未来会不会流失，提前干预高风险用户。

### 4. 反向留存：流失分析
专门分析流失用户：流失前有什么行为特征？流失原因是什么？找到早期流失信号。

## 常见坑和避坑指南

| 坑 | 后果 | 避坑 |
|----|------|------|
| 只看启动留存，不看行为留存 | 留存率虚高，看起来好看但用户其实没用到核心价值 | 同时看启动留存和核心行为留存，核心行为留存更重要 |
| 把"回访"当"留存" | 用户收到push点进来马上就走，也算留存，但没有价值 | 定义合理的活跃标准：停留时长≥N秒，或完成核心行为 |
| 不做Cohort分群，只看整体大盘留存 | 老用户占比高了整体留存看起来涨了，但新用户留存其实一直在跌，问题被掩盖 | 必须按新增日期做Cohort分析，分开看新老用户留存 |
| 留存低只会怪产品，不看渠道质量 | 渠道买了垃圾流量，留存低是必然，产品改半天没用 | 先分渠道看留存，如果是单一渠道低，是渠道问题；全渠道低才是产品问题 |
| 忽略样本量，过早下结论 | 某一天新增只有100个用户，留存波动极大，不能代表趋势 | 看至少连续一周的Cohort趋势，不要看单天数据 |
| 把相关性当因果 | 发现"用了分享功能的用户留存高"，就强制让所有用户分享，反而导致用户反感流失 | 相关性≠因果，要通过AB测试验证引导用户做某个行为是否真能提升留存 |
| 追求短期留存，损害长期体验 | 发骚扰push提升了短期次留，但用户长期关通知甚至卸载 | 留存提升不能以损害用户体验为代价，要找长期健康的方式 |

## 实战案例：内容APP次留提升
**背景**：某内容APP新用户次留只有25%，行业平均35%。
**分析**：
1. 渠道拆解：各渠道次留差不多，排除渠道问题，是产品问题
2. 新用户路径分析：50%新用户启动后看了不到3个视频就退出
3. 行为对比：留存用户首周平均看28个视频，流失用户只看了4个；看了≥10个视频的用户次留高达55%
4. 问题诊断：新用户冷启动推荐内容不精准，用户划了几个不感兴趣的就走了
**优化方案**：
1. 新用户首次启动增加简单兴趣标签选择，3个标签可选
2. 优化新用户冷启动推荐算法，优先给高质量、普适性热门内容
3. 新手引导简化，直接进入内容feed流
**效果**：新用户首周人均观看视频数从4个提升到12个，次留从25%提升到34%，达到行业平均。
