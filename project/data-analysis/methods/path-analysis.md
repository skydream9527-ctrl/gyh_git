# 路径分析（Path Analysis）

## 什么是路径分析
路径分析是挖掘用户在产品内的**行为轨迹序列**，发现用户从A到B实际走了哪些路径、各路径的占比、异常路径、热门路径等，还原真实用户行为地图。

和漏斗分析的区别：
- 漏斗分析：预设固定步骤，看有序转化
- 路径分析：不预设路径，挖掘真实行为流向，适合开放场景

## 适用场景
1. ✅ 用户行为地图还原：用户进入APP后实际都去了哪里？
2. ✅ 导航/入口优化：核心功能的入口是否合理？用户能找到吗？
3. ✅ 异常路径发现：有没有用户在循环打转？有没有走"后门"路径？
4. ✅ 流失后去向：从关键页面退出的用户都去了哪里？
5. ✅ 转化路径探索：完成目标的用户都走了哪些不同路径？占比多少？
6. ✅ 新功能/新页面的流量传导：新上线的页面给哪些页面带了流量？
7. ✅ 漏斗分析前置：在做漏斗前，先看用户真实路径是什么，避免拍脑袋定漏斗步骤

## 核心概念

| 概念 | 定义 |
|------|------|
| Session/会话 | 用户一次启动到退出的连续行为序列，是路径分析的基本单位 |
| 步长/Path Length | 一个会话里包含的行为事件数量 |
| 路径占比 | 走某条特定路径的会话数占总会话数的比例 |
| 转化路径 | 用户从起点到目标终点所经过的步骤序列 |
| 节点 | 路径上的每一个行为/页面 |
| 边/流向 | 从一个节点到下一个节点的跳转，边上的权重是跳转人数 |

## 标准分析流程

### 第1步：明确分析目标，确定分析边界
1. **确定起点（根节点）**：你关心哪类用户的路径？
   - 例：新用户首次启动、从首页进入、支付失败用户、404页面用户
2. **确定终点（可选）**：是否关心到某个目标的路径？
   - 例：完成支付、发布内容、卸载前最后路径
3. **确定会话划分规则**：超时时间多长算新会话？（常见：30分钟无操作算新会话）
4. **确定分析步长**：看前几步？（常见：前5步/前10步，太长路径占比很低意义不大）
5. **确定节点粒度**：
   - 粗粒度：页面级（例：首页→商详→购物车）
   - 细粒度：页面+按钮点击（例：首页→点击搜索→搜索结果页→点击商品）
   - 根据分析目标选择，不要太细否则路径会非常分散

### 第2步：路径计算与桑基图可视化
主流路径可视化用**桑基图（Sankey Diagram）**：
- 左边是起点，每一列是一步，每个块是一个节点
- 块的宽度代表流量大小
- 条带代表流向，宽度代表跳转人数

**计算规则**：
1. 按用户+时间排序所有行为，划分会话
2. 对每个会话，按顺序提取节点序列
3. 统计所有会话中，每一步到下一步的跳转次数
4. 合并相同路径，计算占比

### 第3步：关键发现模式
做路径分析重点找这几类模式：

#### 模式1：预期路径 vs 实际路径对比
你预设的"官方路径"用户走了多少？
- 例：你设计的是首页→分类→商详→下单，但实际80%用户走的是首页→搜索→商详→下单
- → 结论：分类入口设计有问题，搜索才是主流找货路径，优化搜索优先级更高

#### 模式2：高流失节点
哪些节点之后用户大量退出？
- 某一步退出率超过70%就是严重问题
- 看这些页面的用户是不是预期的目标用户，页面信息是否匹配预期

#### 模式3：循环打转/回退路径
有没有用户在两个页面之间来回跳？
- 例：填写表单页→预览页→返回表单页→预览页
- → 说明表单有信息歧义，用户需要反复确认
- 例：搜索结果页→商详→返回搜索结果页→商详
- → 要么搜索结果不精准，要么商详信息不全，用户需要对比

#### 模式4：意料之外的"捷径"
用户有没有走你没想到的路径完成目标？
- 例：你没想到"收藏夹→下单"路径占比20%
- → 说明收藏是重要转化入口，要强化收藏引导和收藏夹体验

#### 模式5："黑洞"节点
哪些页面是流量终点？用户进去之后很少再去其他页面？
- 例：帮助中心页面，90%用户看完直接退出
- → 如果是预期内的（比如完成支付的成功页）没问题，如果是核心流程页面就是问题

#### 模式6：异常路径
有没有违反产品设计预期的奇怪路径？
- 例：支付页→首页→再回到支付页
- → 可能是支付过程中被消息/弹窗打断，或者支付失败引导不对

### 第4步：细分维度对比
和漏斗一样，按核心维度拆分看差异：
- 新用户vs老用户：新用户是不是找不到路径绕远路？
- 高价值用户vs低价值用户：高价值用户的路径有什么共同特征？
- 转化用户vs流失用户：转化用户走了什么路径？流失用户在哪离开？
- 不同渠道用户：渠道质量差异在路径上的体现？
- 不同平台：iOS/Android路径差异，是否有平台特有的问题？

### 第5步：归因问题，提出优化方案
针对发现的问题：
1. 如果是入口找不到：调整入口位置、强化视觉引导、增加引导提示
2. 如果是路径绕远：减少不必要步骤，提供快捷路径
3. 如果是信息歧义：优化页面信息架构、文案、表单设计
4. 如果是用户迷路：增加面包屑导航、进度条、明确的下一步指引
5. 如果是主流路径和设计不符：顺着用户真实路径优化，不要硬掰用户走你设计的路

### 第6步：验证优化效果
上线后再做路径分析，看问题是否解决，新的路径是否符合预期。

## 高级玩法

### 1. 反向路径分析
不看从起点往后走，反过来从终点往前看：
- 例：最终购买的用户，他们之前的3步都来自哪里？
- 找到最高效的转化路径，强化这些路径的流量
- 找到转化用户的共同行为特征

### 2. 流失路径专项分析
专门看从关键节点流失的用户，流失前都做了什么？
- 例：提交订单后未支付的用户，之前5步路径是什么？
- 是去领优惠券了？还是去看运费了？还是被其他消息打断了？

### 3. 路径相似度分群
用序列挖掘算法（如PrefixSpan、GSP）找出高频路径模式，把走相似路径的用户聚成一类：
- 搜索型用户
- 分类浏览型用户
- 推荐feed型用户
- 针对性做个性化运营和产品优化

### 4. 路径效率对比
不同路径的转化效率对比：
- 走路径A的用户最终转化率30%，路径B只有5%
- 引导更多用户走高效路径，减少低效路径的干扰

### 5. 页面流转影响力计算
用PageRank思路计算每个页面在整个路径网络中的影响力：
- 哪些页面是核心枢纽节点？
- 哪些页面是流量分发节点？
- 新功能应该放在哪个页面能获得最大曝光？

## 常见坑和避坑指南

| 坑 | 后果 | 避坑 |
|----|------|------|
| 节点粒度太细 | 路径极度分散，每条路径占比都不到1%，看不出规律 | 先粗粒度（页面级）找整体问题，再细粒度定位细节问题 |
| 步长设置太长 | 后面的步骤流量太小都是噪音 | 先看前3-5步，核心问题一般都在前几步 |
| 不做会话划分 | 跨天/跨场景的行为连在一起，路径完全失真 | 合理设置会话超时时间，按会话分析不要按用户全生命周期串 |
| 只看桑基图不看数据 | 被视觉效果误导，忽略小流量但高转化的重要路径 | 桑基图+具体路径占比表格结合看 |
| 把"高频路径"当"高效路径" | 优化了占比高但转化低的路径 | 路径占比×转化率一起看，找高频且高效的黄金路径 |
| 预设路径偏见 | 只看你期望看到的路径，忽略意外发现 | 先无目的地看全量路径，再聚焦 |
| 只看路径不看用户 | 同样的路径不同用户意图完全不同 | 路径分析+用户分群结合，不要孤立看行为 |

## 实战案例：内容社区新用户路径优化

**背景**：内容社区APP新用户次日留存只有20%，远低于老用户60%。

**分析过程**：
1. 起点设为"新用户首次启动"，看前5步路径
2. 桑基图发现：
   - 40%用户：启动→首页feed→上滑看了2个视频→直接退出
   - 25%用户：启动→首页feed→点击搜索→搜了关键词没结果→退出
   - 15%用户：启动→首页feed→点击评论区→看评论→退出
   - 只有10%用户：启动→首页feed→点赞/关注/收藏→留存
3. 反向路径看留存用户：留存用户70%在首次启动前5步有"点击内容看详情/互动"行为
4. 问题诊断：
   - 新用户冷启动feed内容不匹配，大部分用户划两下就走
   - 搜索无结果率太高，搜索用户直接流失
   - 没有引导新用户互动，纯浏览用户留存低
5. 优化方案：
   - 新用户冷启动feed优化，增加兴趣选择步骤
   - 搜索无结果时推荐相关热门内容
   - 首次启动增加互动引导（"双击点赞"提示）
6. 优化后：新用户次留提升到32%，互动率提升80%。
