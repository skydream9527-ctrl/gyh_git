# 漏斗分析（Funnel Analysis）

## 什么是漏斗分析
漏斗分析是用户行为分析的核心方法，用于衡量用户在**多步骤、连续的目标流程**中每一步的转化和流失情况，像漏斗一样从上到下逐步收窄。

典型应用场景：注册流程、下单流程、内容发布流程、激活流程等。

## 适用场景
1. ✅ 核心转化流程优化：找出流失最高的环节
2. ✅ 不同用户群体转化差异对比：新老用户、渠道、版本
3. ✅ 流程改版效果评估：改版前后漏斗对比
4. ✅ 异常问题定位：某天转化率突然下跌，定位哪个环节出问题
5. ✅ 产品价值验证：核心目标流程整体转化率是否达标

不适用场景：
❌ 用户行为没有固定顺序的开放场景（用路径分析）
❌ 单步转化（直接看转化率即可）

## 核心指标

| 指标 | 定义 | 计算方式 |
|------|------|----------|
| 步骤转化率 | 从第N步到第N+1步的转化比例 | 第N+1步人数 / 第N步人数 |
| 整体转化率 | 从第一步到最后一步的总转化比例 | 最后一步人数 / 第一步人数 |
| 流失率 | 该步骤流失的用户比例 | 1 - 步骤转化率 |
| 转化时长 | 从第一步到完成目标的总耗时分布 | 中位数/分位值 |
| 步骤耗时 | 相邻两步之间的停留时长分布 | 中位数/分位值 |

## 标准分析流程

### 第1步：明确漏斗目标和步骤定义
1. **明确业务目标**：注册？下单？发布内容？
2. **定义关键步骤**：
   - 步骤必须是**用户主动行为**，不要把被动页面曝光算进去
   - 步骤顺序是**严格有序**的，不能跳步
   - 步骤数量建议3-7步，太多太碎不好定位
   - 每一步要明确定义触发条件（比如"提交订单"是点击按钮还是支付成功？）

**示例：电商下单漏斗**
```
步骤1：访问商品详情页 → 步骤2：加入购物车 → 步骤3：提交订单 → 步骤4：支付成功
```

### 第2步：确定统计口径
1. **时间窗口**：用户需要在多长时间内完成整个漏斗算转化？（常见：1天/7天/30天，根据业务定）
2. **去重口径**：按用户ID去重，还是按设备ID？还是按会话？
3. **重复行为**：一个用户多次进入第一步，怎么算？（算一次还是多次？建议按用户首次进入统计）
4. **归因规则**：用户中途退出又回来，从哪一步开始算？

### 第3步：计算并可视化漏斗
输出标准漏斗表格：

| 步骤 | 人数 | 步骤转化率 | 整体转化率 | 流失率 | 平均停留时长 |
|------|------|------------|------------|--------|--------------|
| 1. 访问商品详情 | 100,000 | - | 100% | - | - |
| 2. 加入购物车 | 35,000 | 35% | 35% | 65% | 2分15秒 |
| 3. 提交订单 | 15,000 | 42.9% | 15% | 57.1% | 3分40秒 |
| 4. 支付成功 | 10,500 | 70% | 10.5% | 30% | 1分20秒 |

**可视化**：用条形漏斗图展示每一步宽度，标注转化率。

### 第4步：定位问题环节
判断哪些步骤需要优化：
1. **绝对转化率低**：和行业基准、历史数据比，哪个步骤异常低
2. **耗时异常长**：某一步停留时间远高于其他步骤，说明有困惑
3. **流失人数最多**：绝对流失人数最大的环节，优化ROI最高（不一定是转化率最低的）
   - 比如：第一步到第二步流失6.5万人，第三步到第四步流失0.45万人，优先优化第一步

### 第5步：细分维度对比
按不同维度拆分，找到转化差的群体：
| 拆分维度 | 看什么 |
|----------|--------|
| 新老用户 | 是不是新用户转化特别差？新手引导问题？ |
| 渠道 | 某个渠道过来的用户是不是流量质量差？ |
| 平台 | iOS/Android/小程序转化差异？ |
| 版本 | 新版本上线后哪个步骤转化下跌？ |
| 用户分群 | RFM高价值用户vs普通用户？ |
| 商品品类 | 不同品类商品下单转化差异？ |

### 第6步：诊断流失原因
找到问题环节后，进一步分析：
1. 流失用户去了哪里？（路径分析）
2. 流失用户有什么共同特征？（用户分群）
3. 定性访谈/可用性测试，观察用户卡点
4. 看该步骤的错误日志、页面退出率、按钮点击率

### 第7步：给出优化建议并验证
针对问题提出假设，做A/B测试验证，验证后再看漏斗是否提升。

## 高级玩法

### 1. 转化时长分析
- 不仅看多少人转化，还要看**多久转化**
- 看P50/P90/P99分位的转化时长
- 如果大量用户卡在某一步很久才转化，说明这一步有摩擦
- 如果超过24小时才转化，考虑召回策略

### 2. 多漏斗对比
- 不同路径的漏斗对比：比如"直接购买"vs"加入购物车再购买"哪个转化高
- 不同版本的漏斗对比：改版前后
- 不同运营活动的漏斗对比：活动A vs 活动B

### 3. 漏斗下钻
- 对某个流失高的步骤，再拆成更细的子漏斗
- 比如"支付成功"可以拆成：选择支付方式 → 输入密码/刷脸 → 支付回调 → 成功页

### 4. 智能漏斗/无序漏斗
- 用户行为不是严格顺序时，可以用"有序"vs"无序"漏斗对比
- 看有多少用户跳步了，说明流程设计不合理

## 常见坑和避坑指南

| 坑 | 后果 | 避坑 |
|----|------|------|
| 步骤把"曝光"当"行为" | 虚高转化率，定位不准 | 只算用户主动点击/操作的行为 |
| 步骤太多太细 | 每个步骤流失都不明显，找不到重点 | 合并逻辑相邻的步骤，核心环节单独拆 |
| 时间窗口不合理 | 要么漏算转化，要么把无关行为算进来 | 根据业务场景定：注册用1天，电商用7天，高价商品用30天 |
| 只看转化率不看绝对流失人数 | 优化了转化率最低但流失人少的环节，ROI低 | 转化率 × 流量规模一起看，优先优化流失人数多的 |
| 不做细分就下结论 | 整体没问题但某类用户问题严重，被掩盖 | 必须按核心维度拆分看 |
| 不看转化时长 | 转化率看起来不错，但用户花了很久才完成，体验差 | 时长和转化率一起看 |
| 把相关性当因果 | 某步流失高就说这个步骤有问题，其实可能是流量质量问题 | 多维度对比、控制变量、AB验证 |

## 实战案例：电商下单漏斗优化

**背景**：电商APP整体下单转化率只有2%，远低于行业5%的基准。

**分析过程**：
1. 拆解漏斗：商品详情→加购→提交订单→支付成功
2. 发现问题：商品详情→加购转化率只有8%，行业平均20%，这一步流失了92%的用户，是最大瓶颈
3. 细分维度：发现新用户加购率只有3%，老用户15%，问题集中在新用户
4. 进一步下钻：新用户在商品详情页平均停留12秒就退出，按钮点击率只有5%
5. 定性研究：新用户不知道"加入购物车"按钮在哪，页面信息太多找不到重点
6. 优化方案：新用户进入详情页，加购按钮强化高亮，精简首屏信息
7. AB测试：新用户加购率从3%提升到10%，整体下单转化率提升到3.5%
