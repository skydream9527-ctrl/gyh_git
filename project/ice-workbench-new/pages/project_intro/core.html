Core Mechanics

真正的核心，是一套可控的 Agent 工程系统。

ICE 不是把聊天框套在数据工具上，而是围绕 Agent 执行、工具调用、上下文、记忆、权限和部署可运维性建立闭环。

Agent Kernel

统一 Kernel 将 LLM 回复、工具调用、计划审批、子 Agent、后台任务和压缩策略纳入同一条运行链路。

Kernel 01

有边界的 ReAct 循环

工具调用轮次有默认上限与硬上限，避免无限循环；主对话、子 Agent、后台任务走统一协议。

  • 默认工具调用轮次可由后台配置
  • 超时、取消和二次输入有明确保护
  • 工具结果回灌 LLM 并保留审计轨迹
Kernel 02

Plan Mode 与人工介入

复杂任务先进入只读规划，Agent 输出方案后等待用户审核；需要关键输入时可请求人工补充。

  • 降低误写文件和误发消息风险
  • 适合需求不清晰或影响面较大的任务
  • 计划、待办和执行进度可联动
Kernel 03

子 Agent 与并行工具

通用 Agent 可派单给专业 Agent，数据分析 Agent 也可派发波动、知识库等任务；并行机制用于缩短多命题分析耗时。

  • 按 task_id / conv_id 控制运行中状态
  • spawn_targets 描述可委派对象
  • 并行工具和后台任务通过特性开关灰度

文件优先架构

G3 的关键价值是“可迁移、可热更新、可自托管”：核心数据以文件目录为真源，SQLite 作为索引和查询加速。

G3 File-first

数据不是黑盒数据库资产

用户、任务、文件、Agent、Skill、运行缓存都映射到清晰目录。部署升级时明确保护 users、tasks、files、agents、skills、.cache 与 .env。

任务可携带一个任务目录就包含对话、文件、产物、工具调用和日志。
写入可审计对话消息、工具调用、审计日志采用 append-heavy JSONL，避免大 JSON 锁竞争。
热更新友好打包和部署脚本默认不覆盖运行时用户数据和密钥。
users/

用户资料、偏好和权限相关文件。

tasks/

任务上下文、会话、产物、工具日志。

files/

团队公共文件、指南、规范、方法论。

agents/

Agent 配置、Prompt、知识和默认 Skills。

skills/

Skill 元数据、说明、脚本和外部工具能力。

SQLite

列表、检索和索引缓存；内容仍以文件为准。

模型、工具与错误恢复

项目把“模型不可控”和“工具可能失败”作为工程事实处理，而不是隐藏在 Prompt 里。

LLM Gateway

多协议统一事件流

按模型 ID 前缀路由到不同协议适配器，对上游统一输出 text、tool_use_delta、message_done 等内部事件。

Tool Contract

工具失败不崩服务

Kyuubi、飞书、Mify 等外部 CLI 缺失时返回 NOT_CONFIGURED / NOT_INSTALLED，不阻塞系统启动。

Error Envelope

错误结构化归一

服务端用统一错误包，工具结果标记 success / error_code；后续可扩展 recoverable 与 suggested_action。

超时控制

工具执行设置 timeout floor 与硬上限，子 Agent 和长任务不会无限占用资源。

取消机制

WebSocket 断开不直接取消运行，显式 abort 才会触发 cancel event。

参数校验

优化方向中规划工具调用前 JSON Schema 校验，减少 LLM 幻觉参数。

链路追踪

路线图中规划 tool trace_id、工具成功率和错误类型统计，用于优化 Prompt 和工具设计。

记忆与知识进化

ICE 将“这次分析得出的经验”变成下一次 Agent 能用的上下文，而不是只停留在聊天记录里。

Capture沉淀经验用户对优秀回复点击 💡，补充标题、摘要和标签。
Review管理员审核经验卡片进入后台审核队列，避免低质量知识污染。
Inject注入 Agent批准后的经验卡进入 Agent 知识库，下次自动检索。
Rank生命周期BM25、recency、confidence、使用次数共同影响召回。
Evolve团队记忆个人记忆可晋升为团队知识，形成组织级复用资产。

安全、权限与部署

项目适合内部数据工作台场景，围绕登录、权限、沙箱、密钥和自托管做了明确约束。

Auth

双认证链路

支持 JWT 与 Aegis 代理头，Feishu 首次登录可自动建号或严格白名单。

Roles

三层角色

super_admin、admin、user 分层管理，super_admin 要求 Feishu OAuth。

Sandbox

Python 隔离

分析代码运行在独立 venv，限制 CPU、内存、文件大小、网络和敏感环境变量。

Deploy

自托管友好

支持 deploy.sh、make pack、Docker Compose；.env 不随包扩散，密钥部署后补齐。
