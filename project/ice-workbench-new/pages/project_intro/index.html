项目在解决什么问题？

数据同学的日常工作往往横跨提数、验证、绘图、写报告、发飞书、沉淀经验。 ICE 的目标是把这些动作收敛为可复用、可审计、可持续进化的 Agent 工作流。

Before

传统链路碎片化

需求在聊天里，SQL 在工具里，图表在 notebook 里，报告在飞书里，经验沉淀靠人脑记忆；一旦换人或复盘，路径难以复现。

重复提数同类查询和口径解释反复发生，缺少模板化入口。
决策不透明工具失败、模型输出、文件产物没有统一链路追踪。
经验难沉淀优秀分析结论无法自然进入下一次 Agent 对话。
After
01选择范式从 AB、数据分析、灰度或开放任务进入。
02配置 Agent绑定 Skills、文件、Prompt 和可见性。
03执行分析自然语言驱动 SQL、Python、飞书与文件产物。
04协作审核公共任务、模板、经验卡进入团队资产。
05持续进化记忆、反馈、计划和工具统计反哺 Agent。
