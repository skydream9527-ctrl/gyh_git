# Moxt 前端风格参考

> 来源：moxt.ai/w/...  + 用户提供的两张工作台截图（Spaces 仪表盘 / 文档详情）
> 用途：给 design_v5 一个 **冷调 Notion 风** 的对照样本（与现有 v5 的「Anthropic 暖纸」是两套完全不同的语言，本文不是替换，是对照）

---

## 一、整体调性（一句话）

**冷白纸面 + 极薄边框 + 大量留白 + 单一品牌绿点缀 + 圆润胶囊**——Notion / Linear / Cron 一脉的「现代 SaaS 文档工具」语言。视觉上几乎没有阴影和质感，靠**线 + 留白 + 字号差**组织信息，把所有装饰让给内容。

和 ICE v5 现有的 Claude/Anthropic 暖纸风（铜橙 + Fraunces 衬线 + 印刷感）是**冷暖对立**的两端。

---

## 二、颜色（截图取色，估值）

### 浅色（默认，且基本只有浅色）

```
/* 中性面 —— 几乎全是冷白 / 极浅灰 */
--bg:           #ffffff;   /* 主面板背景（文档区、聊天区） */
--surface-2:    #fafafa;   /* 左侧导航/二级面板 */
--surface-3:    #f4f4f5;   /* hover / 选中态填充 */

/* 边框 —— 极薄，几乎只是「有」而已 */
--border:       #ececec;   /* 卡片、输入框、表格 */
--border-soft:  #f1f1f1;   /* 分隔线 */

/* 文本 */
--text:         #18181b;   /* 标题/正文 */
--text-dim:     #52525b;   /* 二级文字 */
--text-muted:   #a1a1aa;   /* 占位符、时间戳 */

/* 品牌强调（Moxt 那颗星）—— 薄荷/翡翠绿，单点出现 */
--brand:        #34d399;   /* 主绿 (估) */
--brand-deep:   #10b981;   /* hover/按下 */
--brand-soft:   #ecfdf5;   /* 极淡背景 */

/* 头像/装饰用色（出现频率低，仅在 avatar） */
--avatar-coral: #f87171;   /* 「G」头像底色那种珊瑚红 */

/* 信息提示色（"Tips: Save..." 那段） */
--tip-amber:    #d97706;   /* 琥珀文字色 */
```

特点：
- **没有真正的灰阶层级感**，三档背景之间差 4–8 个亮度，靠像素差不是色相差。
- 品牌绿**只在 logo / 主 CTA / 特殊状态**出现，整页 95% 是黑白灰。
- 没有「成功/警告/错误」彩色面板系统——错误也用边框红文字，不铺背景。

### 深色

截图里没出现，按 Notion / Linear 类工具的常见做法推断会是 `#0a0a0a` 主面 + `#18181b` 二级 + `#27272a` hover，文字反白，品牌绿不变（或微微加亮到 `#4ade80`）。

## 三、字体

```
font-family: 'Inter', -apple-system, system-ui, 'PingFang SC', sans-serif;
```

- **统一 Inter / 系统无衬线**，没有衬线副字体，没有正文装饰字。
- 中文回退 PingFang SC，**字号给得偏小**：正文 14–15px，行高 1.55 左右。
- 标题不用大字号炫技，靠**字重 + 字距** (`letter-spacing: -0.01em`) 和留白凸显。
- 数字不刻意 tabular-nums，标题不刻意衬线——这一点和 v5 的 Fraunces 大标题是完全相反的取向。

字号阶梯（估）：
| 角色 | px | weight |
|---|---|---|
| 文档标题 H1 | 30–32 | 600 |
| 章节标题 H2 | 22–24 | 600 |
| 小节标题 H3 | 16–17 | 600 |
| 正文 | 15 | 400 |
| 表格/侧栏 | 13–14 | 400/500 |
| 时间戳/辅助 | 12 | 400 |

## 四、形态语言（圆角/阴影/边框）

- **圆角**：按钮 / 卡片 / 输入框统一 **10–12px**，胶囊按钮 **999px**（Selected 标签、模型选择器 `Pro` 都是胶囊）。比 v5 的 4–6px 圆润很多。
- **阴影**：整页几乎不用 box-shadow。聊天面板悬浮卡片至多 `0 1px 2px rgba(0,0,0,.04)`。**靠边框分层，不靠阴影分层**。
- **边框**：1px solid #ececec，统一一档，不区分 soft/strong。
- **分隔**：列表项之间用 1px 底线，不用间距。

## 五、布局骨架

三栏经典 SaaS 文档工具布局：

```
┌──┬──────────────┬──────────────────────────┬────────────┐
│  │              │                          │            │
│ 图│   左侧树     │       主内容             │  AI Chat   │
│ 标│  (Spaces /   │   (文档 / 仪表盘)         │   (副驾)   │
│ 栏│   System /   │                          │  可折叠     │
│  │   Team)      │                          │            │
└──┴──────────────┴──────────────────────────┴────────────┘
 56px   240–280px           1fr                  340–380px
```

- 最左是 **icon rail**（Spaces / Settings 两枚图标，垂直堆叠），56px 宽，背景同主面，仅靠图标颜色区分。
- 第二栏是树形导航：Personal Space / Team Space / GitHub 三段，每段一个浅灰 section header，子项缩进 12px，hover 是 `#f4f4f5` 圆角填充。
- 主内容区**最大宽度不限**，靠左右内边距控制阅读区（约 64–96px）。
- 右侧 AI 聊天**永远在线**（不像 ICE 现在是切换 Tab）——头部带模型选择 dropdown + 「+」新会话 + 折叠按钮，底部输入框带 attach / mic 图标 + 圆形主 CTA。

## 六、关键组件细节

### 1. Quick Start 卡片（仪表盘空状态）

```
┌────────────────┐  ┌────────────────┐  ┌────────────────┐
│ ＋  New File   │  │ 🗂  New Folder │  │ ⤓  Import      │
└────────────────┘  └────────────────┘  └────────────────┘
```
- 1px 边框 + 12px 圆角，无填充无阴影。
- 图标在左（24x24，圆角方块灰底 `#f4f4f5`），文字 15px medium。
- hover 时仅边框变深一档（`#d4d4d8`），不变背景。

### 2. Recently Viewed 表格

```
Name                              Contributors    Viewed
──────────────────────────────────────────────────────────
□ AI 主题抖音运营方案.md           [G]             1 minute ago
   work/抖音
□ AGENTS.md                        ─               28 days ago
   7amzjkds
```
- 列头小号大写、灰色 `#a1a1aa`、`letter-spacing: 0.05em`。
- 主名称 + 灰色路径双行：路径 12px `#a1a1aa`。
- 行间用 1px 底线分隔，hover 整行背景变 `#fafafa`。

### 3. AI Chat 面板

- 顶部：星形 logo + 会话标题（可点击下拉切换） + 工具图标组 (折叠 / 新建 / 收起)。
- **空状态有大插画**（绿色卡通星星表情）+ 一句 Tips（琥珀色文字 + 琥珀色短下划线）。
- 输入框是单一圆角矩形（12px），`Ask me anything` 占位符浅灰。
- **Selected 引用块**：胶囊形（999px 圆角）灰底 `#f4f4f5`，前缀 `Selected:`，尾部 `×`。
- 底部行：左下 attach 📎 + 模型选择 `Pro v` 胶囊；右下 mic + 圆形发送按钮（品牌绿 `#34d399`，禁用时灰）。

### 4. 文档正文

- **双语对照排版**（中英混排）是这个产品独有的内容组织方式：英文行 + 橙色中文翻译行紧贴，不是 Moxt 通用样式，而是这一篇文档的内容形态。
- 正文大段留白，h2 上下各 32px，h3 上下各 24px。
- 表格：极简 1px 边框 + 内 padding 12–16px，无斑马纹，列宽随内容。
- 列表：圆点 `•` 而非方块，缩进 24px。

### 5. 头像

- 28–32px 圆形纯色底（#f87171 那种珊瑚红 / 品牌绿 / 蓝紫等系统轮换），白色单字母 600 字重。
- 没有阴影没有渐变。

### 6. 图标

- **Lucide / Heroicons outline 风**：1.5px stroke，圆角端点，无填充。
- 颜色继承当前文字色（`currentColor`），hover 不变颜色只变背景。

## 七、与 ICE v5 现行风格的对照

| 维度 | ICE v5（Claude 暖纸） | Moxt（冷白 Notion） |
|---|---|---|
| 主背景 | `#faf4ed` 暖米白 | `#ffffff` 冷白 |
| 强调色 | 铜橙 `#cc785c` | 薄荷绿 `#34d399` |
| 字体 | Fraunces 衬线大标题 + Inter | 全 Inter，无衬线 |
| 圆角 | 4–6px，硬朗印刷感 | 10–12px，圆润 |
| 阴影 | 暖色多层 box-shadow | 几乎无阴影 |
| 装饰 | 范式 5 色 + 渐变图标 | 单品牌色 + 黑白灰 |
| 调性 | 文档纸张 / 印刷品 | SaaS 工具 / 极简 |

## 八、可借鉴的局部模式（不一定要换皮）

如果未来想把 Moxt 的某些**好的局部模式**引入 v5，但保留 v5 的暖纸总调性，最值得借鉴的是：

1. **Quick Start 卡片网格**——空状态/首次使用时的引导，比纯文字 CTA 更友好。
2. **Recently Viewed 行式表格 + 双行（名称 / 路径）**——比单行截断信息密度更好。
3. **AI Chat 永远在线的右侧栏**——比当前 Tab 切换的会话/详情少一次心智切换。注意：v3 后端是「任务为原子单位」，所以右栏内 AI 不能跨任务保留上下文，引入时要有任务边界提示。
4. **Selected 引用胶囊**——把当前选中的文件/段落以胶囊形式挂在输入框上方，比 v5 现有的 attach 列表清晰。
5. **icon rail（最左 56px 列）**——给 Admin / Workspace / 设置三个一级模块独立入口，比 TopNav 横向切换更稳。

> ⚠️ 不建议借鉴的：极薄边框 + 无阴影 + 全冷白 —— 这套语言和 v5 暖纸 + Fraunces 衬线放一起会**两边都拉跨**，要么整套换冷白要么坚持暖纸，不要混。
