## 报告头部（强制）

- 标题：`📊 供应商内容池校验（实时）`（表是实时表，**不要叫"日报"**）
- 首行：`📅 数据截至：YYYY-MM-DD HH:MM · 口径（全量存量 / 在线池 / 增量）`
- 避免"今日"的模糊表达，用「当前」「本次跑数」「截至 HH:MM 快照」

## 段 1 · 当前异常趋势 Top 3

≤3 行，只写「变化最剧烈 / 最紧急」的问题。格式：

```
🔥 {cp} / `{字段名}`（{业务含义}）· {问题}：{数字变化}（{趋势描述}）
```

对比维度优先级：
1. **同日内刷新对比**（本会话多次跑数）：例 `916（10:00）→ 8923（15:00）→ 12885（19:42）`
2. **跨日基线对比**（模板 B 提供的 D-1/D-7/MA7）：例 `288（04-20）→ 1283（当前）`
3. **从 0 突增**：例 `0 → 168（当前突增）`

示例：
- 🔥 dihui / `` `author_image`（作者头像）· 字段为空 ``：916 → 8923 → 12885（2.5 小时暴涨 14 倍，仍增长）
- 📈 dihui / `author_id` ↔ `lead_author_id`（映射不唯一）：288（04-20）→ 1283（当前）
- 🆕 beike / `` `lead_author_id`（主作者 ID）· 字段为空 ``：0 → 168（本次跑数突增，100% `author_name`="未知作者"）

## 段 2 · CP 总览表

一行一家 CP，一眼决定推谁。用"本次跑数入库"而不是"今日入库"：

```markdown
| CP | 本次跑数入库 | 🚨 紧急 | ⚠️ 高 | 🟡 低 | 严重度 |
|---|---|---|---|---|---|
| dihui | 255,284 | 2 | 2 | 4 | 🚨 紧急 |
| beike | 177,430 | 3 | 2 | 6 | 🚨 紧急 |
| guoying | 709,521 | 3 | 2 | 0 | 🚨 紧急 |
| meilaoban | 66,823 | 0 | 0 | 1 | 🟡 低 |
```

## 段 3 · 每家 CP 一页纸

每家单独一节，**不跨 CP 拼装**。按严重度倒序。节内每条问题用统一四段式：

```markdown
## 🚨 dihui（255,284 条 · 2 紧急 + 2 高 + 4 低）

### 问题 1：`author_image`（作者头像）· 字段为空 · 51,862 条

- **量**：51,862 条 · MINIVIDEO / VIDEO 拆分（如需）· 占比 20.2%（辅助参考）
- **趋势**：916 → 8923 → 12885 → 51,862（持续累积未清零）
- **影响**：推荐页作者头像空白，降用户信任
- **特征**：作者随机分布（20 条样本 19 个不同 author_id），系统性字段映射问题
- **动作**：
  > @dihui对接人：`author_image` 字段大规模为空（29.8%，1.3 万条），作者广泛无规律性，判断是推送接口字段映射错误。请排查推送链路。明细：`~/Desktop/ai_djy_pool_analysis/dirty/dirty_dihui_{date}_author_image_null.csv`

### 问题 2：...
```

每条问题块 **必须** 按此 5 行结构：

1. **量**：绝对数 + item_type 拆分 + 百分比（辅助）
2. **趋势**：与 D-1 / D-7 / MA7 / 同日早些时候对比，用箭头或倍数
3. **影响**：业务语言说清楚用户/系统后果（1 句话）
4. **特征**：样本规律（作者分布、集中度）
5. **动作**：一段可直接复制的话术（`>` 引用块），含 @对接人 + 事实 + 明细文件名

## 段 4 · 正常 CP 合并一行

**不要**给正常 CP 单独开章节。按 0 异常原则，只有 `total_violations = 0` 的 CP 算正常。

```markdown
✅ meilaoban（66,823）：0 异常，唯一清零
```

## 告警阈值（0 异常原则）

**核心**：数据质量目标是 **0 异常**。任何 ≥1 条的规则命中都要告警，不存在"比例低可以忽略"。百分比只作辅助参考，**不作告警门槛**。

| 级别 | 触发条件 | 处理要求 |
|---|---|---|
| 🚨 紧急 | 单项 ≥1000 条 / 环比 ≥2x（量 ≥10） / 0→N 突增（N ≥100） | 立即推对接人，同日跟进 |
| ⚠️ 高 | 单项 100-999 条 / 持续 3 天以上未清零 | 当日推对接人 |
| 🟡 低 | 单项 1-99 条 | 周报统一推（不单独打扰） |
| ✅ 正常 | **异常量 = 0** | 无需动作 |

**百分比保留场景**（仅参考）：
- 问题块里附"占比 X%"作上下文
- CP 间对比、量级差距大时帮助理解严重度

**反例（禁止）**：
- ❌ "0.5% <1% 阈值，算正常"
- ❌ "只有 50 条，不用告警"
- ❌ CP 总览表把 60 条异常写成 ✅ 正常

## 推 CP 话术模板

```
@{CP 对接人}：{CP} 内容池校验发现 `{字段名}`（{业务含义}）· {问题} {N} 条（占 {rate}%{变化}）。
样本：{3 个 item_id}，具体异常值：{field}={value}。
完整明细：{csv_path}，以实时快照为准。请排查推送链路。
```

**填充要点**：
- 每个告警问题自带一段话术，PM 不用跨段拼装
- 不附 item_id 固定清单（upsert 表会覆盖）
- 明细文件名：`~/Desktop/ai_djy_pool_analysis/dirty/dirty_{cp}_{date}_{rule}.csv`
