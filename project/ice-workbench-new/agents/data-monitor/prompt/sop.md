## 监控执行 SOP

每次被调度唤起时，按以下步骤执行：

### Step 1 — 解析监控配置

从 prompt 中提取：
- 监控指标名
- SQL 查询语句
- 阈值条件（绝对值阈值 / 环比变化率阈值 / 同比变化率阈值）
- 告警接收人 / 告警渠道
- 时间窗口

### Step 2 — 执行 SQL 获取数据

调用 `kyuubi_query` 执行监控 SQL：
- 如果 SQL 失败，记录错误并跳过本次检测（不告警）
- 如果结果为空，按"无数据"处理（可能本身是异常，视配置决定）

### Step 3 — 对比阈值

将查询结果与阈值条件对比：

| 条件类型 | 判断逻辑 |
|---|---|
| 绝对值 | 当前值 > upper_bound 或 < lower_bound |
| 环比变化率 | \|当前 - 上期\| / 上期 > threshold_pct |
| 同比变化率 | \|当前 - 去年同期\| / 去年同期 > threshold_pct |
| 连续 N 期下跌 | 连续 N 个采样点单调递减 |

### Step 4 — 告警或静默

**触发告警**时：
1. 用 `write_file` 写告警详情到 `alerts/{指标}_{日期}.md`
2. 如果配置了飞书通道，用 `feishu_send_message` 发送告警消息
3. 更新 `todo_write` 标记本次检测结果

**未触发告警**时：
1. 只更新 `todo_write` 记录"本次检测正常"
2. 不发送任何消息

### Step 5 — 输出摘要

最终输出格式：
```
📊 监控检测完成
- 指标: {metric_name}
- 状态: ✅ 正常 / 🔔 告警
- 当前值: {value}
- 检测时间: {now}
```
