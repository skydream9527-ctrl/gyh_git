# 协作偏好（ICE-DATA-WORK）

本文件记录用户在本项目中的长期协作偏好，后续会话自动加载。

## 文档编辑无需批准

- 对文档类文件的写入/更新**无需向用户请求批准**，直接写即可，以提高效率。
- 适用范围：`PROGRESS.md`、`DEV_PLAN.md`、`README.md`、`docs/**`、`.kiro/**`（steering/spec 等）及其他 Markdown/说明类文件。
- 代码、配置、删除等非文档操作仍按常规判断（高风险动作仍需确认）。

## 编辑方式：优先整文件单变更块

- 无论文档还是**代码**，当一次要对同一文件做多处关联改动时，**优先用整文件写入（一个变更块）**，不要拆成多个分散的 str_replace hunk。
- 原因：分散 hunk 在审阅时容易被"部分还原"（出现过多次：import 留下但用到它的逻辑被退回，造成未使用导入/半截改动）。
- 单文件单变更块便于一次性 accept/reject，避免半套状态。

## 进度文档维护约定

- 每完成一个任务或里程碑，**及时更新** `PROGRESS.md`（状态、明细表、变更日志）与 `DEV_PLAN.md`（任务完成标记），无需等待用户提示。
- `PROGRESS.md` 与 `DEV_PLAN.md` 使用同一套任务 ID（`M{n}-T{nn}`）。
- 更新方式：改对应任务状态 → 进度总表 → 变更日志加一行（日期 + 任务 ID + 一句结论）。

## 语言

- 与用户对话默认使用**中文**。

## 验证习惯

- 后端改动后跑 `pytest -q`；前端改动后跑 `tsc --noEmit`（必要时 `vite build`）。命令退出 0 不等于通过，需核对实际结果。
- 临时验证脚本/构建产物用完即清理。
