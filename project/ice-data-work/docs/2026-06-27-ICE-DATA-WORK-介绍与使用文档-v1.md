## 一、一分钟了解

**ICE-DATA-WORK 是一个让"人 + AI 分身 + 工具 Agent"在同一块任务看板上协作的团队数据工作台。**

你不直接面对一堆 AI 工具，而是面对**你自己的数字分身（Wisdom Twin）**。分身长期学习你的目标、偏好、口径和工作方式，替你去协调各类专业 Agent（数据分析、AB 实验、波动归因、编码等），把过程、结论、经验都沉淀回来，并在所有高风险动作上交给你确认。

> 一句话：**User → Twin → Agents**。你对分身说话，分身去指挥 Agent 干活，重要的事回来问你。

**解决什么问题**

| 痛点 | ICE-DATA-WORK 的解法 |
|---|---|
| AI 工具/Agent 太多、各用各的 | 统一看板 + 分身代理调度，Agent 像同事一样被指派 |
| 每次都要重新交代背景、口径、文件 | 四层空间 + 记忆模型，分身长期记得你和团队的上下文 |
| 经验、报告散落各处难复用 | 任务产出可沉淀为个人/项目/团队资产 |
| Agent 不会某个业务就卡住 | 你把代码喂给它跑通，沉淀成 Skill，Agent 当场学会（自演进） |
| 自动化越权、追责难 | 分级权限 + 高风险必确认 + 全程审计 + 一键暂停 |

**适合谁**：数据分析师、数据产品、增长/运营、研究型创作者，以及任何重度使用 AI 工具的团队成员。首期面向小米内部（飞书登录、Kyuubi 查询、米盾鉴权）。

## 二、核心概念（词汇表）

| 概念 | 一句话 |
|---|---|
| **User** | 你本人，拥有最终确认权 |
| **Wisdom Twin（数字分身）** | 你私有的一等 AI 分身，长期懂你、代你协调 Agent、请你确认；能感知页面并被你指令驱动导航 |
| **Tool Agent（工具 Agent）** | 团队共享的专业执行体：数据分析、AB 实验、波动归因、编码等 |
| **Task / Mission（任务）** | 看板上的工作项，可指派给人 / 分身 / Agent |
| **Workspace（工作台）** | 进入任务后的现场，分"数据分析"与"通用"两类；核心是产出物 + 多方对话 |
| **Team / Project（团队/项目）** | 共享单元；每个任务都属于某个项目，没挂的进你的"个人项目" |
| **Skill（技能）** | 可复用、可执行的能力（SQL/Python + 入参 + 版本） |
| **Memory（记忆）** | 沉淀的偏好、口径、经验 |
| **Approval（确认）** | 高风险动作的人工确认卡 |
| **Runtime（运行时）** | 执行环境；v1 在服务端，二期支持本机 daemon 驱动 Claude Code/Cursor |

**四层空间**（你的东西存在哪、谁能看，详见第六节）：
任务空间（一个任务自己的） · Agent 空间（每个 Agent 的） · 用户空间（你和你的分身的） · 团队/项目空间（共享的）。

## 三、平台能做什么（核心能力）

1. **团队任务看板**：按"待办 / 进行中 / 待确认 / 已完成"分列；Agent 和人一样是一等的负责人（assignee）。
2. **多 Agent 协作工作台**：一个任务里，你、你的分身、工具 Agent 同台对话；工具调用过程、结论、图表都在现场可见。
3. **一等数字分身**：右侧常驻，维护上下文、给建议、发起确认；**能感知你切到哪个页面，也能听你指挥切页面**（对它说"带我去看板"即可）。
4. **四层空间 + 记忆隔离与复用**：分身和 Agent 的记忆既能跨任务复用，又严格隔离——你的私有经验默认不外泄，显式"贡献给团队"才共享。
5. **Skill / Agent 自演进**：Agent 不会某业务时，你把代码给它，它沙盒里跑通，再蒸馏成 Skill 绑进 Agent，**下次自动会用**。
6. **权限分级与治理**：平台角色 × 团队/项目成员 × 分身等级（L0–L5）三条轴；写文件、发消息、付费、固化记忆、跨空间读取等高风险动作**必须你确认**；全程审计；一键暂停。
7. **团队/项目共享空间**：共享文件、知识、产物、Agent 经验，按团队或项目隔离。
8. **二期能力**：本机 Runtime（驱动你电脑上的 Claude Code/Cursor）、**邀请同事的分身进同一任务协作**（需对方管理员授权）、Squads（Agent 小队）、Autopilots（定时自动任务）。

## 四、上手使用（端到端）

### 4.1 登录与创建分身
飞书一键登录（或账号密码）。首次登录会引导你：给分身起名、设定工作风格与边界、选初始权限等级（默认 L1）、加入团队/项目（没有就进"个人项目"）。

### 4.2 进入看板、认领或新建任务
登录后默认进 **Workbench**（看运行中的任务、待你确认的事、最近任务、核心产物）。去 **任务看板** 可看全量任务并指派；点 **新建任务**，用自然语言描述目标，分身会给出推荐路线（用哪些 Agent、权限、成本预估），你确认后创建。

### 4.3 在工作台与 Agent + 分身协作
进入任务即进 **Workspace**：左边是产出物（报告/文档/图表），右边是多方对话。你提问 → 分身规划 → 工具 Agent 执行（如 SQL 查询）→ 结果回填报告。高风险动作会以确认卡出现，等你点确认。

### 4.4 让分身帮你导航（D-12）
右侧分身知道你当前在哪个页面。你也可以直接对它说"带我去看板 / 打开数据分析工作台 / 回 Workbench"，它会帮你切过去；或点分身对话框下方的快捷指令。

### 4.5 邀请工具 Agent / 邀请同事的分身（二期）
工作台顶部「+ 邀请」：
- **工具 Agent / 我的分身**：直接加入，无需授权。
- **同事的数字分身（二期）**：填授权范围（限本任务、只读、有效期、可撤销），**发给对方分身的管理员审批**，通过后多方同台协作。

### 4.6 贡献代码、沉淀 Skill、让 Agent 学会（D-13）
当 Agent 缺某业务能力：在对话里把 SQL/代码给它 → 它在沙盒跑通 → 分身提议「贡献为 Skill」→ 你填名称、确认入参、选范围（**个人草稿**立即可用 / **贡献给团队**需沙盒测试 + 审核），并勾选"绑定到该 Agent"。之后同类问题，Agent 直接调用，无需再贴代码。

### 4.7 沉淀记忆并选择范围
任务里产生的好经验/口径，分身会做成"记忆候选"，你确认时选范围：**个人偏好 / 项目 / 团队 / 贡献给某 Agent 的团队经验**。默认进个人，团队级需审核。

### 4.8 审批高风险动作、一键暂停
所有高风险动作进 **审批与审计**：你可确认/修改/拒绝；可查权限日志与工具调用记录；担心失控时**一键暂停**全部执行。

### 4.9 管理我的分身
**数字分身管理** 里看/改：画像与边界、记忆文件、我对各 Agent 的私有经验、Token 与成本、权限等级（L0–L5）。可随时暂停你的分身。

## 六、你的数据存在哪、谁能看（四层空间）

| 空间 | 放什么 | 谁能看 |
|---|---|---|
| **任务空间** | 本任务的对话、文件、过程记忆 | 仅本任务参与者，不跨任务 |
| **Agent 空间** | Agent 的手册、技能、使用经验 | 定义/技能团队共享；**你的使用经验默认只属于你** |
| **用户空间** | 你的分身画像、长期记忆、偏好 | 仅你和你的分身（他人看不到） |
| **团队/项目空间** | 共享文件、知识、产物、团队经验 | 团队/项目成员，按边界隔离 |

**隔离与复用的核心一句话**：默认"个人优先、互不泄漏"；**只有你显式"贡献给团队"，经验/技能才会共享**（团队级还要过审核）。这条规则对记忆和 Skill 完全一致。

## 七、权限与安全（你需要知道的边界）

- **三条权限轴**：平台角色（super_admin/admin/user）× 团队/项目成员角色 × 你的分身等级（L0–L5）。实际能做什么取三者交集。
- **分身等级 L0–L5**：从"只观察"到"在边界内自主"，逐级放权；高风险始终需你确认。
- **高风险动作清单**（必确认）：写本机文件、跑命令、发消息、付费调用、固化记忆、跨空间读取。
- **沙盒**：用户贡献的代码一律在受限沙盒执行（无网络、限资源、无凭证）；SQL 只读 + 审计。
- **审计 + 一键暂停**：每次工具调用都有记录；随时可暂停全部执行。
- **数据可移植**：采用"文件优先"存储，数据以文件为准，换机器/重启不丢。

## 八、原型怎么看（design/）

直接双击 `design/index.html`（纯静态、无需服务器），从封面进入任意页面；或在 `design/` 下 `python3 -m http.server` 起本地预览。

值得点的几处：
- 左侧导航切页 + 顶部项目切换器；右侧分身常驻。
- 右侧分身对话框输入"带我去看板"等可真实跳转（D-12）；下方有快捷指令。
- 工作台「+ 邀请」→ 含"邀请同事分身"的授权弹窗（二期 P2-02）。
- 数据分析工作台对话里有"贴 SQL → 跑通 → 贡献为 Skill"的一幕；点开是蒸馏表单（D-13）。
- Agent 详情的「技能 Skills」标签：团队共享 + 我的草稿 + 贡献给团队。

## 十、常见问题（FAQ）

**分身会替我做决定吗？** 不会。它能讨论、规划、协调、草拟，但写文件、发消息、付费、固化记忆等高风险动作一律交你确认。

**我的代码/数据会泄漏给同事吗？** 不会。任务文件不跨任务，你的分身记忆和你对 Agent 的私有经验默认只属于你；只有你显式"贡献给团队"才共享。

**Agent 学到的东西会自动共享给全团队吗？** 不会自动。你贡献的 Skill/经验默认是个人草稿（仅你可用），贡献给团队需要沙盒测试通过 + 管理员审核。

**没装 Kyuubi/飞书能用吗？** 能。相关能力会提示"未配置"而非崩溃，不影响其它功能。

**换机器或重启会丢数据吗？** 不会。数据以文件为准，可整体打包迁移。

**它和 ICE Workbench、TokenWisdom、Multica 是什么关系？** ICE-DATA-WORK 站在三者肩上：以 **ICE Workbench**（数据分析工作台 + 文件优先存储）为实现底座，吸收 **TokenWisdom** 的"数字分身 + 分级授权"理念，借鉴 **Multica** 的"Agent 即同事 + 看板 + 运行时"组织模型。
