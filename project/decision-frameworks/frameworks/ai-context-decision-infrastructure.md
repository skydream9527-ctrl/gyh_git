# AI 上下文决策基础设施：三件套模型

> 来源：飞书文档《PM 工作区升级：三件套补全指南》(2026-06-19)
> 核心观点：AI 的决策质量取决于它对你业务上下文的认知深度。三件套是提升 AI 决策质量的结构性基础设施。

---

## 一、问题诊断：为什么 AI 决策质量低

### 根因分析

AI 打开工作区时，按固定顺序扫描入口文件：

```
AGENTS.md → MEMORY.md → USER.md → README.md → 目录结构
```

**如果前三个文件不存在或为空**，AI 只能靠目录名和文件名猜测上下文，导致：

| 症状 | 根因 | 影响 |
|------|------|------|
| 方案没调用已有分析结论 | AGENTS.md 缺知识导航 | 重复劳动，产出浪费 |
| 输出风格对不上 | MEMORY.md 缺偏好记录 | 每次都需手动调整 |
| 每次重复解释背景 | USER.md 缺角色信息 | 浪费 token 和时间 |
| 知识产出锁在工作区 | 三件套缺位 → AI 找不到 | 决策基于不完整信息 |

### 判断标准

> 打开工作区，让 AI「基于我已有的知识帮我写一份需求分析」——如果 AI 答不上来你有什么知识在哪，说明它不认识你，你需要补入口文件。

## 二、三件套与决策质量的关系

### 决策链路映射

```
决策输入 ← 知识导航（AGENTS.md）→ AI 知道去哪找你的已有产出
决策偏好 ← 偏好记录（MEMORY.md）→ AI 知道你的决策风格和约束
决策约束 ← 角色环境（USER.md）  → AI 知道你的工具链、数据源、权限
    ↓
决策质量提升 = 上下文完整度 × 信息可发现性
```

### 三文件各自对决策的贡献

| 文件 | 决策贡献 | 缺失后果 | 更新频率 |
|------|----------|----------|----------|
| **AGENTS.md** | 告诉 AI 参考哪些知识路径、遵守什么规则 | 决策缺少已有依据，每次从零开始 | 项目变化时（~月度） |
| **MEMORY.md** | 告诉 AI 当前重点、近期决策、个人偏好 | 决策缺少连续性，风格不一致 | 重要决策后即时（~周度） |
| **USER.md** | 告诉 AI 你的角色、业务线、工作环境 | 工具链/数据源匹配不准 | 角色变化时（低频） |

## 三、三件套模板（决策增强版）

> 在原文档模板基础上，增加了「决策相关」字段。

- 我的工作风格：(1 句话)

## 我的项目

| 项目 | 状态 | 关键产出 | 本周重点 |
|------|:---:|---------|---------| 
| (项目名) | (进行中) | (PRD/原型/分析) | (一句话) |

## 知识导航

- 产品方案 → (路径)
- 数据分析 → (路径)
- 竞品研究 → (路径)
- 决策记录 → decisions/

> AI 帮你做事时遵守的规则。

1. (规则 1，如：PRD 先定义问题再写方案)

## 四、三件套 × 决策框架联动

### 场景：让 AI 辅助产品决策

```
1. AI 读取 AGENTS.md
   → 知道你的「知识导航」里有竞品分析和数据报告
   → 决策时自动调用这些已有产出

2. AI 读取 MEMORY.md
   → 知道你上周决定「优先做留存而非拉新」
   → 不会推荐与已有决策矛盾的方向

3. AI 读取 USER.md
   → 知道你用 DataWorks 做数据分析
   → 决策建议中包含可执行的数据验证路径
```

### 场景：决策记录回流

```
做出重要决策 → 写 ADR（用 templates/decision-record.md）
                ↓
          更新 MEMORY.md「关键决策」段
                ↓
          AI 下次决策时自动参考历史决策
```

## 五、实施步骤（30 分钟）

### 步骤 1：盘点现状（5 分钟）

1. 我已有哪个文件？AGENTS.md □ / MEMORY.md □ / USER.md □
2. 已有文件中哪些段是空的？
3. 我的已有产出在哪？——列出 2-4 个目录路径

### 步骤 2：创建/补全三件套（20 分钟）

- **AGENTS.md**：飞书个人简介 + 工作区目录抄路径
- **MEMORY.md**：本周 OKR + 最近方案评审结论
- **USER.md**：飞书个人页 + 工具链信息

### 步骤 3：提交并验证（5 分钟）

```bash
git add AGENTS.md MEMORY.md USER.md
git commit -m "补全三件套入口文件"
git push origin main
```

**验证方法**：问 AI 一句话：

> 「我现在在做什么？我的决策偏好是什么？有哪些已有知识可以用？」

如果 AI 能答出业务背景 + 至少 1 条知识路径 → 通过。

## 六、维护节奏

| 文件 | 更新触发 | 预计频率 | 维护成本 |
|------|----------|----------|----------|
| AGENTS.md | 项目/方向变化 | ~每月 1 次 | 低 |
| MEMORY.md | 重要决策后 | ~每周 1 次 | 极低（1-2 行） |
| USER.md | 角色/业务线变化 | 极低频 | 最低 |

---

## 七、关键洞察

1. **三件套不是额外工作量，是信息归拢**——你已有的产出里已包含所有信息，只需汇到一个文件里
2. **AI 的决策质量 = 上下文完整度**——没有入口文件，AI 就是瞎子摸象
3. **MEMORY.md 是最活跃的决策资产**——每次重要决策后加 1 行，AI 的上下文就越来越准确
4. **决策记录是闭环关键**——ADR 写完要回流到 MEMORY.md，否则 AI 下次还是不知道你做过什么决策
5. **删掉空段比留占位符好**——AI 看到空段会等待，看到缺失比看到占位符更清晰
