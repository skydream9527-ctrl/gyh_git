## 一、核心决策原则

### 工作理念
1. **AI 做执行，我做判断** —— 把重复劳动交给 AI，把时间留给思考和决策
2. **先结论后细节** —— 任何输出先给结论
3. **表格优于文字** —— 对比、分析尽量用表格
4. **异常必想** —— 方案必须覆盖边界条件和异常流程
5. **数据说话** —— 引用数据标注来源，无数据标「推断」
6. **记录决策** —— 重要决策必须落地成文件，不做脑子记
7. **完成比完美重要** —— 70分就行动，快速迭代
8. **定期复盘** —— 每周/季度回顾决策，更新认知

## 二、决策全景分类

```
┌─────────────────────────────────────────────────────────────┐
│                    产品决策层 (Product)                      │
│  产品方向 → 功能优先级 → 用户体验 → 商业化/价值衡量         │
├─────────────────────────────────────────────────────────────┤
│                    技术决策层 (Tech)                        │
│  架构选型 → Agent/Skill/MCP边界 → 技术债务 → 性能优化       │
├─────────────────────────────────────────────────────────────┤
│                    业务决策层 (Business)                     │
│  AB实验解读 → 数据归因 → 增长策略 → 资源分配                 │
├─────────────────────────────────────────────────────────────┤
│                    个人决策层 (Personal)                     │
│  工作优先级 → 学习方向 → 时间分配 → 复盘成长                 │
└─────────────────────────────────────────────────────────────┘
```

### 各类决策清单

| 决策类型 | 频率 | 核心问题 | 判断依据 |
|---------|------|---------|---------|
| **产品方向** | 季度/月度 | 做不做某个方向？核心价值是什么？ | 用户痛点 × 业务价值 × 可行性 |
| **功能优先级** | 周/双周 | 先做什么后做什么？哪些砍？ | RICE 评分 + 战略对齐度 |
| **交互设计** | 日常 | 流程怎么走？界面怎么放？ | 用户路径最短 × 认知负荷最低 |
| **Agent 能力边界** | 日常 | 什么时候用 Agent？Skill？MCP？ | 复杂度 × 可复用性 × 延迟容忍 |
| **架构选型** | 版本级 | 用什么框架？什么存储？ | 团队熟悉度 × 性能需求 × 维护成本 |
| **模块边界** | 版本级 | CLI / Agent / Skill / MCP 怎么划分？ | 职责单一 × 可复用 × 可独立演进 |
| **技术债务** | 双周/月度 | 现在重构还是以后再说？ | 债务影响范围 × 重构成本 × 业务压力 |
| **AB 实验解读** | 实验周期后 | 效果好不好？推不推全？ | 统计显著性 × 业务指标 × 副作用 |
| **数据归因** | 问题发生时 | 指标波动是什么原因？ | 因果推断方法 × 维度拆解 × 排除法 |

## 三、日常决策 OODA 循环

```
  ┌─────────────┐
  │   Observe   │  观察：收集信息，不做判断
  │   (观察)    │  • 当前现状是什么？
  └──────┬──────┘  • 有哪些数据？
         │         • 历史上类似情况？
         ▼
  ┌─────────────┐
  │   Orient    │  调整：用框架分析，形成判断
  │   (调整)    │  • 这是什么类型的决策？
  └──────┬──────┘  • 用什么决策工具？
         │         • 我的偏见可能是什么？
         ▼
  ┌─────────────┐
  │   Decide    │  决策：做出选择，明确理由
  │   (决策)    │  • 选哪个方案？
  └──────┬──────┘  • 为什么选它？放弃什么？
         │         • 成功标准是什么？
         ▼
  ┌─────────────┐
  │    Act      │  行动：执行决策，快速验证
  │   (行动)    │  • 最小行动是什么？
  └─────────────┘  • 多久复盘？
                   • 失败信号是什么？
```

## 四、不同决策类型的标准流程

### 1. 产品功能决策流程

**Step 1: 问题定义**
- [ ] 要解决的用户痛点是什么？（一句话说清楚）
- [ ] 目标用户是谁？
- [ ] 成功指标是什么？（可衡量）
- [ ] 不做这个会怎么样？（机会成本）

**Step 2: 信息收集**
- [ ] 查现有需求和历史决策
- [ ] 看相关数据（用户行为、反馈、竞品）
- [ ] 有没有现成方案可以复用？

**Step 3: 方案生成**
- [ ] 至少想 3 个方案（不要一上来就拍脑袋）
- [ ] 用表格对比各方案的：收益、成本、风险、复杂度
- [ ] 考虑异常流程和边界条件（必须！）

**Step 4: 决策打分（RICE 或决策矩阵）**
| 维度 | 权重 | 方案A | 方案B | 方案C |
|-----|------|-------|-------|-------|
| 用户价值 | 40% | | | |
| 开发成本 | 25% | | | |
| 战略对齐 | 20% | | | |
| 风险 | 15% | | | |
| **加权总分** | 100% | | | |

**Step 5: 决策记录**
- 记录最终决策和理由
- 记录放弃的方案和原因
- 设定复盘时间点

---

### 2. 技术选型决策流程

**Step 1: 约束条件明确**
- [ ] 性能要求是什么？（延迟、QPS、并发）
- [ ] 团队熟悉度如何？
- [ ] 维护成本谁承担？
- [ ] 数据安全/合规要求？

**Step 2: 备选方案列表**
- [ ] 业界主流方案有哪些？
- [ ] 我们之前用过什么？
- [ ] 至少列出 2-3 个备选

**Step 3: PoC 验证（关键技术决策必须做）**
- [ ] 写最小原型验证可行性
- [ ] 压测看性能指标
- [ ] 验证集成复杂度

**Step 4: 决策并记录**
- 选型结论
- 为什么选这个（对比表格）
- 什么情况下可能需要换方案
- 技术债务预期

---

### 3. AB 实验决策流程

**实验前**
- [ ] 假设是什么？（明确的可证伪假设）
- [ ] 核心指标、护栏指标分别是什么？
- [ ] 样本量够不够？（功效计算）
- [ ] 实验周期多久？（避免提前偷看 peeking）

**实验后解读**
- [ ] 统计显著性够吗？（p 值、置信区间）
- [ ] 核心指标涨了吗？护栏指标跌了吗？
- [ ] 分维度看：不同用户群效果一致吗？
- [ ] 有没有 SRM（样本比率偏差）？
- [ ] 用 CUPED 等方法提升灵敏度了吗？
- [ ] 因果关系成立吗？有没有混淆变量？

**决策**
- 推全 / 迭代 / 放弃
- 记录实验结论和经验

### 4. 优先级排序决策

**四象限法（日常）**
```
              紧急
               │
     3. 优先级高 │ 1. 立刻做
     委派/速战  │ 马上处理
───────────────┼───────────────
     4. 别做   │ 2. 计划做
     减少/砍掉 │ 定时间做
               │
              不紧急
     不重要         重要
```

**RICE 评分（版本规划）**
- Reach（影响用户数）：0-10分
- Impact（影响程度）：0-10分
- Confidence（置信度）：0-100%
- Effort（投入人周）：人周数

**RICE 分数 = (R × I × C) / E**

## 六、决策反模式：这些习惯要避免

| 反模式 | 表现 | 怎么改 |
|-------|------|--------|
| **信息囤积** | 总觉得信息不够，迟迟不决策 | 定义"最小必要信息"，超过阈值就拍板 |
| **方案单一** | 只想了一个方案就开始论证 | 强制至少想 3 个方案再对比 |
| **沉默成本** | 因为已经投了很多就继续错下去 | 问自己：如果现在从零开始，还会选这个吗？ |
| **确认偏误** | 只看支持自己观点的证据 | 主动找反对证据，找个人抬杠 |
| **事后诸葛亮** | 结果出来后说"我早就知道" | 决策前先写下来预测，事后对比 |
| **紧急奴役** | 天天处理救火的事，忘了重要的事 | 每天留整块时间做"重要不紧急"的事 |
| **完美主义** | 追求最优解，迟迟不行动 | 先做 MVP，70分就上线，快速迭代 |
