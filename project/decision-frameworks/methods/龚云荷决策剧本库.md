# 龚云荷高频决策剧本库

> 用途：AI 识别出决策类型后，直接按对应剧本走，不用每次临场设计分析路径。
> 关联：护栏见 `龚云荷决策护栏.md`，流程见 `WORKFLOWS.md` 第 4 节，框架详解见 `../tools/` 和 `../frameworks/`。
> 每个剧本固定四件事：识别信号 · 必问问题 · 调用框架 · 要查的证据。

---

## 剧本 1：需求要不要接 / 要不要插队

- **识别信号**：业务方提新需求、临时加急、"能不能插到前面"
- **必问**：谁提的？截止时间？不做的后果？有没有替代方案？
- **调用框架**：RICE 评分（`../tools/rice-scoring.md`）+ 护栏 G1/G3
- **要查**：`WORK-PLAN.md` 当前 P0/P1 排期、`CURRENT.md` 明确不做
- **输出重点**：插队会挤掉谁、是否服务 `nl-sql`、给"接 / 不接 / 降级排期"三选一

## 剧本 2：口径选 A 还是 B

- **识别信号**：同一指标多个口径、新老口径不一致、"按哪个算"
- **必问**：对外解释口径是什么？和现有看板一致吗？差异多大？
- **调用框架**：决策矩阵（`../tools/decision-matrix.md`）+ 数据校验
- **要查**：`00-知识库/数据资产/` 指标口径字典、历史 case 校验值
- **输出重点**：一致性优先还是准确性优先、差异量化（无则标「推断」并给验证动作）

## 剧本 3：方案 A/B（产品 / 技术 / 分析路径）

- **识别信号**：两个以上做法、"哪个方案更好"
- **必问**：评估维度是什么？各方案成本？失败了会是什么样？
- **调用框架**：决策矩阵 + 预 mortem（`../tools/premortem-analysis.md`）
- **要查**：`decision-frameworks/cases/`、相关历史决策
- **输出重点**：加权评分表 + 每个方案的失败模式 + 推荐

## 剧本 4：要不要投入某 Skill / 项目 / 探索

- **识别信号**：新想法、新工具、"要不要做这个"
- **必问**：服务 `nl-sql` 主线吗？现在做还是 Q4？最小验证是什么？
- **调用框架**：RICE + 第一性原理（`../frameworks/first-principles.md`）+ 护栏 G1/G2
- **要查**：`MEMORY.md` Q3 OKR、`CURRENT.md` 明确不做、`DECISIONS.md` 聚焦决策
- **输出重点**：默认倾向"不做 / 推迟"除非直接服务主线；给最小验证方案

## 剧本 5：资源 / 时间怎么分

- **识别信号**：多任务抢时间、"这周先做什么"、精力分配
- **必问**：哪些是 P0？哪些今天必须闭环？buffer 还剩多少？
- **调用框架**：护栏 S1/S3（保 P0、保核心价值区）+ 决策矩阵
- **要查**：`CURRENT.md` 本周 P0、`05-个人成长/每日记录/DAILY-TODO.md`、`WORK-PLAN.md`
- **输出重点**：P0 优先排、核心价值区（归因 / OKR / 协调）留足自己的时间

---

## 没有匹配剧本时

按通用生产线走：识别决策层级 → 选 `../tools/` 里最合适的框架 → 过护栏 → 输出决策简报。

## 新剧本沉淀规则

- 当某类决策累计出现 **3次** 时，自动提炼为新剧本追加到本文件
- 新剧本固定结构：识别信号 · 必问问题 · 调用框架 · 要查的证据 · 输出重点
- 剧本沉淀后同步更新到决策生产线的"识别类型→选剧本"环节
- 回看时发现某剧本判断偏差大，及时修正剧本内容
