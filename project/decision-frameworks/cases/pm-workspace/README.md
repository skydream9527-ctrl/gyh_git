# PM 工作区决策案例：三件套补全

> 来源：飞书文档《PM 工作区升级：三件套补全指南》(2026-06-19)
> 核心问题：PM 工作区已有丰富产出，但 AI 无法识别和调用 → 决策效率打折

---

## 一、决策背景

### 问题描述

PM 工作区里有 PRD、竞品分析、数据报告、原型、Skill，内容已经够好，但打开工作区时 AI 不认识你——不知道你是谁、做什么、有哪些已有产出。每次让 AI 帮忙都得从头解释业务背景。

### 诊断判断

**简单判断法**：打开工作区，直接让 AI「基于我已有的知识帮我写一份需求分析」——如果 AI 答不上来你有什么知识在哪，说明它不认识你。

**根因分析**：缺 AGENTS.md / MEMORY.md / USER.md 三个入口文件。不是能力问题，是**结构意识问题**。

### 影响范围

分布在 L0、L1、L2 多个层级的 PM，卡点性质完全相同。

---

## 二、决策框架分析

### AI 扫描路径（决策依据链路）

```
AI 打开工作区
    ↓
按固定顺序扫描入口文件：
    AGENTS.md → MEMORY.md → USER.md → README.md → 目录结构
    ↓
如果前三个文件不存在或为空
    ↓
AI 只能靠目录名和文件名猜测上下文
    ↓
导致四类决策失效
```

### 四类决策失效模式

| 失效模式 | 表现 | 根因 |
|----------|------|------|
| 知识不调用 | AI 给出的方案没有调用已有分析结论 | AGENTS.md 知识导航缺失 |
| 风格不匹配 | AI 不知道决策偏好，输出风格对不上 | MEMORY.md 偏好缺失 |
| 上下文重复 | 每次 prompt 里重复业务背景 | USER.md 角色信息缺失 |
| 知识锁定 | 知识产出锁在工作区，AI 找不到 | AGENTS.md 知识导航缺失 |

### 三件套职责矩阵

| 文件 | 核心职责 | 缺失后果 | 更新频率 |
|------|----------|----------|----------|
| **AGENTS.md** | AI 知道你负责什么、参考哪些知识路径、遵守什么规则 | AI 无法定位已有产出，每次从零开始 | 项目/方向变化时（~月/次） |
| **MEMORY.md** | AI 获取当前业务上下文（本周重点、近期决策、个人偏好） | AI 不知道最近做了什么判断，输出缺少连续性 | 重要决策后立即（~周/次） |
| **USER.md** | AI 了解你的角色、业务线、工作环境 | AI 不知道工具链做分析，路径匹配不准 | 换角色/业务线时（极少） |

---

## 三、决策方案：30 分钟三步法

### 步骤 1：盘点现状（5 分钟）

回答三个问题：

1. **已有哪个文件？** AGENTS.md □ / MEMORY.md □ / USER.md □
2. **已有文件中哪些段是空的？**（对照模板逐段检查）
3. **已有产出在哪？** 列出 2-4 个目录路径（如 PRD/、竞品分析/、数据报告/、decisions/）

**决策分支**：
- 三个文件都没有 → 从步骤 2 开始全部新建
- 已有部分文件 → 只补缺的，已有内容不需要重写

### 步骤 2：创建/补全三件套（20 分钟）

**AGENTS.md 填写要点**：

| 段落 | 信息来源 | 填写方法 |
|------|----------|----------|
| 我是谁 | 飞书个人简介 | 抄：岗位、业务线、负责产品 |
| 我的项目 | 当前项目列表 | 列 1-3 个项目：项目名 + 状态 + 关键产出 |
| 知识导航 | 工作区目录 | 对步骤 1 列出的目录，每行写一条路径 |
| 关键规则 | 个人偏好 | 写 2 条希望 AI 遵守的规则 |
| 踩坑记录 | — | 可空（AI 帮你做事后逐步补） |

**MEMORY.md 填写要点**：

| 段落 | 信息来源 | 填写方法 |
|------|----------|----------|
| 当前重点 | 本周 OKR 或周报 | 抄 1-3 条 |
| 关键决策 | 最近一次方案评审/需求评审结论 | 抄 1-2 条（日期 + 决策 + 依据） |
| 我的偏好 | 个人工作习惯 | 写 1-2 条 |
| 近期记忆 | 最近一周的发现或踩坑 | 写 1 条 |

**USER.md 填写要点**：

| 段落 | 信息来源 | 填写方法 |
|------|----------|----------|
| 角色 | 飞书个人页 | 抄：岗位、业务线、产品名 |
| 工作环境 | 个人工具使用 | 写工具和常用数据源 |
| 当前阶段 | 本季度重点 | 写 1 句话 |

### 步骤 3：提交并验证（5 分钟）

```bash
git add AGENTS.md MEMORY.md USER.md
git commit -m "补全三件套入口文件"
git push origin main
```

**验证方法**：push 完成后，打开工作区，问 AI 一句话：

> 「我现在在做什么？我的决策偏好是什么？有哪些已有知识可以用？」

**通过标准**：AI 能答出你的业务背景 + 至少 1 条知识路径。

## 四、决策洞察

### 为什么 30 分钟能解决

三个文件不需要从零创作——已有产出里已经包含了所有信息：

| 文件段 | 信息来源 | 本质操作 |
|--------|----------|----------|
| AGENTS.md 知识导航 | 工作区目录 | 对着目录抄路径 |
| MEMORY.md 当前重点 | 本周 OKR 或周报 | 从周报复制 |
| MEMORY.md 关键决策 | 最近一次方案评审结论 | 从结论复制 |
| USER.md 角色信息 | 飞书个人页 | 从个人页复制 |

> **核心洞察**：你做的事 = 把散落在各处的信息汇到一个文件里，让 AI 能一次读全。不是新增工作量，是**信息归拢**。

### 持续维护模型

| 文件 | 更新触发 | 预期频率 | 维护成本 |
|------|----------|----------|----------|
| AGENTS.md | 项目/方向变化 | ~每月 1 次 | 低 |
| MEMORY.md | 重要决策后立即 | ~每周 1 次 | 中（1-2 行） |
| USER.md | 换角色/业务线 | 极少 | 极低 |

### 生效验证

- **短期**：AI 自动引用 MEMORY.md 里的决策偏好
- **中期**：AI 从「知识导航」里调用已有的分析报告
- **长期**：随着 MEMORY.md 持续更新，AI 的上下文越来越准确

## 五、常见决策 Q&A

**Q：已有 AGENTS.md，还要做什么？**

→ 对照模板逐段检查「知识导航」「关键规则」两段是否有实质内容。MEMORY.md 四段也要检查——AGENTS.md 教 AI 怎么做事，MEMORY.md 教 AI 记什么，两者缺一不可。

**Q：负责多个产品/方向，MEMORY.md 怎么写？**

→ 写当前最重点的 1-3 件事。不需要面面俱到——AI 需要的是优先级信号，不是流水账。

**Q：填完之后怎么知道有没有用？**

→ 下周让 AI 写 PRD 时，如果它自动引用了你 MEMORY.md 里的决策偏好，或者从「知识导航」里调用了你已有的分析报告 → 证明生效了。

## 六、模板

> 以下模板可直接复制到工作区根目录。替换 (...) 内占位符为你的信息。
> 如果一个段确实无法填写，删掉该段比留空好——AI 看到空段会等待，看到空白比看到占位符更清晰。

### AGENTS.md 模板

```markdown
# AGENTS.md — AI 助手行为规则

> AI 读到的第一个文件。

## 我是谁

- 产品经理，负责 (业务线/产品名)
- 当前主攻方向：(1-2 句话)
- 我的工作风格：(1 句话)

## 我的项目

| 项目 | 状态 | 关键产出 | 本周重点 |
|------|:---:|---------|---------|
| (项目名) | (进行中) | (PRD/原型/分析) | (一句话) |

## 知识导航

> AI 从这里找到你的已有产出。对着工作区目录抄路径。

- 产品方案 → (路径)
- 数据分析 → (路径)
- 竞品研究 → (路径)
- 决策记录 → decisions/

## 关键规则

> AI 帮你做事时遵守的规则。

1. (规则 1)
2. (规则 2)

## 踩坑记录

| 日期 | 问题 | 修正方式 |
|------|------|---------|
| | | |
```

### MEMORY.md 模板

```markdown
# MEMORY.md — 业务记忆

> AI 读这个文件了解你当前的业务上下文。

## 当前重点（本周/本月）

- (最重要的 1-3 件事)
- (为什么要做这些事)

## 关键决策（近 4 周）

| 日期 | 决策 | 依据 | 影响 |
|------|------|------|------|
| | | | |

## 我的偏好

- 方案风格：(如「先拉数据再写方案」)
- 决策偏好：(如「宁可慢一点对齐数据」)

## 近期记忆

- (最近一周的关键发现)
```

### USER.md 模板

```markdown
# USER.md — 个人上下文

> 写给 AI 看的「我是谁」。

## 角色

- 岗位：(产品经理 / 策略产品 / 数据产品)
- 业务线：(信息流 / 影音娱乐 / 儿童与教育 / ...)
- 负责产品：(产品名，1 句话描述)

## 工作环境

- 工作区工具：VS Code / Trae / Cursor
- 常用数据源：DataWorks / 飞书多维表格
- 常用 AI 能力：PRD 起草 / 数据分析 / 原型生成

## 当前阶段

- 本季度重点：(1 句话)
```

## 七、给 AI 的指令模板

> 复制下方指令到 AI 助手，让它引导你完成三件套补全。

```
我正在进行工作区结构升级，需要补全 AGENTS.md / MEMORY.md / USER.md 三个入口文件。

请按以下步骤引导我：

1. 先扫描我的工作区根目录，告诉我目前已有哪个文件、缺哪个文件
2. 列出我工作区里已有的产出目录（如 PRD/、竞品分析/、数据分析/ 等），帮我确定「知识导航」段填什么路径
3. 逐个文件引导我填写——每段问我一个问题，我回答后你帮我填入
4. 全部填完后，检查三件套是否完整，并给出验证建议

模板参考上方「附录 A」。
```
