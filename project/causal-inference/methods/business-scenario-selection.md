# 因果推断业务场景方法选型指南

> 不知道什么场景该用什么因果方法？看这一张表就够了。

---

## 一、先判断你遇到的是哪类问题

| 问题类型 | 典型场景 | 首选方法 |
|---|---|---|
| **我做了一个改动，到底有没有效果？** | 产品功能上线、策略调整、运营活动 | AB测试（RCT） |
| **功能已经全量上线了，怎么评估效果？** | 历史功能复盘、无法做AB的场景 | DID / 合成控制 / 断点回归 |
| **用户为什么转化/流失，哪些因素真正起作用？** | 增长归因、问题定位、渠道效果评估 | PSM / IPW / DML |
| **哪些用户对这个策略最敏感？我们该对谁推？** | 精准营销、个性化推荐、Uplift提升 | Uplift模型 / CATE估计 / 因果森林 |
| **指标之间的因果路径是什么？哪个是核心驱动因素？** | 指标体系建设、根因分析 | 因果图（DAG）/ 中介分析 |

---

## 二、方法选型决策树

```
开始
│
├─ 能不能做随机对照实验（AB测试）？
│  ├─ 能 → 用AB测试！别想别的，AB是因果推断的黄金标准
│  └─ 不能（已经全量/成本太高/有伦理问题/网络效应）
│     ├─ 有政策/规则的明确截断点？
│     │  └─ 是 → 断点回归（RDD）
│     ├─ 有面板数据（多个个体多个时间点），且干预在某个时间点一刀切上线？
│     │  └─ 是 → 双重差分（DID）
│     ├─ 只有一个/少数几个干预单元，要评估政策/事件影响？
│     │  └─ 是 → 合成控制法（Synthetic Control）
│     ├─ 混淆变量全都能观测到？
│     │  ├─ 样本量不大 → 倾向得分匹配（PSM）
│     │  └─ 样本量大/高维混淆 → 双重机器学习（DML）
│     └─ 有一个变量只影响是否被干预，不直接影响结果？
│        └─ 是 → 工具变量（IV）
```

---

## 三、各方法详解与适用场景

### ✅ 首选：AB测试（随机对照试验 RCT）
**什么时候用**：只要能做AB，就优先做AB，其他方法都是"没办法的办法"。

**优点**：
- 理论最完善，结论最可靠
- 随机分组自动平衡所有混淆变量（观测到的和没观测到的）
- 解释简单，业务方容易理解

**局限**：
- 很多场景做不了：全量功能、网络效应、双边市场、长期效应、合规问题
- 成本高，需要足够流量和时间

**最佳实践**：参考[ab-testing知识库](../../ab-testing/)的SOP和规范。

---

### 📊 观察性研究方法（不能做AB的时候）

#### 1. 双重差分（DID, Differences-in-Differences）
**适用场景**：
- 干预在某个时间点上线，有干预组和未干预对照组
- 有多期面板数据（上线前+上线后都有数据）
- 典型场景：某城市/某部分用户先上线新功能，其他后上线；政策评估

**核心假设**：平行趋势假设——如果没有干预，干预组和对照组的趋势是一样的。

**怎么做**：
```
效应 = (干预组上线后 - 干预组上线前) - (对照组上线后 - 对照组上线前)
```

**注意事项**：
- 必须做平行趋势检验（上线前多期看两组趋势是否一致）
- 标准误要聚类到个体层面，不能直接用OLS标准误
- 考虑时间固定效应和个体固定效应

---

#### 2. 合成控制法（Synthetic Control）
**适用场景**：
- 干预单元只有一个或少数几个（比如一个国家、一个城市、一个大版本）
- 有很多潜在对照单元
- 典型场景：某地区政策上线、全平台级别的功能上线、重大版本发布评估

**核心思想**：用多个对照单元加权合成一个"虚拟对照组"，这个虚拟对照组在干预前的特征和趋势和干预组几乎一模一样，然后看干预后两者的差异。

**优点**：
- 特别适合只有一个处理单元的场景（AB测试和DID都搞不定的场景）
- 结果可视化好，给业务方一看就懂
- 可做placebo检验（安慰剂检验）验证显著性

**工具**：R的`Synth`包，Python的`SyntheticControlMethods`

---

#### 3. 断点回归（RDD, Regression Discontinuity Design）
**适用场景**：
- 是否接受干预是由某个连续变量（running variable）是否超过一个阈值决定的
- 阈值附近的个体其他特征几乎一样，只是刚好在阈值左边/右边
- 典型场景：分数够线就被录取、消费满X元就给优惠券、版本号灰度按尾号分流

**核心思想**：阈值附近的样本本质上是随机分在两边的，比较阈值左右很近的样本就能得到干净的因果效应。

**优点**：
- 因果识别非常干净，接近随机实验
- 假设容易检验

**注意**：
- 带宽选择很重要（不能离阈值太远）
- 检验断点附近有没有操纵（比如用户为了拿优惠刚好刷到阈值）
- 效应只在断点附近局部有效，不能随便外推

---

#### 4. 倾向得分匹配（PSM, Propensity Score Matching）
**适用场景**：
- 混淆变量都能观测到
- 干预不是随机的，用户"选择"是否接受干预
- 典型场景：评估用户使用某个功能对留存的影响、评估某类用户的行为效果

**核心思想**：
1. 先用用户特征预测每个用户接受干预的概率（倾向得分）
2. 给每个干预组用户找一个/几个倾向得分差不多的对照组用户匹配
3. 匹配后的两组除了是否接受干预，其他特征都平衡了，就可以比结果了

**缺点**：
- 只能平衡观测到的混淆变量，没观测到的平衡不了
- 样本量小的时候匹配效果不好
- 现在工业界更倾向用IPW/DML，PSM适合小样本和解释性场景

---

#### 5. 双重机器学习（DML, Double/Debiased Machine Learning）
**适用场景**：
- 高维混淆变量（很多特征，几十上百个）
- 样本量大
- 要干净估计干预的因果效应，同时控制大量混淆
- 典型场景：复杂业务场景下的策略评估、多因素归因

**核心思想**：
用机器学习方法分别拟合：
1. 混淆变量X预测结果Y
2. 混淆变量X预测干预T
然后用残差做回归，得到干净的T对Y的因果效应。

**优点**：
- 可以处理高维混淆，不需要手动选变量
- 机器学习可以拟合非线性关系，比线性回归准确
- 理论保证了估计量是无偏的

**工具**：EconML的`DML`，CausalML的`BaseDML`

---

#### 6. 工具变量（IV, Instrumental Variables）
**适用场景**：
- 存在未观测到的混淆变量，PSM/DML都搞不定
- 能找到一个"工具变量"：只影响是否接受干预，不直接影响结果，和混淆变量不相关
- 典型场景：价格弹性估计（用成本做工具变量）、医生偏好对治疗效果的影响

**核心假设**：
1. 相关性：工具变量和干预强相关
2. 排他性：工具变量只通过干预影响结果，没有其他路径
3. 独立性：工具变量和混淆变量不相关

**注意**：工具变量非常难找，排他性假设永远无法完全验证，用的时候要非常谨慎。弱工具变量问题比没有工具还糟。

---

### 🎯 异质效应与精准干预：Uplift建模
**适用场景**：
- 不是问"干预平均有没有效果"，而是问"**对哪些用户**效果最好？"
- 要做精准营销/个性化干预，提升ROI
- 典型场景：发Push/优惠券/短信，找真正会被影响的用户，别发给"本来就会买"和"怎么都不会买"的用户

**核心概念**：把用户分成四类：
| 类型 | 不干预 | 干预 | 是否该触达 |
|---|---|---|---|
| 营销敏感用户（Persuadables） | 不转化 | 转化 | ✅ 必须发 |
| 自然转化用户（Sure Things） | 转化 | 转化 | ❌ 别发，浪费钱 |
| 无动于衷用户（Lost Causes） | 不转化 | 不转化 | ❌ 别发，浪费钱 |
| 反作用用户（Sleeping Dogs） | 转化 | 不转化 | ❌ 千万别发，起反作用 |

**常用方法**：
- S-Learner / T-Learner / X-Learner（Meta-Learner系列）
- 因果森林（Causal Forest）
- Uplift树/随机森林

**评估指标**：Qini曲线、AUUC（Area Under Uplift Curve）

**工具**：EconML、CausalML、scikit-uplift

## 四、什么时候用什么方法——速查表

| 业务场景 | 数据条件 | 推荐方法 | 注意事项 |
|---|---|---|---|
| 新功能上线，能分流 | 可随机分流 | AB测试 | 跑够样本量，先AA，不要偷看 |
| 功能已经全量上线了 | 有上线前后面板数据，有对照组 | DID | 检验平行趋势 |
| 全平台大版本/政策评估 | 只有一个处理单元，多期数据 | 合成控制 | 做安慰剂检验 |
| 按阈值规则灰度/发放权益 | 有明确的分配阈值 | 断点回归RDD | 检验阈值操纵，效应局部有效 |
| 评估"用了XX功能的用户价值更高" | 有用户特征数据，能观测到主要混淆 | PSM/IPW/DML | 只能控制观测混淆，不能说严格因果 |
| 高维特征下的策略评估 | 大量用户特征，大样本 | DML | 用ML拟合，结果稳健 |
| 价格弹性/政策评估，有未观测混淆 | 能找到好的工具变量 | 工具变量IV | 工具变量要满足所有假设，非常难 |
| 精准营销/发券，找高ROI人群 | 有历史干预数据和用户特征 | Uplift模型/因果森林 | 别用响应模型，要用增益模型 |
| 指标异动根因分析，找因果路径 | 有指标时序/细粒度数据 | 因果图DAG + 中介分析 | 需要业务知识构建DAG |

## 五、因果推断常见误区

### 1. 拿着相关系数就说因果
"买了A功能的用户留存高" → "A功能提升留存"，这是最大的误区。大概率是"高留存用户本来就更愿意用A功能"。

**记住**：没有合理的识别策略，所有观察性研究的结论都是"相关性"，不是"因果性"。

### 2. 方法越复杂越好
很多人上来就用因果森林、深度学习因果模型，其实大部分场景简单的DID/PSM就够了，而且更稳健、更容易解释。

**记住**：能AB就AB；不能AB先试简单方法（DID/RDD）；简单方法不行再上复杂方法。

### 3. 忽略方法的核心假设
每个因果方法都有不可验证的核心假设（比如DID的平行趋势、IV的排他性、PSM的无未观测混淆），不检验假设就直接跑方法，结果完全没用。

**记住**：用任何方法之前，先搞清楚它的核心假设是什么，能不能验证，不满足会怎么样。

### 4. 过度追求"统计显著"，不看效应大小
p<0.05但效应只有0.01%，业务上没价值；p=0.08但效应有2%，可能是样本量不够。和AB测试一样，因果推断也要看效应量和置信区间，不能只看p值。

### 5. 不做稳健性检验
好的因果分析一定有一堆稳健性检验：换模型设定、换带宽、换匹配方法、安慰剂检验、排除子样本……核心结论在各种设定下都成立才可信。
