# DAG 与 d-分离：因果结构的可视化语言

> **DAG**（Directed Acyclic Graph，有向无环图）是 Pearl 因果推断的核心工具——它让你能在画图阶段就判断"哪些变量必须控制、哪些不能控制"。这是因果分析里最大的杠杆，也是最容易忽略的一步。

---

## 一、为什么必须画 DAG

最常见的因果错误**不在"用了哪个模型"**，而在"控制了不该控制的变量"或"漏控了关键变量"。两个例子：

```
错误 1：控制了中介变量 → 把因果效应"吞掉"
   广告 X → 点击 → 购买 Y
   "我把点击作为控制变量加进回归..."
   → 估出来的广告系数接近 0，但你解读成"广告无效"

错误 2：控制了碰撞变量 → 制造假相关
   X → Z ← Y
   "看起来 Z 和 X、Y 都相关，先控制吧..."
   → 控制 Z 后 X 和 Y 出现假的因果关系
```

DAG 的价值是：**你在跑代码之前，光看图就能看出该控制谁不该控制谁**。

## 二、DAG 的基本元素

### 节点 + 有向边 + 无环

```
节点：变量（X, Y, Z, U, ...）
边  ：A → B 表示 "A 是 B 的直接原因之一"
约束：不能有环（A → B → A 不允许）
```

### 三种基本子结构（**记住这三个**）

整个因果分析的"识字课"。所有复杂图都是这三种的拼接。

#### 1. Chain（链）：A → B → C

```
   A ──→ B ──→ C
```

- **B 是 A 影响 C 的中介**。
- A 和 C 边际相关；**控制 B 后**，A 和 C **变得无关**（B 阻断了通路）。
- 业务例子：
  - 广告 → 点击 → 购买
  - 用户教育水平 → 收入 → 消费

#### 2. Fork（叉）：A ← B → C

```
   A ←── B ──→ C
```

- **B 是 A 和 C 的共同原因**（混淆变量 / Confounder）。
- A 和 C 边际相关（但**没有因果关系**）；**控制 B 后**变得无关。
- 业务例子：
  - 用户活跃度 → App 用 X 功能；活跃度 → 留存
  - 季节 → 冰激凌销量；季节 → 溺水率（虚假相关）

#### 3. Collider（碰撞）：A → B ← C

```
   A ──→ B ←── C
```

- **B 是 A 和 C 的共同结果**。
- A 和 C 边际**无关**；**控制 B 后****反而变得相关**（!）。
- 业务例子：
  - 颜值 → 演员被录用 ← 演技 → 演员被录用
    - 不控制：颜值和演技无关
    - 只看"被录用的演员"（条件控制了 B）：颜值低的演员演技通常更好（假负相关）
  - 用户对 X 的兴趣 → 购买 ← 收入水平 → 购买
    - 只看"买了的用户"，兴趣和收入会出现负相关

## 三、d-分离：可识别性的判定

**d-分离**（d-separation）是 Pearl 给的判断规则：给定要控制的变量集 Z，X 和 Y 之间是否还有"信息流"？

### 简单规则

X → Y 之间有一条路径"打开"（开放）当且仅当：

1. 路径上**所有 chain / fork 中介**都**没**被控制
2. 路径上**所有 collider** 都**已**被控制（或它的后代被控制）

听起来绕。其实就一句话：

```
Chain 和 Fork：控制中间变量 → 关闭通路
Collider     ：控制碰撞变量 → 打开通路（!）  ← 这是 collider 的特殊之处
```

> **黄金启发式**：默认应该控制 chain/fork 上的混淆变量，**默认不应该控制 collider**。如果不确定一个变量是哪类，先画 DAG。

## 四、后门准则：找混淆变量的算法

Pearl 给了一个机械的判定：**Z 是有效的"控制变量集"当且仅当**——

1. **Z 阻断**所有从 X 到 Y 的"后门路径"（即从 X 出发的反向路径）
2. **Z 不包含 X 的任何后代**（避免变成 collider）

### 例子

```
        U （未观测的混淆，如"用户活跃度"）
       ╱ ╲
      ▼   ▼
      X ──→ Y
      ↑      ↑
      │      │
      W ─────┘
```

要估 X → Y 的因果效应，后门路径是 `X ← U → Y` 和 `X ← W → Y`。

- 如果 U 不可观测、W 可观测：必须假设没有 U（强可忽略性），然后控制 W
- 如果 U、W 都可观测：控制 {U, W}
- 如果 U 不可观测且无替代：后门法失效，要走前门法 / IV / DID 等绕路

---

## 五、前门准则（Front-door）

当后门关不掉（U 不可观测），有时还能从"中介"那边走前门。条件：

```
        U （未观测）
       ╱ ╲
      ▼   ▼
      X ──→ M ──→ Y
```

如果中介 M：
1. 完全中介（X → Y 的所有效应都经过 M）
2. M 不被 U 影响
3. 控制 X 后，M → Y 没有混淆

那么可以分两步估：`P(Y | do(X)) = ∑_m P(M=m | X) · ∑_x' P(Y | M=m, X=x') P(X=x')`

> 这个工具在 PM 思考具体业务因果链路时**比想象中常用**：当你怀疑有未观测混淆，问问自己"有没有清晰的中介链路？"。

## 六、四种"控制变量"决策矩阵

把上面所有规则浓缩成一张表：

| 变量类型 | 是否控制 | 原因 |
|---|---|---|
| **混淆变量**（chain/fork 上的共因） | ✅ 必须 | 不控制 → 估计有偏 |
| **中介变量**（X 到 Y 的中间） | ❌ 不要 | 控制 → 把因果效应吞掉 |
| **碰撞变量**（X 和 Y 的共同结果） | ❌ 不要 | 控制 → 制造虚假相关 |
| **工具变量**（影响 X 不直接影响 Y） | ❌ 不要在主回归里 | 它有专门的用法（IV 法），不能简单加进控制 |
| **预测变量**（影响 Y、不影响 X） | ⚪ 可以 | 不影响估计无偏性，但能减小方差（提升精度） |
| **结果变量的子代** | ❌ 不要 | 引入 reverse causation 风险 |

## 七、业务里的 DAG 例子

### 例 1：评估"信息流广告投放对购买的影响"

```
        历史购买行为 H
       ╱             ╲
      ▼              ▼
   投广告 X ──→  点击 ──→ 购买 Y
                  ↑
                价格敏感度 S
                  ↓
              （也影响 X 的投放决策）
```

分析：
- H 是混淆变量 → **必须控制**（不然广告投放本身就偏向高价值用户）
- 点击是中介 → **不能控制**（不然量不到广告效应）
- S 也是混淆变量 → 控制
- 如果 S 不可观测，就要考虑 IV 或前门

### 例 2：评估"完成新手任务对留存的影响"

```
   注册时间 T → 活跃度 A → 留存 Y
                ↓        ↗
               完成任务 X
```

分析：
- A（活跃度）→ X 也 → Y → A 是混淆变量 → 控制
- T → A → X → Y → 同样路径，T 也是混淆
- 如果你只在"完成新手任务的人"里看留存，等于做了 selection on collider → 偏。
- 正确做法：用 PSM / IPW 在控制 A、T 后比较。

### 例 3：评估"客服满意度对复购的影响"（典型陷阱）

```
   产品质量 Q
   ╱        ╲
  ▼         ▼
  客服满意度 X ──→ 复购 Y
```

- Q 是 X 和 Y 的共因（混淆）→ 控制
- 但 Q 通常**不可观测** → 需要替代变量（如订单瑕疵率、客诉记录）

## 八、画 DAG 的实操流程

### 1. 列变量
- X（处理）、Y（结果）、所有可能影响 X 或 Y 的变量

### 2. 画箭头（**最关键**）
- 对每对变量问：**A 在时间上 / 因果上能影响 B 吗？**
- 注意箭头方向用"先发生 / 决定"定，不是"看起来相关"

### 3. 找路径
- 列出所有从 X 到 Y 的路径（含反向）
- 标记每条路径上的 chain / fork / collider

### 4. 应用后门准则
- 找出所有"未关闭的后门路径"
- 设计控制变量集 Z

### 5. 检查 collider
- 确保 Z 里没有 X 或 Y 的共同后代

### 6. 写下假设（**所有不可观测的因子都要列出来**）
- 你做的每个假设都是**可被推翻的依据**——记下来才能复盘

### 7. 跑完模型后做敏感性分析
- "如果有一个未观测混淆 U，强度多大才能翻转结论？"
- E-value、Rosenbaum bounds 等工具

## 九、工具

# DoWhy: 直接接受 DAG，自动判定可识别 + 估计 + 反驳
import dowhy
from dowhy import CausalModel

model = CausalModel(
    data=df,
    treatment='X',
    outcome='Y',
    graph="""digraph {
        X -> M -> Y;
        U -> X; U -> Y;
        W -> X; W -> Y;
    }"""
)
identified = model.identify_effect()  # 自动应用后门 / 前门 / IV
estimate = model.estimate_effect(identified, method_name='backdoor.propensity_score_matching')
refute = model.refute_estimate(identified, estimate, method_name='random_common_cause')

## 十、常见认知陷阱

### 1. "把所有变量都加进回归就稳了"
错。collider、中介都不能加。"控制更多 = 更稳"是数据分析最大的迷思。

### 2. "DAG 就是相关图"
错。DAG 表达**因果方向**，相关图无方向。两个 DAG 可能蕴含同样的相关结构但因果不同。

### 3. "我没办法画 DAG，因为业务太复杂"
错。**复杂正是要画的理由**。先画一个粗的，每次复盘细化——画错的 DAG 比不画好得多。

### 4. "DAG 是因果发现，能从数据自动学出来"
部分对。从数据学 DAG（causal discovery）是研究方向，但工业实践中 DAG 主要靠**领域知识**画。**数据可以辅助验证 DAG 是否与观察分布一致，但不能唯一确定**。

### 5. "线性回归和 DAG 没关系"
错。回归实质就是"控制一组变量后估计偏效应"——你控制的变量集正确不正确，DAG 决定。

## 十一、Checklist

```
□ 1. 我有没有把 X、Y 写在纸上？
□ 2. 我有没有把每个候选变量分类成混淆/中介/碰撞/预测/工具？
□ 3. 不可观测的变量（如"内在动机"）是不是显式标了 U₁、U₂ 写下来？
□ 4. 我控制的变量是混淆变量，还是不小心混进了中介或 collider？
□ 5. 后门路径关闭了吗？
□ 6. 我做的关键假设是哪条？最不放心哪条？
□ 7. 是否能找到"自然实验"避开未观测混淆？
□ 8. 跑完模型有没有做敏感性分析？
```
