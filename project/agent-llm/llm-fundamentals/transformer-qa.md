# Transformer 架构问答：从基础到核心概念

---

## 一、架构基础

### Transformer 架构的两个核心模块是什么？

**编码器（Encoder）** 和 **解码器（Decoder）**。

- 编码器：将输入序列映射为连续的上下文表示（context representation）。
- 解码器：基于编码器输出和已生成的 token，自回归地生成输出序列。

原始 Transformer（Vaswani et al., 2017）是 Encoder-Decoder 结构。后续衍生出：
- 仅编码器：BERT（理解任务，如分类、NER）
- 仅解码器：GPT 系列（生成任务）
- 编解码器：T5、BART（翻译、摘要）

---

### 自注意力机制中的 Q、K、V 分别代表什么含义？

| 符号 | 全称 | 含义 |
|------|------|------|
| Q | Query（查询） | 当前 token 想"问"什么 |
| K | Key（键） | 序列中每个 token "提供"什么信息的索引 |
| V | Value（值） | 序列中每个 token 实际携带的内容 |

计算过程：

```
Attention(Q, K, V) = softmax(QK^T / √d_k) · V
```

直觉：Q 和 K 做点积得到相似度权重（"这个词应该关注哪些词"），再用权重对 V 加权求和，得到每个位置的上下文向量。

除以 `√d_k` 是为了防止点积值过大导致 softmax 梯度消失（缩放点积注意力）。

---

### 多头注意力相比单头注意力的主要优势在哪里？

单头注意力每次只能学习一种"关注模式"。多头注意力将 Q/K/V 分别投影到 h 个子空间，并行计算 h 个注意力，最后拼接。

**核心优势：**

1. **多视角捕捉**：不同头可以同时学习语法依存、指代关系、语义相似性等不同类型的关联。
2. **表达能力更强**：子空间维度 `d_k = d_model / h`，参数量与单头相当，但表达更丰富。
3. **稳定性更好**：多头的聚合效果类似 ensemble，对单一头的噪声不敏感。

```
MultiHead(Q,K,V) = Concat(head_1, ..., head_h) · W^O
其中 head_i = Attention(Q·W_i^Q, K·W_i^K, V·W_i^V)
```

---

### 位置编码起什么作用？常用的实现方式有哪些？

**作用：** 自注意力本身是置换不变的（permutation invariant），即打乱输入顺序不影响计算结果。位置编码向模型注入 token 在序列中的位置信息，使模型能区分"猫吃鱼"和"鱼吃猫"。

**常用实现：**

| 方式 | 特点 | 代表模型 |
|------|------|------|
| 正弦/余弦绝对位置编码 | 固定，可外推到训练时未见长度 | 原始 Transformer |
| 可学习绝对位置编码 | 可训练，但受最大长度限制 | BERT、GPT-2 |
| 相对位置编码（RPE） | 编码 token 对之间的相对距离 | T5、Music Transformer |
| RoPE（旋转位置编码） | 以旋转矩阵形式将位置注入 Q/K，支持长度外推 | LLaMA、Qwen、ChatGLM |
| ALiBi | 在注意力分数上加线性偏置惩罚，天然支持外推 | MPT、BLOOM |

---

### 编码器（Encoder）与解码器（Decoder）在结构上存在哪些关键差异？

| 维度 | 编码器 | 解码器 |
|------|--------|--------|
| 自注意力类型 | 双向（可看到全部输入） | 因果/掩码自注意力（只看到当前及之前的 token） |
| 额外注意力层 | 无 | 有交叉注意力层（Cross-Attention，Q 来自解码器，K/V 来自编码器输出） |
| 典型任务 | 理解、分类、抽取 | 生成、翻译、摘要 |
| 推理方式 | 一次前向传播 | 自回归逐 token 生成 |

解码器的三层结构：
1. 掩码自注意力（Masked Self-Attention）
2. 交叉注意力（Cross-Attention）
3. 前馈网络（FFN）

---

### 为什么要同时引入残差连接和层归一化？

**残差连接（Residual Connection）：**
- 公式：`output = x + Sublayer(x)`
- 解决深层网络梯度消失/爆炸问题，梯度可通过跳跃连接直接反传。
- 使网络更容易学习恒等映射（"至少不比不加差"）。

**层归一化（Layer Normalization）：**
- 对每个样本在特征维度上做归一化（不同于 BatchNorm 在批次维度上归一化）。
- 稳定训练过程，减少对学习率的敏感性。
- 适合变长序列（BatchNorm 在 NLP 中效果差，因为序列长度不一）。

**两者组合：** 残差连接保留信息流，层归一化稳定数值范围，共同保障深层 Transformer 的可训练性。

原始论文用 Post-LN（`LN(x + Sublayer(x))`），现代模型（GPT、LLaMA）多用 Pre-LN（`x + Sublayer(LN(x))`），训练更稳定。

---

### 相比 RNN，Transformer 在处理长序列时有哪些明显优势？

| 维度 | RNN/LSTM | Transformer |
|------|----------|-------------|
| 并行化 | 时序依赖，无法并行 | 全序列并行计算 |
| 长程依赖 | 信息需逐步传递，远距离衰减 | 任意两个 token 直接计算注意力，路径长度为 O(1) |
| 梯度传播 | 长序列易梯度消失/爆炸 | 残差+归一化缓解，梯度路径短 |
| 训练速度 | 慢（串行） | 快（可充分利用 GPU 并行） |
| 上下文窗口 | 理论无限但实际受限 | 由显式上下文窗口决定，可通过架构扩展 |

核心原因：自注意力将序列中任意两个位置的交互路径从 O(n) 压缩到 O(1)，从根本上解决了长程依赖问题。

---

## 二、关键组件

### 前馈神经网络在 Transformer 中承担了怎样的功能？

每个 Transformer 层中，注意力之后跟着一个 **位置独立的前馈网络（FFN）**：

```
FFN(x) = max(0, xW_1 + b_1)W_2 + b_2
```

- 通常扩展维度：`d_ff = 4 × d_model`（如 d_model=512，d_ff=2048）。
- **功能分工**：注意力负责 token 间的信息聚合（"谁关注谁"），FFN 负责对每个位置的表示做非线性变换（"如何处理这个信息"）。
- 研究表明 FFN 层是知识存储的主要场所（"知识神经元"假说），而注意力负责路由和检索。

现代变体：SwiGLU（LLaMA）、GeGLU（PaLM）替换 ReLU，效果更好。

---

### 注意力掩码通常分为哪两类？各自用于什么场景？

**1. 填充掩码（Padding Mask）**
- 用途：屏蔽 padding token（`[PAD]`）对注意力计算的影响。
- 场景：batch 训练时，不同长度的序列用 padding 对齐，需要让模型忽略这些无意义位置。
- 适用：编码器和解码器均需要。

**2. 因果掩码（Causal Mask / Look-ahead Mask）**
- 用途：确保生成时每个位置只能看到它之前的 token，不能"偷看"未来。
- 实现：上三角矩阵设为 -∞，经 softmax 后权重归零。
- 场景：解码器的自注意力层，以及 GPT 类仅解码器模型的训练。

---

### 词嵌入维度一般如何确定？

没有绝对公式，通常基于**模型规模与任务复杂度**权衡：

| 模型规模 | 典型 d_model |
|----------|-------------|
| 小型（<1亿参数） | 256 ~ 512 |
| 中型（1-10亿） | 768 ~ 1024 |
| 大型（10B+） | 2048 ~ 8192 |

**影响因素：**
- 越大的词表（vocab size）通常需要越大的嵌入维度以区分词义。
- d_model 决定了整个模型的隐层宽度，是最核心的超参数。
- 受计算资源约束：参数量和显存随 d_model 平方增长（注意力矩阵）。
- 实践上常遵循缩放定律，与层数、头数协同设计。

---

### 为什么要在模型中使用 Dropout？

**作用：** 训练时随机将一部分神经元的输出置为 0，推理时关闭。

**核心目的：**
1. **正则化**：防止模型对训练数据过拟合，强迫网络学习冗余表示。
2. **隐式集成**：每次 forward 相当于训练一个不同的子网络，推理时相当于这些子网络的 ensemble。

**在 Transformer 中的位置：**
- 注意力权重之后
- FFN 中间激活之后
- 残差连接的输出之后

大模型（GPT-3、LLaMA）训练时通常用较小的 dropout（0.0 ~ 0.1），因为数据量足够大，过拟合风险相对较小。

---

### 解码器在训练阶段，"掩码"机制具体是如何生效的？

训练时使用 **Teacher Forcing**：将真实目标序列（ground truth）全部喂给解码器，而非上一步的预测输出。

**问题：** 如果不加限制，位置 i 的 token 能看到位置 i+1 的真实 token，造成"信息泄漏"，模型学不到真正的生成能力。

**解决方案——因果掩码（Causal Mask）：**

```
位置:  1   2   3   4
  1   可  不  不  不
  2   可  可  不  不
  3   可  可  可  不
  4   可  可  可  可
```

具体实现：在注意力分数矩阵的上三角区域加上 `-inf`，softmax 后这些位置的权重为 0，等效于屏蔽未来信息。

这样模型在并行计算所有位置的同时，每个位置只能利用历史信息，与推理时的自回归行为保持一致。

---

### 预训练模型的核心设计思想是什么？

**核心思想：** 在大规模无标注数据上学习通用表示，再在特定任务上微调（Pre-train → Fine-tune）。

**两个关键观察：**
1. 语言本身包含大量隐式知识（语法、事实、推理），可以通过自监督任务（预测下一个词、填空）从文本中学到。
2. 这些通用知识可以迁移到下游任务，大幅降低标注数据需求。

**典型预训练目标：**

| 目标 | 代表模型 | 适合任务 |
|------|----------|----------|
| 因果语言模型（CLM）：预测下一个 token | GPT 系列 | 生成 |
| 掩码语言模型（MLM）：预测被遮盖的 token | BERT | 理解 |
| 前缀语言模型 / Seq2Seq | T5、BART | 翻译、摘要 |

预训练的本质：用自监督任务构造几乎无限的训练信号，把互联网上的文本转化为模型权重中的"世界知识"。

## 三、核心概念

### 模型的参数量与计算量有什么区别？

| 维度 | 参数量（Parameters） | 计算量（FLOPs） |
|------|---------------------|----------------|
| 定义 | 模型权重矩阵的数值个数 | 完成一次前向传播所需的浮点运算次数 |
| 单位 | 个（通常用 B/亿表示） | FLOPs（通常用 PetaFLOP-days 衡量训练） |
| 决定因素 | 模型架构（层数、宽度） | 参数量 × 序列长度（注意力是 O(n²d)） |
| 影响 | 存储/显存占用 | 推理速度、训练时间 |
| 关系 | 参数量固定；计算量随输入长度变化 | 通常 FLOPs ≈ 2 × 参数量 × token 数（训练） |

**例子：** GPT-3 有 1750 亿参数，训练一次需约 3.14×10²³ FLOPs。参数量决定"模型有多大"，计算量决定"跑一次要花多少资源"。

---

### 什么是生成式 AI？典型任务包含哪些？

**定义：** 生成式 AI 指能够生成新内容（文本、图像、代码、音频等）的模型，与判别式模型（分类、回归）相对。

**典型任务：**

| 模态 | 典型任务 | 代表模型 |
|------|----------|----------|
| 文本 | 对话、写作、翻译、摘要、代码生成 | GPT-4、Claude、Gemini |
| 图像 | 文生图、图像编辑、风格迁移 | Stable Diffusion、DALL-E、Midjourney |
| 代码 | 代码补全、代码生成、调试 | GitHub Copilot、CodeLlama |
| 音频 | 语音合成、音乐生成 | ElevenLabs、Suno |
| 视频 | 文生视频 | Sora、Runway |
| 多模态 | 跨模态生成与理解 | GPT-4o、Gemini 1.5 |

核心能力来源：在海量数据上学习数据分布，推理时从该分布中采样生成。

---

### 大模型为何需要海量的训练数据？

1. **覆盖知识广度**：语言中涵盖科学、历史、文化、代码等几乎所有人类知识领域，稀疏的数据无法学到完整分布。

2. **统计学习的本质**：模型通过最大化训练数据的似然来学习规律，数据越多，学到的规律越可靠、越通用。

3. **缩放定律的驱动**（Chinchilla Law）：计算最优训练需要参数量和数据量同步增长。1B 参数的模型需约 20B token 才能充分训练。

4. **长尾知识**：罕见但重要的知识（小语种、专业领域）需要足够多的数据才能被模型捕获。

5. **泛化能力**：数据越多样，模型在未见过的 prompt 上的泛化能力越强，"涌现能力"的触发也依赖于足够大的数据+模型规模组合。

---

### 什么是过拟合？如何判断模型是否出现过拟合？

**定义：** 模型过度拟合训练数据中的噪声和细节，导致训练集表现好但测试集/真实场景表现差。

**直觉：** 模型"死记硬背"而非"理解规律"。

**判断方法：**

| 信号 | 说明 |
|------|------|
| 训练 loss 持续下降，验证 loss 上升/停滞 | 最直接的过拟合信号 |
| 训练集准确率远高于验证集准确率 | 泛化差距（Generalization Gap）过大 |
| 模型在训练数据上几乎完美，但在新样本上失败 | 常见于小数据集 |
| 学习曲线出现"V 形"或发散 | 早停（Early Stopping）的依据 |

**常用缓解手段：**
- 增加训练数据
- 数据增强
- Dropout、L2 正则化
- Early Stopping
- 减小模型容量

---

### "模型推理"指的是哪个具体过程？

**推理（Inference）** 指训练完成后，用固定权重的模型对新输入进行前向传播、产生输出的过程。

与**训练**的区别：

| 阶段 | 训练（Training） | 推理（Inference） |
|------|----------------|-----------------|
| 权重 | 持续更新 | 固定不变 |
| 梯度 | 需要计算（反向传播） | 不需要 |
| 显存 | 需存储激活值（用于反向传播） | 仅需前向激活 |
| 目标 | 最小化 loss | 生成预测结果 |
| 批量 | 通常大 batch | 通常小 batch 或单条 |

对于自回归生成模型，推理还包含**逐 token 解码**（每次生成一个 token，将其拼接回输入，再预测下一个），是串行过程，延迟受序列长度影响显著。

---

### 如何理解大模型中的"涌现能力"？

**涌现（Emergence）** 指随着模型规模的增大，某些能力在小模型中几乎不存在，但在超过某个规模阈值后突然显现。

**典型涌现能力示例：**
- Few-shot 学习（GPT-3 规模才出现）
- 链式思维推理（Chain-of-Thought）
- 代码生成、算术推理
- 遵循复杂指令

**为什么会涌现？**

目前无定论，主流假说：
1. **组合复杂度**：某些任务需要多步推理，每步都需要足够准确，模型规模不足时每步误差累积导致整体失败，规模超阈值后每步准确率足够，整体成功率骤升。
2. **任务评估的非线性**：多选/精确匹配等评估方式本身是离散的，准确率从 0 跳到非零造成"涌现"假象（Wei et al. 的批评）。

**实践意义：** 无法通过小模型实验预测大模型能力，给模型选型和产品规划带来不确定性。

---

### 解释一下"缩放定律"及其意义

**缩放定律（Scaling Laws）** 描述了模型性能与模型规模（参数量 N）、数据量（D）、计算量（C）之间的幂律关系。

**Kaplan et al. (2020) OpenAI Scaling Law：**
```
Loss ∝ N^(-α)   α ≈ 0.076（参数维度）
Loss ∝ D^(-β)   β ≈ 0.095（数据维度）
Loss ∝ C^(-γ)   γ ≈ 0.050（计算维度）
```

**Chinchilla Law（Hoffmann et al., 2022）—— 计算最优训练：**
- 给定固定计算预算，参数量和训练 token 数应**同比例增长**（约 1:20 比例）。
- 纠正了 GPT-3 时代"只增大模型不增加数据"的错误做法。
- Chinchilla（70B 参数 + 1.4T tokens）在同等计算下超过 Gopher（280B + 300B tokens）。

**意义：**

| 角度 | 意义 |
|------|------|
| 可预测性 | 在训练前可以通过小规模实验预测大规模模型的性能 |
| 资源分配 | 指导如何在参数量和数据量之间分配固定计算预算 |
| 工程决策 | 为模型选型、训练策略提供量化依据 |
| 研究局限 | 不预测涌现，不覆盖微调/对齐阶段，仅适用于预训练 loss |

缩放定律是过去五年大模型快速进步的核心指导原则，推动了从"精心设计架构"到"规模制胜"的范式转变。
