## ❤️ 我的偏好

> 让AI不用每次问风格问题，按我的习惯输出

### 输出要求
- **结构优先**：先结论后细节，结构化输出，避免大段文字
- **表格偏好**：对比、清单、多维度信息优先用表格呈现
- **数据标注**：引用数据必须标注来源和时间；无数据支撑的判断标注「推断」
- **语言**：中文，专业简洁，不啰嗦，不用情绪化表达，不要过度礼貌
- **协作方式**：同一文件多个改动合并为一次提交，避免冲突；不同文件可并行修改；一次给完整结果，不要反复确认小事

### 文件规范
- **目录结构**：严格遵循AGENTS.md定义的目录分类，新文件按分类存放
- **命名规则**：报告`YYYY-MM-DD-主题.md`、SQL`业务主题-指标描述.sql`、会议纪要`MM-DD 会议主题.md`
- **归档要求**：所有正式文档生成后必须归档到`00-知识库/文档归档/`对应目录，更新索引
- **临时文件**：临时分析放`03-数据分析/`，完成后可复用内容沉淀到知识库，临时文件放`99-临时文件/`每周清理

### 工具偏好
- 数据查询：优先用`kyuubi`/`sql`/`nl-sql`Skill，直接执行SQL获取结果
- 文档输出：正式交付优先飞书文档，本地Markdown作为草稿和归档
- 代码修改：先读最新文件内容再编辑，保持现有代码风格，不新增不必要的依赖
