# WORKFLOWS.md — 工作 Agent 运行手册

> 作用：把「记录工作、帮助工作、辅助规划、辅助决策」固化成可重复执行的流程。AI 接到任务时优先按这里的流程运行。

## 0. 启动协议

每次进入本工作区，AI 默认按以下顺序读取上下文：

1. `AGENTS.md`：工作区规则、目录结构、归档要求
2. `USER.md`：用户角色、偏好、工具链、决策风格
3. `CURRENT.md`：当前重点、本周建议、明确不做
4. `WORK-PLAN.md`：P0/P1/P2、阻塞、近期完成项
5. `MEMORY.md`：长期记忆、关键决策、沟通偏好
6. `DECISIONS.md`：重要决策与回看事项
7. `00-知识库/README.md`：知识资产导航；按任务再读对应知识域 README

## 1. 记录工作流程

### 每日待办更新

触发方式：每天开始工作时，或用户说「今天做什么」「更新今日待办」。

默认动作：

1. 先读 `05-个人成长/每日记录/DAILY-TODO.md`
2. 再读 `CURRENT.md`、`WORK-PLAN.md` 和最新周会/飞书输入
3. 更新 `DAILY-TODO.md` 的「今日待办」：优先放 P0，其次放必须今天闭环的 P1
4. 晚上回顾时把完成状态同步到 `WORK-PLAN.md` 和每日工作回顾
5. 未完成但仍需推进的事项移动到「滚动跟进」

固定文档：`05-个人成长/每日记录/DAILY-TODO.md`

### 每日回顾

触发方式：每天 23:00 定时，或用户说「帮我做今日回顾」。

默认动作：

1. 扫描当日变更文件与 Git 提交
2. 生成 `05-个人成长/每日记录/YYYY-MM-DD-每日工作回顾.md`
3. 同步备份到 `04-会议纪要/每日回顾/` 和 `99-临时文件/每日回顾/`
4. 提醒用户补充飞书交互、完成事项、明日计划
5. 从日报提取三类回流建议：`MEMORY.md`、`WORK-PLAN.md`、绩效过程素材

现有脚本：`07-定时任务/scripts/daily_work_review.py`

### 每周总结

触发方式：每周日 22:00 定时，或用户说「帮我做本周总结」。

默认动作：

1. 读取本周每日记录
2. 汇总完成任务、文件变更、关键决策
3. 生成 `05-个人成长/每周总结/YYYY-MM-DD-to-YYYY-MM-DD-周总结.md`
4. 输出下周 P0/P1 建议
5. 抽取战功、内功、可复用资产

现有脚本：`07-定时任务/scripts/weekly_work_review.py`

## 2. 帮助工作流程

### 数据分析

1. 澄清业务问题：目标、时间范围、指标、维度、口径、读者
2. 先查数据资产层复用：`00-知识库/数据资产/`（SQL 模板库、指标口径字典、人群包登记），再看 `00-知识库/data-analysis/`、`TEMPLATES/分析报告/`、`TEMPLATES/SQL脚本/`
3. 需要查询时使用 `kyuubi` / `sql` / `nl-sql` 相关能力，并记录数据来源和时间
4. SQL、数据、报告分别落到 `03-数据分析/SQL查询/`、`03-数据分析/数据源/`、`03-数据分析/分析报告/`
5. 沉淀：可复用取数/圈人 SQL、口径、人群包回填到 `00-知识库/数据资产/`；分析方法沉淀到 `00-知识库/data-analysis/`

### 项目推进

1. 先判断是否属于 Q3 P0；若不是，说明是否会挤占主线
2. 新项目放入 `01-业务项目/项目名/YYYYMMDD-描述/`
3. 使用 `TEMPLATES/项目启动/新项目README模板.md`
4. 重要方案与产出同步归档到 `00-知识库/文档归档/`

### Skill / Agent 建设

1. 先判断是否直接服务 `nl-sql` 或核心数据工作流
2. 新 Skill 放入 `02-Skill开发/`
3. 设计前参考 `02-Skill开发/互联网数据分析Skill体系开发Roadmap.md`
4. 能力成熟后同步到 ICE Data Workbench 对应 `agents/` 或 `skills/`

## 3. 辅助规划流程

触发语句：用户说「帮我规划」「下一步做什么」「本周重点」「重新排优先级」。

默认动作：

1. 读取 `CURRENT.md`、`WORK-PLAN.md`、`MEMORY.md`
2. 按 P0/P1/P2/明确不做四层判断
3. 输出建议时必须包含：推荐动作、为什么现在做、会挤占什么、不做的后果
4. 若调整计划，更新 `WORK-PLAN.md`；若只是建议，先不改文件
5. 重要取舍同步写入 `DECISIONS.md`

## 4. 辅助决策流程（决策生产线）

触发语句：用户说「帮我判断」「要不要做」「怎么选」「给我建议」「用决策生产线」，或发来一张决策请求卡（`TEMPLATES/决策/决策请求卡.md`）。

### 决策生产线 6 步

1. **接收输入**：读用户的决策请求卡。信息不全时，按卡片缺失字段主动追问，不带着空白硬答；能用工作区上下文（`CURRENT.md`、`WORK-PLAN.md`、数据资产层）补齐的先补齐。
2. **识别类型 → 选剧本**：对照 `00-知识库/decision-frameworks/methods/龚云荷决策剧本库.md` 判断属于哪类（需求接/插队、口径选择、方案 A/B、要不要投入、资源分配），按剧本确定分析路径；无匹配走通用框架。
3. **过护栏**：逐条检查 `00-知识库/decision-frameworks/methods/龚云荷决策护栏.md`，任一硬护栏（G1-G5）触发必须显式提示。
4. **检索证据**：按剧本指引去查——数据资产层 `00-知识库/数据资产/`、历史决策 `DECISIONS.md`、相关知识域和案例。数据标来源和时间，无数据标「推断」。
5. **产出决策简报**：按 `TEMPLATES/决策/决策简报.md` 输出（结论/置信度 → 选项对比 → 假设与反对意见 → 护栏检查 → 风险预案 → 下一步与回看）。
6. **回写闭环**：
   - 重要决策写入 `DECISIONS.md` 决策总览表（含回看时间）
   - 完整决策简报归档到 `00-知识库/decision-frameworks/my-decisions/` 对应分类目录，命名为 `YYYY-MM-DD-决策标题.md`
   - 给出 `WORK-PLAN.md`/`CURRENT.md` 更新建议

### 按决策层级提速

- 运营级（可逆小事）：直接给结论，不必跑完整生产线。
- 战术级（影响本周-本月）：跑完整生产线 + 选项对比。
- 战略级（影响 Q3 主线/不可逆）：生产线 + 预 mortem + 第一性原理，必写 `DECISIONS.md`。

### 输出铁律

- 先结论后细节。
- 强制列 1-3 条反对意见，不做单边论证。
- 没有数据支撑的判断标「推断」。
- 每个重要决策给回看时间，到期按 `DECISIONS.md`「决策回填闭环」复盘。

## 5. 文档生成与归档流程

任何正式文档生成后必须：

1. 放入对应业务目录或知识库目录
2. 复制/记录到 `00-知识库/文档归档/` 对应子目录
3. 更新 `00-知识库/文档归档/README.md` 的归档索引
4. 若是飞书文档，用 `.url.md` 记录标题、日期、链接、类型、关键词、关联项目
5. 若内容可复用，在对应知识域 README 中补引用

## 6. 维护节奏

| 文件 | 更新触发 | 频率 |
|------|----------|------|
| `CURRENT.md` | 周会后、当前重点变化 | 每周 |
| `WORK-PLAN.md` | 计划调整、任务完成、阻塞变化 | 每周或即时 |
| `MEMORY.md` | 重要决策、长期偏好、关键项目状态变化 | 每周或即时 |
| `DECISIONS.md` | 做出重要取舍或路线选择 | 即时 |
| `AGENTS.md` | 目录结构或协作规则变化 | 低频 |
| `WORKFLOWS.md` | 自动化流程变化 | 低频 |
