# ICE 消费方自助接入指南

> 读者：数据夜校学员、业务侧 PM、任何需要自助取数的人
> 目标：不用找数据组（龚云荷 / 邵石麒），自己用 ICE 完成日常取数
> 维护：龚云荷 ｜ 2026-07-13

## 一句话

ICE Data Workbench 是一个用自然语言取数的 AI 工作台：你用中文描述要什么数据，它生成并执行 SQL，返回结果和图表。你要做的只有两件事——**把需求和口径讲清楚 + 校验它生成的 SQL**。

## 五步自助取数

1. **登录** → 飞书一键登录，进入任务首页 `/dashboard`
2. **新建任务 / 选对 Agent**

   | 你的需求 | 选哪个 Agent |
   |----------|-------------|
   | 常规取数（DAU / 时长 / 留存 / CTR / UV） | 数据分析 Agent |
   | AB 实验显著性、下钻、放量决策 | 实验分析 Agent |
   | APP 版本灰度发布分析 | 灰度版本 Agent |
   | 都江堰内容池日报 | 日报推送 Agent |
   | 飞书知识库操作 | 知识库 Agent |

3. **描述需求**（口径写清楚）：时间范围 + 指标 + 维度 + 业务线。
   - 示例："查浏览器信息流 6 月分天有效 DAU 和人均时长，按顶 tab / 底 tab 拆开"
4. **校验 SQL**（唯一必须你把关的环节）：Agent 生成 SQL 后，先看
   - 表选对了吗？分区过滤（`date`/`dt`）加了吗？去重逻辑对吗？
   - 这正是数据夜校教的「自助校验 AI 生成 SQL」能力
5. **拿结果**：确认后执行，导出 CSV / 表格 / 图表；需要飞书文档可让它直接生成并归档

## 自助排查（先别找人）

| 现象 | 自己先做 |
|------|----------|
| 口径存疑 | 对照 nl-sql Skill 口径说明 / 数据资产层「指标口径字典」 |
| 查询超时 | 缩短时间范围、按 7 天分窗（kyuubi 硬上限 300s） |
| 权限不足 | 按报错里的数据权限申请地址走审批 |
| SQL 看不懂 | 让 Agent「解释这段 SQL」，或回看数据夜校第二期材料 |

## 什么时候才需要找数据组（升级路径）

自助能覆盖约 80% 的高频取数。**只有这些才升级找人**：

- 跨多域复杂归因，或口径本身有争议、需要业务对齐
- 需要新建埋点 / 接新数据源
- Push 类等有组间干扰的 AB 实验（必须用 DID，见业务踩坑硬规则，不能直接对比 TGI）

> 目标：把「凡事找数据组」变成「先自助，复杂才升级」。这就是数据夜校三阶段的落地：7 月培训 → 8-9 月 AI 工具赋能全员 → Q4 只保留顶级分析师做复杂分析。

## 遇到 badcase 怎么办

把「需求 + 生成的错误 SQL + 正确写法」反馈到 [消费与反馈登记.md](./消费与反馈登记.md) 的第三节。你的每个 badcase 都会被用来优化 nl-sql 范式库，让下次更准——这是工具越用越准的关键。
