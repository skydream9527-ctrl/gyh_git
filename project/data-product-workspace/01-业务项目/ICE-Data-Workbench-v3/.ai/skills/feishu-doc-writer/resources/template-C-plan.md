# 模板 C：规划/方案文档

## 适用场景

- 专题方案
- 某打法的完整规划文档
- 跨团队协作方案

## 骨架结构

```markdown
<callout color="blue">
【方案定位】<一句话说清本方案在战略中的位置>
【关键打法】<最多 3 个>
【三步走时点】<Phase 1 / Phase 2 / Phase 3>
【最需决策】<0-2 条>
</callout>

## 一、背景与判断

### 1.1 当前现状
<200 字以内，数据+判断>

### 1.2 问题识别（MECE）
- 问题 1：<红色，带数据>
- 问题 2：<同上>

### 1.3 根因分析
<用 3 Why 追到可操作层根因>

## 二、关键打法（2-3 个）

### 打法 1：<蓝色命名>
- **核心动作**：<一句话>
- **成功标准**：<可验证指标>
- **资源需求**：<HC / 预算>
- **风险**：<红色>

## 三、三步走路径

| 阶段 | 时点 | 目标 | 验收指标 |
|-----|-----|-----|--------|

## 四、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|-----|-----|-----|-----|

## 五、决策请求

1. <待决策项，给出 2-3 选项 + 推荐>
```

## 填写要领

1. 关键打法 ≤ 3 个
2. 每个 Phase 必须有验收指标
3. 决策请求必须给选项
