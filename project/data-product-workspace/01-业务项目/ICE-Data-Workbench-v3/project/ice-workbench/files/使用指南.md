# ICE Data Workbench 使用指南

> AI 驱动的数据工作流工作台 — 面向 AB 实验、经营分析、波动归因、数据探查、版本灰度等日常数据工作。
> 本指南同步自当前线上版本，覆盖登录、Workspace、Agent/Skill、公共区、定时任务、管理后台六大模块。

---

## 一、快速开始

### 1.1 登录

访问系统首页进入 `/login`，三种登录方式：

1. **账号密码** — 邮箱 / 工号 + 密码（默认管理员 `admin / admin123`，首次登录后请立即修改）
2. **飞书一键登录** — 点「使用小米账号登录」按钮；需管理员在 `.env` 配好 `FEISHU_APP_ID/SECRET` 并在飞书后台加白名单
3. **测试快捷登录** — 本地开发用，默认种子化若干测试用户

> 飞书登录的建号策略由管理员在 `/admin/settings` 控制：
> - `enable_feishu_auto_register`（默认开）：首次飞书登录自动建 `user` 账号
> - `enable_feishu_strict_whitelist`（默认开）：只有 `users` 表中已有的飞书账号能登录，与自动建号互斥

登录后进入 `/dashboard`。

### 1.2 Dashboard 首页布局

顶部四张快捷卡片：

| 入口 | 说明 |
|---|---|
| 📝 **新建任务** | 进入 `/create-task` 3-Step 工坊 |
| ⏱ **定时任务** | 查看 / 管理 cron 任务列表 |
| 📖 **使用指南** | 打开本文 |
| 🌐 **公共任务** | 滚动到团队资产区 |

下方分三段：

- **⚡ 快速开始** — 所有「可一键建任务」的 Agent 范式卡片，点一下自动建任务进入 Workspace
- **📋 我的任务** — 自建 + 被协作的任务
- **🌐 团队资产** — 5 个 Tab：📋 公共任务 · 🤖 Agents · 🧰 Skills · 📁 公共文件 · 📑 任务模板

### 1.3 六种工作范式

| 范式 | 代号 | 典型场景 |
|---|---|---|
| **经营分析** | `biz` | 经营指标拆解、归因、趋势洞察 |
| **AB 实验** | `ab` | 实验分流 / 显著性检验 / 结论产出 |
| **数据分析** | `data` | NL→SQL + 可视化 + 导出 |
| **波动分析** | `wave` | 核心指标异动，多维下钻定位根因 |
| **版本灰度** | `gray` | 灰度版本指标对比 + 放量决策 |
| **开放任务** | `open` | 自由选 Agent + Skills，跨范式 / 自由探索 |

### 1.4 创建任务（3 Step）

进入 `/create-task`：

1. **Step 1 — 起点**：空白 / 基于开放模板 / 基于公共任务复刻 / 从我的模板
2. **Step 2 — 配置**：范式 → Agent → Skills → 初始 Prompt → 可见性（私有 / 开放给团队）→ 可选定时
3. **Step 3 — 确认**：检查配置，点「创建」进入 `/workspace/:taskId`

> 若只想跑一次成熟模板，直接在 Dashboard 「📑 任务模板」选择已有模板「一键使用」即可。

## 二、Workspace

`/workspace/:taskId` 是日常工作主界面，三栏布局。

### 2.1 界面布局

```
┌───────────────────────────────────────────────────────────────┐
│ 顶栏：← · 任务名 · 范式 · 🔗 开放给团队 / 审核中 / 已驳回        │
├────────────┬──────────────────────────────┬───────────────────┤
│ 左侧       │ 对话区                        │ 右侧（5 Tab）     │
│            │                              │                   │
│ 📎 文件    │ 📦 模型  💾 导出  🔁 重新加载 │ 🤖 Agent          │
│ ＋ 上传    │ (聊天消息流，含工具调用卡片)   │ 📄 文件           │
│ 🔗 导入链接│                              │ 💬 对话历史       │
│            │ [输入框 / 附件 / 技能引用]     │ 📋 待办           │
│            │ 🧭 Plan Mode 开关             │ ⚙ 配置            │
└────────────┴──────────────────────────────┴───────────────────┘
```

移动端顶栏提供 📂 文件 / 🤖 详情 切换按钮，顶部操作按钮会收进 `⋯` 菜单。

### 2.2 与 Agent 对话

- 在底部输入框输入需求，`Enter` 发送，`Shift + Enter` 换行
- Agent 会理解意图并自动调用 Skills；每条消息底部会展开工具调用卡片（含参数、耗时、结果）
- 支持**流式输出**，生成过程中可点 **中止**
- 结果以文字 / 表格 / 图表 / Markdown 呈现

**提示：** 如需大段粘贴，直接粘贴即可；多行 Prompt 用 `Shift + Enter` 连续换行。

### 2.3 Plan Mode（只读规划）

点击输入框下方 🧭 按钮切换「Plan Mode」：

- 开启后 Agent 只做**只读调研**（不执行写操作）
- Agent 整理出方案后调用 `exit_plan_mode` 让你审核
- 你批准后才退出 Plan Mode 并执行

适用场景：需求不清晰、涉及改写/删除、想先对齐方案再动手。

### 2.4 文件管理（左侧边栏）

四种方式把文件带入任务：

| 方式 | 入口 | 说明 |
|---|---|---|
| **本地上传** | ＋ 按钮或拖入 | 软上限默认 20MB，硬上限 50MB（管理员可调） |
| **引用公共文件** | 消息输入中 @ 文件 | 团队沉淀的模板、规范、口径文档 |
| **Agent 产出** | 对话自动生成 | 报告、图表、数据表 |
| **🔗 导入链接** | 左侧「🔗 导入链接」按钮 | 飞书文档 / 飞书知识库链接一键导入，可点 🔄 重新抓取最新内容 |

- 点文件名 → 右侧「📄 文件」Tab 预览（支持 md/txt/csv/json/sql/py/pdf/png 等）
- 在对话中输入 `@文件名` 可把文件内容注入 Agent 上下文
- 导入链接的文件条目上会出现 🔗 角标，悬停可见原始 URL

### 2.5 Skills 绑定（右侧「🤖 Agent」Tab）

- Agent 标题下方显示「🧰 Skills（本任务）」
- 任务创建者 / admin 可点 `＋ 添加` 挂载或解绑 Skill
- Skill 列表按类别分组（SQL / 文档 / 数据 / 可视化 / 搜索等）
- 挂载后 Agent 即可通过 Tool Calling 调用

### 2.6 💾 导出对话

点顶栏 **💾 导出对话**，把当前对话（含工具调用摘要）导出为 Markdown 文件下载。

### 2.7 💡 沉淀为经验

在任何 Agent 回复上点 💡 按钮，弹出「沉淀为经验」对话框：

1. 你补充标题 / 摘要 / 标签
2. 提交后进入 admin 审核队列
3. 管理员批准 → 经验卡片注入 Agent 知识库 → 后续对话自动应用

### 2.8 🔗 任务开放给团队

顶栏 🔗 按钮把任务开放到团队。流程：

```
私有 → 点击 →「🕓 审核中」→ admin 批准 →「🔗 已开放给团队」
                          └→ 驳回 →「🚫 已驳回」（可重新提交）
```

已开放任务出现在 Dashboard「🌐 公共任务」Tab，其他同学可 Fork。

### 2.9 右侧 5 Tab 详细

| Tab | 内容 |
|---|---|
| 🤖 **Agent** | 当前 Agent 信息 + 本任务 Skills 管理 |
| 📄 **文件** | 当前打开文件预览 / 下载 |
| 💬 **对话** | 历史 Conversation 切换（同一任务可存多个会话分支） |
| 📋 **待办** | Agent 自动维护的 TODO（长任务分步跟踪） |
| ⚙ **配置** | 任务级参数：模型偏好、温度、协作者管理等 |

## 三、Agent 与 Skill

### 3.1 Agent

Agent 是绑定某个范式的智能助手，具备：

- 领域 System Prompt（固化分析方法论）
- 默认 Skills 集合
- 记忆 + 经验卡片库
- Tool Calling 执行回路（最多 5 轮工具链）

浏览所有 Agent：`/dashboard` 「🤖 Agents」Tab 或 `/agent/:agentId` 进详情。

详情页包含：Agent 简介、默认 Skills、示例任务、近期使用、（admin）Prompt 编辑 / 版本历史 / 回滚 / 沙盒测试。

### 3.2 Skill

Skill 是 Agent 可调用的工具能力。当前内置能力覆盖：

| 分类 | 代表 Skill |
|---|---|
| SQL / 数据 | `nl-sql`（自然语言→SQL）、`kyuubi`（SQL 执行网关）、`nl-mapping-table-sql` |
| 文档输出 | `docx`、`pptx`、`xlsx`、`pdf` |
| 搜索 / 知识 | `feishu`（飞书文档/知识库/多维表格/云盘全家桶） |
| 其它 | 管理员可在 `/admin/skills` 新增自定义 Skill（JSON Schema + 沙盒测试） |

Agent 在对话中自动选择合适的 Skill，用户一般无需手动触发。

### 3.3 Agent 进化闭环

```
对话 ──► 💡 沉淀为经验 ──► admin 审核 ──► 经验卡注入 Agent ──► 下次对话自动应用
```

加速进化：

- 对优秀回答点 👍 / 对错误回答点 👎
- 手动 💡 沉淀关键对话段
- admin 定期在 `/admin/experience-cards` 审批

## 五、定时任务

进入 `/scheduled-tasks`：

- 列表展示所有 cron 任务（自建 + 被授权的）
- 每条可看：cron 表达式（人类可读）、下一次执行时间、最近 N 次运行、运行结果
- 后端调度器每 **20 秒** 扫描一次，到点自动触发

### 5.1 创建方式

1. 在 `/create-task` Step 2 勾选「启用定时执行」
2. 填 cron 表达式（默认 `0 9 * * *` = 每日 09:00）
3. 填 schedule_prompt（调度时向 Agent 发送的 Prompt）
4. 保存 → 任务即登记到调度器

### 5.2 历史回放

每次触发会生成一条新 Conversation，可在 `/scheduled-tasks` 点「历史回放」查看完整执行上下文（Prompt / Agent 输出 / 工具调用 / 产物文件）。

## 七、管理员后台（admin / super_admin）

顶部菜单点 🛡 **管理** 进入 `/admin`，左侧分组：

### 概览 / 监控

- **总览** — 用户数、任务数、24h 对话量、告警、热门 Agent 排行
- **用量与成本**（`/admin/usage`）— 5 Tab：Token / 请求 / 用户 / Agent / 模型，支持月度预算 + CSV 导出
- **SQL 审计**（`/admin/sql-audit`）— 按风险三级分类，可 CSV 导出
- **操作审计**（`/admin/audit`）— 全量后台操作日志

### 审核中心

- **审核中心**（`/admin/review-center`）— 聚合三类待办计数
- **经验卡片**（`/admin/experience-cards`）— 批准后注入 Agent
- **公共任务**（`/admin/public-tasks`）— 审核 Workspace 🔗 开放的任务
- **任务模板**（`/admin/templates`）— 审核同学提交的模板

### 资源维护

- **Agents**（`/admin/agents` + `/admin/agents/:id`）— 4 Tab：基础 / Prompt / Skills / 版本，支持回滚 + 沙盒测试
- **Skills**（`/admin/skills`）— CRUD + JSON Schema 验证 + 沙盒 test-run
- **知识库**（`/admin/knowledge-bases`）— 飞书 wiki 同步 + 同步日志
- **公共文件**（`/admin/files`）— 上传、编辑、置顶

### 用户与系统

- **用户管理**（`/admin/users`）— CRUD；角色编辑限 `super_admin`
- **系统设置**（`/admin/settings`，仅 super_admin）— 全局开关 / LLM 默认参数 / 文件大小限额 / 公告 / 飞书自动建号与白名单

## 八、常见问题

### Q: Agent 回复不准确？

1. 描述更具体的指标名 / 时间窗口 / 维度
2. 用 `@文件` 把口径文档带入上下文
3. 切换到更强模型（Claude Opus 4.7 / GPT-5.4）
4. 复杂任务先开 🧭 **Plan Mode**，让 Agent 先出方案再执行
5. 对错误回答点 👎 并备注

### Q: 如何上传数据进行分析？

- 左侧「📎 文件」→ ＋ 按钮 或 直接拖入聊天区
- 软上限默认 **20MB**，超过警告但可继续；硬上限 **50MB**，超过拒绝（管理员可调）
- 常见格式：`csv / md / txt / json / sql / py / html / pdf / png / xlsx / docx / pptx`
- CSV 自动解析列结构；二进制只保留元信息供引用

### Q: 如何引入飞书文档？

- 左侧「🔗 导入链接」→ 粘贴飞书文档 / 知识库 / 多维表格 链接
- 系统拉取内容落盘为本任务文件，可在对话中 `@` 引用
- 点文件行的 🔄 重新抓取最新内容

### Q: 启动后登录页的飞书按钮点不动？

`.env` 缺 `FEISHU_APP_ID` / `FEISHU_APP_SECRET`。填好后重启后端即可。未配置时按钮返回 `FEISHU_NOT_CONFIGURED`，不影响账号密码登录。

### Q: 对话卡住 / 流式断开？

- 顶栏横幅若显示 **WS_DISCONNECTED** 会自动重连
- 若显示 **LLM_KEY_MISSING**，说明 `.env` 缺 `MIFY_GATEWAY_API_KEY` 或 `ANTHROPIC_API_KEY`
- 其它异常可点顶栏 **🔁 重新加载** 重拉任务详情（保持 WS 连接）

### Q: 我的对话数据安全吗？

- 对话与文件落到本机 `tasks/<task_id>/` 和 `users/<user_id>/` 目录，G3 文件优先存储
- 权限隔离：创建者 + 被授权协作者可见；管理员后台看得到元数据与操作日志，但默认不直读用户对话正文
- SQLite 索引损坏时管理员可用 `python scripts/rebuild_index.py` 重建

### Q: 公共任务 / 模板 / 经验卡片的审核在哪？

`/admin/review-center` 聚合三类待办。admin 可直接批准 / 驳回 / 退回修改。
