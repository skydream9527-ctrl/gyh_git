# 反馈与稳态

> 反馈（feedback）是控制论的最小核心概念。一旦你建立起"反馈循环"的思维，你会在自动驾驶、人体调温、企业 OKR、LLM Agent、生态平衡等几十个看似毫无关系的系统里看到**同一种结构**——这是控制论最深刻的洞察。

---

## 一、反馈的最小定义

```
反馈 = 系统输出的一部分被引回作为输入

  Input ─→ ┌─────┐ ─→ Output
           │ Sys │       │
           └─────┘ ←─────┘
                  feedback
```

**只此一图，就是控制论的 95%**。剩下的内容都是这个结构的精细化、变种、复合。

---

## 二、负反馈 vs 正反馈

### 负反馈（Negative Feedback）

输出**反向**修正输入，把系统拉向目标。

```
设定温度 22°C ──┐
               ▼
              误差 = 22 - 当前温度
               │
               ▼
         (空调控制器决定开关)
               │
               ▼
              温度变化
               │
               └──── 测量温度 ────┘  闭环
```

**功能**：稳定。负反馈是世界上 **99% 的"自动调节"** 的本质：

| 系统 | 负反馈机制 |
|---|---|
| 恒温器 | 温度高 → 降温；温度低 → 升温 |
| 巡航控制 | 速度低 → 加油；速度高 → 减油 |
| 人体血糖 | 血糖高 → 胰岛素 → 降；血糖低 → 胰高血糖素 → 升 |
| 市场价格 | 价高 → 供给增 + 需求减 → 价格降 |
| Agent ReAct | 错了 → reflexion → 修正下一步 |
| RLHF reward | 输出"差" → reward 负 → 模型调整 |

### 正反馈（Positive Feedback）

输出**同向**放大输入。结果是失控（爆炸 / 共振 / 雪崩）。

```
设定 ──→ 误差 ──→ 控制 ──→ 输出 ──→ 误差更大 ──→ ...
                              ↑                         │
                              └─────────────────────────┘
```

| 系统 | 正反馈机制 |
|---|---|
| 麦克风啸叫 | 声音 → 麦 → 喇叭 → 更大声音 |
| 银行挤兑 | 担心倒闭 → 取钱 → 真倒闭 |
| 双边市场 | 用户多 → 商家多 → 用户更多（"网络效应"） |
| 病毒 / 谣言传播 | 转发越多 → 看到越多 → 转发更多 |
| 通胀预期 | 预期涨价 → 提前买 + 涨工资 → 真涨价 |
| 增长引擎（产品） | 用户带用户 |

正反馈不一定是"坏"——网络效应、产品增长靠它。但**正反馈系统必须配上"刹车"（边界 / 饱和 / 抑制）**，否则会崩溃。

> **设计任何带正反馈的系统时，第一个问题永远是：刹车在哪？**

## 三、稳态（Homeostasis）

**稳态**是负反馈的最终状态——系统在内外干扰下仍能维持关键变量在某个范围。

```
干扰    ── ┌────────┐
（环境）   │  系统   │ ── 关键变量被维持
扰动    ── │ + 控制器│    （即使有干扰）
            └────────┘
              ↑
              └── 反馈环
```

来源：1929 年 Walter Cannon 提出 *homeostasis* 描述生物体内部环境的稳定。Wiener 把它推广到所有反馈系统。

**例子**：

- 人体核心温度 37°C ± 0.5°C（无论你在沙漠还是冰川）
- 公司利润率维持在某个区间（无论市场如何）
- LLM Agent 的"步数"被限制在 max_iterations
- 互联网协议的 TCP 拥塞控制把吞吐量维持在最优带宽附近

### 稳态 vs 平衡态

```
平衡态（Equilibrium）：    死的稳定，比如冷却的咖啡
稳态（Homeostasis）：       动态稳定，主动消耗能量来维持
```

**生命系统、复杂系统都是稳态而非平衡态**——它们靠不断"消耗"能量来维持秩序。

## 四、Ashby 的 Ultrastability（高阶稳态）

W. Ross Ashby 把稳态再推一阶：

```
普通稳态：    系统能在已知干扰下回到目标
Ultrastability：当干扰超出预期时，系统能"自我重组"控制策略
```

例子：
- 人在突遇陌生环境时，不是用旧策略硬撑，而是**学习新策略**
- LLM Agent 在第一种工具失败后，自己换工具尝试
- 大型组织在市场剧变时，不是优化原有流程，而是重组业务

**Ultrastability 是适应性、学习能力、智能的形式化前身**。AI 强化学习里的 meta-learning 就是这一思想的现代版。

## 五、PID 控制器：工程版的负反馈

工程上把负反馈具体化成一个公式——**PID 控制器**：

```
u(t) = K_p · e(t) + K_i · ∫e(t)dt + K_d · de(t)/dt

       │            │              │
       │            │              └── 微分项：误差变化的趋势
       │            └── 积分项：累计误差
       └── 比例项：当前误差
```

直觉：

- **P（比例）**：差多少补多少（响应快但有"稳态误差"）
- **I（积分）**：长时间持续的误差累计补偿（消除稳态误差，但响应慢、可能震荡）
- **D（微分）**：根据误差变化趋势"提前刹车"（防止过冲）

**这是工业自动化最常用的控制器，覆盖几乎所有恒温、流量、转速、姿态控制场景**。

LLM Agent 视角的类比：

- P：当前 step 的偏差（"现在跑偏了多少"）
- I：累计偏差（"这次任务跑偏多次了"）→ Reflexion 类方法
- D：趋势（"越跑越偏吗？"）→ 早停 / 升级到更强模型

## 六、反馈系统的几个关键性质

### 1. 增益（Gain）

反馈环每跑一圈"放大多少"。增益过大 → 震荡或失稳；过小 → 响应慢。
- 调 PID = 调增益参数
- 调 LLM Agent 的 max_steps、reflexion 频率 = 调隐含增益

### 2. 时延（Dead Time）

从"输出变化"到"反馈到达"的时间。
- 时延大 → 系统震荡 / 失稳（控制器在"过期信息"上做决策）
- 业务例子：财报季度性发布 = 大时延 → 经营调整迟滞

### 3. 饱和（Saturation）

实际系统的输出受物理 / 资源 / 预算限制——不能无限放大。
- 含义：负反馈遇到饱和会失效
- 例子：空调到达功率上限后，温度持续上升

### 4. 多变量耦合

真实系统通常**多个反馈环互相纠缠**（温度影响湿度，湿度影响温度）。
- 这是为什么"开放的复杂巨系统"难以单一控制器解决——需要 [钱学森的综合集成方法](../qian-xuesen/meta-synthesis.md)

## 七、反馈思维的业务应用

### 1. OKR / KPI 系统

OKR / KPI 本质是反馈控制器：

```
目标 (O)        ──── setpoint
   ↓
关键结果 (KR)   ──── 测量值
   ↓
偏差             ──── 是否需要调整执行
```

**陷阱**：
- 测量周期太长 → 大时延 → 决策滞后
- KR 选错（测了"外表"不是"实质"）→ Goodhart's Law
- 太多 KR → 多变量耦合，互相干扰

### 2. 产品增长循环

产品增长引擎都是正反馈 + 负反馈的组合：

```
正反馈：用户增长 → 推荐 → 更多用户  (engine)
负反馈：服务质量下降 → 流失 → 用户减少  (brake)

设计良好的产品 = 让正反馈可控放大，让负反馈晚一点起作用
```

### 3. LLM Agent 设计

```
Thought-Action-Observation 是反馈环
Reflexion 是二阶反馈
max_steps 是饱和保护
工具调用错误率监控 = 增益调节
```

→ 详见 [../../agent-llm/agents/react-and-variants.md](../../agent-llm/agents/react-and-variants.md)

### 4. 企业组织

钱学森的"系统工程方法论"基本就是把反馈控制论用到组织管理：

- 总体设计部 = 主控制器
- 各分系统 = 局部反馈环
- 综合集成 = 多反馈环协调

→ 详见 [../qian-xuesen/systems-engineering-method.md](../qian-xuesen/systems-engineering-method.md)

## 八、几个反直觉的事实

### 1. "目标更明确 ≠ 系统更稳"
- 太严格的 setpoint 让系统在小扰动下也大动作
- 工程上叫 "stiff control" → 容易震荡
- 启发：OKR 的"必达"目标 + "挑战"目标的两层设计有道理

### 2. "提高响应速度 ≠ 系统更好"
- 反馈太快 + 增益高 → 容易震荡
- 反馈太慢 + 增益低 → 响应迟钝
- 控制论给的答案：响应速度要匹配系统的固有时间常数

### 3. "正反馈不全是坏的"
- 学习曲线、复利、网络效应都是正反馈的好处
- 但必须配 brake；无 brake 的正反馈 = 灾难

### 4. "反馈不一定要数字"
- 反馈可以是定性信号（好评 / 差评、用户访谈、语义判断）
- LLM-as-Judge 就是定性反馈数字化

### 5. "Ashby 必要多样性定律"（重要）
> 只有控制器的多样性 ≥ 干扰的多样性，才能维持系统稳定。

直白：复杂世界需要复杂的控制器。
推论：组织 / 系统的"简化"必有代价。

详见 [ashby-requisite-variety.md](ashby-requisite-variety.md)。

## 九、Checklist：用反馈思维诊断系统

```
□ 1. 这个系统的目标（setpoint）是什么？明确可测量吗？
□ 2. 测量值（feedback signal）从哪里来？时延多大？
□ 3. 是负反馈（求稳）还是正反馈（求增长）？
□ 4. 正反馈的"刹车"在哪？
□ 5. 增益（响应力度）合适吗？是否震荡 / 迟滞？
□ 6. 系统是否有饱和限制？接近限制时怎么办？
□ 7. 多个反馈环之间有耦合 / 冲突吗？
□ 8. 当干扰超出预期，系统能自我调整吗（ultrastability）？
□ 9. 测量是否被"游戏化"（Goodhart's Law）？
□ 10. 控制器多样性 ≥ 干扰多样性吗？（Ashby）
```
