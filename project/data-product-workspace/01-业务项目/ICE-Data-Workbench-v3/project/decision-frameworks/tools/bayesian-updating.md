# 贝叶斯更新 (Bayesian Updating)

> 决策不是一次性判断，而是持续更新信念的过程。

## 什么是贝叶斯更新

贝叶斯更新是一种基于新证据不断修正先验判断的决策方法。核心公式：

```
P(假设|证据) = P(证据|假设) × P(假设) / P(证据)
```

即：

```
后验概率 = 似然度 × 先验概率 / 证据概率
```

**关键思想**：

- **先验 (Prior)**：决策前的初始信念
- **似然 (Likelihood)**：在假设为真的条件下，观察到当前证据的概率
- **后验 (Posterior)**：结合新证据后更新的信念
- **迭代**：后验成为下一轮的先验

## ASCII 流程图

```
先验信念 P(H)
    │
    ▼
观察到新证据 E
    │
    ▼
计算似然 P(E|H)
    │
    ▼
贝叶斯公式计算
    │
    ▼
后验信念 P(H|E)
    │
    ▼
后验 → 新一轮先验 ──→ 回到顶部
```

## 使用步骤

### 步骤 1：明确假设和先验

**操作**：将模糊的判断转化为可量化的假设。

**方法**：

1. 写出你要判断的假设 H
2. 基于已有信息给出先验概率 P(H)
3. 给出置信区间或最低/最高估计

**示例**：

| 假设 | 先验概率 | 依据 |
|------|----------|------|
| 新功能上线后转化率提升 >10% | 60% | 类似功能历史数据 |
| 竞品下个月跟进同样功能 | 40% | 竞品迭代周期分析 |
| 技术方案能在2周内完成 | 70% | 团队历史交付速度 |

**常见错误**：

- ❌ 先验 = 0 或 1（绝对确定，无法被证据更新）
- ❌ 忽略基础概率（Base Rate Neglect）
- ✅ 先验应有数据或经验支撑，允许被修正

### 步骤 2：识别和收集新证据

**操作**：明确哪些信息能改变你的判断。

**方法**：

1. 列出能影响假设成立概率的证据类型
2. 设计最小化验证实验
3. 按信息价值排优先级

**信息价值排序**：

| 证据类型 | 信息价值 | 获取成本 | 推荐度 |
|----------|----------|----------|--------|
| A/B测试结果 | 高 | 中 | ★★★★★ |
| 用户访谈 | 中高 | 低 | ★★★★ |
| 竞品动作 | 中 | 低 | ★★★ |
| 行业报告 | 中低 | 低 | ★★ |
| 内部直觉 | 低 | 无 | ★ |

### 步骤 3：计算后验概率

**操作**：用贝叶斯公式更新信念。

**简化计算法（避免复杂概率运算）**：

使用** odds ratio **方法：

```
后验 odds = 先验 odds × 似然比

先验 odds = P(H) / P(¬H)
似然比 = P(E|H) / P(E|¬H)
```

**实战示例**：

> 假设：新推荐算法能提升CTR 15%以上
> 
> - 先验概率 P(H) = 0.40（历史类似项目成功率约40%）
> - 灰度3天后，CTR提升18%
> - P(提升18%|H为真) = 0.70（如果真的有效，大概率能看到这个结果）
> - P(提升18%|H为假) = 0.15（如果实际无效，灰度偏差导致这个结果的概率低）
> 
> 先验 odds = 0.40 / 0.60 = 0.67
> 似然比 = 0.70 / 0.15 = 4.67
> 后验 odds = 0.67 × 4.67 = 3.13
> 后验概率 = 3.13 / (3.13 + 1) = 76%
> 
> 结论：从40%提升到76%，可以增加灰度范围继续验证

### 步骤 4：决策与迭代

**操作**：基于更新后的概率做出决策，并继续收集证据。

**决策阈值建议**：

| 后验概率 | 决策建议 | 行动 |
|----------|----------|------|
| < 30% | 放弃或大幅调整 | 换方向或降低投入 |
| 30% - 50% | 继续验证，低成本试错 | 小规模实验 |
| 50% - 70% | 条件性推进 | 带对冲方案推进 |
| 70% - 90% | 积极推进 | 正常投入资源 |
| > 90% | 全力执行 | 最大化投入 |

**迭代规则**：

- 后验 → 新一轮先验
- 每轮至少获取一条新证据
- 连续 2 轮概率变化 < 5% → 判断已收敛，可以决策
- 概率在 45%-55% 徘徊 → 证据不足以区分，需要更有区分度的证据

## 实战案例

### 案例：AI功能是否值得投入

**背景**：产品团队考虑是否投入2个月开发AI智能推荐功能。

**步骤 1 - 先验**：

| 假设 | 先验 | 依据 |
|------|------|------|
| H：AI推荐能提升用户留存5%+ | 35% | 行业报告显示同类功能留存提升中位数3-8% |
| ¬H：AI推荐无法提升留存5%+ | 65% | 公司此前AI功能留存提升均低于3% |

**步骤 2 - 第一轮证据**：

- 用户调研（N=200）显示62%用户表示会使用AI推荐
- 但调研意愿 ≠ 实际行为，历史转化率约40%

**似然估计**：

- P(调研62%正向|H为真) = 0.80
- P(调研62%正向|H为假) = 0.45（即使无效，用户也倾向正向回答）

**更新**：

- 先验 odds = 0.35/0.65 = 0.54
- 似然比 = 0.80/0.45 = 1.78
- 后验 odds = 0.54 × 1.78 = 0.96
- 后验概率 = 0.96/1.96 = **49%**

→ 不确定性仍然很高，继续验证

**步骤 3 - 第二轮证据**：

- MVP灰度2周，实验组留存提升3.2%（未达5%目标）
- 但使用频次Top 30%用户留存提升7.8%

**似然估计**：

- P(Top30%用户提升7.8%|H为真) = 0.65
- P(Top30%用户提升7.8%|H为假) = 0.12

**更新**：

- 先验 odds = 0.96（上一轮后验）
- 似然比 = 0.65/0.12 = 5.42
- 后验 odds = 0.96 × 5.42 = 5.20
- 后验概率 = 5.20/6.20 = **84%**

→ 置信度显著提升，决定对高频用户优先上线

**决策**：

| 阶段 | 时间 | 目标 | 关键任务 |
|------|------|------|----------|
| 定向优化 | 2周 | 提升低频用户使用率 | 优化推荐触达策略 |
| 全量上线 | 4周 | 全用户留存提升5%+ | 分层推荐策略 |
| 效果评估 | 2周 | 验证全量效果 | A/B测试 + 贝叶斯更新 |

## 常见错误与规避

| 错误 | 后果 | 规避方法 |
|------|------|----------|
| 确认偏误：只寻找支持假设的证据 | 后验被人为抬高 | 主动寻找反驳证据，计算 P(E\|¬H) |
| 锚定先验：先验过于极端 | 证据无法有效更新信念 | 先验概率避免 <10% 或 >90% |
| 忽略基础概率 | 高估个别案例的概率 | 始终从基础概率出发 |
| 过度自信：似然估计过于极端 | 后验波动剧烈 | 似然比控制在 0.1-10 范围内 |
| 一次定论：不做迭代 | 错过收敛过程 | 至少2-3轮更新后再做最终决策 |

## 高级技巧

### 1. 贝叶斯A/B测试

传统A/B测试需要预设样本量，贝叶斯方法可以：

- 任意时间点查看结果
- 量化"B比A好的概率"
- 设置决策阈值自动停止

```
B优于A的概率 > 95% → 选用B
B优于A的概率 < 5%  → 选用A
中间区间 → 继续测试
```

### 2. 多假设贝叶斯更新

同时维护多个竞争假设：

| 假设 | 先验 | 第1轮后验 | 第2轮后验 | 第3轮后验 |
|------|------|-----------|-----------|-----------|
| H1：效果显著 | 0.30 | 0.45 | 0.52 | 0.61 |
| H2：效果一般 | 0.40 | 0.35 | 0.30 | 0.25 |
| H3：无效 | 0.30 | 0.20 | 0.18 | 0.14 |

### 3. 贝叶斯与决策树结合

在每个决策节点用贝叶斯更新概率，而非使用静态概率。

## 与其他工具的结合

### 贝叶斯 + RICE

- RICE中的 Confidence 分数可以用贝叶斯后验概率替代
- 每次获取新证据后更新 RICE 评分

### 贝叶斯 + 预-mortem

- 预-mortem识别的失败原因作为先验假设
- 针对每个失败假设收集证据并贝叶斯更新
- 更新后概率 > 阈值的失败原因需要主动对冲

### 贝叶斯 + 决策矩阵

- 决策矩阵中的权重或概率参数用贝叶斯后验替代主观估计
- 敏感性分析时，用贝叶斯后验分布替代点估计

## 总结

贝叶斯更新的核心价值：

1. **结构化更新信念**——避免"拍脑袋"修正
2. **量化不确定性**——用概率而非确定性表达判断
3. **迭代逼近真相**——每一轮证据都让判断更准确
4. **避免极端判断**——先验和似然防止过度自信
5. **决策可追溯**——每次更新都有据可查

**记住**：
- 先验不是偏见，是已有知识的总结
- 证据的区分度比数量更重要
- 不更新 = 不学习，固执 ≠ 坚定
- 贝叶斯是思维框架，不是计算工具——近似估计也远好于不做更新
