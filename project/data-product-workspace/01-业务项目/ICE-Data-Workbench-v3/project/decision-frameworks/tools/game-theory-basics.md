# 博弈论基础 (Game Theory Basics)

> 理解利益相关方的策略互动，预测行为，设计机制。

## 什么是博弈论？

博弈论 (Game Theory) 是研究**策略性决策**的数学框架。当你的决策结果不仅取决于你自己，还取决于他人的选择时，博弈论帮你系统化地分析这些互动。

**核心假设**：
- 每个参与者都是理性的（追求自身利益最大化）
- 每个参与者都知道其他人也是理性的
- 参与者的策略互动决定了最终结果

## 基本模型分类

### 1. 按参与方数量

| 类型 | 参与方 | 示例 |
|------|--------|------|
| 双人博弈 | 2方 | 买卖双方谈判 |
| 多人博弈 | 3方及以上 | 平台生态中多方博弈 |

### 2. 按利益关系

| 类型 | 特征 | 示例 |
|------|------|------|
| 零和博弈 | 一方收益 = 另一方损失 | 市场份额竞争 |
| 非零和博弈 | 可双赢或双输 | 供应链合作 |
| 正和博弈 | 总收益增加 | 生态系统共建 |
| 负和博弈 | 总收益减少 | 价格战 |

### 3. 按信息结构

| 类型 | 特征 | 示例 |
|------|------|------|
| 完全信息博弈 | 各方知道所有规则和收益 | 棋类游戏 |
| 不完全信息博弈 | 部分信息私有 | 谈判、拍卖 |
| 完美信息博弈 | 能看到对方所有行动 | 象棋 |
| 不完美信息博弈 | 看不到对方部分行动 | 扑克 |

## 五大核心概念

### 概念 1：纳什均衡 (Nash Equilibrium)

> 没有任何一方能通过单方面改变策略来增加自己的收益。

```
        竞争对手
        降价    不降价
我  ┌─────────┬─────────┐
方  │ 3, 3   │ 1, 5    │
降  │ (双输)  │ (我输)  │
价  ├─────────┼─────────┤
不  │ 5, 1   │ 4, 4    │ ← 纳什均衡
降  │ (对手输)│ (双赢)  │
价  └─────────┴─────────┘
     (收益格式：我方, 对手)
```

**互联网应用**：
- 价格战：双方降价是纳什均衡（双输），但谁都不敢单方面不降价
- 平台补贴：补贴大战是典型的囚徒困境

**识别方法**：
1. 列出所有参与方的策略
2. 画出收益矩阵
3. 找出"没有单方面偏离动力"的策略组合

### 概念 2：囚徒困境 (Prisoner's Dilemma)

> 个体理性导致集体非理性。

```
        参与者B
        合作    背叛
    ┌─────────┬─────────┐
合  │ 3, 3   │ 0, 5    │
作  │ (双赢)  │ (被坑)  │
A   ├─────────┼─────────┤
背  │ 5, 0   │ 1, 1    │ ← 纳什均衡
叛  │ (坑人)  │ (双输)  │
    └─────────┴─────────┘
```

**互联网典型案例**：
- 外卖平台补贴战：不补贴 → 对手补贴 → 流量流失 → 只能补贴
- 短视频内容同质化：不做 → 对手做 → 用户流失 → 跟着做

**破解方法**：
1. **重复博弈**：建立长期合作关系（多次互动让合作更有利）
2. **惩罚机制**：背叛者会被惩罚（差评系统、信用体系）
3. **信号传递**：提前表明合作意愿（公开承诺、保证金）
4. **改变收益结构**：让合作收益 > 背叛收益（联盟规则、行业标准）

### 概念 3：信号传递 (Signaling)

> 通过可观察的行为传递不可观察的信息。

**类型**：

| 信号类型 | 机制 | 互联网案例 |
|----------|------|-----------|
| 分离均衡 | 高质量者愿意发信号，低质量者不愿 | 付费认证、保证金 |
| 混同均衡 | 所有人都发同样的信号 | 统一UI风格、品牌标准 |
| 半分离均衡 | 部分能区分 | 高级会员标识 |

**设计原则**：
- 信号成本必须与质量负相关（高质量者发信号成本更低）
- 信号必须可观察
- 信号必须难以伪造

### 概念 4：机制设计 (Mechanism Design)

> 博弈论的逆问题：设计规则让参与者的自利行为产生期望结果。

```
传统博弈论：规则 → 预测结果
机制设计：  期望结果 → 设计规则
```

**设计步骤**：

1. **定义目标**：你想要什么结果？
2. **识别参与方**：谁会参与？他们的偏好是什么？
3. **设计激励**：让参与方的自利行为导向目标
4. **验证均衡**：在纳什均衡下，结果是否符合目标？
5. **检查稳健性**：信息不对称时还能工作吗？

**互联网案例**：

| 机制 | 目标 | 设计 | 均衡行为 |
|------|------|------|----------|
| 拍卖广告位 | 最大化广告收入 | GSP拍卖 | 广告主如实出价 |
| 评价系统 | 获取真实评价 | 双盲评价 | 用户如实评价 |
| 推荐系统 | 最大化用户满意 | 多臂老虎机 | 探索-利用平衡 |
| 补贴规则 | 防止刷单 | 最低消费门槛 | 真实用户使用 |

### 概念 5：序贯博弈 (Sequential Games)

> 参与方轮流行动，后动者能看到前动者的选择。

**分析工具：逆向归纳法 (Backward Induction)**

```
从最后一步开始推理：
  → 最后行动者选什么？
    → 倒数第二个行动者会怎么回应？
      → ...回推到第一步
```

**互联网案例**：
- 先发优势 vs 后发优势：先发者定义规则，后发者学习改进
- 平台进入决策：大平台进入 → 小平台是否还进入？

## 使用步骤

### Step 1：识别博弈结构

回答以下问题：

- [ ] 有哪些参与方？
- [ ] 每方的可选策略是什么？
- [ ] 各方在不同策略组合下的收益是什么？
- [ ] 信息结构是怎样的？（完全/不完全信息）
- [ ] 行动顺序是怎样的？（同时/序贯）

### Step 2：构建博弈模型

**收益矩阵模板**（双人博弈）：

```markdown
        参与者B
        策略1   策略2   策略3
    ┌────────┬────────┬────────┐
策  │ a, b   │ c, d   │ e, f   │
略  ├────────┼────────┼────────┤
1   │ g, h   │ i, j   │ k, l   │
A   ├────────┼────────┼────────┤
策  │ m, n   │ o, p   │ q, r   │
略  └────────┴────────┴────────┘
3   (格式：A的收益, B的收益)
```

**收益量化指南**：

| 方法 | 适用场景 | 操作 |
|------|----------|------|
| 绝对数值 | 可量化收益 | 直接用收入/利润 |
| 相对排序 | 难量化收益 | 1=最好，2=次之... |
| 效用函数 | 风险偏好不同 | 用U(x)衡量主观价值 |

### Step 3：求解均衡

1. **寻找占优策略**：是否存在"无论对手怎么做都最优"的策略？
2. **剔除劣策略**：去掉永远不会被选择的策略
3. **找纳什均衡**：找出"没有单方面偏离动力"的策略组合
4. **多重均衡处理**：如果有多个均衡，用聚点、帕累托优势等筛选

### Step 4：分析结果

- [ ] 均衡结果是否是帕累托最优？
- [ ] 是否存在囚徒困境（个体理性导致集体非理性）？
- [ ] 信息不对称如何影响结果？
- [ ] 重复博弈能否改善结果？

### Step 5：设计干预

如果均衡不理想：

| 问题 | 干预策略 | 示例 |
|------|----------|------|
| 囚徒困境 | 改变收益结构 | 引入惩罚/奖励机制 |
| 信息不对称 | 信号传递/筛选 | 认证、担保、测试 |
| 搭便车 | 排他机制 | 付费墙、会员制 |
| 恶性竞争 | 合作机制 | 行业联盟、标准制定 |

## 实战案例：SaaS 定价博弈

**场景**：两家 SaaS 公司争夺同一市场，需要决定定价策略。

**Step 1：识别博弈结构**

| 维度 | 分析 |
|------|------|
| 参与方 | 公司A（先发者）、公司B（后发者） |
| 策略 | 高价/低价 |
| 信息 | 双方知道对方定价（完全信息） |
| 顺序 | 序贯博弈：A先定价，B后定价 |

**Step 2：构建博弈模型**

```
市场总容量：1000客户
高价 = ¥299/月，低价 = ¥199/月
产品差异不大，客户主要看价格

收益矩阵（月收入，万元）：
           公司B
        高价    低价
    ┌─────────┬─────────┐
高  │ 15, 10  │ 5, 12   │
价  │ (各得半)│ (B抢客户)│
A   ├─────────┼─────────┤
低  │ 12, 5   │ 8, 6    │
价  │ (A抢客户)│(各得半) │
    └─────────┴─────────┘
```

**Step 3：求解均衡**

- 逆向归纳：B看到A高价 → B选低价（12>10）；B看到A低价 → B选低价（6>5）
- A预判B总选低价 → A选低价（8>5）
- **纳什均衡**：(A低价, B低价)

**Step 4：分析结果**

- 均衡(A低价, B低价)收益(8,6) < 合作(A高价, B高价)收益(15,10)
- 典型囚徒困境

**Step 5：设计干预**

| 策略 | 方法 | 预期效果 |
|------|------|----------|
| 差异化 | A做高端功能，B做性价比 | 改变收益结构，走出囚徒困境 |
| 用户锁定 | A推年付优惠 | 提高切换成本，减少价格敏感 |
| 价值信号 | A发布ROI白皮书 | 信号传递，让用户认可高价 |
| 市场细分 | A聚焦大客户，B聚焦小客户 | 各自成为细分领域的领导者 |

## 高级技巧

### 1. 混合策略

当纯策略没有纳什均衡时，使用概率化策略：

```
参与者A以概率p选策略1，概率(1-p)选策略2
使得对手对两种策略无差异（收益相等）
```

**案例**：A/B测试中的探索-利用平衡——以概率ε随机探索，概率(1-ε)选择当前最优。

### 2. 重复博弈与合作

| 博弈类型 | 合作条件 | 策略 |
|----------|----------|------|
| 有限次重复 | 几乎不可能合作 | 逆向归纳导致每轮都背叛 |
| 无限次重复 | 贴现因子足够大 | 针锋相对(Tit-for-Tat) |

**Tit-for-Tat 策略**：
1. 第一轮合作
2. 之后每轮模仿对手上一轮的行为
3. 简单、善意、可激怒、宽容

### 3. 机制设计深入

**VCG机制**：让每个参与者如实报告自己的估值

```
支付 = 没有你时其他人的总收益 - 有你时其他人的总收益
```

**应用**：广告位拍卖、频谱拍卖、资源分配

### 4. 演化博弈

放弃完全理性假设，用"适者生存"替代：

```
策略表现好 → 更多参与者采用 → 均衡逐步形成
```

**应用**：市场趋势分析、商业模式演化、平台生态发展

## 常见误区

| 误区 | 问题 | 修正 |
|------|------|------|
| 假设对手不理性 | 高估自己的优势 | 永远假设对手至少和你一样聪明 |
| 忽略重复效应 | 单次博弈结论直接套用 | 实际大多是重复博弈 |
| 过度量化收益 | 强行给不可比因素赋值 | 用相对排序或效用函数 |
| 静态分析 | 只看当前均衡 | 考虑动态演化和学习 |
| 忽略心理因素 | 纯理性假设不成立 | 结合行为经济学修正 |

## 与其他工具的结合

### 博弈论 + SWOT分析

```
SWOT识别环境 → 博弈论分析策略互动 → 预测均衡结果
```

适用于：竞争策略制定

### 博弈论 + 决策矩阵

```
决策矩阵评估选项 → 博弈论分析对手反应 → 更新决策
```

适用于：定价、市场进入

### 博弈论 + 风险评估

```
风险评估识别威胁 → 博弈论分析威胁来源的动机 → 设计防范机制
```

适用于：安全策略、合规策略

## 记住

- 博弈论帮你看清**互动的影响**，不是给你唯一答案
- 纳什均衡≠最优结果（可能是囚徒困境）
- 机制设计比改变人性更可靠
- 重复博弈中，声誉和信任有价值
- 好的机制让自利行为导向共赢
