# 功效与样本量

> 本文目标：让你能独立回答"这个实验我要跑多少天 / 多少用户才靠谱"，并理解为什么很多实验"跑了等于没跑"。

---

## 一、为什么这是 AB 测试里最重要的一节

如果一个实验**功效不够**，无论你怎么"看 p 值"都没意义——因为即使新版本真的更好，你也大概率检测不出来。

业界一个被反复引用的数据（Kohavi）：很多公司的实验 **70% 以上是不显著的**，其中相当一部分根本不是因为没效果，而是因为**样本量从设计之初就不够**。这是"假阴性"的工业级浪费。

**核心原则**：实验的样本量要**事先**算好。开实验之前没算样本量，等于上战场不带子弹。

---

## 二、四要素的相互制约

样本量计算的四要素：

```
   ┌────────────────────────────────────────┐
   │                                        │
   │    α (显著性水平)        β (II 类风险) │
   │       │                      │         │
   │       └────┐         ┌───────┘         │
   │            ▼         ▼                 │
   │            样 本 量  N                  │
   │            ▲         ▲                 │
   │       ┌────┘         └───────┐         │
   │       │                      │         │
   │   MDE (最小可检测效应)    σ (方差)     │
   │                                        │
   └────────────────────────────────────────┘
```

**互为牺牲**：
- 想 α 更严（0.05 → 0.01）→ N 增大
- 想功效更高（80% → 95%）→ N 增大
- 想检出更小的差异（MDE 5% → 1%）→ N **平方级**增大
- 数据本身波动越大（σ 越大）→ N 增大

记住一条**黄金经验**：**MDE 砍一半，样本量翻四倍**（因为 N ∝ 1/MDE²）。这是为什么"我想抓到 0.5% 的小提升"在多数业务里不现实——你要付出 100 倍的样本量。

---

## 三、最小可检测效应（MDE）

MDE = Minimum Detectable Effect。**给定样本量、α、β，能稳定检出的最小真实差异。**

它不是"我希望差异多大"——而是"我事先承诺，**只有**真实差异达到这个值，我才有 80% 概率发现它"。

### 怎么定 MDE（业务视角）

不是统计问题，是产品问题。问自己：

> "**新版本至少要好多少**，才值得我们承担研发、运维、风险成本去上线？"

- 这个 ROI 阈值就是你的 MDE。
- 如果你的 MDE 是 1%，但当前数据下需要 200 万样本，而你日均 UV 只有 5 万——那就要么扩大流量、要么承认这个改动不值得做实验。

### 一个产品组常犯的错

```
❌ 想法：MDE 越小越好，能抓到任何细微提升
✅ 现实：MDE 太小 → 样本量爆炸 → 实验跑不动 → 业务停滞
```

**多数公司的"健康 MDE"在 1%-5% 之间**。低于 1% 通常意味着你需要 CUPED、分层等方差缩减手段。

## 四、样本量公式

### 1. 比例指标（转化率、点击率）

双边、双样本同等大小：

```
              2 · (z_{α/2} + z_β)² · p̄(1 − p̄)
       n  =  ────────────────────────────────────  （每组）
                          δ²

其中：
  p̄ = (p_A + p_B)/2          ≈ 基线转化率
  δ = p_B − p_A              = MDE 绝对值
  z_{α/2} = 1.96  (α=0.05)
  z_β     = 0.84  (功效=0.80)
```

代入 z 值，常用版本：

```
n ≈ 16 · p̄(1 − p̄) / δ²    （α=0.05, 功效=0.80）
n ≈ 21 · p̄(1 − p̄) / δ²    （α=0.05, 功效=0.90）
```

例：基线转化率 5%，MDE = 5% 相对（即 δ = 0.05 × 5% = 0.25 个百分点）：

```
n ≈ 16 × 0.05 × 0.95 / 0.0025² ≈ 121,600 (每组)
两组合计 ≈ 24 万，按日均 UV 5 万 → 约 5 天
```

### 2. 均值指标（人均时长、人均 GMV）

```
            2 · (z_{α/2} + z_β)² · σ²
     n  =  ───────────────────────────  （每组）
                    δ²

σ = 指标的标准差（要从历史数据估出来）
δ = 你想检出的均值差
```

### 3. Python 一键算

```python
from statsmodels.stats.power import NormalIndPower, zt_ind_solve_power

# 比例：转化率 5%，要检出 +1% 绝对（即 6%）
from statsmodels.stats.proportion import proportion_effectsize
es = proportion_effectsize(0.06, 0.05)
n = NormalIndPower().solve_power(
    effect_size=es, alpha=0.05, power=0.80, alternative='two-sided'
)
print(f"每组 n = {n:.0f}")

# 均值：σ = 30，要检出 δ = 1.5
n = zt_ind_solve_power(
    effect_size=1.5/30, alpha=0.05, power=0.80, alternative='two-sided'
)
print(f"每组 n = {n:.0f}")
```

> 在线计算器推荐：https://www.evanmiller.org/ab-testing/sample-size.html

## 五、把样本量翻译成"实验跑多少天"

样本量算出来还不能直接定时间，还要走三步：

```
每组 n  →  总样本量  →  日均唯一访客（UV）  →  天数
                          ↑
                  可能还要除以分流比例
```

例：实验需要每组 12 万，分流 50/50。日均 UV 5 万，但只有 60% 是目标人群（移动端）：

```
有效日均 UV = 50,000 × 60% = 30,000
每天每组样本 ≈ 15,000
所需天数 ≈ 120,000 / 15,000 = 8 天
```

再加业界经验：

- **最少跑满 1 个完整周（周一到周日）**：周末和工作日行为差异大，跑半周容易偏。
- **理想 2 周**：观察新奇效应、初见效应是否衰减。
- **不超过 4 周**：太久会被外部变化、实验疲劳污染，且占用流量。

## 六、易错点（高频踩坑）

### 1. 把"相对 MDE"和"绝对 MDE"混淆
- 相对 MDE = 5% 意味着：5% × 5%（基线）= 0.25 个百分点
- 绝对 MDE = 5% 意味着：5 个百分点（从 5% 涨到 10%）
- **后者样本量需求是前者的 1/400**！

❗ 工程同学和产品同学开会前先对齐这个，否则两人在算两个完全不同的实验。

### 2. 用历史平均估的方差，但实验组方差更大
新功能上线初期波动通常更大。建议样本量计算时**额外乘 1.2-1.5 的安全系数**。

### 3. 没考虑"分流单位 ≠ 指标单位"
分流是按 user_id，但你看的是 session 级或 event 级指标。这种情况下指标方差被低估，实际功效远低于计算值。
- 解法：用聚合到 user 级的指标算样本量；或用 cluster bootstrap。

### 4. SRM 后再补样本
SRM 出现意味着分流逻辑出错。**不要简单地"再多跑几天就好"**——污染过的数据基本要丢掉重做。

### 5. 多个变体（ABCD）忘了校正
4 个变体两两比较 = 6 次比较。如果还按 α=0.05，实际 family-wise 错误率约 26%。
- 解法：α 收紧到 0.05/6 ≈ 0.008（Bonferroni）；或预先指定主对比对。

### 6. 长尾指标用了 t 检验，但样本不够大
GMV、时长这类右偏严重的指标，t 检验依赖 CLT 收敛。每组 < 5000 时容易给出虚假显著。
- 解法：bootstrap、log 变换、winsorize（截尾）。

## 八、进阶：用 CUPED 把样本量降一半

**CUPED**（Controlled-experiment Using Pre-Experiment Data, Deng et al. 2013）的本质：用实验前的协变量解释当前指标的一部分方差，剩余方差更小 → 样本量需求降低。

经验上：

- 转化率类：方差降 20-40%
- 人均时长类：方差降 50% 以上常见
- 高度依赖前期行为的指标：可降 70%+

CUPED 几乎是免费的优化（不影响无偏性、不需要重新分流），但你需要：
1. 用户在实验前的历史数据
2. 计算 θ = Cov(Y, X) / Var(X)
3. 调整后的指标 Y' = Y − θ(X − E[X])

详见 [methods/cuped.md](../methods/cuped.md)。

## 九、Checklist：开实验前必答

```
□ 1. 我的主指标（OEC）是什么？基线值多少？
□ 2. 我的 MDE 是相对的还是绝对的？数值是多少？
□ 3. MDE 的业务依据是什么？（不是拍脑袋）
□ 4. α、β 怎么定的？
□ 5. 算出来每组样本量是多少？
□ 6. 每天有效流量多少？预计跑几天？
□ 7. 是否覆盖至少一个完整周（周末效应）？
□ 8. 多变体或多指标比较是否做了校正？
□ 9. 分流单位 = 分析单位吗？
□ 10. 用不用 CUPED？
```
