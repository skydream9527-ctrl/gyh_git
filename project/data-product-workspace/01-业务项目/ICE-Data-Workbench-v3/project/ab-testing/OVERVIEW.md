# AB 测试 — 开篇导读

> 这是 [ab-testing/](.) 目录的总入口。读完这一篇，你应该能回答：AB 测试解决什么问题、不解决什么问题、由哪些核心模块构成、应该按什么顺序学。

---

## 一、AB 测试在解决什么问题

**一句话**：AB 测试是用随机分流让"如果上线 vs 如果不上线"两个平行宇宙在统计意义上变得可比，从而把"相关性"变成"因果性"的工程化工具。

回到产品决策的本质：你做了一个改动 X，业务指标 Y 变了。你想知道 Y 变是不是 X 导致的。如果只看上线前后对比（Before / After），你无法排除：

- **季节性 / 周末 / 大促**：用户行为本身在变
- **同时上线的其他改动**：竞品涨价、运营推广、其他功能上线
- **新奇效应**：用户对"新东西"的短期好奇

AB 测试通过随机分流（50% 用户看 A，50% 用户看 B），让两组用户在 X 之外的所有其他变量上**期望一致**——这就是它最核心的价值。

> 这套思路在统计学和经济学里叫**随机对照试验（RCT）**，是因果推断的金标准。详见 [../causal-inference/](../causal-inference/)。

---

## 二、AB 测试 *不* 解决什么问题

很多人把 AB 测试当万能钥匙，这是一个产品组反复踩的坑。它不解决：

| 不解决 | 原因 / 替代方案 |
|---|---|
| **战略方向选择** | 实验只能比较"现在的 A vs 现在的 B"，无法告诉你"应该做 A 还是做 C" |
| **长期效应** | 实验通常跑 1-4 周，长期价值（留存、品牌）难以观测 → 用 Surrogate Index、合成控制 |
| **网络效应强的双边市场** | 司机/乘客分流会互相污染 → 用 Switchback、地理隔离实验 |
| **稀有事件 / 小流量** | 检测不出统计显著 → 加大效应量、合并指标、贝叶斯方法 |
| **发现"为什么"** | 实验只能告诉你 A 比 B 好，不能告诉你为什么 → 漏斗分析、用户访谈 |
| **0→1 创新功能** | 没有用户反馈数据，先上才有数据 → 灰度、用户调研、北极星指标 |

**判断要不要做实验的关键**：你能否清楚说出"假设、指标、效应量、样本量、停止规则"。说不出来就别开实验。

## 三、心智模型：三根支柱

把整个 AB 测试体系简化成三件事，剩下的都是细节。

```
┌──────────────────────────────────────────────────────────┐
│                      AB 测试                              │
├──────────────┬──────────────────┬────────────────────────┤
│  实验设计     │   统计推断        │   工程实现             │
│  (Design)    │   (Inference)    │   (Engineering)        │
├──────────────┼──────────────────┼────────────────────────┤
│ 假设          │ 假设检验          │ 分流引擎               │
│ 分流单位      │ p 值 / 置信区间   │ 指标计算               │
│ 指标体系      │ 功效 / 样本量     │ SRM 监控               │
│ 互斥分桶      │ 多重检验校正      │ 元数据管理             │
│ MDE           │ 序贯检验          │ 报告平台               │
│ 实验时长      │ 方差缩减 (CUPED)  │ 实验治理               │
└──────────────┴──────────────────┴────────────────────────┘
```

学习时也按这三根柱子走：先想清楚"我要回答什么"（设计），再学"怎么严谨地下结论"（推断），最后看"怎么规模化"（工程）。

## 四、最常见的五个陷阱（先记住，后细学）

我把这五个放在导读，是因为新手 80% 的错误都来自这里。详见 [pitfalls/peeking.md](pitfalls/peeking.md)（偷看陷阱）。

1. **偷看（Peeking）**
   - 现象：每天看一次报表，第 3 天看到显著就停。
   - 后果：实际 I 类错误膨胀到 20-30%（标称 5%）。
   - 解法：固定样本量后跑完，或用序贯检验（mSPRT、AGS）。

2. **多重比较（Multiple Comparisons）**
   - 现象：一个实验看 20 个指标，总有一个 p < 0.05。
   - 后果：发现的"显著"是噪音。
   - 解法：预先指定主指标（OEC），其他做 FDR 校正。

3. **SRM（样本比例失配）**
   - 现象：分流设的 50/50，跑出来 48/52，p 值小于 0.001。
   - 后果：分流坏了，所有结论作废。
   - 解法：每个实验 day-1 自动跑卡方检验，SRM 报警直接下线。

4. **Simpson 悖论**
   - 现象：总体上 A 赢，但每个子群里 B 都赢。
   - 后果：方向性结论反了。
   - 解法：分层看（新老用户 / iOS-Android / 渠道），警惕分流不平衡。

5. **新奇效应 / 初见效应**
   - 现象：第一周新功能涨 20%，第三周回落到 +1%。
   - 后果：基于第一周决策上线，长期没收益。
   - 解法：实验跑满一个用户周期（典型是两周），看曲线趋势再下结论。

## 五、推荐的学习顺序

### 入门（1-2 周可读完）
1. **本目录的 [concepts/hypothesis-testing.md](concepts/hypothesis-testing.md)** — 假设检验逻辑
2. **本目录的 [concepts/power-and-sample-size.md](concepts/power-and-sample-size.md)** — 样本量与功效
3. Kohavi 的两篇 KDD 短文（"Seven Pitfalls"、"Twelve Trustworthy"）

### 进阶（1-2 个月）
4. Kohavi & Tang & Xu — *Trustworthy Online Controlled Experiments*（必读，工业界圣经）
5. Microsoft ExP / Netflix / Airbnb 工程博客系列
6. CUPED 论文（Deng et al., 2013）→ [methods/cuped.md](methods/cuped.md)

### 高阶（按需）
7. 序贯检验：mSPRT、AGS（Always Valid Inference, Optimizely 论文）
8. 异质效应：Causal Forest, Meta-Learner（与 [../causal-inference/](../causal-inference/) 共读）
9. 双边市场：Switchback / Time-split（Doordash, Uber 博客）

## 八、把这一页当 Checklist

在你下次开实验之前，回答这 7 个问题：

1. 我的**假设**是什么？（一句话，可证伪）
2. 我的**主指标（OEC）**是什么？护栏指标是什么？
3. 我估计的**效应量（MDE）**是多少？依据是什么？
4. 按 α=0.05、β=0.2、上面的 MDE，我需要多少**样本量** / 跑多少天？
5. 我的**分流单位**是什么？是否会有溢出 / 网络效应？
6. **停止规则**是什么？（达到样本量？时间到？SRM 报警？）
7. 实验结果显著但**反直觉**时，我会怎么排查？

回答不出来其中任何一个，先回去 [concepts/](concepts/) 补课。
