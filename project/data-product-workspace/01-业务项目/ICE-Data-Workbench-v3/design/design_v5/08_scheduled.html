📐 这张稿的布局重设计

  • 原版：max-width 1080，header（标题 + 创建按钮）→ 过滤栏（chip + 搜索）→ 任务卡片流。每卡：状态 badge + 名字 + 操作按钮组 + 4 字段网格 + 可展开 prompt + 历史。
  • v5：保留单栏 max-width 1080，加入更印刷感的 Hero（Mono 大写 tag「CRON · AGENT-DRIVEN」+ 40px Inter H1「定时任务 自动跑」+ 副标题）+ 右上 ＋创建按钮
  • 4 stat 卡条：活跃 / 暂停 / 今日已跑 / 本周失败（border 1px 分隔，Inter 28px 数字 tnum，趋势 Mono 字）—— 原版没有数据维度
  • 过滤栏：4 个 chip pill（全部 30 / 活跃 23 / 暂停 7 / 失败 3）+ 右侧搜索框；active chip 铜橙实色
  • 任务卡：状态 chip（Mono 大写 RUNNING / ACTIVE / PAUSED / ERROR，run 状态 dot 脉冲动画）+ 衬线 16px 卡名 + 操作按钮组（▶ 立即执行用 accent-soft 突出）。Body 4 字段 grid（CRON 用 mono code 块 + 译文 / 所属任务 link / 下次时间 / 上次时间）
  • cron 字段：Mono 等宽 + accent-soft 底 + 1px border code 块（如 0 9 * * 1-5），下一行小灰字译文（"工作日 09:00"）
  • 详情展开（虚线 border-top）：「PROMPT 预览」用暖墨深色块（# 标题铜橙）+「最近 3 次执行」紧凑历史 row（status chip + 时间 + 输出摘要 + tokens 用量）
  • Modal 表单：680px 宽，Inter H3 标题 + 字段（所属任务 / 名称 / 提示 / ✨ AI 生成按钮 / cron + 译文 hint + 5 个 cron preset Mono 按钮 / AI 模型 / 启用 toggle）。新增 cron preset（原代码没有）
  • 移动端：Profile 风 hero + 2×2 stats 网格 + chip 横滚 + 紧凑卡片（status RUN/ON/OFF/ERR 缩写）+ FAB 浮动新建

和 v4 / 当前实现的差异

  • Hero：单行 h1 + 按钮 → Mono tag + 40px 衬线 H1 + 副标
  • 新增 4 stat 卡条（活跃 / 暂停 / 今日 / 失败）
  • 状态 badge：text 直显 → Mono 大写 + 状态色 chip + 脉冲 dot（运行中）
  • cron：纯文本 → Mono code 块 + accent-soft + 译文
  • 历史：badge + 行 → 暖墨 prompt 块 + 紧凑 row（status / 时间 / 输出 / tokens）
  • Modal：原版无 cron preset → 新增 5 个 Mono preset 按钮（每天 09:00 / 工作日 09:00 / 每周一 09:00 / 每小时 / 每天 18:00）
  • 「✨ AI 生成 cron+todo」按钮：保留功能，改用 1px dashed accent border 突出
