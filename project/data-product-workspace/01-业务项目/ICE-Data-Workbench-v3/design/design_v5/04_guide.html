使用指南

从零到上手 ICE Workbench —— 一份覆盖核心概念、工作流程、运维边界的完整文档。

💡
第一次来？ 先读「理解工作范式」和「创建第一个任务」就够开始工作了。其余章节按需查阅。

理解工作范式

ICE Workbench 把数据工作流抽象成 5 种范式，每个范式预绑定了 Agent + Skills + Prompt。你做的事永远是 「选范式 → 描述问题 → Agent 给结论」。

范式典型场景绑定 Agent
dataSQL 取数 · 可视化data-analysis
ab实验显著性 · ARPU 复盘ab-experiment
biz指标拆解 · 异动归因biz-insight
wave多维下钻定位wave-attribution
gray分桶决策gray-release

创建第一个任务

任务是 ICE Workbench 的原子单位 —— 一次完整的数据工作流，包括对话、文件、产出。

三种入口

  1. 从 Dashboard 点 + 新任务（推荐，可选范式 + 命名）
  2. 从 Dashboard 「⚡ 快速开始」点任意范式卡（一键以默认 Agent 创建）
  3. 定时任务每次执行也会创建一个任务（自动命名 {cron} · {时间戳}）
# 示例：用 data 范式创建一个任务 POST /api/v1/tasks { "name": "双十一 AB 实验复盘", "paradigm": "ab", "agent_id": "ab-experiment", "visibility": "private" }

Workspace 三栏

进入任务后，Workspace 是三栏布局：左侧文件 · 中间对话 · 右侧文件预览。三栏宽度可拖拽，移动端折叠为顶部三段切换。

⚠️
同一时刻只能有一个对话 当 (task_id, conv_id) 已有进行中的回合，新消息会返回 CONVERSATION_INFLIGHT。如需中断，先点对话区右上 ⏹ 取消。

左侧 · 数据源 / 文件

挂载到本任务的文件、飞书知识库、引用片段。Agent 在回答时会自动检索这些数据源，生成的引用会以 [1] [2] chip 出现在正文里。

中间 · 流式对话

WebSocket 连接 /api/v1/ws/conversations/{{cid}}?task_id=...，事件协议：text / tool_use_delta / message_done。所有事件落地为 jsonl，崩溃可重放。

中间 · Tool Call 卡片

Agent 在调用工具时会展开一张工具卡，显示参数 / 输出 / 耗时。手机端默认折叠，点击展开。
