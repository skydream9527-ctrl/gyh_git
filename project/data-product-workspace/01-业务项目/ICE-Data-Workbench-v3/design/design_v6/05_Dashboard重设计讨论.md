# ICE Workbench v6 - Dashboard (首页) 重设计讨论

根据用户的反馈，现有的 Dashboard 结构（按任务状态平铺）不够聚焦，且混合了“新用户引导”、“日常工作管理”、“公共资产获取”等不同意图。

**新的设计方向：基于用户意图的三 Tab 架构**

我们将 Dashboard 的首屏收拢为 3 个一级 Tab，以适配用户在不同场景下的诉求：

## Tab 1: 新建与发现 (Discover & Create) - 默认首屏
**定位：发号施令、快速启动、灵感启发。**
*   **超大核心输入框 / 动作条**：一进来就是一个极具 AI 感的对话框或操作区，输入自然语言直接拉起一个新的任务（Task）。
*   **场景引导 (Quick Starts)**：在输入框下方，放置 3-4 个高频场景模板卡片（例如：“👉 生成上周财务复盘”、“👉 根据本地 CSV 进行数据透视”）。
*   **可用智能体 (Agent Market)**：展示平台目前搭载了哪些核心能力（如 Data Analyst, Research Agent），点击直接以该 Agent 初始化任务。

## Tab 2: 我的工作流 (My Workflows)
**定位：日常进度管理、异步监控、异常处理（Human-in-the-loop）。**
*   **状态概览**：顶部展示今日数字（待审批: 2 | 运行中: 5 | 已完成: 18）。
*   **聚合列表/看板**：
    *   **高优先级置顶**：将「需人工干预 (Action Required)」和「后台运行中」的任务悬停在最上方。
    *   **混合列表**：下方用高密度表格或紧凑卡片展示「已有任务」、「运行中的定时任务 (Cron)」，一站式掌握自己负责的所有流转节点。

## Tab 3: 公共大厅 (Public Hub)
**定位：资产沉淀、经验复用、平台知识。**
*   **公共任务 (Public Tasks)**：团队内其他人公开的优质分析报告或长效工作流，支持一键 Fork。
*   **公共文件与数据盘 (Public Files)**：跨团队共享的只读元数据字典、参考口径 PDF、公共表结构说明。
*   **平台介绍与变更 (Changelog & Intro)**：ICE 框架的版本更新公告、使用指南。

---
**讨论与下一步**：
这个三 Tab 结构将平台的能力切分得非常清晰（我要干活 -> 我要看进度 -> 我要找资源）。
接下来，我将基于这个结构，使用 v6 的「Claude 暖白 + 铜橙高亮」风格，重新输出一版 `dashboard_v6_restyle.html` 原型。
