# ICE Workbench v6 - 核心功能演进讨论

基于数据工作流工作台的定位，v6 版本的核心功能演进建议聚焦于以下几个高价值方向，提升平台处理复杂数据任务和自动化工作流的能力。

## 方向一：多智能体协同与复杂工作流 (Multi-Agent Workflows)
目前工作台以单任务单 Agent 对话为主（虽然底层有 sub-agent 实验机制）。v6 可以将其显性化。
*   **规划模式 (Plan Mode) 升级**：Agent 自动将复杂需求拆解为可执行的子任务树，并在界面上实时呈现进度（Todo List 机制从后台走向前台）。
*   **子 Agent 派发调度**：主 Agent (General/Planner) 负责理解意图，随后自动派发给专门的 Data Agent (写 SQL/Python)、Research Agent (查飞书文档)、Review Agent (质检输出)，形成协同网络。
*   **UI 映射**：在 Chat 界面外，增加「执行流图」或「任务甘特图」面板。

## 方向二：深度 Human-in-the-loop (HITL) 与中间态干预
数据任务往往不能 100% 放心让 AI 自动执行，需要人工介入。
*   **关键节点审批**：在执行高危操作（如线上库 DML、大批量飞书消息推送、耗费大量 Token 的全库扫描）前，挂起任务，向用户发起「审批卡片」。
*   **数据中间态修正**：沙盒 (Sandbox) 跑完第一步数据清洗后，允许用户直接在 UI 上预览 DataFrame（或表格），手动修改几行异常数据，然后再让 Agent 继续执行下一步。
*   **UI 映射**：Chat 气泡中嵌入可交互的表单、数据预览表格和「批准/拒绝」按钮。

## 方向三：飞书 / 外部数据源的「活知识库」化
目前的飞书能力多表现为单次工具调用（fetch/publish）。
*   **实时数据源绑定**：允许任务直接绑定一个「飞书多维表格 (Bitable)」作为实时输入输出源，工作台能感知 Bitable 的表头变更。
*   **知识库增强**：支持将特定目录下的飞书文档一键建立为 RAG 知识库，Agent 问答时自动附带引用来源（Citation）。
*   **UI 映射**：任务配置页新增「挂载数据源」模块，Chat 气泡中支持直接渲染飞书文档 / Bitable 的摘要视图卡片。

## 方向四：离线异步与长耗时任务接管
数据分析任务（尤其是跑复杂 SQL 或训练简单模型）耗时较长，当前的 WebSocket 连接容易在用户切后台时断开。
*   **纯异步模式**：用户下发指令后即可关闭页面。后端接管会话，利用后台队列执行。
*   **进度与通知回捞**：任务跑完后，通过邮件、飞书机器人或系统的 Notification 中心通知用户回来看结果。
*   **UI 映射**：任务列表增加「运行中 (Running in background)」状态，不再强依赖打开该任务的 Chat 页面才往下走。
