# ICE Workbench v6: 前台工作区与后台全局重构蓝图

通过一连串的 HTML 原型验证与体验设计，我们已经为 ICE Workbench v6 确立了从“视觉风格”到“产品心智”的完整落地方案。

本文档汇总了所有的重设计产出，并为接下来的真实 React/Vite 代码开发提供导航指南。

## 一、 视觉交付物 (HTML 高保真原型)

为了方便前端开发者直接复制 UI 结构与 Tailwind 类名，我们将设计切分为两套独立的 SPA (单页面应用) 原型文件：

1. 🧑‍💻 **前台用户端 (`frontend_v6_spa.html`)**
   - 包含：登录与认证 (Login)、控制台大盘 (Dashboard)、任务工作区 (Workspace)、数据资产 (Data Assets)、我的智能体 (Agents)、定时调度 (Scheduled)。
2. ⚙️ **后台管理员端 (`admin_v6_spa.html`)**
   - 包含：系统监控大盘 (Overview)、用户管理 (Users)、全局智能体配置 (Admin Agents)、系统设置 (Settings)。

*(你可以直接在浏览器中打开这两个文件，它们具备基础的点击路由和切换交互。)*

## 二、 核心重构页面与现有代码对照

在实施开发时，请按照以下映射关系去修改 `frontend/src/` 中的文件：

### 1. 登录与初始化 (Login & Onboarding)
*   **原文件**：`pages/login/LoginPage.tsx`
*   **v6 改造点**：
    *   移除过时的剧中白色小卡片设计。
    *   采用 **左右分栏布局**（参考 `frontend_v6_spa.html`）。左侧为品牌深色插画区，右侧为表单区。
    *   提升“密码登录”与“SSO(Feishu)登录”的 Tab 切换视觉体验。

### 2. 前台：导航与布局 (Layout)
*   **原文件**：`components/shell/` 相关组件 (如 `TopNav.tsx`, `Sidebar.tsx`)
*   **v6 改造点**：
    *   左侧边栏 (Sidebar) 重新划分层级：“核心工作区” / “能力中心”。
    *   底部悬浮展示当前用户信息，并暴露管理员的专属入口（`进入系统后台`）。

### 3. 前台：任务工作区 (Task Workspace)
*   **原文件**：`pages/workspace/Workspace.tsx` + `components/chat/ChatBox.tsx`
*   **v6 改造点 (最重磅)**：
    *   放弃双栏，改为 **三栏布局 (Three-column Layout)**。
    *   **左栏 (Context)**：管理该 Task 绑定的数据源 (Feishu Bitable/SQLite) 以及工作智能体。
    *   **中栏 (Chat)**：负责渲染对话流。必须引入 `HITLRichCard` 组件，以拦截形式请求用户决策。
    *   **右栏 (Plan)**：渲染 `ExecutionPlanTree`，实时反馈后台智能体的流转节点。

### 4. 前台：数据资产 (Data Assets)
*   **原文件**：可能对应 `pages/public_file/` 或是全新的页面。
*   **v6 改造点**：
    *   将连接的飞书多维表格、SQLite 库、本地 CSV 统一以“卡片 (Cards)”形态管理，展示状态 (Live Sync) 和更新时间。

### 5. 后台：整体架构 (Admin Layout & Pages)
*   **原文件**：`pages/admin/AdminLayout.tsx` 以及其子路由页面。
*   **v6 改造点**：
    *   `AdminLayout` 左侧边栏强行使用 **深色模式 (`bg-slate-900`)**，与前台工作区的浅色拉开差距。
    *   `AdminOverview.tsx`：重构为顶层 Alert、中层 Stats、底层 Table/Rank 的块状结构。
    *   `AdminUsers.tsx`：优化表格样式，高亮 `Pending` 状态用户，操作按钮就近收敛。
    *   `AdminSettings.tsx`：将生硬的输入框转化为分区块 (LLM Config / Feishu Config / System Limits) 的现代 SaaS 表单设置页。

## 三、 开始开发的建议步骤

1. **配置 Tailwind Theme**：先修改 `tailwind.config.ts`，加入 `ice` (铜橙色主题) 调色板。
2. **重构 Layouts**：优先完成前端两个核心外壳 `AppLayout` (白) 和 `AdminLayout` (黑)。
3. **改造 Login**：给用户第一眼的震撼，先重构 LoginPage。
4. **攻坚 Workspace 三栏**：这部分逻辑最复杂（涉及 `taskStore` 的 executionPlan 与 messages 管理）。建议先切静态 HTML，再接入 WebSocket 数据。
5. **平刷后台 Admin**：按照 `admin_v6_spa.html` 的卡片和表格风格，逐个替换原有的 Admin 页面。
