# ICE Workbench v6: 管理员后台重设计规范

在将前台工作区升级为“多智能体数据舱”的同时，管理后台（Admin Panel）也需要从视觉和交互架构上向现代企业级 SaaS 对齐。

本设计规范基于项目中真实存在的 `frontend/src/pages/admin/*.tsx` 页面功能，重构了信息层级和 UI 表现。

## 一、 后台导航架构重组 (Sidebar)

由于 Admin 包含多达十几个细分页面（Overview, Users, Agents, Skills, Templates, KnowledgeBases, Settings, AuditLogs, Diagnostics 等），我们需要将它们归类，降低用户的寻找成本。

在 v6 设计中，左侧导航被划分为四大板块（深色主题，以区别于前台浅色工作区）：

1. **系统监控 (System Monitoring)**
   - 管理大盘 (Overview) -> `AdminOverview.tsx`
   - 使用统计 (Usage) -> `AdminUsage.tsx`
   - 系统诊断 (Diagnostics) -> `AdminDiagnostics.tsx`
2. **资源与权限 (Resources & Roles)**
   - 用户管理 (Users) -> `AdminUsers.tsx` (带红点提醒待审批用户)
   - 体验卡管理 (Exp Cards) -> `AdminExperienceCards.tsx`
3. **业务管理 (Business Management)**
   - 全局智能体 (Agents) -> `AdminAgents.tsx` / `AdminAgentEdit.tsx`
   - 全局技能 (Skills) -> `AdminSkills.tsx`
   - 提示词模板 (Templates) -> `AdminTemplates.tsx`
   - 知识库 (Knowledge Base) -> `AdminKnowledgeBases.tsx`
4. **运营与安全 (Ops & Security)**
   - 审批中心 (Review) -> `AdminReviewCenter.tsx`
   - 审计日志 (Audit Logs) -> `AdminAuditLogs.tsx`
   - SQL 审计 (SQL Audit) -> `AdminSqlAudit.tsx`
   - 系统设置 (Settings) -> `AdminSettings.tsx`

## 二、 核心页面设计与数据映射

我们为两个最重要的高频使用页面设计了高保真原型：

### 1. 管理大盘 (AdminOverview.tsx)
过去可能只是一堆散乱的数字，v6 将其分为三个层次：
*   **顶层预警 (Alerts)**：直接映射原代码中的 `alerts.pending_users`、`alerts.scheduled_failed` 和 `alerts.budget_alert`。设计成点击即可跳转的彩色预警卡片（红/黄/蓝）。
*   **核心指标 (Stats)**：映射 `stats.users`、`stats.tasks`、`stats.messages`。采用加大字号的卡片，并带有增长趋势角标。
*   **趋势与排行榜 (Rank & Recent)**：
    *   底部右侧放置 `rank` (Agent 满意度和调用次数排名)。
    *   底部左侧放置 `recent` (最近注册用户列表)，方便管理员第一时间感知系统活跃度。

### 2. 用户管理 (AdminUsers.tsx)
这是一个典型的 CRUD 表格页。设计亮点：
*   **状态强对比**：把新注册等待审批的用户 (`status === 'pending'`) 用淡红色的高亮行背景和 `Action Required` 按钮凸显出来。
*   **标签化 (Badges)**：将 `role` (Super Admin, Admin, User) 变为带背景色的标签。
*   **操作收纳**：通过/拒绝按钮在特定行才会出现，其他常规操作（编辑/删除）收敛为右侧的图标，使整个表格（Data Table）显得整洁专业。

## 三、 v6 视觉语言在后台的落地

1. **深浅模式切换 (Dark/Light Context)**
   - Admin Sidebar 采用了暗色主题 (`bg-slate-900`)。这不仅是为了看起来高端，更是为了与前台工作区 (`bg-white` 侧边栏) 形成强烈的上下文隔离。当用户切入后台，立刻能感受到自己在“全局管理”的语境下。
2. **主题色延续**
   - 依然使用铜橙色 (`#f97316`) 作为最高亮的点缀色（如当前选中的导航项），但在危险操作（如待审批用户的警告）上果断使用红色 (`#ef4444`)。
3. **卡片式承载 (Card-based UI)**
   - 表格、图表和列表全都被包裹在 `border-slate-200` 并带有 `shadow-sm` 的白底圆角卡片中。不再有内容直接裸露在背景色上。

## 四、 开发实施建议

开发同学可以直接参考 `design/design_v6/admin_v6_spa.html` 中的结构，将原有的 `frontend/src/pages/admin/AdminLayout.tsx` 替换为新的深色 Sidebar。

然后，提取 HTML 中的 `Card` 和 `Table` 样式类名，应用到现有的 `AdminOverview` 和 `AdminUsers` 的 TSX 结构上，即可快速完成 v6 风格的平滑过渡。
