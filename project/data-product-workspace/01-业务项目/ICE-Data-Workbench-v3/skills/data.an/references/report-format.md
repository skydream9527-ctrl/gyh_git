# 写作标准、飞书写入规范、交付自检

## 写作规范（金字塔原理 — 严格执行）

遵循 SKILL.md 金字塔三原则（结论先行 / MECE / 逻辑递进）。以下为深度报告的补充规则：

- **章节按业务主题划分**，不按分析方法划分
- **每章固定结构**：章节结论（callout 高亮）→ 各观点子标题（编号 N.1/N.2/N.3，每个观点含文字分析 + 对应图表）→ 行动建议。禁止在章节标题下堆砌大段无子标题内容
- **AI 深度分析融入每章观点支撑**，不单独列章。主动做：维度交叉、异常捕捉、隐藏关联、风险预警、具体策略（"对 Y 人群在 X 时间做 Z"，不说"建议关注"）。数据不够时主动补充查询（Hive: SQL / 本地: pandas）。**AI 是分析师，不是数据搬运工。**

## 飞书写入规范

> **飞书操作优先级（强制）**：所有飞书文档操作（创建、写入文字、插入图片）必须**优先使用 feishu 技能**（CLI 命令），不得直接调用飞书 MCP。原因：飞书 MCP 不支持图片插入，若混用会导致图文交付不完整。仅当 feishu 技能不可用（未安装或命令执行失败）时，才降级使用飞书 MCP，并提前告知用户"图片需手动插入"。

通过 **feishu 技能** CLI 创建和写入文档。**写入前必须用 AskUserQuestion 询问用户写入知识库还是个人空间**（不得默认选择）。结论用 `<callout>` 高亮；长报告分段写入；**飞书文档标题已由 title 参数设置，正文不再写一级标题（封面是 HTML 报告专属），直接从正文内容开始**。

**图片插入**：通过 feishu 技能 CLI（`feishu docx upload-image`）上传图片。仅当 feishu 技能不可用时，降级尝试飞书 MCP（需告知用户 MCP 图片插入能力有限）。

## 飞书文档章节格式规范（严格执行）

1. **章节标题命名**：使用「第一章 XXX」「第二章 XXX」格式，如 `## 第一章 用户活跃趋势分析`
2. **每个观点必须有子标题**：一个章节中分析出多个观点时，每个观点对应一个子标题，禁止在章节标题下堆砌大段无标题内容
3. **子标题编号格式**：使用 `N.1`、`N.2`、`N.3` 格式（N 为章节序号），如：
   - `### 1.1 暑期活跃峰值创历史新高`
   - `### 1.2 开学季回落幅度超预期`
   - `### 2.1 核心渠道贡献率持续下降`
4. **章节结构示例**：
   ```
   ## 第一章 用户活跃趋势分析
   <callout>章节结论</callout>
   ### 1.1 观点一标题
   结论文字（一句话判断）
   图表图片（PNG）
   数据表格（明细/对比数据）
   ### 1.2 观点二标题
   结论文字
   图表图片
   数据表格
   ### 1.3 观点三标题
   结论文字
   图表图片
   数据表格
   行动建议
   ```

## 图文并茂写入规则（严格执行）

每个观点严格按「结论 → 图表 → 表格」顺序写入，凡是有表格数据的地方都必须配图：

1. **每个观点的固定顺序**（不可调换）：
   - ① **结论文字**：一句话判断 + 数据解读
   - ② **图表图片**：通过 feishu 技能插入对应 PNG（直观展示趋势/对比/分布）
   - ③ **数据表格**：补充明细数据或对比数值（表格是图表的数据支撑，不能替代图表）
2. **有表必有图**：凡是写入了数据表格的地方，表格前面必须有对应的图表图片。禁止"只有表格没有图"
3. **严禁集中堆图**：禁止将所有图片集中插入到文档末尾或任何单一位置。每张图片必须紧跟其所属观点的结论文字之后
4. **写入顺序示例**：章节结论(callout) → 观点1结论 → 观点1图表 → 观点1表格 → 观点2结论 → 观点2图表 → 观点2表格 → ... → 章节行动建议
5. **自检**：每个章节写入完成后，确认每个观点的图片都在结论之后、表格之前

## 报告逻辑结构

```
KPI 概览 → 核心结论与战略建议 → 各分析章节（按业务主题组织）
```

- **核心结论**（≤5 条）：每条一句话判断 + 关键数据支撑
- **战略建议**（≤3 条）：具体到人群、时间、动作
- **不在末尾设独立总结章**：各分析章节末尾的行动建议仅放该章局部建议

## matplotlib 图表交付自检

生成图表后、写入飞书前，必须执行以下检查：

- [ ] `dar-charts/` 目录存在
- [ ] `ls -lh dar-charts/` 列出所有 PNG 文件
- [ ] 每个 `chart_N.png` 都存在（数量应与分析观点数一致）
- [ ] 所有 PNG 文件大小 > 0 字节（空文件说明生成失败，需重试）
- [ ] 缺失文件处理：列出缺失清单 → 重试生成一次 → 仍失败则告知用户

## 飞书文档交付自检

飞书写入完成后，必须确认：

- [ ] 核心结论前置于报告开头（不在末尾）
- [ ] 每章结构完整：章节结论(callout) → 各观点(子标题+结论+图表+表格) → 行动建议
- [ ] 每张图表 PNG 已通过 feishu 技能正确插入到对应观点位置
- [ ] 无"有数据无观点"的段落（每段必须回答"所以呢"）
- [ ] 策略建议具体到人群、时间、动作（不说"建议关注"）
