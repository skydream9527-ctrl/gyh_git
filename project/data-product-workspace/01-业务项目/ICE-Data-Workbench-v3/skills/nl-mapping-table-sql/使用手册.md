# NL Mapping Table SQL — 使用手册

## 1. 简介

`nl-mapping-table-sql` 是一个自然语言转映射表 SQL 的技能。你只需用自然语言描述数据需求，技能会自动：

1. 确认业务线和查询类型
2. 将你的描述映射到标准表名和字段名
3. 生成可直接执行的 SQL 语句
4. 保存 SQL 文件到桌面

---

## 3. 支持的业务线

| 业务线 | 说明 | 数据库前缀 | 数据层级 |
|--------|------|-----------|---------|
| 浏览器 | 浏览器整体数据 | `iceberg_zjyprc_hadoop.browser` | 中间表 + 底表/宽表 |
| 浏览器信息流 | 浏览器信息流子集，复用浏览器dm表 | `iceberg_zjyprc_hadoop.browser`（通过 `is_dau_feed_dapan_2024 = 1` 筛选） | 中间表 + 底表/宽表 |
| 内容中心 | 内容中心独立数据 | `iceberg_zjyprc_hadoop.newhome` | 中间表 + 底表/宽表 |
| 搜索 | 搜索业务数据 | `iceberg_zjyprc_hadoop.dw` | 仅底表/宽表 |
| 小说 | 小说/多看业务数据 | `iceberg_zjyprc_hadoop.dw` / `iceberg_zjyprc_hadoop.duokan` | 仅底表/宽表 |

> **数据层级说明**：
> - **中间表**（dm/ads/dim 层）：预聚合表，查询效率高，维度有限。文件命名无前缀。
> - **底表/宽表**（dwm/dwd 层）：原始聚合表，支持更灵活的维度筛选，查询性能较低。文件命名以 `raw-` 前缀区分。
> - **适用场景**：优先使用中间表查询；当中间表不支持所需维度或指标时，回退到底表/宽表。

## 4. 三种查询类型

### 4.1 维度枚举值查询

查询某个维度字段包含哪些值。

**使用场景**：想了解某个维度有哪些取值

**示例对话**：
```
你：帮我查一下浏览器多维指标表的启动方式有哪些值
AI：请选择业务线 → 浏览器 → 确认表和字段 → 生成SQL
```

**生成的 SQL 示例**：
```sql
SELECT  app_launch_way,
        COUNT(DISTINCT CASE WHEN da_pv > 0 THEN did END) AS dau
FROM    iceberg_zjyprc_hadoop.browser.dm_browser_multi_dimension_indicators_di
WHERE   date = '20260512'
        AND is_app_dau_2024 = 1
GROUP BY app_launch_way
ORDER BY dau DESC
```

### 4.2 指标聚合查询

基于映射表查询聚合指标，支持不筛选维度和按维度筛选两种模式。

**使用场景**：查询DAU、时长、留存、收入等指标

**示例对话**：
```
你：查一下浏览器昨天的DAU
你：查一下信息流按体裁的时长
你：查一下内容中心按新老用户的次日留存
你：查一下浏览器财收按广告位场景
```

**支持的指标分类**：

| 分类 | 指标 | 数据表 |
|------|------|--------|
| 核心指标 | DAU、应用使用时长、信息流时长、消费时长、内容曝光量、内容点击量 | dm_*_multi_dimension_indicators_di |
| 留存指标 | 次日留存、7日留存、30日留存 | dm_*_multi_dimension_retain_indicators_di |
| 商业化指标 | 财收、流水、计费曝光、计费点击、广告下载量、广告请求次数、广告填充次数 | ads_*_finance_core_indicators_di |

**支持的维度筛选**：

| 维度类别 | 维度字段 | 适用指标 |
|----------|---------|---------|
| 设备/人口维度 | 新老用户(is_new_2024)、启动方式(app_launch_way)、体裁(item_type)、App版本(app_version)、系统版本(os_version)、机型(phone_model)、频道(feed_channel) | 核心指标 |
| 用户类型维度 | 新老用户(is_new_2024)、启动方式(app_launch_way)、体裁(item_type) | 留存指标 |
| 广告位维度 | 广告位场景(ad_position_scene)、广告位(tag_id) | 商业化指标 |

### 4.3 多表关联查询

跨表 JOIN 查询，如多维指标表 JOIN 留存表。

**使用场景**：需要同时查询多个表的指标

**示例对话**：
```
你：帮我关联多维指标表和留存表，查一下DAU和次日留存
```

### 4.4 底表/宽表指标查询

当中间表不支持所需维度或指标时，可使用底表/宽表（dwm/dwd 层）查询。

**使用场景**：中间表维度不够灵活、需要查询埋点事件指标、搜索/小说等无中间表的业务线

**示例对话**：
```
你：用底表查一下浏览器主启DAU
你：查一下浏览器搜索SUG页UV
你：查一下多看的DAU
你：查一下内容中心小说的uv和时长
```

**底表/宽表支持的指标**：

| 业务线 | 底表/宽表数据源 | 支持指标 |
|--------|---------------|---------|
| 浏览器 | dwm_browser_event_aggregation_label_di + dwd_ot_event_di_31000000442 | 主启DAU、有效DAU、消费UV、一级页VV、短/长会话UV、图文/视频详情页VV时长、搜索SUG页UV、开屏UV |
| 浏览器信息流 | 同浏览器底表 | 同浏览器底表指标 |
| 内容中心 | dwm_newhome_event_aggregation_label_di + dwd_ot_event_di_31000000297 | DAU、消费UV、总VV、总时长、有效用户DAU&时长、有效DAU7日留存、有效DAU、激励DAU、激励页面时长 |
| 搜索 | dwd_ot_event_di_31000000442 | 搜索SUG页UV |
| 小说 | dwd_ot_event_di_1004465 + dwd_ot_event_di_31000000442 + dwd_ot_event_di_31000000297 | 多看DAU、多看阅读时长、多看有效阅读UV、多看网文/出版阅读时长和UV、SDK有效阅读UV、浏览器小说uv/vv/时长、短故事vv、内容中心小说uv/vv/时长 |

**底表/宽表查询示例**（浏览器搜索SUG页UV）：
```sql
SELECT
    date,
    COUNT(DISTINCT distinct_id) AS uv
FROM
    iceberg_zjyprc_hadoop.dw.dwd_ot_event_di_31000000442
WHERE
    date = '20260512'
    AND event_name = 'search_sugpage_expose'
GROUP BY date
```

## 6. 不在dm表中的维度（替代方案）

以下 5 个维度不在 dm 映射表中，但可以通过 dwm 源表查询：

| 维度 | 字段 | dwm 源表 |
|------|------|---------|
| 实验组 | exp_id | dwm_*_event_aggregation_label_di |
| 三方调起包名 | third_packagename | dwm_*_event_aggregation_label_di |
| 是否冷启 | is_coldstart | dwm_*_event_aggregation_label_di |
| 是否有上划行为 | is_feed_sliding | dwm_*_event_aggregation_label_di |
| 内容源 | source | dwm_*_event_aggregation_label_di |

**替代查询示例**（按实验组查DAU）：
```sql
SELECT  date,
        exp_id,
        COUNT(DISTINCT did) AS dau
FROM    iceberg_zjyprc_hadoop.browser.dwm_browser_event_aggregation_label_di
WHERE   date = '20260512'
        AND is_app_dau = 1
GROUP BY date, exp_id
ORDER BY dau DESC
```

## 8. 重要规则

### 8.1 日期格式
- 统一使用 `'YYYYMMDD'` 整型格式，如 `'20260512'`
- 禁止使用连字符格式（如 `2026-05-12`）

### 8.2 DAU 计算方式
- dm 多维指标聚合表中**没有 dau 字段**
- 正确的 DAU 计算：`COUNT(DISTINCT CASE WHEN da_pv > 0 THEN did END)`
- dwm 源表中使用：`COUNT(DISTINCT did)`
- `ads_*_user_type_core_indicators_di` 表中有 dau 字段，可直接使用

### 8.3 留存表查询规则
- 留存表是did粒度指标表，留存需要通过自关联计算（JOIN自身，按did匹配，目标日期 = 基准日期 + N）
- 按维度筛选时，直接在WHERE/GROUP BY中使用对应字段（如 is_new_2024, app_launch_way, item_type）

### 8.4 高基数维度
- App版本、系统版本、机型、广告位ID 等高基数维度自动加 `ORDER BY ... DESC LIMIT 50`

### 8.5 浏览器信息流特殊规则
- 浏览器信息流**没有独立的 dm 表**，复用浏览器整体的 dm 表
- 通过 `is_dau_feed_dapan_2024 = 1` 筛选信息流大盘DAU用户

### 8.6 表/字段范围约束
- 严格禁止为参考文件中不存在的表或字段生成 SQL
- 遇到不支持的表/字段时，会终止执行并引导联系开发者 gongyunhe

## 10. 常见问题

**Q：我想查的维度不在dm表中怎么办？**
A：技能会自动检查 `unsupported-dimensions.md`，如果该维度在不支持列表中，会提供 dwm 源表的替代查询SQL。

**Q：中间表和底表/宽表有什么区别？什么时候用底表/宽表？**
A：
- **中间表**（dm/ads/dim 层）：预聚合表，查询效率高，但维度有限。适合常规指标查询。
- **底表/宽表**（dwm/dwd 层）：原始聚合表，支持更灵活的维度筛选和更多指标，但查询性能较低。
- **使用建议**：优先使用中间表；当中间表不支持所需维度或指标时（如搜索SUG页UV、多看阅读时长等），回退到底表/宽表。参考文件中以 `raw-` 前缀标识底表/宽表文件。

**Q：搜索和小说业务线为什么没有中间表？**
A：搜索和小说业务线目前尚未建设中间表（dm/ads/dim 层），仅支持通过底表/宽表（dwm/dwd 层）查询。如需建设中间表，请联系开发者 gongyunhe。

**Q：我想查的指标不在支持列表中怎么办？**
A：技能会终止执行，告知当前支持的指标列表，并引导联系开发者 gongyunhe 补充。

**Q：可以一次查多个指标吗？**
A：可以，只需在描述中说明多个指标名称，技能会生成包含多个指标的 SQL。

**Q：时间周期怎么指定？**
A：支持多种方式："昨天"、"最近7天"、"20260301~20260325" 等，技能会自动转换为标准日期格式。

**Q：浏览器和浏览器信息流有什么区别？**
A：浏览器信息流是浏览器的子集，复用同一套 dm 表，通过 `is_dau_feed_dapan_2024 = 1` 筛选信息流用户。
