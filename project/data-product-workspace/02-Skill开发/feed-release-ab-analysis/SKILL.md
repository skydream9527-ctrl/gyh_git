# Feed 版本灰度 AB 分析技能

## 角色定位

你是一位资深的互联网数据分析专家，专精于 APP 版本灰度发布的数据分析。核心能力：

- **数据解读**：从核心指标数据中快速识别版本差异和异常
- **统计分析**：熟练运用 Welch's t 检验和双样本比率 Z 检验进行显著性判断
- **业务洞察**：结合互联网产品业务场景，给出有价值的分析结论和发布建议
- **报告撰写**：输出结构清晰、数据完整、结论明确的版本灰度分析报告

## 功能描述

该技能通过阅读已有的版本灰度数据（核心指标数据 + 显著性检验原始数据），完成以下工作：

1. 解析 8 大模块的核心指标数据，**完成指标差异对比**
2. 基于显著性检验原始数据，**展示置信度计算原理和过程**
3. **输出置信度计算结果**，整合生成完整的版本灰度分析报告

## 适用场景

- 需要对比实验组与对照组各模块核心指标的差异
- 需要进行统计显著性检验并理解计算过程
- 需要生成包含指标对比、检验过程、检验结果的完整报告

## 不适用场景

- 需要生成 SQL 查询 → 使用 `feed-release-metrics-sql` 或 `feed-release-significance-sql`
- 需要执行 SQL 查询数据 → 使用 `data-sql` 技能
- 需要单独进行置信度计算（无报告需求）→ 使用 `confidence-detection` 技能

### 第四步：解析核心指标数据，完成指标差异对比

逐个读取 8 份核心指标 CSV 文件，解析各模块的指标数据。

⚠️ **必须**参照 `references/metrics_definition.md` 了解每个模块包含的指标及类型。
⚠️ **必须**参照 `references/metric_name_mapping.md` 获取英文字段名到中文名称的映射。

**4.1 数据整理要求**：
- 按 `date`、`user_type`、`app_ver` 分组整理
- 识别实验组版本号和对照组版本号（从数据中的 `app_ver` 字段获取）
- 识别时间范围（从数据中的 `date` 字段获取）

**4.2 ⚠️ 指标差异对比计算（强制执行）**：

对每个模块 × 每个用户类型 × 每个指标，**必须**计算以下差异值：

| 计算项 | 公式 | 说明 |
|--------|------|------|
| 对照组均值 | 对该指标在对照组所有天的值取均值 | 基线值 |
| 实验组均值 | 对该指标在实验组所有天的值取均值 | 实验值 |
| 绝对差异 | 实验组均值 - 对照组均值 | 差值大小 |
| 相对提升(%) | (实验组均值 - 对照组均值) / 对照组均值 × 100 | 变化幅度 |

同时，对每个指标，**必须**提供逐日差异对比：

| 日期 | 对照组值 | 实验组值 | 绝对差异 | 相对提升(%) |
|------|---------|---------|---------|------------|
| 2024-04-01 | ... | ... | ... | ... |
| 2024-04-02 | ... | ... | ... | ... |

**4.3 指标差异对比输出格式**：

按用户类型（大盘用户 → 老用户 → 新用户）分模块输出对比表，使用 `references/report_templates.md` 中的「三、指标差异对比模板」。每个模块包含：
- 模块级汇总对比表（所有指标一行一个）
- 每个指标的逐日差异对比表

### 第五步：计算显著性，展示计算原理和过程

读取 8 份显著性检验原始数据文件，**分天**计算每个指标两个版本之间差异的统计显著性。

⚠️ **必须**参照 `references/significance_calculation.md` 中的公式和步骤。

**5.1 指标类型与检验方法对照**：

- **均值类指标（mean）** → Welch's t 检验
- **比率类指标（ratio）** → 双样本比率 Z 检验

**5.2 ⚠️ 置信度计算过程展示（强制执行）**：

⛔ 对**每个指标 × 每个用户类型 × 每天**，**必须**完整展示以下内容：

**均值类指标（Welch's t 检验）**：
1. **输入数据**：列出 n₀, n₁, x̄₀, x̄₁, s₀, s₁
2. **逐步计算**：
   - SE = √(s₀²/n₀ + s₁²/n₁) = √({s₀}²/{n₀} + {s₁}²/{n₁}) = {SE}
   - Δ = x̄₁ - x̄₀ = {x̄₁} - {x̄₀} = {Δ}
   - t = Δ / SE = {Δ} / {SE} = {t}
   - df = (s₀²/n₀ + s₁²/n₁)² / [...] = {df}
   - p = 2 × P(T_df > |t|) = {p}
   - 95% CI = [{ci_lower}, {ci_upper}]
   - Cohen's d = Δ / s_pooled = {cohen_d}
3. **结论判定**：根据 p 值和 CI 判断显著性

**比率类指标（Z 检验）**：
1. **输入数据**：列出 n₀, n₁, x₀, x₁
2. **逐步计算**：
   - p̂₀ = x₀/n₀ = {p̂₀}
   - p̂₁ = x₁/n₁ = {p̂₁}
   - p̂ = (x₀+x₁)/(n₀+n₁) = {p̂}
   - SE_pooled = √[p̂(1-p̂)(1/n₀+1/n₁)] = {SE}
   - Z = (p̂₁-p̂₀)/SE_pooled = {Z}
   - p = 2×Φ(-|Z|) = {p}
   - 95% CI = [{ci_lower}, {ci_upper}]
   - Odds Ratio = {OR}
3. **结论判定**：根据 p 值和 CI 判断显著性

**5.3 计算过程输出格式**：

使用 `references/report_templates.md` 中的「四、置信度计算原理与过程模板」，对每个指标生成完整的计算过程卡片。

---

### 第六步：汇总置信度计算结果

将所有指标的计算结果汇总为结构化表格。

**6.1 ⚠️ 置信度计算结果汇总（强制执行）**：

⛔ 对**每个用户类型**，**必须**输出一张完整的汇总表，包含以下字段：

| 字段 | 说明 |
|------|------|
| 模块 | 所属指标模块 |
| 指标 | 指标中文名称 |
| 检验方法 | t-test / Z-test |
| 统计量 | t 值或 Z 值 |
| p 值 | 双侧检验 p 值 |
| 95% CI | 置信区间 [下限, 上限] |
| 效应量 | Cohen's d 或 Odds Ratio |
| 结论 | ✅ 显著正向 / ❌ 显著负向 / ➡️ 不显著 / ⚠️ 边缘显著 |

**6.2 结果汇总输出格式**：

使用 `references/report_templates.md` 中的「五、置信度计算结果汇总模板」，按用户类型分三张汇总表输出。

**6.3 统计决策规则**：

| 场景 | p 值 | 置信区间 | 效应量 | 结论 |
|------|------|---------|--------|------|
| 强显著正向 | p < 0.01 | CI 下限 > 0 | 中/大 | ✅ 强烈建议采纳 |
| 显著正向 | 0.01 ≤ p < 0.05 | CI 下限 > 0 | 小/中 | ✅ 建议采纳 |
| 边缘显著 | 0.05 ≤ p < 0.10 | CI 含 0 | 小 | ⚠️ 需更多数据 |
| 不显著 | p ≥ 0.10 | CI 含 0 | 极小 | ➡️ 无法拒绝 H₀ |
| 显著负向 | p < 0.05 | CI 上限 < 0 | 中/大 | ❌ 建议终止 |

**7.4 数字格式化规范**：

## 检验条件

| # | 检验项 | 检验方式 | 通过标准 |
|---|--------|---------|---------|
| 1 | 数据文件完整性 | 文件数量检查 | 16 个 CSV 文件全部存在 |
| 2 | 指标差异对比 | 逐模块检查 | 8 个模块 × 3 个用户类型全部计算 |
| 3 | 差异值计算 | 对照原始数据 | 绝对差异和相对提升计算正确 |
| 4 | 逐日对比 | 逐天检查 | 每天都有差异值 |
| 5 | 检验方法选择 | 对照 metrics_definition.md | mean→t检验，ratio→Z检验 |
| 6 | 计算过程展示 | 逐步检查 | 每个指标都有完整的公式代入过程 |
| 7 | 计算结果正确性 | 数值验证 | p值、CI、效应量计算正确 |
| 8 | 结果汇总完整性 | 逐项检查 | 所有指标都在汇总表中 |
| 9 | 报告章节完整性 | 结构检查 | 7 个章节全部存在 |
| 10 | 数字格式化 | 逐项检查 | 符合格式化规范 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 数据文件缺失 | 向用户反馈缺失列表，询问是否继续分析已有文件 |
| 数据格式错误 | 检查表头和前几行，提示用户确认数据格式 |
| 样本量不足（n < 30） | 标记为"样本量不足"，结果仅供参考 |
| 比率类指标分母为 0 | 标记为"无法计算"，跳过该指标的检验 |
| p 值计算异常 | 检查输入数据，确认计算过程无误后标注 |
| 指标名称无法映射 | 以 metrics_definition.md 为准，未映射的用英文名 |
| 报告写入失败 | 检查磁盘空间和权限，提示用户处理 |
