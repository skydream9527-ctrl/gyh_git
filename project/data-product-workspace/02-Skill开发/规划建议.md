# 三、给你的 2 个月详细规划

# 第1-2周：搭框架 + 定标准

## 目标

把整套 Skill 体系的“骨架”搭出来，避免后面越做越乱。

## 本阶段产出

* Skill 总地图
* Skill 命名规范
* 输入输出规范
* 分析方法库
* 通用提示词框架
* 统一输出模板

## 要做的事

### 1）梳理互联网数据分析全景能力图

输出一版总表，字段建议包括：

* Skill 名称
* 解决的问题
* 适用场景
* 输入内容
* 处理逻辑
* 输出内容
* 依赖数据
* 优先级
* 是否可自动化

### 2）统一 Skill 标准结构

每个 Skill 都按同样格式写：

* **Skill 名称**
* **目标**
* **适用业务场景**
* **输入**
* **核心分析步骤**
* **输出**
* **约束条件**
* **示例问题**
* **示例输出**

### 3）建立通用分析规范

比如：

* 必须先明确口径再分析
* 必须先看整体趋势再做拆解
* 异常分析必须回答“何时开始、影响多大、谁贡献、为什么”
* 结论必须区分“事实、判断、建议”

### 4）建立基础模板库

优先沉淀这几类模板：

* 趋势分析模板
* 异常分析模板
* 留存分析模板
* 漏斗分析模板
* AB 实验分析模板
* 埋点需求模板
* 看板设计模板
* 复盘模板

## 第1-2周建议优先完成的 Skill 文档

先不急着全部上线，先把这 8 个定义清楚：

* 指标定义生成
* 指标树拆解
* 趋势分析
* 异常分析
* 留存分析
* 漏斗分析
* AB 实验结果分析
* 埋点方案设计

# 第3-4周：先做第一批高频 Skill

## 目标

上线第一批“最常用”的 Skill，让这套体系开始能真正工作。

## 建议优先开发的 10 个 Skill

这是最值得先做的一批：

### A. 基础分析核心 Skill

1. **指标定义生成 Skill**
   输入业务目标，输出指标定义、计算公式、维度、注意事项。

2. **指标树拆解 Skill**
   输入核心指标，输出一级二级驱动因素拆解。

3. **趋势分析 Skill**
   输入时间序列数据，输出趋势、波动、周期性、拐点判断。

4. **异常波动分析 Skill**
   输入异常指标和时间段，输出异常定位、拆因思路、可能原因。

5. **结构拆解 Skill**
   输入某指标及维度拆分数据，输出结构变化和贡献分析。

### B. 用户与增长核心 Skill

6. **留存分析 Skill**
   输出新老用户留存、分群留存、关键流失节点和改善方向。

7. **漏斗分析 Skill**
   输出转化率、流失点、关键环节诊断。

8. **用户分层分析 Skill**
   对活跃、消费、价值、频次进行分层分析。

### C. 业务分析核心 Skill

9. **内容消费分析 Skill**
   针对资讯/视频/内容平台，分析曝光、点击、消费时长、完播、互动。

10. **日报周报生成 Skill**
    根据输入数据自动生成管理层可读的日报/周报。

## 这一阶段重点

* 每个 Skill 都至少要有 3 个测试 case
* 每个 Skill 输出不要太学术，要更像业务能看懂的分析报告
* 开始区分：给分析师看的版本、给业务方看的版本、给老板看的版本

### 2）数据建设类

* 埋点方案设计 Skill
* 埋点校验 Skill
* 数据看板设计 Skill
* 看板优化建议 Skill
* 数据需求文档生成 Skill

### 3）经营诊断类

* 渠道质量分析 Skill
* Push 召回效果分析 Skill
* 用户生命周期分析 Skill
* 增长策略建议 Skill
* 风险预警 Skill

## 这一阶段的重点

你要开始做“**Skill 之间的衔接**”。

比如形成这样的链路：

* 业务问题输入
  → 指标树拆解
  → 趋势分析
  → 异常分析
  → 留存/漏斗专项分析

也就是从单点 Skill，升级为**组合式分析工作流**。

# 第7-8周：联调、打磨、产品化

## 目标

## 本阶段产出

## 要做的事

### 1）做统一入口

用户不需要知道该用哪个 Skill。
可以设计一个总调度逻辑：

* 识别用户问题类型
* 自动选择合适 Skill
* 必要时串联多个 Skill
* 输出一份完整分析结果

### 2）补齐案例库

至少准备：

* DAU 下滑分析案例
* 留存下降案例
* 内容消费下降案例
* Push 拉活效果分析案例
* 渠道拉新质量分析案例
* AB 实验结果解读案例
* 埋点需求设计案例

### 3）做输出分层

同一个 Skill，建议至少支持 3 种输出风格：

* **简洁版**：给业务同学
* **专业版**：给分析师
* **汇报版**：给管理层

### 4）做质量评估机制

每个 Skill 检查：

* 问题理解是否准确
* 指标口径是否清晰
* 分析路径是否完整
* 结论是否有证据
* 建议是否可执行

# 七、你这套 Skill 最终推荐的产品形态

建议不是做成一堆散 Skill，而是做成“三层结构”：

## 第一层：基础原子 Skill

单一能力，比如：

* 计算环比同比
* 做漏斗拆解
* 做留存分群
* 输出显著性检验

## 第二层：分析任务 Skill

一个完整分析任务，比如：

* DAU 波动分析
* 留存专项分析
* 内容消费专项分析
* AB 实验复盘

## 第三层：总控 Agent

负责：

* 理解问题
* 选择 Skill
* 串联流程
* 输出完整结果

这样后面扩展会更容易。
