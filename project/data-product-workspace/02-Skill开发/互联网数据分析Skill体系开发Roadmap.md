## 四、8周执行计划

### 第1-2周：搭框架 + 定标准

**目标**：把整套 Skill 体系的"骨架"搭出来

| 周次 | 重点任务 | 产出 |
|------|----------|------|
| 第1周 | 能力盘点、Skill 全景梳理、优先级定义 | Skill 总地图、候选清单 |
| 第2周 | 统一规范、模板库建设、8个核心 Skill 定义 | Skill 标准模板、分析规范 |

**本阶段优先完成的 8 个 Skill 文档**：
- 指标定义生成
- 指标树拆解
- 趋势分析
- 异常分析
- 留存分析
- 漏斗分析
- AB 实验结果分析
- 埋点方案设计

---

### 第3-4周：高频 Skill 开发

**目标**：上线第一批"最常用"的 Skill

| 周次 | 开发 Skill | 说明 |
|------|------------|------|
| 第3周 | 指标定义、指标树拆解、趋势分析、异常分析、结构拆解 | 基础分析核心 Skill |
| 第4周 | 留存分析、漏斗分析、用户分层、内容消费分析、日报周报生成 | 用户与增长、业务分析 Skill |

---

### 第5-6周：中高阶 Skill 开发

**目标**：补齐专项分析和实验分析能力

| 周次 | 开发 Skill | 说明 |
|------|------------|------|
| 第5周 | AB实验设计、样本量估算、显著性检验、实验结果分析、实验复盘 | 实验分析类 |
| 第6周 | 埋点方案设计、埋点校验、数据需求文档、数据看板设计、看板优化建议 | 数据建设类 |

**关键串联**：开始做 Skill 之间的衔接，形成完整分析链路

```
业务问题 → 指标树拆解 → 趋势分析 → 异常分析 → 留存/漏斗专项 → 策略建议 → 汇报输出
```

---

### 第7-8周：联调与产品化

**目标**：把这套 Skill 从"能写"变成"能稳定用"

| 周次 | 重点任务 | 产出 |
|------|----------|------|
| 第7周 | Skill编排、多Skill串联、统一调度入口、完善案例库 | 工作流编排、案例库 |

## 五、Skill 优先级清单

### P0（必须先做）- 8个

| Skill | 解决的问题 |
|-------|-----------|
| 指标定义生成 | 输入业务目标，输出指标定义、计算公式 |
| 指标树拆解 | 输入核心指标，输出驱动因素拆解 |
| 趋势分析 | 输入时间序列，输出趋势、拐点判断 |
| 异常波动分析 | 输入异常指标，输出定位、拆因、原因 |
| 留存分析 | 输出新老用户留存、流失节点 |
| 漏斗分析 | 输出转化率、流失点、关键环节 |
| 日报周报生成 | 自动生成管理层可读的周期性报告 |
| 埋点方案设计 | 输出完整埋点需求文档 |

### P1（第二批重点）- 8个

| Skill | 解决的问题 |
|-------|-----------|
| 用户分层分析 | 对活跃、消费、价值、频次分层 |
| 内容消费分析 | 曝光、点击、消费时长、完播、互动 |
| AB 实验设计 | 实验方案、分组策略、评估指标 |
| 实验结果解读 | 显著性判断、效果估算、结论输出 |
| 数据看板设计 | 看板布局、指标选择、可视化建议 |
| 渠道质量分析 | 渠道ROI、用户质量评估 |
| Push召回效果分析 | 召回率、转化率、疲劳度分析 |
| 经营诊断与策略建议 | 问题定位+业务动作建议 |

### P2（后续增强）- 余下 Skill

- 准实验分析（DID/PSM/IV/RDD）
- 用户生命周期分析
- 风险预警
- 高层汇报材料生成
- 自动汇报 PPT/文档生成

## 八、核心原则

> **不要一上来追求"全"，要追求"深"和"稳"**

真正好用的互联网数据分析 Skill，关键是三点：
1. **输入标准** - 口径统一、格式清晰
2. **分析路径稳定** - 每次输出质量一致
3. **输出真的能指导业务动作** - 结论可执行

**2个月最重要的事**：先把 8~12 个高频核心 Skill 做深做稳，再逐步扩展
