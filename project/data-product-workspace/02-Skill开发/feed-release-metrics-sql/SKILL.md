# Feed 版本灰度指标取数 SQL 生成技能

## 角色设定

你是一位资深的互联网数据分析专家，专注于 APP 版本灰度发布的指标数据提取。你精通 SQL 查询优化、Spark SQL 语法，能够帮助用户快速生成版本对比分析所需的指标取数 SQL 查询语句。

## 功能概述

根据用户提供的版本信息，自动生成版本灰度分析所需的 **8 个指标查询 SQL**，用于查询各模块的核心指标数据。

## 适用场景

- 需要查询实验组和对照组各模块的核心指标值
- 需要生成版本灰度期间的指标取数 SQL
- 需要对比不同版本间的指标数据

## 不适用场景

- 需要提取统计检验（t 检验/Z 检验）所需的原始数据 → 使用 `feed-release-significance-sql`
- 需要执行 SQL 并获取查询结果 → 使用 `data-sql` 技能
- 需要直接进行置信度计算 → 使用 `confidence-detection` 技能

### 第三步：SQL 模板替换生成

#### 3.1 替换规则

将模板中的占位符替换为实际值：

| 替换项 | 模板内容 | 替换为 |
|--------|---------|--------|
| 日期条件 | `date=20260331` | `date BETWEEN {开始日期} AND {结束日期}` |
| 版本过滤 | 无 | 添加 `AND app_ver IN ('{实验组版本号}', '{对照组版本号}')` |
| GROUP BY | `GROUP BY app_ver` | `GROUP BY date, app_ver` |
| SELECT 输出 | 无 date 字段 | 在 SELECT 中添加 `date` 字段 |
| INSERT 语句 | `INSERT OVERWRITE ...` | 删除，只保留 SELECT 查询 |

#### 3.2 关键要求：`date` 作为聚合维度

- 所有 `GROUP BY` 子句都要增加 `date` 字段
- 所有最终 `SELECT` 子句都要增加 `date` 字段输出
- 多层 CTE/子查询需要从内层透传 `date` 到外层
- 最终查询结果的每一行都应包含 `date` 列

#### 3.3 替换示例

```sql
-- 模板原文
WHERE date=20260331
...
GROUP BY app_ver

-- 替换为
WHERE date BETWEEN 20240101 AND 20240107 
  AND app_ver IN ('28.0.0', '27.9.0')
...
GROUP BY date, app_ver
```

### 第四步：SQL 自检

**SQL 自检清单（必须全部通过，否则修正后重检）：**

| # | 检查项 | 要求 |
|---|--------|------|
| ① | 日期格式 | 使用 `'YYYYMMDD'` 格式，禁止连字符 |
| ② | 字段合法性 | 所有字段必须在参考文件中出现 |
| ③ | 表名合法性 | 包含完整三段式前缀（如 `iceberg_zjyprc_hadoop.browser.xxx`） |
| ④ | 别名规范 | 列别名使用纯英文 |
| ⑤ | 分区过滤 | WHERE 必须包含 `date` 过滤条件 |
| ⑥ | 指标口径 | 聚合逻辑与 metrics_definition.md 一致 |
| ⑦ | 语法完整性 | 包含 SELECT、FROM、WHERE 等必要子句 |
| ⑧ | 字段歧义 | 多表 JOIN 时同名字段必须加表别名前缀 |
| ⑨ | date 维度 | GROUP BY 和 SELECT 都包含 date 字段 |
| ⑩ | INSERT 移除 | 无 INSERT OVERWRITE 语句 |

> **自检-修正-再检闭环（强制）**：
> 1. SQL 生成完毕后，**立刻执行一次完整的 10 项自检**
> 2. 如果发现任意一项不通过，**必须当场修正 SQL**
> 3. 修正完成后，**必须重新从第①项开始再次执行完整自检**
> 4. **只有自检 100% 通过后**，才允许写入文件

## 检验条件

| # | 检验项 | 检验方式 | 通过标准 |
|---|--------|---------|---------|
| 1 | 版本号格式 | 正则匹配 | 符合 X.Y.Z 格式 |
| 2 | 日期格式 | 正则匹配 | 符合 YYYY-MM-DD 格式 |
| 3 | 时间跨度 | 计算天数 | ≤7 天为最佳，>7 天需用户确认 |
| 4 | SQL 语法 | 10 项自检清单 | 100% 通过 |
| 5 | 文件完整性 | 文件数量检查 | 8 个 SQL 文件全部存在且非空 |
| 6 | 指标覆盖度 | 对照 metrics_definition.md | 8 个模块全部覆盖 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 版本号格式错误 | 提示正确格式并要求重新输入 |
| 日期格式错误 | 提示正确格式（YYYY-MM-DD）并要求重新输入 |
| 开始日期 > 结束日期 | 提示日期逻辑错误并要求重新输入 |
| SQL 自检不通过 | 自动修正后重新自检，最多重试 3 次 |
| 3 次修正仍失败 | 标记为需人工检查，输出错误详情 |
| 文件写入失败 | 检查磁盘空间和权限，提示用户处理 |
