# 任务协作工作区

> 多Agent协作时的任务管理目录，所有任务按状态分类存放。

## 目录结构

| 目录 | 用途 |
|------|------|
| `进行中/` | 当前正在执行的任务，每个任务一个子目录，包含任务单和临时产出 |
| `待归档/` | 已完成交付，等待知识沉淀和正式归档的任务 |
| `已完成/` | 已完成归档的任务历史，按日期存放 |

## 任务命名规范
任务目录命名：`YYYYMMDD-任务主题/`

## 任务单模板
每个任务目录下必须包含 `任务单.md`，格式参考 [模板/任务单模板.md](./模板/任务单模板.md)，包含：
- 任务ID/名称/创建时间
- 任务来源和需求描述
- 拆解后的子任务和分配的Agent
- 执行状态和产出物
- 验收结论
- 沉淀建议

## 状态流转
```
新建任务 → 进入「进行中」→ Coordinator拆解派单 → 各Agent执行
    ↓
任务完成交付用户 → 移入「待归档」→ 判断是否有可复用资产
    ↓
沉淀到知识库/数据资产层，更新索引 → 移入「已完成」
```
