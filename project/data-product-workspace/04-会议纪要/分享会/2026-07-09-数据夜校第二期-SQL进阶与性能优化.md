## 📚 课程核心内容

### 1. 数据地图使用指南
- 如何通过数据地图查找表、理解表结构
- 表血缘关系查看，理解数据流转
- 分区字段使用注意事项
- 常用表推荐：流量表、用户表、内容表、消费表等核心表介绍

### 2. SQL高级语法实战
- **JOIN操作**：INNER JOIN/LEFT JOIN/RIGHT JOIN/FULL JOIN的区别与适用场景
- **WITH子句(CTE)**：拆分复杂SQL，提高可读性，避免嵌套过深
- **聚合函数**：SUM/COUNT/AVG/MAX/MIN + GROUP BY分组统计
- **窗口函数**：ROW_NUMBER()/RANK()/SUM() OVER()等常用窗口函数使用场景
- **时间函数**：日期格式化、时间加减、日期截断等常用时间处理

### 3. SQL性能优化原则
| 优化项 | 具体做法 |
|--------|----------|
| 分区裁剪 | 必须带分区字段（如dt）过滤，避免全表扫描 |
| 列裁剪 | 只SELECT需要的字段，避免SELECT * |
| 小表关联大表 | 关联时小表放前面，减少shuffle数据量 |
| 避免数据倾斜 | 注意NULL值、热点key导致的数据倾斜 |
| LIMIT先行 | 测试时加LIMIT先看样本，确认逻辑正确再跑全量 |
| 执行计划查看 | 通过EXPLAIN查看执行计划，识别性能瓶颈 |

## 💡 常见问题与解答

1. **Q：怎么判断SQL写的对不对？**
   A：先跑1天数据，与已知看板口径对比，交叉验证多维度汇总是否一致。

2. **Q：大表关联慢怎么办？**
   A：先过滤再关联，利用分区裁剪，必要时拆分SQL分步执行。

3. **Q：nl-sql生成的SQL能不能直接用？**
   A：必须人工校验口径和逻辑，AI是辅助工具，最终责任人是使用者。
