有了想法，下一步是"找到数据"

模块1 学会用数据发现问题、自证。要落地，得先找到数据——你想要的指标，藏在哪张表里？

👉 销售/运营：不一定自己写 SQL，但要会"看懂有哪些数据、找对表、把需求讲清楚"。

数据地图 = 数据的"图书馆检索系统"

能查什么说明
表信息、负责人谁维护、出问题找谁
字段含义、类型、枚举每列是什么意思、能取哪些值
分区规则、更新频率多数按 date、T+1
表血缘、样例数据从哪来被谁用、真实长什么样
产出信息（加工逻辑）点任务名，看字段是怎么算出来的

💡 先查清"数据在哪张表、字段什么含义、怎么加工的"，再动手写 SQL。

💡 优先找名字带 _di（日分区）的聚合表；浏览器业务表在 browser. 库下。

核心两表（覆盖 80% 需求）

表（简称）粒度用途
…miui_didid+dateAPP 启动、版本、启动方式、整体时长
…label_didid+date信息流曝光/点击/播放、消费时长、标签

两表用 date + did 关联；算人数 COUNT(DISTINCT did)；时长 /60000 转分钟；自然流量 is_top=0。

📖 全名与字段清单见《内容生态业务线核心表速查手册》：browser.dwm_browser_event_aggregation_miui_di / …_label_di

数据工坊：写完 SQL 在哪里跑

功能要点
执行方式光标停某段 = 只跑该段；选中部分 = 只跑选中
查询历史关键词检索跑过的 SQL（如"热点卡片人群圈选"），相似需求先搜历史复用
表管理写 CREATE 或上传 CSV/Excel 自动建表；集群 sbook、库 new_home；表名别重复
执行引擎默认选"自动"即可

🔴 分号分隔的多段代码会"并行提交"：有先后依赖的（先建临时表再查）别一起跑，并行会取到空数据——一段段按顺序执行。

权限申请：按流程办下来

环节怎么做
入口数据工厂 → 空间权限申请
选空间桌面内容中心-产品
有效期选 365 天（免得频繁重申）
审批链上级 → 表管理员 → 空间管理员（多级）

💡 看字段加工逻辑（数据地图"产出信息"里点任务名）是单独权限；被驳回 → 截图发群、联系研发确认口径。

📖 完整操作见《数据工坊与权限操作手册》。
