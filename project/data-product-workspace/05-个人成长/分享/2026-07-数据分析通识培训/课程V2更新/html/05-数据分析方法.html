从"报数"到"给结论"：排查、漏斗、分群

⏱️ 1.5 小时🎯 产品重点，销售/运营选学

从"报数"到"给结论"

模块 2-4 你已经会找数、写 SQL / 让 AI 取数了。这节课再进一步：拿到一个业务问题（如"CTR 为什么降了"），怎么把查询结果系统地拆解、定位、给结论。

⚠️ 动手分析前先做一件事：对齐口径、校验数据。现场就踩过——自己算的 DAU 和看板差一截，多半是过滤/口径没对齐。别拿错的数做分析。

排查问题的四把尺子

① 好维度

强区分度：按 PUSH 类型拆 3% vs 7.8% 一眼定位；按性别拆没用

② 好参照

基准线：时间参照 / 对照组 / 大盘 / 目标

③ 好视角

对的样本（最小作用颗粒度）+ 对的周期

④ 好起止点

链路分析：改动只影响该功能"往后"的环节

四种常用分析方法

方法一句话
对比分析同比/环比/对照/细分——找差异、判好坏
趋势分析看周期、拐点、异常，别被单点误导
分群分析按属性/行为拆，定位问题群体
漏斗分析多步链路转化，找流失瓶颈

💡 分群落到 SQL 就是 CASE WHEN 打标（呼应模块3）：把用户按新老 / 活跃度 / 来源打成组再聚合。

⚠️ 辛普森悖论：整体降但分群都涨——大盘异动先拆维度，别只看平均。

A/B

PUSH 全链路：下发 → 到达 → 展示 → 点击 → 落地页 → 消费。CTR 降别只盯点击，拆漏斗看哪环掉。

案例：PUSH CTR 降 2pct 排查

步骤发现
拆时间周四 8.3% → 周日 6%，凌晨 2-4 点特别低
拆类型新热点推送 CTR 仅 3%，贡献 80% 下降
拆用户新用户从 7% 降到 3.5%，重灾区
拆漏斗问题在"展示→点击"

结论：新策略对老用户正向、对新用户严重负向——不必全量回滚，立刻调整。

👉 这些"拆时间 / 类型 / 用户 / 漏斗"的取数，就让 AI 用 label_di 生成分维度 SQL、你按 4 步校验，最快。

常见分析误区

  • 唯数据论（异常先查埋点/口径）
  • 忽略样本量（几百人的组别别过度解读）
  • 相关当因果
  • 过度拆解 / 只看相对不看绝对
