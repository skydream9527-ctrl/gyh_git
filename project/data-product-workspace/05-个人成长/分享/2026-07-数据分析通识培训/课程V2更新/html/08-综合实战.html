PUSH CTR 突然下降怎么查？

周日 CTR 从 8% 降到 6%，是不是上周四的新策略有问题？要不要回滚？

明确问题

定指标(CTR=点击/到达)、时间、拆分维度

校验数据

先排除延迟/埋点/口径变化

对比定位

拆时间/类型/人群/漏斗

深入原因

内容/受众/时间/灰度没覆盖新用户

拆维度，锁定问题

拆解发现
拆时间逐步下降，凌晨 2-4 点特别低
拆类型新"热点推送"CTR 仅 3%，拉低整体
拆用户新用户 7%→3.5%，重灾区
拆漏斗问题在"展示→点击"

结论先行：不必全量回滚，新用户 7 天内不推时政 + 凌晨不批量发 + 灰度覆盖新用户。

现场命题：一个刁钻取数怎么拆

命题：昨天，浏览器自建源、视频沉浸式场景的"人均评论发布次数"。条件一多就懵——拆步骤 + 让 AI 补。

-- 人均 = 评论总次数 / 发过评论的人数（先圈场景，再算） SELECT SUM(comment_pv)*1.0 / COUNT(DISTINCT CASE WHEN comment_pv>0 THEN did END) AS avg_cmt FROM …内容评论表 WHERE date = '${date-1}' AND source='自建源' AND item_tab IN('短视频','小视频') AND page='沉浸式';

🎯 步骤：① 先取评论总量验证 → ② 加"自建源/视频/沉浸式"场景条件 → ③ 再算人均。一步步来，别一口气写完（字段以数据地图为准）。

业务版：给一次活动做复盘

不写复杂 SQL 也能走完整流程，把自证三板斧用起来：

  • 明确 + 找数：不会写 SQL 就按"提需求五要素"找数据同学 / 让 AI 取
  • 对比定位：活动前后 + 分渠道，排除大盘自然增长
  • 结论建议：结论先行——"净拉新 X，A 渠道最优、C 亏损，砍 C 保 A"

常见卡点速查

卡点怎么办
算的和看板对不上先对口径，再找数据同学核对
SQL 跑很久90% 没加 date；先查 1 天验证
找不到根因MECE 拆：时间→人群→模块→链路
实验不显著看样本量、效果大小、是否某细分显著
