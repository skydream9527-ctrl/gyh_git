# AB 实验判断 Checklist

> 用途：拿到一份实验报告，按这个清单逐条过一遍，都答得上，结论才可信。配套模块6《AB 实验原理与判断》。

---

## 一、快速判断清单（按顺序检查）
- [ ] **实验周期**：够长吗？覆盖工作日 + 周末吗？过了新奇效应期吗？
- [ ] **样本量**：达到 MDE 要求的最小样本量吗？（能检测出你关心的最小效果吗）
- [ ] **SRM（样本比例失衡）**：两组实际流量比例符合设计（如 50:50）吗？偏差在合理范围吗？
- [ ] **基线 / AA 测试**：实验前两组的新老占比、活跃度、版本分布、核心指标基线一致吗？
- [ ] **核心指标**：提升幅度多少？p 值 < 0.05 吗？置信区间不包含 0 吗？
- [ ] **护栏指标**：有没有显著负向（通知关闭率、卸载率、留存、整体时长、负反馈）？
- [ ] **分维度效果**：新老 / 高低活 / 版本 / 渠道效果一致吗？有没有辛普森悖论？
- [ ] **实验干扰**：有没有其他实验同时开在相同用户层 / 相同位置？流量正交吗？
- [ ] **业务价值**：绝对值提升多少？影响多少用户？ROI 划算吗？
- [ ] **结论逻辑**：结论符合数据吗？有没有其他可能的解释？

---

## 二、三条判断"红线"（不满足就别急着上线）
1. **不显著别上**：p > 0.05 或置信区间包含 0 → 不能排除随机波动，延长时间/扩流量再看。
2. **护栏负向别上**：核心指标涨、但护栏指标（关闭率/卸载/留存）显著负 → 短期赢长期输。
3. **辛普森别只看大盘**：大盘不显著，但分群有显著正负 → 分群处理，不要一刀切全量。

---

## 三、常见不可信结论的"信号词"
| 信号 | 问题 | 正确姿势 |
|------|------|----------|
| "才开 1 天就涨了" | 新奇效应、样本不足 | 开够一个完整周再看 |
| "虽然 p=0.08 但还是涨了" | 不显著硬解读 | 不显著就是不显著 |
| "相对提升 20%" | 可能基数极小 | 同时看绝对值和业务影响 |
| "大盘没降，全量吧" | 可能有辛普森悖论 | 分维度拆开看 |
| "灰度时是涨的" | 灰度人群不全（如只测老用户） | 灰度要覆盖全人群 |

---

## 四、统计三件事（不用背公式，够看懂结果）
- **p < 0.05** = 结果是巧合的概率低于 5%，可认为显著。
- **置信区间不包含 0** = 效果方向可信（含 0 = 不显著）。
- **样本量要够** = 达到 MDE，否则真效果也测不出。

> 💡 记住：p 值小 ≠ 效果大。p 值管"可不可信"，提升幅度和置信区间管"效果多大"。
