# 数据分析常用框架Checklist
> 做分析前对照检查，避免遗漏

---

## 一、数据分析完整流程Checklist
- [ ] **明确分析目标**：解决什么问题？谁用？做什么决策？
- [ ] **明确时间范围**：分析什么时间段的数据？对比基准是什么？
- [ ] **确定核心指标**：看什么指标？指标口径是什么？
- [ ] **确定拆分维度**：需要按什么维度拆分？新老用户/渠道/功能/时间？
- [ ] **先校验数据准确性**：数据有没有延迟？口径对不对？和已知数据对比是否一致？
- [ ] **做对比分析**：和基准比差异多大？是否显著？
- [ ] **做分群分析**：拆分维度看，是整体问题还是局部问题？有没有辛普森悖论？
- [ ] **做漏斗分析**：链路问题要拆转化漏斗，定位流失最大的环节
- [ ] **关联业务动作**：异常时间点有没有上线/活动/bug/外部事件？
- [ ] **验证原因**：数据是否支持你的结论？有没有其他可能的解释？
- [ ] **输出结论**：结论先行，数据支撑，原因拆解
- [ ] **给出建议**：建议具体可落地，有预估收益

---

## 二、常见分析场景框架

### 1. 指标异动排查框架
1. 确认数据是否准确（口径/更新/埋点）
2. 对比历史看是否异常（同比/环比/周期性）
3. 分维度下钻，定位问题范围（时间/人群/业务维度）
4. 拆漏斗定位流失环节（链路类指标）
5. 匹配近期业务动作，找到根因
6. 评估影响范围和影响程度
7. 给出解决建议

### 2. 新功能/新策略效果评估框架
1. 明确评估指标：核心指标、辅助指标、反向指标（避免副作用）
2. 确定对比基准：AB实验看对照组，没有AB看上线前后时间+相似人群对比
3. 看整体效果：核心指标是否显著提升？
4. 分维度看：对哪些用户群效果好？哪些效果差？为什么？
5. 看长期效果：是不是新奇效应？有没有长期留存/价值？
6. 看负面影响：有没有损害其他指标？
7. 结论：全量/灰度/回滚/优化后再测

### 3. 用户分群分析框架
常用分群维度：
- 用户生命周期：新用户/成长/成熟/流失/召回用户
- 用户活跃度：高活/中活/低活
- 行为特征：有没有用某功能/点击过PUSH/消费过某类内容
- 渠道来源：主启/三方/PUSH/搜索
- 设备属性：版本/机型/系统/地域

【行动建议】
