## 一、数据地图：找数与溯源

1. **搜表进详情页**：按表名（如 `label 表`）搜索 → 看字段描述、样例数据、分区规则。
2. **产出信息（溯源加工逻辑）**：详情页的"产出信息"里**点任务名**，可看到字段是怎么算出来的（如 `is_dau_2024`、视频播放量的计算规则）。
   > ⚠️ 查看产出信息的代码**需要权限**。被驳回时：截图发群，联系研发确认口径。
3. **数据地图是"数据的图书馆检索系统"**——先查清"数据在哪张表、字段什么含义、怎么加工的"，再动手写 SQL。

## 二、数据工坊：执行 SQL 的工具

### 2.1 执行方式
- **执行单段**：光标停在某段，点执行只跑这一段。
- **选中执行**：选中部分代码，只跑选中的。
- 🔴 **分号分隔的多段代码会"并行提交"**：**有先后依赖的多段（如先建临时表再查）千万别一起跑**，并行会导致后面的取到空数据。有依赖就一段段按顺序执行。
- **执行引擎**：默认选"自动"即可。

### 2.2 查询历史（复用过往代码）
- "查询历史"支持**关键词检索**过往跑过的 SQL（如搜"热点卡片人群圈选"）。
- 🎯 遇到相似需求，先搜历史，复用别人/自己写过的代码，比从零写快得多。

### 2.3 结果验证（每次必做）
- 看**数量级是否合理**：DAU 百万级、CTR 百分之几、人均时长几十分钟。
- **单位换算**：时长字段是**毫秒**，`/60000` 才是分钟。
- 对不上就回去查口径/分区/去重（呼应模块3 的"4 步校验"）。

## 三、权限申请流程

| 步骤 | 说明 |
|------|------|
| 入口 | 数据工厂 → 空间权限申请 |
| 选空间 | **桌面内容中心-产品** |
| 有效期 | 选 **365 天**（免得频繁重申） |
| 审批链 | **上级 → 表管理员 → 空间管理员** 多级审批 |
| 申请理由 | 写清楚用途（如"XX 分析需要 XX 表"），审批更快 |

> 💡 查看字段加工逻辑（产出信息代码）是**单独的权限**，和查数权限不同，可能需要单独找研发。

## 四、表管理与建表

- **两种建表方式**：① 写 `CREATE TABLE` 代码建表；② 上传文件（CSV / Excel）**自动建表**。
- **常用位置**：集群 **sbook**，库 **new_home**（内容中心）。
- ⚠️ **表名不要重复**——建表前先确认库里没有同名表。

## 五、平台特有语法与高频坑

| 项 | 说明 | 示例 / 提醒 |
|----|------|-------------|
| **动态日期参数** | 调度/复用时用参数代替写死日期 | `WHERE date = '${date-1}'`（昨天） |
| **中文括号报错** | 全角 `（）` 会导致语法错误 | 一律用英文半角 `()`，中文标点也当心 |
| **必带日期分区** | 不加 date = 全表扫描，几小时跑不完 | `WHERE date = 20260628` 放最前 |
| **CASE WHEN** | 分类打标、条件分支 | 见下方 5.1 |
| **is_dau_2024 口径** | 内容中心"当日活跃"的定义 | 曝光+点击+播放+进入空间次数 > 0 **且非置顶(is_top=0)** |

### 5.1 CASE WHEN（现场高频用到，务必会）
从上往下匹配，命中即停（**互斥**），必须 `END` 结尾：
```sql
SELECT
  CASE
    WHEN score >= 90 THEN '优秀'
    WHEN score >= 60 THEN '及格'      -- 走到这说明 <90
    ELSE '不及格'
  END AS grade
FROM ...;
```
> ⚠️ 三个坑：① 区间从上往下写、别重叠；② 一定 `END` 结尾；③ 中文标点会报错。分类打标、把长表转宽表都靠它。

## 六、一次取数的完整落地流程（串起来）
```
① 数据地图搜表、看字段/产出信息   → 找数（模块2）
② 没权限就按流程申请（内容中心-产品/365天）
③ 数据工坊里写 SQL（AI 生成 + 4 步校验）→ 模块3/4
④ 先跑单日验证逻辑、看数量级         → 结果验证
⑤ 没问题再放大时间范围；复用查询历史
```
