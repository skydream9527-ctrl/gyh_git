# 第3章：计算字段与函数

> 对应原书：第 7–8 课（创建计算字段、使用函数处理数据）
> 目标：会在查询里"现算"新列，会用函数加工文本/日期/数值。

---

## 3.1 计算字段是什么

数据库里存的是**原始数据**，报表要的常是**加工后**的值（拼接、算术、格式化）。这些不改动数据、只在查询时"现算"出来的列，就是**计算字段**。

---

## 3.2 拼接字段 + 别名

把供应商名和国家拼成 `名 (国家)` 的展示格式：
```sql
-- 标准 SQL 各库拼接写法不同：Access/SQL Server 用 +，DB2/Oracle 用 ||
SELECT vend_name + ' (' + vend_country + ')' FROM Vendors;   -- 书里(SQL Server)
-- 我们的 Hive/Spark：用 CONCAT
SELECT CONCAT(vend_name, ' (', vend_country, ')') AS vend_title
FROM Vendors;
```
> 💡 **别名 `AS`**：给计算列起个名字，否则列名是一串表达式、没法引用。`AS vend_title` 后，结果列就叫 vend_title。

---

## 3.3 算术计算

```sql
SELECT prod_id,
       quantity,
       item_price,
       quantity * item_price AS expanded_price   -- 现算每个订单物品的小计
FROM OrderItems
WHERE order_num = 20008;
```
支持 `+ - * /`，可用括号控制优先级。

---

## 3.4 函数：加工数据

> ⚠️ **函数是 SQL 里最不通用的部分**——文本/日期函数各数据库名字差异很大。学思路，落地时查你所用平台（Hive/Spark）的函数名。

**文本函数（示例）：**
| 作用 | 常见函数 |
|------|----------|
| 转大小写 | `UPPER(x)` / `LOWER(x)` |
| 去空格 | `TRIM(x)` / `LTRIM` / `RTRIM` |
| 截取子串 | `SUBSTR(x, 起, 长)` |
| 长度 | `LENGTH(x)` |

**日期函数（示例）：** 提取年份做筛选是最高频需求：
```sql
-- 书里(不同库)：DATEPART / YEAR / EXTRACT / strftime 各不同
-- 思路统一：取 order_date 的年份 = 2020
SELECT order_num FROM Orders
WHERE YEAR(order_date) = 2020;      -- Hive/Spark 有 year() 函数
```
> 🔴 我们平台还有个大坑：分区列 `date` 和业务日期是两码事，**查询先靠 `WHERE date=…` 做分区裁剪**，再用日期函数做业务筛选。

**数值函数：** `ABS()`、`ROUND(x,n)`（保留小数，算率常用）、`SQRT()` 等，跨库较一致。

---

## 本章要点
1. 计算字段 = 查询时现算的列（拼接、算术、函数），不改原数据。
2. 拼接我们用 `CONCAT(...)`；计算列**记得用 `AS` 起别名**。
3. 函数最不通用，尤其文本/日期——**落地查所用平台的函数**。
4. 取年份等日期筛选很常用，但别忘了先加分区 `date` 过滤。

## 小练习（附参考答案）
1. 把 Vendors 的 `vend_name` 和 `vend_country` 拼成 `名 - 国家`，列名叫 title。
2. 算订单 20005 里每个物品的小计（数量×单价）。
3. 取 2020 年下的订单号。

<details><summary>参考答案</summary>

```sql
-- 1
SELECT CONCAT(vend_name, ' - ', vend_country) AS title FROM Vendors;
-- 2
SELECT prod_id, quantity*item_price AS subtotal
FROM OrderItems WHERE order_num = 20005;
-- 3
SELECT order_num FROM Orders WHERE YEAR(order_date) = 2020;
```
