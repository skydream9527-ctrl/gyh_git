# 第5章：子查询与联结

> 对应原书：第 11–13 课（子查询、联结表、高级联结）
> 目标：会用子查询"套娃"，会用 JOIN 把多张表拼起来——这是跨表分析的核心。

---

## 5.1 子查询：查询里套查询

**用子查询做过滤**：先查出一批 id，再拿去过滤。
需求：找出订购了产品 `RGAN01` 的所有顾客名。沿着 OrderItems → Orders → Customers 一层层套：
```sql
SELECT cust_name FROM Customers
WHERE cust_id IN (
  SELECT cust_id FROM Orders
  WHERE order_num IN (
    SELECT order_num FROM OrderItems WHERE prod_id = 'RGAN01'
  )
);
```
> 💡 从最里层往外读：里层找出含该产品的订单号 → 中层找出这些订单的顾客 id → 外层拿顾客 id 查名字。

**用子查询做计算字段**：给每个顾客算订单数：
```sql
SELECT cust_name,
       (SELECT COUNT(*) FROM Orders
        WHERE Orders.cust_id = Customers.cust_id) AS orders
FROM Customers;
```
> ⚠️ 子查询直观但**性能常不如 JOIN**，能用 JOIN 写的优先 JOIN（尤其大表）。

---

## 5.2 联结（JOIN）：把多表拼起来

需求：产品来自哪个供应商——产品在 Products、供应商名在 Vendors，靠 `vend_id` 拼：
```sql
SELECT vend_name, prod_name, prod_price
FROM Vendors, Products                       -- 老式写法：FROM 两表 + WHERE 关联
WHERE Vendors.vend_id = Products.vend_id;
```
**推荐用 `INNER JOIN ... ON`（更清晰、不易漏关联）：**
```sql
SELECT vend_name, prod_name, prod_price
FROM Vendors
INNER JOIN Products ON Vendors.vend_id = Products.vend_id;
```
> 🔴 **一定写关联条件**！漏了就是**笛卡尔积**（每行配对方所有行，行数爆炸）——主课模块4 案例：100万×80万=8000亿行。

**联结多个表**（顾客 + 订单 + 物品 + 产品，四表串起来）：
```sql
SELECT cust_name, order_num, prod_name, quantity
FROM Customers
JOIN Orders     ON Customers.cust_id = Orders.cust_id
JOIN OrderItems ON Orders.order_num  = OrderItems.order_num
JOIN Products   ON OrderItems.prod_id = Products.prod_id;
```

## 5.3 高级联结

**表别名**（简化书写）：
```sql
SELECT c.cust_name, o.order_num
FROM Customers AS c
JOIN Orders   AS o ON c.cust_id = o.cust_id;
```
**内联结 vs 外联结：**

| 类型 | 保留什么 |
|------|----------|
| `INNER JOIN` | 只保留两表**都匹配**的行 |
| `LEFT JOIN` | **左表全保留**，右表没匹配填 NULL |
| `RIGHT JOIN` | 右表全保留（少用，常改写成 LEFT） |

需求：列出**所有**顾客及其订单数——包括**没下过单**的顾客（用 LEFT JOIN 才不会漏）：
```sql
SELECT c.cust_name, COUNT(o.order_num) AS orders
FROM Customers AS c
LEFT JOIN Orders AS o ON c.cust_id = o.cust_id
GROUP BY c.cust_name;
```
> ⚠️ 没下单的顾客，`o.order_num` 是 NULL；用 `COUNT(o.order_num)` 会算成 0（COUNT 跳过 NULL），正是我们要的。这正是主课"LEFT JOIN 后配 COALESCE / 注意 NULL"的道理。

**自联结**：同一张表自己和自己联（如"找和某产品同供应商的其他产品"），用别名区分两个副本。

---

## 本章要点
1. 子查询"套娃"直观，但大表优先用 JOIN（性能更好）。
2. JOIN 用 `... JOIN ... ON 关联条件`，**必写 ON**，否则笛卡尔积爆炸。
3. `INNER JOIN` 只留都匹配的；`LEFT JOIN` 左表全留、右表缺失为 NULL。
4. 多表 JOIN 沿着外键一路串；用表别名让 SQL 更短。

## 小练习（附参考答案）
1. 用 JOIN 列出每个产品名及其供应商名。
2. 列出所有顾客名及其订单数（没下单的算 0）。
3. 找出订购了产品 'BR03' 的顾客名（子查询或 JOIN 均可）。

<details><summary>参考答案</summary>

```sql
-- 1
SELECT p.prod_name, v.vend_name
FROM Products p JOIN Vendors v ON p.vend_id = v.vend_id;
-- 2
SELECT c.cust_name, COUNT(o.order_num) AS orders
FROM Customers c LEFT JOIN Orders o ON c.cust_id=o.cust_id
GROUP BY c.cust_name;
-- 3（JOIN 写法）
SELECT DISTINCT c.cust_name
FROM Customers c
JOIN Orders o     ON c.cust_id=o.cust_id
JOIN OrderItems i ON o.order_num=i.order_num
WHERE i.prod_id='BR03';
```
