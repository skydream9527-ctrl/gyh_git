# 第4章：汇总与分组

> 对应原书：第 9–10 课（汇总数据、分组数据）
> 目标：会用聚集函数算总数/平均，会用 GROUP BY 分组、HAVING 过滤分组。**这一章是数据分析最高频的部分。**

---

## 4.1 聚集函数：把很多行"算成一个数"

| 函数 | 作用 |
|------|------|
| `COUNT(*)` | 行数（含 NULL） |
| `COUNT(列)` | 该列**非 NULL** 的行数 |
| `COUNT(DISTINCT 列)` | 该列不同值的个数（算"多少人/多少种"） |
| `SUM(列)` | 求和 |
| `AVG(列)` | 平均（**自动跳过 NULL**） |
| `MAX(列)` `MIN(列)` | 最大/最小 |

```sql
SELECT COUNT(*)        AS num_items,      -- 有多少个产品
       MIN(prod_price) AS price_min,
       MAX(prod_price) AS price_max,
       AVG(prod_price) AS price_avg
FROM Products;
```

> ⚠️ 两个高频坑（主课也反复强调）：
> - **`COUNT(*)` vs `COUNT(列)`**：后者不数 NULL。算"多少人"用 `COUNT(DISTINCT 用户id)`。
> - **`AVG` 会跳过 NULL**：想算"人均"（含没消费的人当 0）时，`AVG` 会偏高，应 `SUM(x)/COUNT(DISTINCT 用户)`。

**聚集不同值：** `AVG(DISTINCT prod_price)` 只对不同价格求平均（少用，注意语义）。

---

## 4.2 GROUP BY：分组聚集

不加 GROUP BY，聚集函数把整表算成 1 个数；加了 GROUP BY，就**按分组各算一个**：
```sql
-- 每个供应商各有多少个产品
SELECT vend_id, COUNT(*) AS num_prods
FROM Products
GROUP BY vend_id;
```
规则：
- `SELECT` 里出现的**非聚集列**，必须都在 `GROUP BY` 里。
- NULL 会单独成一组。

---

## 4.3 HAVING：过滤"分组"

`WHERE` 过滤**行**，`HAVING` 过滤**分组**（对聚集结果加条件）：
```sql
-- 只要"产品数≥2"的供应商
SELECT vend_id, COUNT(*) AS num_prods
FROM Products
GROUP BY vend_id
HAVING COUNT(*) >= 2;
```
> 🎯 一句话记住：**WHERE 在分组前筛行，HAVING 在分组后筛组。** 两者可同时用（先 WHERE 再分组再 HAVING）：
```sql
SELECT cust_id, COUNT(*) AS orders
FROM Orders
WHERE order_date >= '2020-01-01'   -- 先筛行
GROUP BY cust_id
HAVING COUNT(*) >= 2;               -- 再筛组
```

---

## 4.4 SELECT 子句顺序（务必记牢）

```
SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY
```
写错顺序直接报错。这也是主课"五段式"的完整版。

---

## 本章要点
1. 聚集函数：COUNT/SUM/AVG/MAX/MIN；算人数用 `COUNT(DISTINCT)`。
2. `COUNT(列)` 和 `AVG` 都**跳过 NULL**，算人均当心偏高。
3. `GROUP BY` 分组，SELECT 的非聚集列必须都在 GROUP BY 里。
4. `WHERE` 筛行、`HAVING` 筛组；子句顺序 SELECT→FROM→WHERE→GROUP BY→HAVING→ORDER BY。

## 小练习（附参考答案）
1. 算 OrderItems 表里订单 20005 的物品总数量和总金额。
2. 每个订单（order_num）各有多少个物品。
3. 找出"物品数≥3"的订单号。

<details><summary>参考答案</summary>

```sql
-- 1
SELECT SUM(quantity) AS qty, SUM(quantity*item_price) AS total
FROM OrderItems WHERE order_num = 20005;
-- 2
SELECT order_num, COUNT(*) AS items FROM OrderItems GROUP BY order_num;
-- 3
SELECT order_num, COUNT(*) AS items
FROM OrderItems GROUP BY order_num HAVING COUNT(*) >= 3;
```
</details>
