## 二、样例数据库（全课统一使用）

原书用一个**假想玩具经销商的订单系统**做例子，共 5 张表。本课所有示例都基于它：

| 表 | 一行代表 | 关键列（主键**加粗**） |
|----|----------|------------------------|
| **Vendors** 供应商 | 一个供应商 | **vend_id**、vend_name、vend_country… |
| **Products** 产品 | 一个产品 | **prod_id**、vend_id(FK)、prod_name、prod_price |
| **Customers** 顾客 | 一个顾客 | **cust_id**、cust_name、cust_contact、cust_email… |
| **Orders** 订单 | 一个订单（不含明细） | **order_num**、order_date、cust_id(FK) |
| **OrderItems** 订单物品 | 订单里的一个物品 | **order_num+order_item**、prod_id(FK)、quantity、item_price |

**表间关系（外键）：**
```
Vendors ──1:N──> Products ──1:N──> OrderItems <──N:1── Orders <──N:1── Customers
（一个供应商多个产品）（一个产品出现在多个订单物品里）      （一个订单多个物品）（一个顾客多个订单）
```

> 🎯 记住这张关系图，后面所有 JOIN 都是沿着这些外键"拼"起来的。

## 四、一个重要提醒：书是标准 SQL，我们用的是 Hive/Spark

原书讲的是通用标准 SQL，语法 90% 通用，但落到公司的 Hive/Spark 平台有几处差异，每章会标注，先记住这几条：

| 主题 | 书里（标准 SQL / 多数数据库） | 我们的 Hive/Spark |
|------|------------------------------|-------------------|
| 限制行数 | `SELECT TOP 5` / `FETCH FIRST` 等各不同 | 用 `LIMIT 10` |
| 字符串拼接 | `+` / `||` / `Concat()` 各库不同 | 用 `CONCAT(a,b)` |
| 日期处理 | 各库函数差异极大 | 用 Hive/Spark 的日期函数，且注意分区列 `date` |
| 查询必带分区 | 书里不涉及 | **必加 `WHERE date=…`**（主课铁律） |
| 写操作 | 书里详讲 INSERT/UPDATE/DELETE | 我们**多为只读**，写操作了解即可 |

> ⚠️ 学语法看这门课，落到实际取数以主课的公司表和口径为准。
