## 7.1 创建和操纵表

```sql
CREATE TABLE Products (
  prod_id    CHAR(10)      NOT NULL,
  vend_id    CHAR(10)      NOT NULL,
  prod_name  CHAR(255)     NOT NULL,
  prod_price DECIMAL(8,2)  NOT NULL DEFAULT 0,
  prod_desc  VARCHAR(1000) NULL
);
ALTER TABLE Vendors ADD vend_phone CHAR(20);   -- 加列
DROP TABLE Products;                           -- 删表（危险，慎用）
```
- `NOT NULL` 不允许空、`DEFAULT` 默认值、指定数据类型。
- 我们数仓建表多由数据同学/调度任务负责，分析同学一般只查。

## 7.2 视图（View）

**视图 = 一条被"存起来、起了名字"的查询**，本身不存数据，用时才跑。作用：把复杂查询封装成一个"虚拟表"，反复复用。
```sql
CREATE VIEW ProductCustomers AS
SELECT c.cust_name, c.cust_contact, p.prod_id
FROM Customers c
JOIN Orders o     ON c.cust_id=o.cust_id
JOIN OrderItems i ON o.order_num=i.order_num
JOIN Products p   ON i.prod_id=p.prod_id;

-- 之后像查表一样用它
SELECT cust_name FROM ProductCustomers WHERE prod_id='RGAN01';
```
> 💡 视图能简化复杂 JOIN 的复用；但视图查询每次都实际执行，别在视图上叠太多层。公司 BI 里"复用报表 SQL"就是类似思路。

## 7.3 其余进阶特性（各一句话即可）

| 特性 | 一句话 | 分析同学 |
|------|--------|----------|
| **存储过程** | 一组预编译、可反复调用的 SQL（带参数/逻辑） | 了解 |
| **事务处理** | 一组操作"要么全成、要么全回滚"（COMMIT/ROLLBACK），保证一致性 | 了解概念 |
| **游标** | 逐行处理结果集（分析里几乎不用，能用集合操作就别用游标） | 了解 |
| **约束** | 主键、外键、UNIQUE、CHECK、NOT NULL——保证数据完整性 | 了解 |
| **索引** | 给列建"目录"加速查找（代价是占空间、拖慢写入） | 了解；我们数仓靠**分区**加速 |
| **触发器** | 某个增删改动作自动触发的一段 SQL | 了解 |

> 🎯 对照主课：我们用的 Hive/Spark 数仓**不靠索引、靠分区（date）加速**；事务/触发器/游标在离线分析里基本用不到。这些概念知道是什么就够了。

---

## 本章要点
1. `CREATE/ALTER/DROP TABLE` 建改删表——分析同学多为只读，了解即可。
2. **视图** = 存起来的查询，用来封装复用复杂 JOIN（呼应 BI 复用报表 SQL）。
3. 存储过程/事务/游标/约束/索引/触发器：知道各是什么即可；我们数仓靠**分区**而非索引加速。

## 小练习（附参考答案）
1. 用一句话说清"视图"和"表"的区别。
2. 我们的数仓为什么不强调索引？

<details><summary>参考答案</summary>

1. 表**真实存数据**；视图**不存数据**，只是一条起了名字的查询，用时才实际执行。
2. 数仓是海量离线数据、几乎只读不改，靠**按 date 分区**来加速（只扫需要的分区），所以不像业务库那样依赖索引。
</details>
