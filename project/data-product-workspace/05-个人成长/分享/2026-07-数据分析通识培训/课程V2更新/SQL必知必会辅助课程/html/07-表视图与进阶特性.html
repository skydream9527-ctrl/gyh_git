表、视图与进阶特性

建表 · 视图 · 事务 · 索引 · 存储过程

📖 对应原书课 17–22🎯 了解级：知道是什么即可

创建和操纵表

CREATE TABLE Products ( prod_id CHAR(10) NOT NULL, prod_price DECIMAL(8,2) NOT NULL DEFAULT 0, prod_desc VARCHAR(1000) NULL ); ALTER TABLE Vendors ADD vend_phone CHAR(20); -- 加列 DROP TABLE Products; -- 删表（危险）

💡 我们数仓建表多由数据同学/调度负责，分析同学一般只查。

视图 = 存起来的查询

CREATE VIEW ProductCustomers AS SELECT c.cust_name, p.prod_id FROM Customers c JOIN Orders o ON … JOIN OrderItems i ON … JOIN Products p ON …; -- 之后像查表一样用它 SELECT cust_name FROM ProductCustomers WHERE prod_id='RGAN01';

💡 视图本身不存数据，用来封装复用复杂 JOIN（类似 BI 里复用报表 SQL）。

其余特性：各一句话

特性一句话
存储过程预编译、可反复调用的一组 SQL
事务处理一组操作"要么全成、要么全回滚"
游标逐行处理（分析里几乎不用）
约束主键/外键/UNIQUE/CHECK 保证完整性
索引给列建"目录"加速查找
触发器增删改时自动触发的 SQL

🎯 我们数仓不靠索引、靠分区(date)加速；事务/触发器/游标离线分析基本用不到。

本章要点

  • 建改删表：CREATE/ALTER/DROP，分析多只读、了解即可
  • 视图 = 存起来的查询，封装复用复杂 JOIN
  • 存储过程/事务/索引等知道是什么即可；数仓靠分区加速
