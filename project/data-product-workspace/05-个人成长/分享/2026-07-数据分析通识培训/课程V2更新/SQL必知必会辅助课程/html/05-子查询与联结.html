子查询与联结

子查询"套娃" · JOIN 把多表拼起来 —— 跨表分析核心

📖 对应原书课 11–13🎯 子查询 · JOIN · LEFT JOIN

子查询：查询里套查询

-- 找订购了 RGAN01 的顾客：从里往外读 SELECT cust_name FROM Customers WHERE cust_id IN ( SELECT cust_id FROM Orders WHERE order_num IN ( SELECT order_num FROM OrderItems WHERE prod_id='RGAN01' ) );

⚠️ 子查询直观，但大表性能常不如 JOIN，能用 JOIN 就用 JOIN。

JOIN：把多表拼起来

SELECT vend_name, prod_name, prod_price FROM Vendors INNER JOIN Products ON Vendors.vend_id = Products.vend_id;

🔴 一定写 ON！漏了 = 笛卡尔积，行数爆炸（主课案例：100万×80万=8000亿行）。

沿外键串多个表

SELECT cust_name, order_num, prod_name, quantity FROM Customers JOIN Orders ON Customers.cust_id = Orders.cust_id JOIN OrderItems ON Orders.order_num = OrderItems.order_num JOIN Products ON OrderItems.prod_id = Products.prod_id;

💡 顾客 → 订单 → 物品 → 产品，沿着外键一路拼。

内联结 vs 外联结

类型保留什么
INNER JOIN只留两表都匹配的行
LEFT JOIN左表全留，右表没匹配填 NULL
-- 所有顾客及订单数（含没下单的，算 0） SELECT c.cust_name, COUNT(o.order_num) AS orders FROM Customers c LEFT JOIN Orders o ON c.cust_id=o.cust_id GROUP BY c.cust_name;

⚠️ 没下单的 order_num 是 NULL，COUNT 跳过 NULL 正好算成 0（呼应主课 COALESCE）。

本章要点

  • 子查询直观，大表优先 JOIN
  • JOIN 必写 ON，否则笛卡尔积爆炸
  • INNER 只留都匹配的；LEFT 左表全留、右表缺失为 NULL
  • 多表沿外键串联，用表别名让 SQL 更短
