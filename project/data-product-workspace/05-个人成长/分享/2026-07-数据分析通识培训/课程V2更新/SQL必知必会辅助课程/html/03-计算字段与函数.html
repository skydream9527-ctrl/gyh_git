计算字段与函数

在查询里"现算"新列，用函数加工文本/日期/数值

📖 对应原书课 7–8🎯 拼接 · 算术 · 函数

什么是计算字段

数据库存的是原始数据，报表常要加工后的值（拼接/算术/格式化）。这些不改数据、只在查询时"现算"的列 = 计算字段。

拼接字段 + 别名

-- 书里各库拼接不同(+ 或 ||)；我们 Hive/Spark 用 CONCAT SELECT CONCAT(vend_name, ' (', vend_country, ')') AS vend_title FROM Vendors;

💡 计算列一定用 AS 起别名，否则列名是一串表达式、没法引用。

算术计算

SELECT prod_id, quantity, item_price, quantity * item_price AS expanded_price -- 每个物品小计 FROM OrderItems WHERE order_num = 20008;

支持 + - * /，用括号控制优先级。

函数：加工数据

文本

  • UPPER/LOWER 大小写
  • TRIM 去空格
  • SUBSTR(x,起,长)
  • LENGTH 长度

日期 / 数值

  • YEAR(d) 取年份
  • ROUND(x,2) 保留小数
  • ABS() 等

⚠️ 函数是 SQL 最不通用的部分（尤其文本/日期）——学思路，落地查所用平台。

🔴 我们平台：分区列 date ≠ 业务日期，先 WHERE date=… 裁剪，再用日期函数筛业务。

本章要点

  • 计算字段 = 查询时现算的列，不改原数据
  • 拼接用 CONCAT；计算列记得 AS 起别名
  • 函数最不通用，落地查平台函数；日期筛选别忘先加分区 date
