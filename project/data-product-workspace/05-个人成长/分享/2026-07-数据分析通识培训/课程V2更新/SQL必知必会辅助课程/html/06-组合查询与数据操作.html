组合查询与数据操作

UNION 并结果 · INSERT/UPDATE/DELETE（了解级）

📖 对应原书课 14–16🎯 分析多为只读，重在看懂

组合查询：UNION

SELECT cust_name, cust_contact FROM Customers WHERE cust_state='CA' UNION SELECT cust_name, cust_contact FROM Customers WHERE cust_id IN (SELECT cust_id FROM Orders);
写法区别
UNION自动去重
UNION ALL不去重，更快

🎯 UNION 上下拼行，JOIN 左右拼列，别搞混；ORDER BY 放最后一个 SELECT 之后。

INSERT / UPDATE / DELETE（了解级）

INSERT INTO Customers(cust_id, cust_name) VALUES('...', '小米'); UPDATE Customers SET cust_email='x' WHERE cust_id='...'; DELETE FROM Customers WHERE cust_id='...';

🔴 最危险的坑：忘写 WHERE！UPDATE 无 WHERE=改全表，DELETE 无 WHERE=删光表。

写操作铁律

  • INSERT 建议写明列名，别用省略列名的写法
  • UPDATE/DELETE 必带 WHERE
  • 改之前先用同样 WHERE 写一条 SELECT 确认影响的行
💡 分析同学多为只读，写数据走公司规范，别在生产表上随手改。

本章要点

  • UNION 拼行去重、UNION ALL 不去重更快
  • 写操作必带 WHERE，先 SELECT 验证
  • 分析重点是 UNION 和"看懂写操作"
