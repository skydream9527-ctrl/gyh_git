汇总与分组

聚集函数 · GROUP BY · HAVING —— 数据分析最高频

📖 对应原书课 9–10🎯 重点章

把很多行"算成一个数"

函数作用
COUNT(*)行数（含 NULL）
COUNT(列)该列非 NULL 的行数
COUNT(DISTINCT 列)不同值个数（算"多少人/多少种"）
SUM/AVG/MAX/MIN求和/平均/最大/最小

🔴 COUNT(列) 和 AVG 都跳过 NULL；算人数用 COUNT(DISTINCT)，人均别用 AVG。

GROUP BY：分组聚集

-- 每个供应商各有多少个产品 SELECT vend_id, COUNT(*) AS num_prods FROM Products GROUP BY vend_id;

⚠️ SELECT 里的非聚集列，必须都出现在 GROUP BY 里。

HAVING：过滤"分组"

SELECT cust_id, COUNT(*) AS orders FROM Orders WHERE order_date >= '2020-01-01' -- 先筛行 GROUP BY cust_id HAVING COUNT(*) >= 2; -- 再筛组

🎯 WHERE 在分组前筛行，HAVING 在分组后筛组。

SELECT 子句顺序（务必记牢）

SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT

写错顺序直接报错——这就是主课"五段式"的完整版。

本章要点

  • 聚集：COUNT/SUM/AVG/MAX/MIN；算人数用 COUNT(DISTINCT)
  • COUNT(列)/AVG 跳过 NULL，算人均当心偏高
  • WHERE 筛行、HAVING 筛组；牢记子句顺序
