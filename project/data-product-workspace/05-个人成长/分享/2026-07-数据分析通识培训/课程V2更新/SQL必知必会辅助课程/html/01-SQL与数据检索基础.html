SQL 与数据检索基础

知道 SQL 是什么，会用 SELECT 把数据取出来

📖 对应原书课 1–2🎯 SELECT · DISTINCT · LIMIT

SQL 是什么

  • 数据库装数据，表是结构化的数据清单
  • 表 = 行（记录） + 列（字段），主键唯一标识一行（如 prod_id）
  • SQL = 和数据库对话的语言，各库大同小异
💡 呼应主课：表就是二维表格，SQL 就是在表上"筛选 + 分组 + 计算"。

SELECT：检索列

-- 单列 / 多列 / 所有列 SELECT prod_name FROM Products; SELECT prod_id, prod_name, prod_price FROM Products; SELECT * FROM Products;

⚠️ 除非真要所有列，别用 SELECT *——多取列既慢又浪费（列裁剪）。

DISTINCT：只要不同的值

SELECT DISTINCT vend_id FROM Products; -- 有哪些供应商

⚠️ DISTINCT 作用于后面所有列——DISTINCT vend_id, prod_price 是"两列组合去重"，不是只对 vend_id 去重。

限制行数：各库差别最大

环境写法
SQL ServerSELECT TOP 5 …
DB2FETCH FIRST 5 ROWS ONLY
MySQL / Hive / Spark（我们）… LIMIT 5

🎯 我们平台用 LIMIT n；探查数据养成加 LIMIT 的习惯。

本章要点

  • SELECT 列 FROM 表 是起点；平时只取需要的列
  • DISTINCT 去重，作用于后面所有列
  • 限制行数我们用 LIMIT n
