排序与数据过滤

把结果排好序，用 WHERE 精准筛出想要的行

📖 对应原书课 3–6🎯 ORDER BY · WHERE · IN/NOT · LIKE

ORDER BY：排序

SELECT prod_name, prod_price FROM Products ORDER BY prod_price DESC, prod_name; -- 价格降序，同价再按名字升序

💡 ORDER BY 放最后；取"最贵的 5 个"→ ORDER BY prod_price DESC LIMIT 5。

WHERE：过滤行

操作符例
= <> !=prod_price <> 3.49
BETWEEN a AND bprod_price BETWEEN 5 AND 10
IS NULLprod_price IS NULL

🔴 NULL 用 IS NULL，不能 =NULL；<> 3.49 一般也查不出 NULL 行。

高级过滤：AND / OR / IN / NOT

-- 🔴 AND 优先级高于 OR，混用必加括号 WHERE (vend_id='DLL01' OR vend_id='BRS01') AND prod_price>=10; -- IN 比一串 OR 更清晰 WHERE vend_id IN ('DLL01','BRS01'); WHERE vend_id NOT IN ('DLL01');

⚠️ 不加括号，A OR B AND C 会被读成 A OR (B AND C)，结果全错。

通配符过滤：LIKE

通配符匹配例
%任意多个字符LIKE 'Fish%'
_单个字符LIKE '_ inch'

⚠️ LIKE 慢——'%xxx'（% 在开头）用不上索引最慢；能不用就不用、% 别乱放开头。

本章要点

  • ORDER BY … DESC 配 LIMIT 取 TopN
  • NULL 用 IS NULL
  • AND 比 OR 优先，混用必加括号；一组值用 IN
  • LIKE 模糊匹配但慢，% 别乱放开头
