# 第6章：组合查询与数据操作

> 对应原书：第 14–16 课（组合查询、插入数据、更新和删除数据）
> 目标：会用 UNION 把多个查询结果并起来；了解 INSERT/UPDATE/DELETE（分析同学多为**只读**，重在看懂）。

---

## 6.1 组合查询：UNION

把**多条 SELECT 的结果上下拼起来**（并集）。要求各 SELECT 的**列数、列顺序、类型对应一致**。
```sql
-- 把"加州的顾客"和"下过单的顾客"两批并到一起
SELECT cust_name, cust_contact FROM Customers WHERE cust_state = 'CA'
UNION
SELECT cust_name, cust_contact FROM Customers WHERE cust_id IN
       (SELECT cust_id FROM Orders);
```

| 写法 | 区别 |
|------|------|
| `UNION` | 自动**去重**（合并重复行） |
| `UNION ALL` | **不去重**，全部保留（更快） |

> 💡 `ORDER BY` 只能放在**最后一个** SELECT 之后，对最终合并结果排序。
> 🎯 UNION 是"上下拼行"，JOIN 是"左右拼列"，别搞混。

---

## 6.2 插入数据：INSERT（了解级）

```sql
-- 明确给出列名（推荐，安全）
INSERT INTO Customers(cust_id, cust_name, cust_city)
VALUES('1000000010', '小米科技', '北京');
```
> ⚠️ 别用省略列名的 `INSERT INTO Customers VALUES(...)`——一旦表结构变化就错位。
> 从别的表复制：`INSERT INTO 表A(列...) SELECT 列... FROM 表B;`

---

## 6.3 更新与删除：UPDATE / DELETE（了解级，高危）

```sql
UPDATE Customers SET cust_email = 'new@x.com' WHERE cust_id = '1000000005';
DELETE FROM Customers WHERE cust_id = '1000000006';
```
> 🔴 **最危险的坑：忘写 `WHERE`！**
> - `UPDATE Customers SET cust_email='x'` （无 WHERE）→ **改掉所有行**
> - `DELETE FROM Customers` （无 WHERE）→ **删光整张表**
>
> 铁律：写 UPDATE/DELETE 前，**先用同样的 WHERE 写一条 SELECT 确认影响的行**，没问题再改。

---

## 6.4 对我们意味着什么

- 分析取数几乎只用到 **6.1 的 UNION**（偶尔把多段结果并起来）。
- INSERT/UPDATE/DELETE 在数仓分析场景**基本用不到**（你一般只有查询权限），了解语法、能看懂即可。
- 真要写数据（如建临时结果表），走公司规范流程，别在生产表上随手改。

---

## 本章要点
1. `UNION` 上下拼行（去重），`UNION ALL` 不去重更快；列要对应一致，ORDER BY 放最后。
2. `INSERT` 建议写明列名；可用 `INSERT ... SELECT` 从表复制。
3. `UPDATE/DELETE` **必带 WHERE**，先 SELECT 验证影响行——这是最容易闯大祸的地方。
4. 分析同学重点是 UNION 和"看懂写操作"，写数据走规范。

## 小练习（附参考答案）
1. 把"CA 州顾客"和"IL 州顾客"的名字并成一个结果（不去重）。
2. （判断）`DELETE FROM Orders` 会发生什么？怎么避免？

<details><summary>参考答案</summary>

```sql
-- 1
SELECT cust_name FROM Customers WHERE cust_state='CA'
UNION ALL
SELECT cust_name FROM Customers WHERE cust_state='IL';
```
2. `DELETE FROM Orders` 没有 WHERE，会**删光 Orders 整张表**。避免：写 DELETE/UPDATE 前先用同样条件写 `SELECT` 确认要影响的行，且永远带 WHERE。
</details>
