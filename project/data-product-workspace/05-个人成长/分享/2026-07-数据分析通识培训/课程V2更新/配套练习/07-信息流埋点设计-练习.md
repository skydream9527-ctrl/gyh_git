# 模块7 配套练习：信息流埋点设计

> 面向：产品（尤其信息流方向，⭐专业）。销售/运营了解"数据从埋点来、口径在埋点就定了"即可。
> 说明：本练习练"四核心事件 + 时长原理 + 分题材 + 埋点到指标闭环 + 验收避坑"。每题附**参考答案/示范解**。

---

## 练习1：辨认四核心事件
下面这些用户动作，分别对应哪个核心事件（曝光 / 点击 / 进入 / 退出）？
1. 卡片在屏幕上露出一半、停留了 0.8 秒
2. 用户手指点了这张卡片
3. 详情页加载完成、开始播放视频
4. 用户滑走、离开了播放页

**参考答案：** 1. 曝光（且满足"有效曝光"：露出≥50% 且停留≥500ms） 2. 点击 3. 进入 4. 退出

> 💡 为什么点击和进入要拆开？因为点了不一定进得去（加载失败/秒退），拆开才能发现"展示→点击→进入"哪一环出问题。

## 练习2：时长怎么算 + 挑坑
1. 消费时长的计算公式是什么？
2. 用户看视频时 APP 崩溃了，只有"进入"没有"退出"，时长会怎样？怎么兜底？
3. 用户看一半切到微信 5 分钟再切回来，这 5 分钟算不算时长？

**参考答案：**
1. **消费时长 = 退出时刻 − 进入时刻**（算出来的，不是直接采的）。
2. 只进入没退出 → 时长丢失或异常大。兜底：**心跳埋点**（每 15 秒上报"还在看"）+ 超时截断。
3. 一般算**前台有效时长**，切后台的 5 分钟不算（否则时长虚高）。

## 练习4：从埋点到指标（闭环）
下面的指标分别来自哪些埋点、对应哪个字段？
1. CTR
2. 消费时长
3. VV

**参考答案：**
1. CTR = 点击/曝光 → 来自点击、曝光事件 → `click_cnt` / `expos_cnt`
2. 消费时长 = Σ(退出−进入) + 心跳兜底 → `feed_dura`
3. VV = 有效进入/播放计数 → `news_vv_cnt` / `short_vv_cnt` / `mini_vv_cnt`

> 🎯 你设计的埋点，就是前面模块 2/3 里查的那些字段的源头——埋点定义错了，后面分析全错（GIGO）。

## 练习5：埋点验收挑错
某信息流埋点上线后，数据同学发现"曝光量是点击量的 100 倍还多，明显偏高"。列出你会怀疑的 3 个原因，以及怎么验收。

**示范解：**
- 怀疑原因：① **曝光未去重/重复上报**（同一张卡反复曝光都记了）② **有效曝光标准没生效**（快速划过也算曝光）③ 埋点触发时机错（列表渲染即算曝光，而非真正露出）。
- 验收方法：小流量试跑看曝光/点击比是否在合理区间；检查曝光是否按"露出≥50% 且停留≥500ms"触发；检查是否对同一卡片同一次露出去重。

## 练习6：设计一个埋点需求（综合）
你要给"信息流卡片新增【长按收藏】功能"设计埋点。写出埋点需求表的关键行（事件名 / 触发时机 / 关键参数 / 是否必填）。

**示范解：**

| 事件名 | 触发时机 | 关键参数 | 是否必填 |
|--------|----------|----------|----------|
| `card_longpress` | 用户长按卡片≥500ms | content_id、content_type、position、channel | 是 |
| `card_collect` | 点击"收藏"成功 | content_id、content_type、collect_source=longpress | 是 |
| `card_collect_cancel` | 取消收藏 | content_id | 否 |

> 关键：参数里一定要有 `content_type`（分题材分析要用）、`position`（位置分析）；口径（什么算一次有效收藏）要在需求文档写清。
