# 模块8 配套练习：综合实战

> 面向：产品走**完整版**；销售/运营走**业务版**。
> 说明：用一个新问题，端到端走完"明确问题 → 找数取数 → 拆维度定位 → 判断原因 → 结论建议"。附**示范解**。

---

## 完整版练习（产品）：分析某类 PUSH 对信息流留存的影响
**任务**：老板想知道"点击过 PUSH 的用户，是不是比不点 PUSH 的用户更爱看信息流、留存更好？"请设计完整分析并输出结论。

**要求**：写出你的分析设计（不必真跑数），至少覆盖：
1. 明确问题：核心指标、对比对象、时间范围
2. 找数取数：用哪些表/字段，怎么分组（可让 AI 拆步骤生成 SQL）
3. 拆维度定位：怎么对比、怎么排除干扰
4. 判断原因：可能的坑（相关≠因果、用户本身差异）
5. 结论建议：结论先行

**示范解：**
1. **明确问题**：核心指标=信息流人均时长、7 日留存；对比 PUSH 点击用户 vs 非点击用户；时间取近 30 天。
2. **找数取数**：miui_di 取活跃 + 标记是否点击 PUSH 启动（`app_launch_way IN ('点击push','subscribe_push')`）；label_di 取信息流时长；留存关联未来 7 天活跃表。用 WITH 拆 4 步（呼应模块4 练习1、3），让 AI 生成后按 4 步校验。
3. **拆维度**：分新老用户看（防辛普森）；PUSH 组 vs 非 PUSH 组对比人均时长/留存。
4. **判断原因（关键坑）**：**相关≠因果**——点 PUSH 的用户本身可能就更活跃！直接对比会高估 PUSH 价值。要么分层对比（同活跃度段内比），要么用 AB/DID（呼应模块6 的 PUSH 增量实验）扣掉基线差异。
5. **结论建议**：结论先行——"PUSH 点击用户信息流时长/留存更高，但需扣除用户本身活跃度差异；建议用 DID 或 AB 测净增量，再决定加大 PUSH 投入"。

## 业务版练习（销售/运营）：复盘一次活动/一段业绩
**任务**：选你负责的一次真实活动或一个季度业绩，用**自证三板斧 + 排除大盘**写一份约 400 字复盘。

**要求覆盖：**
1. 亮成绩（前后对比 或 有无对比，**排除大盘**）
2. 找差距（对标目标/大盘/上期）
3. 讲原因 + 下一步建议
4. 结论先行

**示范解骨架：**
> **结论**：本次活动净拉新 X（已扣大盘自然增长），A 渠道最优、C 渠道亏损，建议下次砍 C 保 A。
> **亮成绩**：活动期日均新增比活动前一周高 40%；大盘同期只涨 10%，净贡献 30%。
> **找差距**：整体转化率 5%，低于目标 8%，差在 C 渠道（转化 2%、ROI 0.6）。
> **讲原因 + 建议**：C 渠道人群不精准；下次预算挪给 A，并优化落地页提转化。
