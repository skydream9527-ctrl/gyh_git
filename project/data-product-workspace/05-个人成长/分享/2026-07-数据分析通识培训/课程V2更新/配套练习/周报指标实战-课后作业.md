# 课后作业：周报指标实战（手写 SQL）

> 来源：2026-07-09 培训课后作业。目标：**手写 SQL 算出周报常用指标**，练熟"找数 → 写 SQL → 验证"全流程，也为后续"AI 写周报 skill"打基础。
> 覆盖模块：2（找数）、3（SQL 基础）、4（进阶）+《数据工坊与权限操作手册》。

---

## 一、作业要求

算出**内容中心近 7 天**的周报核心指标，**按天**输出，能自己讲清每个数怎么来的。

**必做指标：**
1. 每天 DAU（活跃用户数）
2. 每天人均 APP 使用时长（分钟）
3. 每天信息流 CTR
4. 每天人均信息流消费时长（分钟）

**进阶（选做）：**
5. 新老用户分别的 DAU 与人均时长
6. 环比：本周 vs 上周同期，DAU 涨跌多少

**交付**：结果 + 你写的 SQL 发群；对不上的地方记录下来一起答疑。

## 二、开工前检查清单（现场踩过的坑）
- [ ] 加了 **date 分区**了吗？（不加=全表扫描，跑不出来）
- [ ] 用了 **`is_dau_2024` 等有效活跃过滤**吗？（口径：曝光+点击+播放+进入空间>0 且非置顶）
- [ ] 算人数用 **`COUNT(DISTINCT did)`** 了吗？
- [ ] 按天要 **`GROUP BY date`**（现场最常漏这个）
- [ ] 时长做 **`/60000` 单位换算**（毫秒→分钟）了吗？
- [ ] CTR 除法 **`*1.0`** 了吗？

## 三、参考 SQL 框架（对照检查，不要照抄）

**指标 1–2：APP 侧 DAU + 人均时长（miui 表）**
```sql
SELECT
  date,
  COUNT(DISTINCT did) AS dau,
  ROUND(SUM(app_dura) / COUNT(DISTINCT did) / 60000, 2) AS avg_app_min
FROM iceberg_zjyprc_hadoop.browser.dwm_browser_event_aggregation_miui_di
WHERE date BETWEEN 20260703 AND 20260709      -- 近 7 天（换成你的日期）
  AND is_app_dau_2024 = 1
  AND COALESCE(did,'') != ''                   -- 过滤空 did，防倾斜
GROUP BY date
ORDER BY date;
```

**指标 3–4：信息流 CTR + 人均消费时长（label 表）**
```sql
SELECT
  date,
  ROUND(SUM(click_cnt) * 1.0 / SUM(expos_cnt), 4)             AS ctr,
  ROUND(SUM(feed_dura) / COUNT(DISTINCT did) / 60000, 2)      AS avg_feed_min
FROM iceberg_zjyprc_hadoop.browser.dwm_browser_event_aggregation_label_di
WHERE date BETWEEN 20260703 AND 20260709
  AND is_dau_feed_dapan_2024 = 1
  AND is_top = 0                               -- 只看自然流量
  AND COALESCE(did,'') != ''
GROUP BY date
ORDER BY date;
```

**指标 5（进阶）：新老用户拆分** —— 用 CASE WHEN 打标后分组
```sql
SELECT
  date,
  CASE WHEN is_new_2024 = 1 THEN '新用户' ELSE '老用户' END AS user_type,
  COUNT(DISTINCT did) AS dau,
  ROUND(SUM(app_dura)/COUNT(DISTINCT did)/60000, 2) AS avg_min
FROM …miui_di
WHERE date BETWEEN 20260703 AND 20260709 AND is_app_dau_2024 = 1 AND COALESCE(did,'')!=''
GROUP BY date, CASE WHEN is_new_2024 = 1 THEN '新用户' ELSE '老用户' END
ORDER BY date, user_type;
```
