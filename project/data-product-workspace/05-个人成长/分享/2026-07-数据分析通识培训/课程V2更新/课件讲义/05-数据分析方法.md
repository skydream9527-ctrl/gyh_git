# 模块5：数据分析方法

> 课时：1.5小时 | 层级：🔷 进阶
> 目标：掌握排查数据问题的"四把尺子"和四种分析方法（对比/趋势/分群/漏斗），能把业务问题拆解、定位、给结论。
> 前置：模块1（数据思维与业务自证，DIKW 和基础对比在那讲过）。
> 👉 本节是**分析师视角**的进阶方法，产品/深入分析的同学重点掌握；销售/运营选学。

## 一、开场：从"报数"到"给结论"

模块 2-4 你已经会**找数、写 SQL / 让 AI 取数**了；模块1 建立了数据思维（用数据说话、自证）。这节课再进一步：拿到一个业务问题（比如"CTR 为什么降了"），**怎么把查询结果系统地拆解、定位、给结论**。

> ⚠️ **动手分析前先做一件事：对齐口径、校验数据。** 现场实操就踩过——学员自己算的内容中心 DAU 和看板差一截（3423 vs 3398），多半是过滤条件 / 口径没对齐。拿错的数做分析，越分析越错。遇到"我算的和看板对不上"，先对口径（过滤 / 表层级 / 去重），再找数据研发核对官方口径。

## 二、排查数据问题的四把尺子（重点）

排查（如"日活下降怎么查"）的关键，是选对拆解的尺子：

- **① 好维度 = 强区分度**：在该维度上指标变化区分度强，一拆就看出"是谁出了问题"。
  - 强区分（好）：按 PUSH 类型拆，热点类 3% vs 其他 7.8%，一眼定位。
  - 弱区分（差）：按性别拆，男 6.1% vs 女 5.9%，帮不上忙。
- **② 好参照 = 基准线**：没有对比就没有结论。参照有：时间参照（上周同日/上月/去年同期）、对照组、大盘/目标。
- **③ 好视角 = 对的样本 + 对的周期**：
  - 样本 = 最小作用颗粒度（优化播放流畅度，样本应是"播放会话"而非"用户"）。
  - 周期由目标定：点击类几天，留存类 1~2 周。
- **④ 好起止点 = 链路分析**：核心原理——**数据变化若和某功能有关，只会影响该功能"往后"的环节**。从改动点往后逐环节看转化，就能判断是不是它造成的。

## 三、四种常用分析方法

| 方法 | 做什么 | 一句话 |
|------|--------|--------|
| 对比分析 | 同比/环比/对照/细分 | 找差异、判好坏 |
| 趋势分析 | 时间序列看周期、异常点 | 看规律、找异动 |
| 分群分析 | 按属性/行为拆分 | 定位问题群体 |
| 漏斗分析 | 多步链路转化 | 找流失瓶颈 |

- **对比分析**（模块1 已讲基础）：口径一致 · 单变量 · 既看绝对值也看相对值。
- **趋势分析**：拉长时间看周期性、拐点、异常值，别被单点波动误导。
- **分群分析**：分群落到 SQL 就是 `CASE WHEN` 打标（呼应模块3）——把用户按新老 / 活跃度 / 来源 / PUSH 类型打成组再聚合。**辛普森悖论**：整体 CTR 降了，但分人群看每个群体都在涨——因为高 CTR 人群占比下降拉低了整体平均。**大盘异动时先拆维度**，别只看平均。
- **漏斗分析 · 六步法**：明确目标(SMART) → 设计路径 → 采集数据 → 算转化率 → 定位瓶颈 → 优化验证(A/B)。
  - PUSH 全链路：`下发 → 到达 → 展示 → 点击 → 落地页 → 消费`。CTR 降别只盯点击，拆漏斗看是"到达少了"还是"展示了没人点"。

## 四、真实案例：PUSH CTR 下降 2pct 排查（四把尺子应用）

| 步骤 | 发现 |
|------|------|
| 拆时间 | 周四上线 8.3% → 周日 6%，逐步下降；凌晨 2-4 点特别低 |
| 拆 PUSH 类型（好维度） | 新热点推送 CTR 仅 3%，贡献 80% 下降 |
| 拆用户 | 新用户从 7% 降到 3.5%，是重灾区 |
| 拆漏斗 | 到达/展示正常，问题在"展示→点击" |

> **结论**：新策略对老用户正向、对新用户严重负向——不必全量回滚，但要立刻调整（新用户前 7 天不推时政、避开凌晨、灰度含新用户）。

> 👉 **取数提示**：这些"拆时间 / 拆类型 / 拆用户 / 拆漏斗"全是模块3/4 的分组 SQL——让 AI 用 `label_di`（信息流表）生成分维度 SQL，你按 4 步校验（字段 / 分区 / 去重·单位 / 数量级），比手写快。

## 五、常见分析误区
- 唯数据论（异常先查埋点/口径）
- 忽略样本量（几百人的组别别过度解读）
- 相关当因果
- 过度拆解（拆到每组几百人没意义）
- 只看相对不看绝对

---

## 本节课小结
1. **四把尺子**：好维度（强区分）、好参照（基准线）、好视角（对的样本+周期）、好起止点（链路分析）
2. 四种方法：对比 / 趋势 / 分群 / 漏斗
3. 大盘异动先拆维度（防辛普森悖论）
4. 漏斗定位瓶颈，别只盯单点
