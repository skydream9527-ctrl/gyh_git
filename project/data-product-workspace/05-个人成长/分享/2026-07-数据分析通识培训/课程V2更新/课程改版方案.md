# 数据分析通识培训 · 课程改版方案（讨论稿）

> 本文是经过多轮讨论形成的**课程整体设计方案**，用于对齐方向。确认后，再按"一次一个文件"逐步落地。
> 一句话定位：**给产品 / 销售 / 运营新人的数据分析通识课——建立数据思维、会用数据自证业务、能借助 AI 上手取数。**

---

## 一、课程定位与目标

- **面向对象**：产品新人、销售同学、运营同学（非技术为主）
- **授课形式**：每周 2~3 节，每节 1~1.5 小时
- **核心目标**：
  1. **系统了解**：知道数据从哪来、指标口径怎么定、怎么找数用数
  2. **能实操**：会基本取数（复杂靠 AI），能独立完成简单分析
  3. **有数据思维**：能用数据发现问题、给业务"自证"（算账 + 复盘）
  4. **了解 AI 与自动化**：会用 AI 辅助取数分析，了解自动化能干嘛
- **设计原则**：以学习者为中心 · 一条主线 + 分层标签 · 全程真实业务场景 · AI 融入 · 练习有反馈

---

## 二、受众与能力画像（分层的依据）

| | 产品新人 | 销售 / 运营 |
|---|----------|-------------|
| 学习重点 | 完整链路：取数 → 分析 → 实验 → 埋点 | **数据思维 + 业务自证** |
| SQL | 会写（含多表） | 基本会看会改，复杂**交给 AI**、自己校验 |
| 分析 | 分析师视角（排查、漏斗、实验） | 经营者视角（**给自己业务算账、复盘、汇报**） |
| 埋点 / AB | 需要掌握 | 不要求 |
| 共同必备 | 数据从哪来、指标口径、用数据说话 | 同左 |

**销售 / 运营能力画像（一句话）**：用数据说话、给业务复盘和算账——**亮成绩（动作→指标变化）、找差距（对标发现不足）、讲原因**，而不是当"取数工具人"。

## 五、重点变化说明

### 5.1 🆕 新增《数据思维与业务自证》（模块1，全员必会核心）
- **数据思维**：用数据说话、相关≠因果、先假设后验证
- **发现问题**：我的业务关键指标是什么、怎么判断好坏
- **自证三板斧**：① 亮成绩（做了 X → 指标涨了 Y，前后/有无对比）② 找差距（对标大盘/目标/上期/同事）③ 讲原因（简单归因）
- **对比方法论**：同比、环比、对标、分群对比
- **把数据讲成故事**：结论先行、一图胜千言、看人下菜（向上汇报）
- **业务案例**：运营复盘一次活动效果、销售用数据证明区域业绩

### 5.2 第4课拆成两层
- 原第4课的 **DIKW / 数据驱动思维** → 上移并入模块1（必会）
- 原第4课的 **排查四把尺子 / 漏斗 / 分群 / 辛普森** → 留在模块5"数据分析方法"（进阶）

### 5.3 埋点两课合并精简
- 原第7课（核心事件模型）+ 第8课（分题材+指标闭环+规范）→ 合并为**模块7 一节**，定位 ⭐专业。

### 5.4 埋点基础 vs 埋点设计分离
- 模块2 保留"埋点是什么"的**基础认知**（必会）；埋点"**怎么设计**"归到模块7（专业）。

### 5.5 AI：融入各课 + 选修手册
- **融入**：模块3/4 的 AI 辅助写 SQL、模块1 的 AI 帮做对比分析、启动课的"AI 写作业人检查作业"理念
- **选修手册**：系统讲 AI 在"查口径 → 生成 SQL → 执行验证 → 一致性检测"全链路的用法，并教 6 个 skill 怎么用

### 5.6 自动化：轻提及，不单独成课
- 在综合实战或 AI 手册里用一小节"了解级"带过：`datum-cli` 定时任务、报表自动推送飞书等，不要求上手。

## 六、各岗位学习路径

**产品新人（全量）**：导览 → 0 启动课 → 1 数据思维 → 2 找数 → 3 SQL基础 → 4 SQL进阶 → 5 分析方法 → 6 AB实验 → 7 埋点 → 8 实战 →（9 答疑 / 选修手册）

**销售 / 运营（精简）**：导览 → 0 启动课(轻) → **1 数据思维与业务自证（重点）** → 2 找数 → 3 SQL基础+AI →（5 分析方法 / 8 实战业务版 选学）
> 核心压在模块 1，取数够用即可、复杂交给 AI。

| 模块 | AI 融入点 | 出场 skill |
|------|-----------|-----------|
