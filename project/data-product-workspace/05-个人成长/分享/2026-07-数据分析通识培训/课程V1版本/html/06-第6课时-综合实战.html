SECTION 01

PUSH CTR 突然下降
怎么办？

STEP 1 · 明确问题

先想清楚要解决什么

场景：周一早上发现周日 PUSH 整体 CTR 从 8% 降到 6%（↓2pct），老板让你查是不是上周四上线的新个性化推送策略有问题，要不要回滚。
分析目标

定位下降原因 · 判断是否新策略导致 · 给回滚 / 优化 / 观察建议。

分析范围

上周四~周日对比上周同期；核心 CTR=点击/到达；拆时间 / PUSH 类型 / 用户 / 漏斗。

STEP 2 · 数据校验

先确认数据本身没问题

不要上来就分析，先排除数据自身的坑：

  • 数据是否更新延迟？换查询方式重跑确认
  • 埋点上报有没有问题？点击 / 到达埋点有没有丢
  • 口径有没有变化？CTR 公式是不是被改了
→ 确认数据没问题，CTR 确实降了 2 个点。（呼应第 4 课：异常先怀疑埋点 / 口径）
STEP 3 · 对比分析，拆维度

搞清是整体降还是局部降

📅 时间对比
周四上线 8.3%→周日 6%，逐步下降；周日凌晨 2-4 点特别低。
📰 PUSH 类型
新上线热点推送周日量大，CTR 仅 3%，贡献 80% 下降。
👥 用户分群
老用户降 0.5pct 正常；新用户从 7% 降到 3.5%，重灾区。
🔻 漏斗拆解
到达 / 展示正常，展示→点击掉 2.5pct，问题在点击环节。

🎯 初步定位：新热点推送凌晨给新用户批量发，点击率仅 3%，拉低整体。

STEP 4 · 深入分析原因

为什么热点类对新用户点击这么低

  • 内容：都是较深度的时政社会新闻，标题严肃
  • 新用户特征：刚装 APP，之前看娱乐 / 生活类，对时政兴趣低
  • 发送时间：周日凌晨 2 点发，用户睡觉，起来一堆推送直接划走
  • 灰度盲区：上线前只测老用户，全量后加进新用户就崩了
💡 呼应第 5 课：灰度测试的人群覆盖不全，是这次事故的根源之一。
STEP 5 · 结论与建议

结论先行 + 可落地建议

CTR 降 2pct，80% 因凌晨给新用户全量推不匹配的时政内容（点击率仅 3%）。新策略对老用户正向、对新用户严重负向 —— 不必全量回滚，但要立刻调整。
立刻做

新用户前 7 天不推时政热点，只推泛娱乐；凌晨 0-8 点不批量发。

后续做

灰度必须含新用户，观察 3 天再全量；建 PUSH 分人群内容池。

预计调整后 CTR 恢复到 7.5% 以上。

这个案例，用到了前 5 课

环节用到的知识来自
取数拆维度分渠道 / 分人群 SQL、CASE WHEN、JOIN第 2·3 课
数据校验异常先查埋点 / 口径第 1·4 课
定位病灶好维度（区分度）、对比 / 分群 / 漏斗第 4 课
灰度与验证灰度覆盖人群、AB 判断第 5 课
输出结论结论先行、可落地建议第 4 课

SECTION 02

高频问题 Q&A

取数类问题

Q1SQL 跑出来和看板对不上？
按序排查：① 口径（过滤条件、时间范围是否一致）② 表（看板可能用 ADS 汇总表，口径不同）③ 去重（COUNT DISTINCT）④ 更新时间（看板是否延迟）。
Q2SQL 跑很久没出来？
① 先查是不是漏了 date 分区（90% 是这个）② 先查 1 天验证逻辑 ③ 别 SELECT * ④ 先在子查询过滤再 JOIN ⑤ 大数据量拆月跑或找数据同学。

AI 与实验类问题

Q3AI 生成的 SQL 敢直接用吗？
简单单表校验后可用；复杂多表一定要：① 查字段是否真实存在 ② 过滤是否加全 ③ 先查 1 天看数值是否合理 ④ 对比历史 / 看板对得上再用。
Q4AB 开了一周还不显著？
① 算现在样本量能测多大 MDE，不够就扩流量 / 延长 ② 效果是不是真的很小（<0.5% 价值不大）③ 分维度看是否某群体显著 ④ 不要为了显著硬凑 p 值。

Q5 · 怎么找到问题的根因？

用 MECE 原则拆维度：互相独立、完全穷尽 —— 一层层往下拆，最后匹配业务动作。
  • 先拆时间：何时开始出问题？突发还是逐步？
  • 再拆人群：新 / 老 / 高低活 / 版本 / 渠道？
  • 再拆业务模块：哪个功能 / 内容 / 入口？
  • 再拆链路：漏斗哪个环节掉了？
  • 最后匹配业务动作：上线 / 活动 / bug / 运营动作 —— 基本就能定位。
