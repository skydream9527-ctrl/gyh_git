## 一、数据驱动思维：从数据到决策
### 1.1 DIKW模型：数据怎么变成价值
我们每天看的原始数据只是一堆数字，要经过四层加工才能变成决策依据：

| 层级 | 名称 | 说明 | 例子 |
|------|------|------|------|
| D（Data） | 数据 | 原始的、未加工的数字 | 昨天PUSH点击量100万次 |
| I（Information） | 信息 | 加工后有意义的指标 | 昨天PUSH CTR是8%，比前天下降了2个百分点 |
| K（Knowledge） | 知识 | 信息对比后发现规律/问题 | CTR下降主要是新用户群体下降，新的推送策略对新用户不友好 |
| W（Wisdom） | 智慧/决策 | 基于知识给出行动方案 | 把新用户的推送策略回滚，或者针对新用户单独调整内容，预计能恢复CTR |

💡 产品同学做分析的目标，不是停留在"报数据"阶段，而是要走到K和W层：**发现问题、定位原因、给出建议**。

### 1.2 数据驱动的核心特征
- **用数据说话，而不是凭感觉**："我觉得这个功能好" → "数据显示这个功能渗透率30%，提升了人均时长5分钟"
- **相关性不等于因果性**：看到两个指标一起涨/跌，不要急着下结论，有可能是第三个因素同时影响了两者
- **数据是验证假设的工具，不是拍脑袋的佐证**：不要先有结论再去找数据支持，而是先提出假设，用数据验证对不对

## 二、数据分析完整流程
遇到一个业务问题，不要急着写SQL取数，按照这个流程走，避免做无用功：

```
1. 明确业务目标 → 2. 拆解指标和维度 → 3. 取数验证 → 4. 分析定位原因 → 5. 输出结论和建议
```

### 步骤1：明确业务目标（最容易被忽略，也是最重要的一步）
先问清楚/想清楚：
- 这个分析要解决什么问题？
- 谁看这个分析？要做什么决策？
- 时间范围是什么？要拆分哪些维度？
- 什么结果算"好"，什么结果算"坏"？

**反例**："帮我看看PUSH数据怎么样" → 根本没法做，不知道看什么指标，看多长时间，对比什么
**正例**："帮我看看本周一PUSH CTR为什么比上周一下降了2个百分点，是不是新上线的推送策略有问题，需要定位到具体是哪类PUSH、哪类用户出的问题，决定要不要回滚"

### 步骤2：拆解指标和维度
把大问题拆成可量化的指标和可拆分的维度：
- **指标**：看什么数？DAU、CTR、留存、人均时长？
- **维度**：怎么拆分对比？时间、新老用户、渠道、启动方式、PUSH类型、版本？
- **拆解方法**：公式拆解，比如CTR = 总点击 / 总曝光 = Σ(各人群CTR × 各人群曝光占比)，可以一层层拆下去

### 步骤3：取数验证
按照拆解的维度写SQL取数，这时候前3节课学的SQL和AI工具就用上了。

### 步骤4：分析定位原因
用对比、分群、漏斗等方法，定位问题出在哪里，为什么会出现。

### 步骤5：输出结论和建议
结论要明确，建议要可落地：
- 不要说"可能是XX原因"，要说"数据显示XX原因贡献了80%的下降"
- 不要说"建议优化"，要说"建议先回滚新策略，预计CTR能恢复1.5个百分点"
- 给出数据支撑和预估收益

## 三、4种常用数据分析方法
这4种方法能覆盖你日常90%的分析需求。

### 3.1 对比分析：没有对比就没有结论
单个数字没有任何意义，必须要有对比基准：
- **同比**：和去年同期比，看长期趋势
- **环比**：和上一个周期比（上周/昨天/上月），看短期变化
- **对照组对比**：实验组vs对照组，看AB实验效果
- **目标对比**：实际值vs目标值，看完成进度
- **细分群体对比**：新用户vs老用户、不同渠道vs大盘平均

**对比分析原则**：
1. 保证对比口径一致：都是自然流量？都是同一时间段？有没有排除节假日/异常活动？
2. 单变量对比：对比时其他条件尽量一致，才能看出差异来源
3. 既要比绝对值，也要比相对值："DAU涨了10万" + "涨幅2%"，不同量级的产品看哪个更有意义

**案例**："PUSH CTR从10%降到8%"，怎么对比找原因？
- 和上周同一天比：是不是周几规律？
- 和历史平均水平比：是异常波动还是正常范围？
- 分PUSH类型比：是所有类型都降了，还是某一类降了？
- 分用户群比：是新用户降了还是老用户降了？是某个版本的用户降了？

### 3.2 趋势分析：看变化规律和异常点
把指标按时间序列排开，看走势：
- 看周期性：周内规律（周末高还是工作日高？）、节假日效应
- 看趋势：是持续上涨/下跌，还是突然异动？
- 看异常点：哪天突然涨/跌了，回忆当天有没有上线新功能、做活动、改策略

**趋势分析常见场景**：
- 监控核心指标日常波动，判断是不是正常
- 看功能上线后指标有没有持续提升，还是只是新奇效应
- 判断问题是暂时的还是持续的

### 3.3 分群分析：不要只看大盘平均值
大盘指标经常会掩盖问题，一定要拆分维度看不同群体的差异：
- **用户属性分群**：新老用户、高低活跃度用户、不同机型/版本/地域
- **行为分群**：有没有点击过PUSH、有没有使用过某功能、不同消费深度用户
- **业务维度分群**：不同PUSH类型、不同频道、不同入口、不同内容类型

**典型案例：辛普森悖论**
> 整体CTR下降了，但分人群看每个群体的CTR都涨了，为什么？
> 原因：高CTR的人群占比下降了，拉低了整体平均值。这时候如果只看大盘就会得出错误结论，必须分群看。

💡 核心原则：**大盘异动时，先拆分维度，看是所有群体都变差了，还是只是某个群体/某个部分出问题了**。

### 3.4 漏斗分析：看转化链路哪里漏了
用户完成一个目标要经过多个步骤，每个步骤都会有流失，把每个步骤的转化率排成漏斗，就能找到瓶颈在哪里。

#### 漏斗分析六步法（参考产运班培训材料）：
1. **明确业务目标（核心前提）**：你要提升什么？PUSH点击率？Push→信息流的转化？还是付费转化？
2. **设计漏斗路径（关键环节）**：梳理用户从进入到完成目标的完整路径，不要漏关键步骤，也不要加无关步骤
3. **采集数据（数据基础）**：确认每个步骤的数据都能拿到，埋点正确
4. **搭建漏斗并计算指标**：算每个步骤的转化率、总转化率、流失率
5. **深度分析瓶颈（核心环节）**：哪个步骤流失最多？为什么流失？和历史比/和对照组比这个步骤是不是变差了？
6. **优化验证与迭代**：针对瓶颈做优化，AB实验验证效果，再看漏斗有没有提升

**案例：PUSH全链路漏斗**
```
PUSH下发 → 到达 → 展示 → 点击 → 落地页加载完成 → 信息流消费 → 后续留存
```
每个步骤都有流失：
- 下发→到达：和推送通道、用户关闭通知权限有关
- 到达→展示：和用户手机状态、通知拦截有关
- 展示→点击：和文案、标题、推送时间、用户匹配度有关
- 点击→加载完成：和网络、落地页性能有关
- 加载→消费：和落地页内容、用户体验有关
- 消费→留存：和内容质量、长期价值有关

CTR下降不要只盯着点击，要拆漏斗看是哪个环节出了问题：是到达少了？还是展示了没人点？

## 四、真实案例：PUSH CTR下降排查
我们用完整流程走一遍，看真实工作中怎么分析问题：

### 问题背景
周一早上你发现：昨天（周日）PUSH CTR从平时的8%降到了6%，下降了2个百分点，需要排查原因。

### 步骤1：明确目标
- 定位CTR下降的原因，判断是正常波动还是异常问题
- 给出是否需要回滚/调整策略的建议

### 步骤2：拆解指标和维度
- 核心指标：CTR=点击/到达
- 拆分维度：时间（小时级）、PUSH类型、用户分群（新老/活跃度/版本）、链路拆漏斗（到达率→展示率→点击率）
- 对比基准：上周日同时段、历史周日平均水平、上周六数据

### 步骤3：取数验证（用前3节课学的SQL写查询）
先按维度拆解开，看问题出在哪里：
1. **时间维度**：看是全天都低还是某个时段低？发现凌晨0-2点低，其他时间正常
2. **PUSH类型**：发现是新上线的"热点新闻PUSH"CTR特别低，只有3%，其他类型正常
3. **用户分群**：新用户CTR下降明显，老用户基本正常
4. **漏斗拆分**：到达率正常，展示率正常，问题出在展示→点击环节，文案点击率低

### 步骤4：分析定位原因
- 周日凌晨发的一批热点新闻PUSH，面向全量用户（包含新用户）推送，但标题过于标题党，和用户匹配度低，导致新用户点击率特别低，拉低了整体CTR
- 这批PUSH贡献了当天20%的发送量，但点击率只有平均水平的一半，贡献了80%的CTR下降

### 步骤5：结论和建议
- **结论**：周日凌晨上线的热点新闻批量推送策略有问题，标题与新用户匹配度低，导致整体CTR下降2个百分点
- **建议**：
  1. 热点新闻PUSH先小流量灰度，不要直接全量发
  2. 新用户单独适配更温和的内容和标题，不要一开始就推标题党内容
  3. 预计调整后CTR能回到7.5%以上

## 五、分析结论怎么呈现？
给业务方/老板汇报时，遵循**结论先行**原则：
1. **一句话总结论**：先把最核心的结论放最前面
2. **核心数据支撑**：用3-5个关键数据证明结论
3. **原因拆解**：问题出在哪里，贡献度多少
4. **可落地建议**：具体做什么，预期收益多少
5. **附录**：详细数据表格、SQL、明细

❌ 反面汇报："我取了数，PUSH CTR周六是8%，周日是6%，分类型看A类是5%，B类是7%，C类是9%..."（全是数据，没有结论，听的人不知道你要说什么）
✅ 正面汇报："周日PUSH CTR下降2个点，80%是因为凌晨上线的热点新闻全量推送对新用户点击率过低导致，建议先回滚该策略，小流量灰度测试后再放量，预计CTR可恢复到7.5%以上。核心数据如下：..."

## 六、常见分析误区避坑
1. **不要唯数据论**：数据是工具不是目的，要理解数据背后的业务逻辑，异常数据先确认是不是埋点问题/数据口径问题
2. **不要忽略样本量**：只有几百个用户的小样本，涨跌幅波动大，不要急着下结论，统计上不显著
3. **不要把相关性当因果**：冰淇淋销量上升和溺水死亡人数上升正相关，但不是冰淇淋导致溺水，是因为夏天温度高，两者都上升
4. **不要过度拆解**：维度拆分到能定位问题就行，不要拆到太细的维度，每个群体只有几百人，没有统计意义
5. **不要只看相对值不看绝对值**："某功能转化率提升了100%"，听起来很厉害，但如果基数是100个人，其实也就多了100人用，影响不大

## 七、本节课小结
1. 数据分析的目标是从数据走到决策，不是只报数字
2. 完整分析流程：明确目标→拆解指标→取数→定位原因→输出可落地建议
3. 四大核心方法：对比分析找差异、趋势分析看规律、分群分析拆人群、漏斗分析找链路瓶颈
4. 遇到问题先拆分维度，不要只看大盘，辛普森悖论经常出现
5. 汇报结论先行，用数据支撑，建议要具体可落地
