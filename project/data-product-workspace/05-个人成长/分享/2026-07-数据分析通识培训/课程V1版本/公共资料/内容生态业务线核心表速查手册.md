# 内容生态业务线核心表速查手册

> 更新时间：2026-07-04
> 适用场景：浏览器/PUSH/信息流日常取数

---

## 核心表清单

| 库名.表名 | 表描述 | 粒度 | 核心用途 |
|-----------|--------|------|----------|
| `browser.dwm_browser_event_aggregation_miui_di` | MIUI端用户行为聚合表 | did+date | APP启动、版本、启动方式、整体使用时长 |
| `browser.dwm_browser_event_aggregation_label_di` | 信息流标签聚合表 | did+date | 信息流曝光/点击/播放/消费时长 |
| （待补充PUSH相关表） | PUSH发送点击明细表 | - | PUSH到达率、点击率、下发效果 |

---

## 一、miui_di表常用字段

### 维度字段
| 字段名 | 说明 | 备注 |
|--------|------|------|
| date | 日期分区 | 格式yyyyMMdd，查询必须加 |
| did | 设备ID | 用户唯一标识，DAU计算用 |
| app_ver | APP版本号 | 如 20.21.1050617 |
| os_ver | 操作系统版本 | |
| model | 机型 | 可分高端/低端机 |
| app_launch_way | 启动方式 | 枚举：点击icon/第三方调起/点击push/新全搜调起等 |
| third_packagename | 三方调起包名 | 仅app_launch_way=第三方调起时有值 |
| is_app_dau_2024 | APP DAU标记 | =1为有效活跃用户，**必须加过滤** |
| is_new_2024 | 新用户标记 | =1为当日新用户 |

### 指标字段
| 字段名 | 说明 | 单位 |
|--------|------|------|
| app_open_cnt | APP打开次数 | 次 |
| app_dura | APP总使用时长 | **毫秒** → 转分钟/60000 |

---

## 二、label_di表常用字段

### 维度字段
| 字段名 | 说明 | 备注 |
|--------|------|------|
| date | 日期分区 | 同miui_di |
| did | 设备ID | 同miui_di |
| is_dau_feed_dapan_2024 | 信息流DAU标记 | =1为信息流活跃用户 |
| feed_status | 信息流消费状态 | 'true'=有消费，'false'=无消费 |
| is_top | 是否置顶内容 | 0=自然推荐，1=置顶，自然流量过滤is_top=0 |
| feed_channel | 信息流频道 | 分频道分析用 |
| page/from_page | 当前页/来源页 | 路径分析用 |

### 指标字段
| 字段名 | 说明 | 单位 |
|--------|------|------|
| expos_cnt | 信息流曝光次数 | 次 |
| click_cnt | 信息流点击次数 | 次 |
| news_vv_cnt | 图文有效阅读VV | 次 |
| short_vv_cnt | 短视频VV | 次 |
| mini_vv_cnt | 小视频VV | 次 |
| feed_dura | 信息流消费时长 | **毫秒** → 转分钟/60000 |
| consum_dura | 总消费时长 | 毫秒 |

💡 总VV计算公式：`news_vv_cnt + short_vv_cnt + mini_vv_cnt`

## 三、通用过滤条件模板
每次查询记得加上这些条件，避免数据异常：
```sql
WHERE date BETWEEN '${start_date}' AND '${end_date}'  -- 替换成你的日期范围
  AND is_app_dau_2024 = 1                              -- 过滤有效活跃用户
  AND COALESCE(did, '') != ''                          -- 过滤空设备ID
  -- 查自然信息流时加上：
  -- AND is_top = 0
```

## 四、常见问题排查
1. **查询跑不出结果/特别慢**：检查是不是没加date分区过滤
2. **DAU数据特别大**：检查是不是没加is_app_dau_2024=1过滤
3. **CTR异常偏高**：检查是不是把置顶内容算进去了，加is_top=0
4. **时长数据太大**：注意是毫秒单位，转分钟要除以60000
