# 导览 · 数据从哪来 & 指标口径与度量（新人必看串讲）

> 约 30 分钟 | 全员必看 | 全课程"地图"，重点讲两件最容易被忽略、又最重要的事：**① 数据从哪来 ② 指标口径与度量**。

---

## 一、新人的三个困惑
1. 数据从哪冒出来的？（我查的表里的数是谁、怎么放进去的？）
2. 这个指标到底怎么算？（CTR 分母用哪个？）
3. 为什么我和你算的对不上？

> 根子都在两件事：**数据从哪来** + **指标口径**。

## 二、一个数字的一生（全景）
`用户行为 → 埋点采集 → 日志加工 → 分层成表 → 取数 SQL → 指标 → 分析决策`
- 前半段 = 数据从哪来；关键节点 = 指标口径。

---

## 第一篇 · 数据从哪来
- **数据的一生**：用户操作 → 埋点采集(SDK记行为) → 日志加工(清洗/去重/统一口径) → 分层聚合成表。数据不是凭空来的，每个数字背后都是"真实行为 + 一套加工"。
- **埋点四事件**：曝光 / 点击 / 进入 / 退出；**消费时长 = 退出 − 进入**（算出来的）。
- **数仓分层**：ODS(原始)→DWD(明细,一行=一次行为)→**DWM(汇总,一人一天一行，日常首选)**→ADS(应用)。
- **一张表的关键**：粒度 did+date（一人一天一行）· 分区 date（查询必加）· T+1（今天数据明天出）· 时长是毫秒。
- **一句话**：数据 = 用户行为被埋点记录、经清洗聚合、存成按天的表。

## 第二篇 · 指标口径与度量（重点）

### 为什么口径这么重要
同一个「CTR」，口径不同结果能差一倍多。你说 3%、他说 5%，其实数据一样，只是分母不同。**"数对不上"90% 是口径问题，不是数据错。**

### 核心指标口径速查

| 指标 | 口径 / 公式 | 关键注意 |
|------|------------|----------|
| DAU | `COUNT(DISTINCT did)` | 必须去重 |
| 信息流 CTR | `SUM(click_cnt)/SUM(expos_cnt)` | 用 SUM/SUM 不用 AVG；自然流量 is_top=0 |
| PUSH CTR | `点击 / 到达` | 分母是"到达"不是"发送"！ |
| 总 VV | `news+short+mini_vv_cnt` | 三类相加 |
| 人均时长(分) | `SUM(feed_dura)/COUNT(DISTINCT did)/60000` | 毫秒转分钟 |
| 渗透率 | `用了的人 *1.0 / 总人数` | 分子记得 *1.0 |

### 度量的六个坑（新人必踩）
1. 时长是毫秒 → 转分钟 `/60000`，不转大 6 万倍
2. 算人数漏 `DISTINCT` → DAU 翻倍
3. 除法漏 `*1.0` → 率算成 0
4. 漏 `date` 分区 → 扫全表跑不出来
5. 漏有效用户过滤（`is_app_dau_2024=1`）→ 混进测试/异常
6. 自然流量漏 `is_top=0` → CTR 虚高

### "为什么我和你算的对不上"——口径对齐清单
查同一指标前，先对齐 6 项：① 同一张表吗（ADS vs DWM）② 都去重了吗 ③ 过滤条件一样吗（有效用户/is_top）④ 时间范围一样吗 ⑤ 单位一样吗（毫秒/分钟）⑥ 分子分母定义一样吗（CTR 口径）。**先对口径，再对数。**

### 指标度量"三件套"
定义一个指标要说清：① 口径定义（是什么、算哪些人）② 计算公式（分子分母、怎么聚合）③ 注意事项（单位/过滤/去重）。
> 示例·人均信息流时长(分)：① 当日信息流活跃用户人均消费时长 ② SUM(feed_dura)/COUNT(DISTINCT did)/60000 ③ feed_dura 是毫秒；只算 is_dau_feed_dapan_2024=1；is_top=0。
> 建议团队维护一份"指标口径字典"。

---

## 记住两句话
1. **每个数字背后都是"真实行为 + 加工口径"，先搞清它从哪来。**
2. **算指标前先对齐口径，再动手** —— 别闷头算完才发现对不上。
