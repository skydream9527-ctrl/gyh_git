# 数据分析通识培训 · 课程交付包（Markdown 源包）

> **这是什么**：一套完整的 Markdown 源包，含课程全部内容 + 生成指令。
> **用途**：交给**另一个环境的 AI（Kiro）**，据此在稳定环境中生成完整课程——每个模块产出【课件 md + 独立 PPT html + 练习 md】。
> **⚠️ 背景**：原环境疑似文件系统同步问题，多文件写入/移动会丢失。故本包用少数几个 MD 承载全部原始信息，交由稳定环境重新生成、拆分。

## 二、目标产物结构（请另一个 Kiro 生成成这样）
```
数据分析通识培训/
├── 课件讲义/   00~09.md          （10 个模块课件，逐模块独立）
├── html/       00~09.html + 串讲.html （演示 deck，逐模块独立）
├── 配套练习/   00~08 各模块-练习.md   （答疑无练习）
└── 公共资料/   5 份辅助材料
```

## 三、生成步骤（给另一个 Kiro 的指令）
1. **拆课件 + 练习**：打开 `3-全部课时内容.md`，按 `# ═══ 模块X ═══` 分隔。每个模块内：
   - `## 【课件】…` 段落 → 存为 `课件讲义/0X-模块名.md`
   - `## 【练习】…` 段落 → 存为 `配套练习/0X-模块名-练习.md`
2. **生成每模块 PPT deck**：打开 `5-PPT演示引擎与各模块大纲.md`，复制里面的**完整 deck 引擎模板**（HTML+CSS+JS），按每个模块的 **slide 大纲**填充 `<section class="slide">` 内容，生成 `html/0X-模块名.html`。所有模块共用同一引擎，视觉统一。
3. **串讲**：`2-串讲导览.md` 内容 → 用同引擎生成 `html/串讲.html`，并保留 md。
4. **公共资料**：`4-公共资料.md` 按 `# ═══` 拆成 `公共资料/` 下各份。
5. **每生成一个文件，读盘验证一次**（防同步丢失）。

## 五、注意事项
1. **口径优先**：涉及指标必写"口径 + 公式 + 注意事项"。
2. **AI 写作业、人检查作业**：贯穿理念——工具/AI 负责写，人判断口径对不对、结果合不合理。
3. **场景化**：全程用浏览器 / 信息流 / PUSH 真实业务，不干讲语法。
4. **分层不劝退**：销售/运营抓必会段（数据思维 + 自证 + 基本 SQL + AI），复杂交给 AI。
5. **生成 deck 复用统一引擎**，别每个重新设计，保证视觉一致。
