#### KA1：【P0核心】nl-sql自然语言取数场景打透
- **目标**：覆盖日常80%取数需求，准确率达到90%+，成为团队取数首选方式
- **核心动作**：
  1. 完成浏览器/内容中心核心业务域数据资产知识库建设（表结构、字段枚举、指标口径、关联逻辑）
  2. 基于实际使用badcase持续优化nl-sql准确率，重点解决多表关联、复杂条件、维度聚合等常见场景
  3. 查询结果自然语言解读，支持结果追问、修正、二次加工，交互流畅无阻塞
  4. 建立用户反馈闭环，badcase快速响应迭代
- **验收标准**：团队50%以上常规取数需求通过AI完成，无需人工写SQL，用户满意度NPS>80

#### KA2：【P0平台】整合集团AI CLI数据能力，产品化提供数据服务
- **目标**：打通集团现有各类AI CLI数据工具，形成统一的产品化数据服务能力，降低使用门槛
- **核心动作**：
  1. 梳理集团现有AI CLI数据能力（Kyuubi CLI、DataWorks CLI、各类数据工具CLI等）
  2. 基于MCP标准化协议做能力整合和封装，统一调用入口，抹平工具差异
  3. 打通飞书机器人、ICE工作台、终端CLI多场景使用通路，在哪里都能便捷调用
  4. 封装常用数据操作场景（取数、查元数据、跑任务、看报表等）为统一产品化服务
- **验收标准**：主流数据场景无需切换多个工具，通过统一入口即可完成，CLI/飞书/工作台三端体验一致

#### KA3：【P0产品】ICE 数据工作台 V2.0 正式上线
- **目标**：完成ICE工作台从v3原型到V2.0正式版的产品化打磨，成为团队数据AI工作统一入口
- **核心动作**：
  1. 工作台核心流程优化：取数、Agent任务、结果查看、历史记录全链路体验打磨
  2. 基础能力完善：用户权限、任务管理、使用统计、错误处理、稳定性保障
  3. 整合nl-sql、版本分析、实验分析、日报等核心AI能力到工作台统一承载
  4. 产品化收尾：交互细节优化、性能优化、可用性测试，达到团队日常使用标准
- **验收标准**：V2.0正式发布上线，稳定运行，团队成员可以顺畅使用工作台完成日常数据工作

#### KA4：【P0认知】沉淀迭代AI Native认知和方法论
- **目标**：在实践中总结沉淀AI Native数据产品的方法论和认知，指导后续工作，形成可复用经验
- **核心动作**：
  1. 实践中持续总结：AI落地数据场景的路径、方法、踩坑经验、防幻觉方案
  2. 沉淀AI Native工作方法论：如何用AI重构数据工作流，如何识别高价值场景，如何评估效果
  3. 形成文档/分享：将实践经验系统化沉淀，输出团队可复用的方法论和最佳实践
  4. 认知持续迭代：边做边复盘，每两周更新一次认知和方法论
- **验收标准**：形成一套可复用的AI Native数据产品落地方法论文档，团队内部对齐认知
