# 决策框架 — 开篇导读

> 这是 [decision-frameworks/](.) 目录的总入口。读完这一篇，你应该能回答：为什么「凭感觉决策」长期必输、什么时候该用什么框架、AI在决策里到底扮演什么角色、怎么建立不依赖运气的决策系统。

---

## 一、为什么你需要决策框架（一段扎心的论述）

工作里最常见的三个场景：

> "这个需求优先级排高一点吧，老板昨天提了一嘴。"
> "A方案看起来更酷，我们做A吧。"
> "上次这么做成了，这次应该也没问题。"

这些都是**真实存在的决策习惯**，但作为长期工作方式是**灾难性的**——你在用"即时情绪和最近记忆"做"影响几个月资源投入"的决策。

更可怕的是：**坏决策经常带来好结果（运气），好决策经常带来坏结果（运气差）**。如果只看结果复盘，你会把运气当能力，把偶然当必然。

决策框架要解决的就是：**在信息永远不全、时间永远不够、未来永远不确定的情况下，怎么持续提高决策质量，把胜率拉到统计意义上占优。**

> 当你有明确数据可以做AB测试时，请优先做。详见 [../ab-testing/](../ab-testing/)。决策框架是当**实验做不了、信息不全、需要拍板**时的操作系统——但这套系统用得好，长期决策质量是数量级的提升。

## 二、决策的三层心智模型

把所有决策分成三层，用错层级是新手最大的错误：

```
┌───────────────────────────────────────────────────────────┐
│  3. 战略决策（年/季度级）    方向、路线、做不做            │ ← 第一性原理/系统思维/反脆弱
│  2. 战术决策（月/周级）      资源分配、优先级、怎么做      │ ← RICE/决策矩阵/预-mortem
│  1. 运营决策（天/小时级）    日常优化、快速迭代            │ ← AB测试/贝叶斯更新/数据看板
└───────────────────────────────────────────────────────────┘
            不确定性↑                           可逆性↓
```

**重点**：
- 第1层运营决策，错了代价小，可以快速试错，不用过度思考
- 第2层战术决策，需要结构化量化比较，避免拍脑袋
- 第3层战略决策，不能靠算分，需要深度思考和反脆弱设计

**记一条铁律**：用运营决策的方法（拍脑袋/看最近案例）做战略决策，等同于闭眼开车。这是绝大多数团队半年后回头看发现走偏了的根本原因。

## 三、AI时代的决策：人机协作新范式

很多人问："AI都能分析数据写方案了，还要决策框架干嘛？" 答案是：AI是增强器，不是替代者，核心问题变成四个：

| 问题 | 答案 |
|---|---|
| 什么时候该信AI，什么时候该信人？ | 高确定性/有数据的场景信AI；高不确定性/价值判断/黑天鹅场景信人 |
| 怎么让AI给出靠谱的决策建议？ | 给足上下文 → 明确约束 → 要求给出反对意见 → 用框架结构化输出 |
| 怎么避免AI的偏见带偏决策？ | 永远让AI同时给出「支持方案」「反对方案」「风险点」三个版本 |
| 怎么把决策流程沉淀成可复用的系统？ | 用 [AI上下文决策基础设施](./frameworks/ai-context-decision-infrastructure.md) 构建决策记忆 |

详见 [frameworks/ai-decision-making.md](./frameworks/ai-decision-making.md)。

### 常用工具速查表

| 工具 | 适用场景 | 核心逻辑 |
|---|---|---|
| **RICE评分** | 需求优先级排序 | Reach × Impact × Confidence / Effort |
| **决策矩阵** | 多选项多维度对比 | 加权打分，量化主观判断 |
| **SWOT分析** | 战略环境扫描 | 优势/劣势/机会/威胁 |
| **波特五力** | 行业竞争格局分析 | 五种力量决定行业利润率 |
| **PESTEL** | 宏观环境分析 | 政治/经济/社会/技术/环境/法律 |
| **安索夫矩阵** | 增长方向选择 | 现有/新市场 × 现有/新产品 |
| **蓝海战略** | 差异化竞争 | 剔除-减少-增加-创造四步动作 |
| **预-mortem** | 风险预判 | "假设项目失败了，是什么原因导致的？" |
| **贝叶斯更新** | 动态调整判断 | 新证据进来时更新先验概率 |
| **ADR决策记录** | 所有重要决策 | 记录结论/依据/假设/回看时间 |

## 五、最常见的五个决策陷阱（先记住，后细学）

我把这五个放在导读，是因为团队80%的决策错误都来自这里。

1. **确认偏误（Confirmation Bias）**
   - 现象：只看支持自己观点的数据，自动忽略反对证据
   - 后果：越分析越觉得自己对，最后在错的路上越走越远
   - 解法：强制要求列出「3个反对理由」，专门找反对意见

2. **锚定效应（Anchoring）**
   - 现象：第一个说出来的数字/方案会锚定所有人的判断
   - 后果：先开价的人占便宜，需求评审被第一个方案带偏
   - 解法：先独立写下判断，再一起讨论；先列标准再看方案

3. **沉没成本谬误（Sunk Cost）**
   - 现象："都已经投了这么多了，现在停太亏了"
   - 后果：在烂项目上持续投入，越陷越深
   - 解法：问自己"如果现在从零开始，我还会做这个吗？"

4. **幸存者偏差（Survivorship Bias）**
   - 现象："别人这么做成都了，我们做也能成"
   - 后果：只看到成功案例，没看到99个用同样方法失败的死人
   - 解法：找失败案例，算基准成功率，看必要条件而非充分条件

5. **结果偏误（Outcome Bias）**
   - 现象："结果好就是决策好，结果差就是决策差"
   - 后果：把运气当能力，复盘学不到真东西
   - 解法：复盘先看决策过程对不对，再看结果好不好

## 七、和本工作区其他模块的关联

- **AB测试**（[../ab-testing/](../ab-testing/)）：运营决策的黄金标准，能量化的决策尽量做实验
- **因果推断**（[../causal-inference/](../causal-inference/)）：不能做实验时，因果方法帮你从观察数据里找靠谱结论
- **Agent + LLM**（[../agent-llm/](../agent-llm/)）：AI辅助决策的技术基础，怎么让Agent帮你做决策分析
- **数据分析**（[../data-analysis/](../data-analysis/)）：决策的输入是数据，分析质量决定决策质量
- **产品方法论**（[../product/](../product/)）：产品场景下的决策落地，需求、优先级、增长的具体方法
- `DECISIONS.md`：重要决策必须沉淀到这里，包含结论/依据/风险/回看时间

## 八、心法

如果只能记三句话：

1. **决策质量 ≠ 决策结果**——好过程可能有坏结果，坏过程可能有好结果，长期来看好过程必胜。
2. **重要决策先写下来**——想的时候觉得想清楚了，写的时候才发现逻辑漏洞百出。
3. **小决策要快，大决策要慢**——可逆的决策快速试错，不可逆的决策慢下来想透。

下次做重要决策前，Checklist：
1. 这个决策是战略/战术/运营哪一层？
2. 我列了反对意见吗？有没有只看支持证据？
3. 如果现在从零开始，我还会做同样选择吗？
4. 失败了最可能是什么原因？有预案吗？
5. 决策依据写下来了吗？什么时候回看复盘？

回答不出来，先别拍板。
