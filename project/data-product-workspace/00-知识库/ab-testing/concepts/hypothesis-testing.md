# 假设检验基础

> 本文目标：把"假设检验"这件抽象的事拆成一个可操作的决策流程，让你在看实验报告时知道每个数字代表什么、什么时候能信、什么时候要警觉。

---

## 一、核心心智模型：反证法

假设检验的本质是统计学版的**反证法（proof by contradiction）**。

正常逻辑：我想证明"新版本更好"。
统计逻辑：我假设"新版本和旧版本一样"，然后看观测数据在这个假设下出现的概率有多小。如果太小（低于阈值 α），我就**拒绝**这个假设——也就是有证据说明它们**不**一样。

```
      想证明的事             先假设它的反面成立        看数据有多反常
   ─────────────────  →   ────────────────────  →  ──────────────────
    "新版更好"              "新旧一样" (H₀)         反常程度 < α？
                                                     ↓
                                            是 → 拒绝 H₀（有证据）
                                            否 → 不拒绝（证据不足）
```

**关键点**：你永远没法"证明"H₀ 为真，只能"找不到证据反对它"。这是新手最容易混淆的地方。

---

## 二、四个必备概念

### 1. 原假设 H₀（Null Hypothesis）
你**预期希望被推翻**的假设。在 AB 测试里通常是"两组没差异"。

- 例：`H₀: 转化率_B - 转化率_A = 0`

### 2. 备择假设 H₁（Alternative Hypothesis）
你**真正想要的结论**。

- 双边：`H₁: 转化率_B ≠ 转化率_A`（B 可能更好也可能更差）
- 单边：`H₁: 转化率_B > 转化率_A`（B 更好）

> 实践默认用**双边**。单边只在你**事先**有明确方向性预期时才用，否则容易主观挑边导致结论失真。

### 3. 显著性水平 α（Significance Level）
你愿意承担**冤枉好人**的最大风险。工业界默认 0.05，金融、医药里有时会用 0.01。

α = 0.05 的真实含义：**当 H₀ 为真时**，我们错误地拒绝它的概率是 5%。
不是：H₀ 为真的概率是 5%（这是常见误读）。

### 4. 检验统计量与 p 值
- **检验统计量**：把数据浓缩成一个数字（如 z 值、t 值），衡量观测到的差异有多"反常"。
- **p 值**：在 H₀ 为真的前提下，观测到至少这么反常（或更反常）的数据的概率。

判决规则：**p < α → 拒绝 H₀**。

---

## 三、p 值的三大常见误读（重要）

| 误读 | 真相 |
|---|---|
| ❌ p = 0.03 意味着"H₀ 为真的概率是 3%" | ✅ p 是**给定 H₀ 为真**时观测到这种数据的概率，不是 H₀ 本身为真的概率（要算这个需要贝叶斯） |
| ❌ p = 0.06 比 p = 0.04 "差很多" | ✅ 0.05 是人为划线，统计上 0.04 与 0.06 差异并不本质 |
| ❌ p > 0.05 意味着"两组一样" | ✅ 只意味着**证据不足**以拒绝 H₀。可能是真的没差异，也可能是样本量不够 |

第三条尤其危险：很多产品决策错在"实验不显著就当成功能没效果，下线"，但其实是 MDE 设得不切实际、样本量不够。这种情况要做的是**重新设计实验**，不是直接下结论。

## 四、两类错误

| 实际情况 \ 决策 | 拒绝 H₀（说有差异） | 不拒绝 H₀（说没差异） |
|---|---|---|
| **H₀ 真**（其实没差异） | I 类错误（α）❌ 冤枉 | 正确 ✅ |
| **H₀ 假**（其实有差异） | 正确 ✅ | II 类错误（β）❌ 漏判 |

- **I 类错误（α）**：把没用的功能上线。
- **II 类错误（β）**：把有用的功能错杀。
- **功效（Power）= 1 − β**：真有差异时，我们能识别出来的概率。

> α 和 β 互相博弈：固定样本量下，降低 α 必然抬高 β。要同时压低，唯一办法是加样本量。详见 [power-and-sample-size.md](power-and-sample-size.md)。

## 五、AB 测试里最常用的三种检验

### 1. 比例检验（Z-test for Proportions）
**场景**：转化率、点击率、留存率——分子分母都是计数。

公式（双比例 z 检验）：

```
       p̂_B − p̂_A
z = ─────────────────────
    √( p̂(1−p̂) · (1/n_A + 1/n_B) )

其中 p̂ = (x_A + x_B) / (n_A + n_B)  ← 合并比例
```

Python：

```python
from statsmodels.stats.proportion import proportions_ztest

# A 组 1000 人 50 转化；B 组 1000 人 70 转化
count = [50, 70]
nobs  = [1000, 1000]
z, p = proportions_ztest(count, nobs, alternative='two-sided')
print(f"z = {z:.3f}, p = {p:.4f}")
```

### 2. 均值检验（Welch's t-test）
**场景**：人均时长、人均 GMV、人均播放数——连续值。

为什么用 Welch 而非 Student：实验里两组方差几乎不会相等，Welch 对此鲁棒。

```python
from scipy.stats import ttest_ind

t, p = ttest_ind(group_a_values, group_b_values, equal_var=False)
```

⚠️ **重尾分布要小心**：GMV、时长往往是右偏长尾的。t 检验依赖 CLT，需要每组样本量足够大（经验阈值 ≥ 10000）。否则用 Mann-Whitney U 或 bootstrap。

### 3. 比率指标（Delta Method）
**场景**："人均订单价"、"人均广告点击率"——分子分母都是随机变量。

错误做法：直接对每个用户算 ratio 再做 t 检验（忽视了分母也是随机的）。

正确做法：用 **Delta Method** 修正方差。`statsmodels` 没有现成函数，业界平台基本都自己实现。简化版：

#### Delta Method 原理

比率指标 R = X̄ / Ȳ 的分子 X̄ 和分母 Ȳ 都是随机变量，直接对每个用户算 ratio 再做 t 检验会**低估方差**——因为分母的波动被忽略了。

Delta Method 的核心思路：对 R = g(X̄, Ȳ) = X̄ / Ȳ 做**一阶泰勒展开**，用线性近似推导 R 的方差。

**推导过程**：

```
1. 定义：R = X̄ / Ȳ，其中 X̄ = Σxᵢ/n, Ȳ = Σyᵢ/n

2. 对 g(X̄, Ȳ) = X̄ / Ȳ 在 (μx, μy) 处一阶泰勒展开：
   g(X̄, Ȳ) ≈ μx/μy + (1/μy)(X̄ - μx) - (μx/μy²)(Ȳ - μy)

3. 取方差（常数项方差为 0）：
   Var(R) ≈ Var[(1/μy)X̄ - (μx/μy²)Ȳ]
          = (1/μy²)Var(X̄) + (μx²/μy⁴)Var(Ȳ) - 2(μx/μy³)Cov(X̄, Ȳ)

4. 代入样本估计量（μx → X̄, μy → Ȳ, R = X̄/Ȳ）：
   Var(R) = [Var(X̄) + R²·Var(Ȳ) - 2R·Cov(X̄, Ȳ)] / Ȳ²
```

**关键洞察**：协方差项 `-2R·Cov(X̄, Ȳ)` 是 Delta Method 和"直接算 ratio"的本质区别。当分子分母正相关时（如"人均 GMV"——花钱多的人订单也多），协方差项为正，会**缩小**方差估计；如果忽略它，方差会被**高估**，导致检验过于保守。反之，负相关时忽略协方差会**低估**方差，导致假阳性升高。

**典型比率指标**：

| 指标 | 分子 X | 分母 Y | X 与 Y 的关系 |
|---|---|---|---|
| 人均 GMV | 总 GMV | 用户数 | 无关（Y 固定） |
| 人均订单价 | 总订单额 | 订单数 | **正相关** |
| 人均广告点击率 | 点击次数 | 曝光次数 | **正相关** |
| 完播率 | 完播次数 | 播放次数 | **正相关** |

> 当分母 Y 在用户间变化时（如"人均订单价"——不同用户订单数不同），Delta Method 必须使用。如果分母对每个用户都相同（如"人均 GMV"——分母恒为 1），则退化为普通的均值检验。

```python
import numpy as np

def delta_method_ratio(num, den):
    """对单组算 ratio = sum(num)/sum(den) 的方差"""
    n = len(num)
    mean_num, mean_den = num.mean(), den.mean()
    var_num, var_den = num.var(ddof=1), den.var(ddof=1)
    cov = np.cov(num, den, ddof=1)[0, 1]
    ratio = mean_num / mean_den
    var_ratio = (var_num + ratio**2 * var_den - 2 * ratio * cov) / (n * mean_den**2)
    return ratio, var_ratio

def delta_method_ab_test(num_a, den_a, num_b, den_b):
    """两组比率指标的 AB 检验"""
    r_a, var_a = delta_method_ratio(num_a, den_a)
    r_b, var_b = delta_method_ratio(num_b, den_b)
    diff = r_b - r_a
    se = np.sqrt(var_a + var_b)
    z = diff / se
    p = 2 * (1 - norm.cdf(abs(z)))
    ci_low = diff - 1.96 * se
    ci_high = diff + 1.96 * se
    return {"diff": diff, "z": z, "p": p, "ci": (ci_low, ci_high)}
```

> **这是工业级实验平台和教科书最大的差距**——教科书几乎不讲 Delta Method，但你打开任何主流实验平台的源码都会看到它。

## 六、双边 vs 单边：实践建议

```
双边（默认）：H₁: μ_B ≠ μ_A
  ├─ p_two_sided = 2 × p_one_sided（在同方向下）
  ├─ 适合：你不确定 B 是变好还是变坏
  └─ 95% 例子用这个

单边：H₁: μ_B > μ_A
  ├─ 样本量需求约比双边少 25-30%
  ├─ 适合：理论上不可能变差的改动（修 bug、降延迟）
  └─ 风险：如果方向猜错，结论无效；监管 / 论文场景禁用
```

**红线**：**严禁先看数据再决定方向！** 这叫 p-hacking，会让 I 类错误率翻倍。

## 七、置信区间与假设检验的关系

很多人把"看 p 值"当唯一动作，但**置信区间（CI）信息密度更高**：

- p < α  ↔  双边 CI 不包含 0（等价）
- 但 CI 还告诉你**效应量的范围**

例：
- A 实验：差异 +2.0%，95% CI = [+0.1%, +3.9%]，p = 0.04
- B 实验：差异 +2.0%，95% CI = [+1.8%, +2.2%]，p = 0.0001

p 值都 "显著"，但 A 实验的 CI 横跨"几乎没用"和"很有用"，B 实验的 CI 锁得很紧。**A 实验的 +2% 实际上不可信**——可能是 0.1%。

> **报告实验结果时，永远连带 CI 一起给**。光给 p 值是耍流氓。

## 八、最小可行检查清单

每次拿到一份 AB 报告，按这个顺序问自己：

```
□ 1. 主指标（OEC）是什么？是不是上线前就定好的？
□ 2. p 值多少？α 是 0.05 还是更严格？
□ 3. 置信区间多宽？业务上"窄到能决策"了吗？
□ 4. SRM 检查通过了吗？（A/B 流量比例是否符合预期）
□ 5. 有没有偷看？（提前停止、在 p=0.05 边缘）
□ 6. 有没有多重比较？（看了多少指标、做了校正吗）
□ 7. 子群分析显著的，是事先指定还是事后挖出来的？
```

任何一项答不清楚，回去找数据科学复核。
