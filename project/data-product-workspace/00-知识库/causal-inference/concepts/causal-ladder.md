# 因果阶梯（The Ladder of Causation）

> Judea Pearl 在 *The Book of Why* 中给整个因果推断领域提供了一张共享的地图。这张地图把因果问题分成三层，每层能力不同、要求不同、解决方法也不同。读懂它，你看待数据的视角会永久改变。

---

## 一、为什么需要一张"地图"

在 Pearl 提出阶梯之前，因果讨论里充满"是因果还是相关"的口水仗——大家都在用同一套词汇描述本质不同的问题。

阶梯的真正贡献是：**给"不同层级的因果问题"起了不同的名字**，让人们能精确地说出自己在问什么、需要什么数据、能用什么方法。

---

## 二、三层全貌

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│   3.  反事实  Counterfactual                          ↑ 难度       │
│       "如果当时没投广告，他会买吗？"                  ↑ 假设       │
│       关键词：反事实推理、因果机制、个体决策          ↑ 数据要求   │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│   2.  干预  Intervention                                          │
│       "如果我们给所有人投广告，整体会怎样？"                       │
│       关键词：do(X)、AB 测试、政策评估                            │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│   1.  关联  Association                                           │
│       "投广告的人和没投的人，购买率有什么差异？"                   │
│       关键词：相关、条件概率、回归、监督学习                       │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

**两个核心规则**：

1. **能力是单向的**：上层能回答下层的问题，反过来不行。
2. **数据要求是反向的**：只用观察数据 → 只能爬到第一层。再上去要么靠实验（第二层），要么靠强结构假设（第三层）。

---

## 三、第 1 层：关联（Association）

### 在问什么

> "看到 X，Y 是什么样？"

形式化：`P(Y | X)`

### 例子

- 看到广告的人，购买率是多少？`P(buy | saw_ad)`
- 时长 > 30 分钟的用户，留存率多高？
- 大数据里"穿黑色衬衫的人"和"喝咖啡的人"是否相关？

### 能用什么数据 / 工具

- 观察数据
- 描述统计、相关分析、回归、监督学习、几乎所有"经典"机器学习

### 致命限制

**所有第 1 层的发现都是相关，不是因果**。模型预测得再好，也只能说"看到 X 时大概率出现 Y"——不能说"加大 X 会让 Y 增加"。

经典翻车案例：

> 早期的"鞋码越大、阅读能力越强"——大数据里这是真相关，但显然不是因果。混淆变量是年龄。

> Netflix 早期发现"看 House of Cards 的用户留存最高"——并不是说每个新用户都该被推 House of Cards。

### 业务里的"第 1 层冒充第 2 层"

```
错误（很常见）：
  "数据显示用了我们 X 功能的用户留存率高 30%，所以我们要让更多人用 X！"

为什么错：
  P(留存 | 用了X) = 80%   ←  这只是相关
  P(留存 | do(用X))       ←  这才是因果，且通常 ≠ 上面的值
  
什么导致差异：
  - 用 X 的人本来就是高活用户（混淆）
  - 强制让低活用户用 X 不会有同样效果
```

## 四、第 2 层：干预（Intervention）

### 在问什么

> "如果我**主动**让 X 发生，Y 会变成什么样？"

形式化：`P(Y | do(X))`

`do(·)` 算子是 Pearl 引入的关键符号——它代表**外部强制**让 X 取某个值，**切断 X 原本的所有"成因"**，模拟一次随机干预。

### 关联 vs 干预的本质差异

```
P(Y | X = x)        ← 看到 X = x 的那部分人里，Y 的分布
                       这部分人的"被选择"机制可能影响 Y

P(Y | do(X = x))    ← 强制让所有人 X = x，然后看 Y 的分布
                       切掉了选择机制
```

```
        混淆变量 U
       ╱        ╲
      ▼          ▼
      X ───────► Y       ← 自然观察：U 既影响 X 也影响 Y
                            P(Y|X) 包含 U 的"间接贡献"

      U
       ╲
        ▼
      X ────╳───► Y      ← do(X)：手术刀切断 U → X 这条线
            ↑               P(Y|do(X)) 只剩 X 的"纯因果"
        do(X=x)
```

### 例子

- 如果对所有人投广告，整体购买率会变成多少？`P(buy | do(ad=1))`
- 如果把所有用户的推荐内容换成新算法，留存会涨多少？
- 如果把最低工资从 8 元/h 提到 12 元/h，失业率会怎么变？

### 能用什么数据 / 工具

- **AB 测试**：do(X) 的工程实现，黄金标准
- **观察数据 + 因果识别**：当你能列出所有混淆并满足 Ignorability，可以用 PSM、IPW、DML 等估出 P(Y|do(X))
- **自然实验**：IV、RDD、DID、合成控制——借助外生扰动实现"伪 do"

### 业务里第 2 层的核心结论

**ATE / ATT / ATC / CATE 全都是第 2 层的量**。它们都在回答"如果干预，会怎样"，而不是"看到时是怎样"。

→ 详见 [potential-outcome.md](potential-outcome.md)

## 五、第 3 层：反事实（Counterfactual）

### 在问什么

> "**对这个具体的个体**，如果当时（事实是 X=x）做的是 X=x'，Y 会是什么？"

形式化：`P(Y_{x'} | X=x, Y=y)`

念一下："给定我们观察到 X=x 且 Y=y，反事实地把 X 改成 x' 后 Y 的分布是多少。"

### 例子

- 这个用户买了。**如果当时没给他推送，他还会买吗？**（增量归因）
- 这个学生考了 80 分。**如果他当时没参加补习班，会考多少？**
- 这名病人吃了药、康复了。**如果不吃药，他会康复吗？**
- A 公司涨价后销量没掉。**如果没涨价，销量会更高吗？**

注意：这些都是**针对**已经发生事实的个体，问"如果"的问题。

### 第 2 层 vs 第 3 层的区别

```
第 2 层（干预 / 群体）：
  "如果给所有用户都推送，平均会多产生多少购买？"
  → 整体策略的 ROI

第 3 层（反事实 / 个体）：
  "对刚才买了的这个用户，如果没推送，他还会买吗？"
  → 个体归因：这次推送是不是必要的
```

第 3 层是**第 2 层的个体化版本**。每个个体的反事实 Y_x(u) 在原潜在结果框架里就是 Y_i(x)。

### 能用什么数据 / 工具

要回答第 3 层，需要**结构因果模型（SCM）**——不只画 DAG，还要给每条边赋予函数关系：

```
Y = f_Y(X, U_Y)
X = f_X(U_X)
```

有了完整 SCM，就能：
1. 看到事实数据 → 倒推外生噪声 U（ABDuction）
2. 切掉边、强制 X = x'（Action）
3. 重算 Y（Prediction）

这就是 Pearl 三步法（Abduction-Action-Prediction）。

### 现实中的第 3 层方法

- **Uplift Modeling**：估 CATE 是个"折中第 3 层"——不是完全的反事实，但已经能回答"对**像这个人这样**的子群，干预会带来多少差异"。
- **结构方程模型（SEM）**：经济学传统方法，假设强但能回答反事实。
- **反事实推理在 LLM 上**："如果换一个 prompt，这个回答会变成什么"——这是一个还在快速演进的研究方向。

### 业务里的第 3 层

最直接的应用是**归因**：
- 这次成单 200 元，多少要归功于广告？多少归功于优惠券？多少是用户本来就要买？
- 真实的归因永远是反事实问题，不是"漏斗中谁是最后一步"那种简单切分。

## 六、阶梯总图：能力 vs 要求

| 层级 | 在问什么 | 数据要求 | 假设强度 | 典型方法 | 业务用途 |
|---|---|---|---|---|---|
| **1. 关联** | 看到 X，Y 怎么样 | 观察数据 | 几乎没有 | 回归、ML、监督学习 | 预测、特征发现 |
| **2. 干预** | 强制 X，Y 怎么变 | 实验 / 观察+识别 | 中等 | AB 测试、PSM、DML、IV、DID | 策略评估、ROI |
| **3. 反事实** | 如果当时不一样会怎样 | 观察数据 + 完整结构模型 | 强 | SCM、Uplift、SEM | 个体归因、政策模拟 |

## 七、几个常见误解

### 1. "我做了 RCT，是不是直接到第 3 层了？"

不是。RCT 给你**第 2 层的 ATE**——能回答"如果给所有人都干预"。要到第 3 层（个体反事实），还需要假设结构和异质性模型。

### 2. "我加了控制变量做回归，是不是就到第 2 层了？"

**不一定**。回归是第 1 层的工具，要把它变成第 2 层结论，必须额外满足 Ignorability + Positivity 等假设。控制变量错了（控制了 collider）反而把第 1 层结论搞得更糟。

### 3. "AB 测试既然是金标准，为啥还要因果推断？"

- 很多场景做不了实验：政策、风控、宏观、伦理、不可逆。
- 实验也只到第 2 层；第 3 层（反事实归因、个体策略）实验本身解决不了。
- 甚至实验设计本身（要测什么、控制什么）也需要因果思维。

### 4. "有大数据就能爬到第 3 层吗？"

**不能**。第 3 层的关键不在数据量，在**结构假设**。10TB 数据没有正确的因果图，连第 1 层都未必爬得稳。

## 八、把阶梯用在日常工作上

下次任何分析、报告、产品决策，先问自己：

```
我此刻在哪一层？
  - 我在做"特征发现 / 找规律"？  → 第 1 层 OK
  - 我在做"决定要不要上线 / 投不投钱"？ → 必须第 2 层
  - 我在做"个体归因 / 个性化干预"？ → 第 3 层

我用的方法和数据，能撑到这一层吗？
```

我见过太多团队拿着第 1 层的相关性，做着第 2 层的决策，写出第 3 层的"个体推荐理由"。这不是数据不行，是**对自己在哪一层没有感知**。
