## 二、核心指标口径
> 指标口径以数鲸/都江堰官方口径为准，临时取数必须和官方口径对齐
| 指标 | 定义 | 口径说明 | 当前水位 |
|------|------|----------|----------|
| **有效DAU** | 日活跃用户数 | 信息流核心指标，基于有效曝光/消费行为去重统计，双端稳定在 **2500万** 量级 | 2500万 |
| **消费时长** | 用户内容消费总时长 | 核心指标，浏览器人均约 **18分钟**，内容中心人均约 **14分钟**；注意：沉浸态图文二期曾有埋点bug导致重复计算 | 浏览器18min/人，内容中心14min/人 |
| **PUSH调起DAU** | Push消息唤起的日活 | Push核心指标，7/12数据：505.15万，周同比+24.82% | 505万（周同比+24.8%） |
| **PUSH渗透率** | Push调起DAU / 总DAU | 7/12数据：15.83%，周同比+20.30% | 15.8%（周同比+20.3%） |
| **CTR** | Click-Through Rate 点击率 | 内容点击次数 / 曝光次数 | - |
| **ARPU** | Average Revenue Per User 每用户平均收入 | 收入核心指标 | - |
| **TGI** | Target Group Index 目标群体指数 | 反映目标群体在特定研究范围(如地理区域、人口统计领域、媒体受众、产品消费者)内的强势或弱势，注意：存在系统偏差的实验（如Push）禁止直接用TGI对比 | - |
| **UV** | Unique Visitor 独立访客 | 去重用户数 | - |
| **PV** | Page View 页面浏览量 | 页面访问次数，不去重 | - |
| **留存率** | 新增用户在后续一段时间内再次启动的比例 | 常用次留、7留、30留 | - |
| **转化率** | 完成目标行为的用户数 / 总曝光/访问用户数 | 漏斗分析核心指标 | - |

## 三、分析方法与术语
| 术语 | 定义 | 适用场景/强制规则 |
|------|------|------------------|
| **DID / 双重差分法** | Differences-in-Differences | **强制规则**：Push类、存在组间干扰/系统偏差的AB实验必须使用DID分析，禁止直接对比实验组对照组TGI<br>公式：政策效应=（政策后实验组-政策前实验组）-（政策后对照组-政策前对照组）<br>核心假设：平行趋势假设 |
| **DIKW模型** | Data → Information → Knowledge → Wisdom | 数据分析层级：事实数据→是什么信息→为什么知识→怎么办智慧 |
| **AB实验** | 对照实验 | 核心科学方法，评估策略效果 |
| **CUPED** | Controlled-experiment Using Pre-Experiment Data | AB实验方差缩减方法，提升实验灵敏度 |
| **mSPRT** | 混合序贯概率比检验 | 序列检测方法，支持实验提前结束 |
| **漏斗分析** | 用户转化路径分析 | 6步流程：明确目标→设计路径→采集数据→搭建计算→分析瓶颈→优化验证 |
| **平行趋势** | DID核心假设 | 政策实施前，实验组和对照组的趋势必须平行，否则DID结果无效 |
| **PEEKING** | 偷看实验结果 | AB实验陷阱：实验未达到样本量提前看结果下结论，会导致假阳性率飙升 |
| **Few-shot / 范式库** | 少样本学习 | nl-sql核心路线，沉淀真实业务SQL案例作为样例，比复杂RAG效果更准 |
| **RAG** | Retrieval-Augmented Generation 检索增强生成 | nl-sql明确不做复杂RAG，优先范式库覆盖高频场景 |

## 五、业务规则与红线
| 规则 | 内容 |
|------|------|
| **人群包规则** | 实验平台人群包功能存在重大稳定性风险，可能导致服务不可用，禁止新增配置，存量必须尽快推全/下线/迁移到通用后台 |
| **产品评审规则** | 所有涉及信息流产品形态/消费路径改动的方案，终审必须邀请算法同学参与评审，评估对数据埋点和算法特征的影响，否则禁止上线 |
| **SQL执行规则** | kyuubi_query超时配置必须≥300s，大数据量查询必须分批提交，数据工场写入必须用户自行建表+分段提交，禁止直接insert生产表 |
| **优先级规则** | 严格遵循Q3 P0/P1/P2划分，P2和明确不做清单内任务禁止投入精力，资源冲突优先保P0 |
| **nl-sql路线规则** | nl-sql优先用真实业务需求沉淀few-shot范式库，不做复杂RAG，优先覆盖80%高频场景 |
