# 用户故事拆解 Prompt 模板

> 来源：pm-workspace-starter prompts/user-story.md
> 沉淀日期：2026-06-29

## 使用方式

复制以下 Prompt，替换 `{ }` 中的内容，发给 AI 即可。

---

## Prompt

> 执行前先读 USER.md 和 MEMORY.md，已有信息（产品名、用户群、当前项目）直接引用，不要重复问用户。只问缺失的信息。

我是一名产品经理，请帮我把以下功能拆解为用户故事（User Story）。

### 功能描述

{功能名称和一句话描述}

### 目标用户

{用户角色，例如：新用户、活跃用户、付费用户}

### 用户场景

{用户在什么场景下使用这个功能？想达成什么目标？}

### 约束条件

- 平台：{iOS / Android / Web / 全平台}
- 依赖：{需要哪些上游能力}

### 输出要求

1. 按「作为 {角色}，我想要 {功能}，以便 {价值}」格式写用户故事
2. 每个用户故事包含验收标准（AC）
3. 覆盖主流程 + 异常流程 + 边界条件
4. 标注优先级（P0 / P1 / P2）
5. 如果用户故事之间有依赖关系，请标明
