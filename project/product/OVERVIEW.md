# 产品方法论全景

> 作为数据产品经理，我需要掌握的核心方法论体系。

## 一、产品经理的三个核心能力

| 能力 | 说明 | 对应知识 |
|---|---|---|
| **判断力** | 判断做什么、不做什么 | 需求分析、优先级排序、决策框架 |
| **执行力** | 把决定落地 | PRD、项目管理、跨团队协调 |
| **洞察力** | 从数据和用户中发现机会 | 数据分析、用户研究、竞品分析 |

## 二、产品生命周期与对应方法

| 阶段 | 核心问题 | 方法/工具 |
|---|---|---|
| 发现机会 | 做什么？ | 用户访谈、竞品分析、市场分析 |
| 验证假设 | 值得做吗？ | MVP、AB 测试、数据验证 |
| 设计方案 | 怎么做？ | PRD、原型、技术方案评审 |
| 迭代优化 | 做得好吗？ | 数据分析、漏斗分析、留存分析 |
| 规模化 | 怎么做大？ | 增长模型、渠道策略、平台化 |

## 三、数据产品的特殊性

数据产品（如 ICE Workbench、nl-sql）与 C 端产品的差异：

| 维度 | C 端产品 | 数据产品 |
|---|---|---|
| 用户 | 海量终端用户 | 内部分析师/产品经理/运营 |
| 价值衡量 | DAU/转化/收入 | 提效（节省的人时）/ 准确率 / 覆盖率 |
| 需求来源 | 用户行为数据 | 内部工作流痛点 |
| 迭代节奏 | A/B 测试驱动 | 需求驱动 + 技术突破驱动 |
| 竞品 | 市场上的竞争产品 | 现有工作方式（手动 SQL / Excel） |

## 四、AI Native 产品设计要点

在 AI 时代做产品需要新的思维：

1. **交互范式变化**：从 GUI 到对话式，从用户操作到意图表达
2. **容错设计**：AI 输出不确定，需要设计验证/修正/反馈闭环
3. **渐进信任**：用户从不信任到信任 AI 需要过程，产品要提供可解释性
4. **数据飞轮**：用户使用 → 积累数据 → 模型更好 → 更多使用
