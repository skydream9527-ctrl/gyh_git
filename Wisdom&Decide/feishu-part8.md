## 📊 数据分析方法手册

---

### 核心心智模型
数据分析的本质是**用数据回答业务问题**，不是炫技：
```
业务问题 → 拆解问题 → 选择方法 → 取数计算 → 业务解读 → 决策建议
```
不要拿着锤子找钉子，方法是工具，不是目的。

---

### 一、用户行为分析类

#### 1. 漏斗分析（Funnel Analysis）
**适用场景**：核心转化流程优化、流失环节诊断、版本效果评估

**核心指标**：步骤转化率、整体转化率、流失率、转化时长、步骤耗时

**分析流程**：
1. 明确目标和步骤（3-7步，主动行为，严格有序）
2. 确定统计口径（时间窗口、去重、归因规则）
3. 计算可视化漏斗，定位问题环节
4. 多维度细分（新老/渠道/版本/平台）
5. 诊断流失原因（路径/分群/可用性测试）
6. 优化建议+AB验证

**关键原则**：
- 优先优化**绝对流失人数最多**的环节，不是转化率最低的
- 转化率+停留时长一起看，耗时长说明有摩擦
- 不要把被动曝光算进步骤，会虚高转化率

**典型漏斗**：商品详情→加购→提交订单→支付成功

---

#### 2. 路径分析（Path Analysis）
**适用场景**：用户行为轨迹还原、导航优化、异常路径发现、流失去向分析、探索真实转化路径

**和漏斗区别**：漏斗预设固定步骤，路径不预设，挖掘真实行为流向。

**核心模式发现**：
- 预期路径vs实际路径差异：用户是不是不走你设计的"官方路径"？
- 高流失节点：哪些页面退出率超过70%？
- 循环打转：用户在两个页面来回跳，说明信息歧义
- 意外捷径：用户走了没想到的高效路径，要强化
- 黑洞节点：进去就再也不出来的页面
- 异常路径：违反设计预期的奇怪跳转

**可视化**：桑基图（Sankey Diagram）

---

#### 3. 留存分析（Retention Analysis）
**核心价值**：留存是产品生命线，是PMF的核心验证指标。拉新再多留存低也是漏水的桶。

**核心指标**：
- 次日留存(D1)：新手体验、首次Aha时刻
- 7日留存(D7)：习惯养成
- 30日留存(D30)：长期价值、PMF验证

**留存曲线解读**：
- 前几天快速下滑正常，7-14天应该进入平稳平台期
- 平台期高度：工具类20%+合格、内容类25%+合格、电商10%+合格
- 如果持续下滑没有平台期，说明产品没找到PMF

**诊断逻辑**：
- D1低 → 新用户首访体验差/新手引导问题/流量不精准
- D7低 → 没有养成使用习惯，缺回来的钩子
- D30低 → 长期价值不足，没有解决核心需求

**魔法数字**：对比留存vs流失用户首周行为，找到"完成某个行为的用户留存率大幅提升"的阈值，引导新用户尽快完成（如Facebook"10天加7个好友"）。

### 二、用户价值分析类

#### RFM分析
经典用户价值分层模型，三个维度：
- **R(Recency)**：最近一次消费/活跃距今天数 → 最近有没有来
- **F(Frequency)**：周期内消费/活跃次数 → 来的勤不勤
- **M(Monetary)**：周期内累计消费/价值 → 创造多少价值

**经典8层用户分群**：
| 用户类型 | R | F | M | 运营策略 |
|----------|---|---|---|----------|
| 重要价值用户 | 高 | 高 | 高 | VIP专属服务，重点维护 |
| 重要发展用户 | 高 | 低 | 高 | 提升频次，交叉销售 |
| 重要保持用户 | 低 | 高 | 高 | 高优先级召回 |
| 重要挽留用户 | 低 | 低 | 高 | 最高优先级召回，调研原因 |
| 一般价值用户 | 高 | 高 | 低 | 提升ARPU，引导付费 |
| 一般发展用户 | 高 | 低 | 低 | 新手引导，激活转化 |
| 一般保持用户 | 低 | 高 | 低 | 低成本召回 |
| 一般挽留用户 | 低 | 低 | 低 | 减少投入，低成本触达 |

**原则**：M高的都是重要用户，资源向高价值用户倾斜，不要在低价值用户身上浪费钱。

### 三、对比归因类

#### 对比分析
**没有对比就没有结论**，是最基础最常用的分析方法。

**5种对比角度**：
1. **时间对比**：环比（和上周期比，看短期）、同比（和去年比，除季节性）
2. **横向对比**：新老用户、渠道、平台、版本、品类
3. **纵向对比**：全链路各环节转化对比
4. **目标对比**：和KPI/预测值比
5. **外部对比**：行业基准、竞品

**指标异动诊断流程**：
```
确认真实性（先排除数据问题！）→ 确认幅度范围 → 维度拆解下钻 → 关联事件匹配 → 验证因果 → 给出结论建议
```

**四大原则**：
1. 口径一致再对比，苹果比苹果
2. 一次对比只变一个变量（控制变量）
3. 相对数+绝对数一起看，不要只看比例
4. 小样本看统计显著性，不要把波动当结论

---

#### 维度下钻分析
核心逻辑：**整体异常一定是某个局部异常导致的，按维度一层一层拆，找到贡献最大的问题点。**

**核心拆解维度**：用户（新老/渠道/地域）、产品（端/版本/功能）、业务（品类/业务线）、时间（小时级）

**贡献度计算**：
```
指标变化贡献度 = (该部分本期值 - 该部分基期值) / 整体变化量
```
优先看贡献度最大的部分，不要盲目拆所有维度。

### 四、分析常见陷阱

| 陷阱 | 表现 | 规避方法 |
|------|------|----------|
| 相关当因果 | 两个指标一起涨就认为A导致B | 混淆变量排查、因果推断、AB验证 |
| 幸存者偏差 | 只看成功/留存用户特征 | 全量用户对比，分析流失样本 |
| 辛普森悖论 | 总体和分组结论相反 | 必须分层下钻看 |
| 口径不一致 | 同一个指标不同人算出来不一样 | 统一指标字典，文档化口径 |
| 忽略基数 | 只看转化率不看流量规模 | 绝对数+相对数一起看 |
| 过度拟合 | 过度解读小样本波动 | 显著性校验，样本足够再下结论 |
| 为解释而解释 | 强行给正常波动找原因 | 先判断是否在正常波动区间 |

---

### 心法
1. **业务第一，数据第二**：不懂业务的分析师只是取数工具
2. **结论先行**：先说"是什么、为什么、怎么办"，再放过程
3. **好分析能落地**：不能指导决策的分析是自嗨
4. **质疑数据**：拿到数据先校验，错数据比没数据更可怕
5. **保持简单**：能用简单方法说清楚的，不要搞复杂模型
6. **讲好故事**：数据是论据，逻辑是骨架，业务价值是结论
