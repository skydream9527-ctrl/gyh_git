# 洞见与思考沉淀

存放不定期的思考、洞见、复盘总结。日常心得的默认落点。

## 命名约定

- 格式：`YYYY-MM-DD-主题.md`
- 示例：`2026-06-30-AI-agent时代交互方式变化思考.md`
- 日期用实际思考日期；主题简短中文，不加空格
- 一文一主题，写完即归档，不反复回改（新认知另起一篇并互相链接）

## 写作建议

- 先结论后细节，开篇一句话给核心判断
- 对比、分析尽量用表格
- 引用数据标注来源和时间，无数据标「推断」
- 想清楚的写结论，没想清楚的写问题与待验证假设
