# Wisdom&Decide — 我的智慧与决策中心

> 这是个人决策的大脑，所有重要判断、决策记录、方法论沉淀都在这里。

---

## 📂 目录结构

```
Wisdom&Decide/
├── README.md                    # 本文件（索引）
├── 00-我是谁与我的决策地图.md     # 个人定位 + 决策全景分类
├── 01-我的项目全景导航.md         # 整个工作区的内容地图和索引
├── 02-决策工作流SOP.md           # 各类决策的标准流程和模板
├── decisions/                   # 决策记录归档（按月存放）
│   └── README.md
└── insights/                    # 洞见和思考沉淀
    └── README.md
```

---

## 🧭 怎么用这个目录？

### 1. 做决策前
- 先看 `00-我是谁与我的决策地图.md`，确认这是什么类型的决策
- 打开 `02-决策工作流SOP.md`，走对应的决策流程
- 去 `project/decision-frameworks/tools/` 找合适的决策工具

### 2. 做完决策后
- 在 `decisions/YYYY-MM/` 下按模板记录决策
- 踩坑了或有新认知，更新到 00/01/02 文档

### 3. 找不到东西时
- 打开 `01-我的项目全景导航.md`，快速定位文件位置

## ⚡ 核心决策原则（反复提醒自己）

1. **AI 做执行，我做判断** —— 把重复劳动交给 AI，把时间留给思考和决策
2. **先结论后细节** —— 任何输出先给结论
3. **表格优于文字** —— 对比、分析尽量用表格
4. **异常必想** —— 方案必须覆盖边界条件和异常流程
5. **数据说话** —— 引用数据标注来源，无数据标「推断」
6. **记录决策** —— 重要决策必须落地成文件，不做脑子记
7. **完成比完美重要** —— 70分就行动，快速迭代
8. **定期复盘** —— 每周/季度回顾决策，更新认知
