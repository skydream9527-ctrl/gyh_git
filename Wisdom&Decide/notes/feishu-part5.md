### 1. SWOT 分析框架

> SWOT 是一种战略规划工具，用于识别优势、劣势、机会和威胁。

#### 什么是SWOT？

SWOT是四个因素的首字母缩写：

- **S**trengths（优势）：内部积极因素，你可以控制的
- **W**eaknesses（劣势）：内部消极因素，你可以改善的
- **O**pportunities（机会）：外部积极因素，你可以利用的
- **T**hreats（威胁）：外部消极因素，你无法控制但可以应对的

#### SWOT四象限

```
┌─────────────────────────────┬─────────────────────────────┐
│       内部因素               │       外部因素               │
├─────────────────────────────┼─────────────────────────────┤
│  S 优势 (Strengths)         │  O 机会 (Opportunities)     │
│  - 你做得好的地方            │  - 外部环境中的有利因素      │
│  - 独特的资源/能力           │  - 可以抓住的趋势            │
│  - 竞争优势                 │  - 未被满足的需求            │
├─────────────────────────────┼─────────────────────────────┤
│  W 劣势 (Weaknesses)        │  T 威胁 (Threats)           │
│  - 需要改进的地方            │  - 外部环境中的不利因素      │
│  - 资源/能力不足             │  - 竞争对手的行动            │
│  - 竞争劣势                 │  - 政策/市场变化             │
└─────────────────────────────┴─────────────────────────────┘
```

#### 使用步骤

**第1步：分析内部因素（S+W）**

| 维度 | 需要问的问题 |
|------|-------------|
| 资源 | 我有什么独特资源？资金？人才？技术？ |
| 能力 | 我擅长什么？核心竞争力是什么？ |
| 经验 | 过去哪些事做成了？哪些失败了？ |
| 文化 | 团队文化优势/劣势？ |

**第2步：分析外部因素（O+T）**

| 维度 | 需要问的问题 |
|------|-------------|
| 市场 | 市场趋势如何？有什么新机会？ |
| 竞争 | 竞争对手在做什么？ |
| 政策 | 有什么政策利好/风险？ |
| 技术 | 有什么新技术可以利用？ |

**第3步：制定策略**

| | 优势（S） | 劣势（W） |
|--|----------|----------|
| **机会（O）** | **SO策略**：利用优势抓住机会 | **WO策略**：克服劣势利用机会 |
| **威胁（T）** | **ST策略**：利用优势应对威胁 | **WT策略**：减少劣势规避威胁 |

#### 实战示例：个人职业SWOT

| | 内容 |
|--|------|
| **S 优势** | AI产品经验丰富、数据敏感度高、懂技术、学习能力强 |
| **W 劣势** | 管理经验不足、公开演讲能力待提升、行业人脉有限 |
| **O 机会** | AI行业高速增长、公司重视数据产品、内部转岗机会 |
| **T 威胁** | 技术更新快需要持续学习、外部竞争激烈、岗位要求提高 |

**策略**：
- SO：利用AI产品经验，抓住公司数据产品机会，牵头重点项目
- WO：通过项目积累管理经验，参加演讲培训，主动拓展人脉
- ST：持续学习保持技术敏感度，用数据能力建立竞争壁垒
- WT：聚焦核心领域，避免分散精力，建立个人品牌

### 2. 波特五力模型

> 波特五力模型用于分析行业竞争态势，帮助理解行业的盈利潜力和竞争强度。

#### 五力详解

```
            潜在进入者威胁
                │
                ▼
┌─────────────────────────────┐
│  供应商议价能力 ──── 现有竞争者 ──── 购买者议价能力  │
│    (上游)        竞争强度        (下游)     │
└─────────────────────────────┘
                │
                ▼
            替代品威胁
```

##### 力1：现有竞争者竞争强度

**决定因素**：
- 竞争者数量和规模
- 行业增长率
- 产品差异化程度
- 退出壁垒
- 固定成本高低

**评分标准**：高/中/低
- 高：众多竞争者、增长慢、同质化、退出难
- 低：少数竞争者、增长快、差异化大、退出易

##### 力2：潜在进入者威胁

**进入壁垒**：
- 规模经济
- 品牌忠诚度
- 资本要求
- 政府政策/牌照
- 技术专利
- 渠道控制
- 学习曲线效应

**评分标准**：高/中/低
- 高壁垒 = 低威胁（好事）
- 低壁垒 = 高威胁（坏事）

##### 力3：替代品威胁

**分析维度**：
- 替代品的性价比
- 买方转换成本
- 买方对替代品的接受度
- 技术迭代速度

**示例**：
- 传统出租车的替代品：网约车、共享单车、公共交通
- 线下零售的替代品：电商、直播带货、社区团购

##### 力4：供应商议价能力

**供应商强势的信号**：
- 供应商集中度高（少数几家垄断）
- 没有替代品
- 对供应商来说你不是重要客户
- 供应商产品对你很关键
- 转换成本高
- 供应商可能前向整合（自己做你的业务）

##### 力5：购买者议价能力

**购买者强势的信号**：
- 购买者集中度高（少数大客户）
- 产品标准化/同质化
- 转换成本低
- 购买者可能后向整合（自己做你的业务）
- 价格敏感度高
- 购买者掌握充分信息

#### 使用步骤

**第1步：定义行业边界**
- 明确分析哪个行业/细分市场
- 确定地理范围
- 明确时间范围

**第2步：逐一分析五力**
- 为每一种力打分（高/中/低）
- 列出关键证据
- 识别关键驱动因素

**第3步：评估整体行业吸引力**
- 五力都强 → 行业吸引力低（盈利难）
- 五力都弱 → 行业吸引力高（盈利易）

**第4步：制定竞争策略**
- 定位：找到五力都弱的细分市场
- 影响：通过策略改变五力平衡
- 利用：预见变化并提前布局

#### 实战示例：AI大模型行业五力分析

| 力量 | 强度 | 理由 |
|------|------|------|
| 现有竞争 | 高 | 国内外众多玩家，价格战激烈，产品趋同 |
| 进入威胁 | 中高 | 开源模型降低门槛，但算力和人才壁垒高 |
| 替代品威胁 | 中 | 传统软件和SaaS是替代，但AI赋能是趋势 |
| 供应商议价 | 高 | GPU供应垄断（英伟达），算力成本高 |
| 购买者议价 | 中高 | 客户选择性多，对价格敏感，转换成本降低 |

**结论**：行业竞争激烈，但增长快。策略建议：
1. 垂直细分：聚焦特定行业/场景做深做透
2. 生态构建：通过应用和数据建立壁垒
3. 成本控制：优化模型推理成本，建立算力优势

### 3. PESTEL分析框架

> PESTEL是一种宏观环境分析工具，用于评估影响组织或项目的六大外部因素。

#### 什么是PESTEL？

PESTEL是六个因素的首字母缩写：

- **P**olitical（政治因素）
- **E**conomic（经济因素）
- **S**ocial（社会因素）
- **T**echnological（技术因素）
- **E**nvironmental（环境因素）
- **L**egal（法律因素）

#### 六大因素详解

##### 1. 政治因素 (Political)

**关键因素**：政府政策、政治稳定性、贸易政策、税收政策、劳工政策、外交关系

**分析清单**：
- [ ] 政府对行业的态度如何？
- [ ] 有哪些产业扶持政策？
- [ ] 税收政策如何？
- [ ] 贸易政策有何影响？

##### 2. 经济因素 (Economic)

**关键因素**：GDP增长率、利率、汇率、通胀率、失业率、消费者信心、可支配收入

**分析清单**：
- [ ] 经济增长趋势如何？
- [ ] 利率和汇率走势？
- [ ] 通胀水平如何？
- [ ] 消费者信心如何？

##### 3. 社会因素 (Social)

**关键因素**：人口结构、文化趋势、生活方式、教育水平、健康意识、社交媒体

**分析清单**：
- [ ] 人口结构变化趋势？
- [ ] 消费观念有何变化？
- [ ] 生活方式有何趋势？

##### 4. 技术因素 (Technological)

**关键因素**：技术创新、研发投入、自动化、数字化转型、技术标准、知识产权

**分析清单**：
- [ ] 有哪些关键技术趋势？
- [ ] 技术创新速度如何？
- [ ] 自动化对行业的影响？

##### 5. 环境因素 (Environmental)

**关键因素**：环保法规、气候变化、可持续发展、资源稀缺、能源转型

**分析清单**：
- [ ] 环保法规有何要求？
- [ ] 气候变化有何影响？
- [ ] 可持续发展趋势？

##### 6. 法律因素 (Legal)

**关键因素**：行业法规、劳动法、消费者保护法、竞争法、数据保护法、知识产权法

**分析清单**：
- [ ] 行业有哪些法规要求？
- [ ] 数据保护要求？
- [ ] 知识产权保护？

#### 使用步骤

**第1步：明确分析范围**：国家/地区、行业、时间范围

**第2步：逐一分析六大因素**，评估每个因素的影响方向（正面/负面）、影响程度（高/中/低）

**第3步：识别机会与威胁**

**第4步：制定应对策略**

#### PESTEL与其他工具的结合

- **PESTEL + SWOT**：先做PESTEL识别宏观机会威胁，纳入SWOT外部因素
- **PESTEL + 波特五力**：先了解宏观环境，再分析对行业竞争的影响
- **PESTEL + 情景规划**：基于关键不确定因素构建不同情景

### 4. 决策矩阵 (Decision Matrix)

> 决策矩阵是一种结构化的决策工具，帮助你在多个选项中做出客观、可追溯的选择。

#### 基本结构

```
┌─────────────────┬────────┬────────┬────────┬────────┐
│ 评估维度         │ 权重   │ 方案A  │ 方案B  │ 方案C  │
├─────────────────┼────────┼────────┼────────┼────────┤
│ 维度1            │ 30%    │ 评分   │ 评分   │ 评分   │
│ 维度2            │ 25%    │ 评分   │ 评分   │ 评分   │
│ 维度3            │ 20%    │ 评分   │ 评分   │ 评分   │
│ 维度4            │ 15%    │ 评分   │ 评分   │ 评分   │
│ 维度5            │ 10%    │ 评分   │ 评分   │ 评分   │
├─────────────────┼────────┼────────┼────────┼────────┤
│ 加权总分         │ 100%   │ 计算   │ 计算   │ 计算   │
└─────────────────┴────────┴────────┴────────┴────────┘
```

#### 使用步骤（8步法）

##### 第1步：明确决策问题
示例："我们应该选择哪个前端框架来重构现有系统？"

##### 第2步：确定评估维度（5-8个）

常见维度：
| 类别 | 示例维度 |
|------|----------|
| 技术 | 性能、可扩展性、安全性、兼容性 |
| 业务 | 成本、时间、风险、战略对齐 |
| 团队 | 学习曲线、团队技能、维护成本 |
| 生态 | 社区活跃度、文档质量 |

##### 第3步：确定权重（总和100%）

权重分配方法：
1. **直接分配法**：团队讨论直接分配（推荐快速决策）
2. **配对比较法**：两两比较哪个更重要，统计得分
3. **AHP层次分析法**：系统化方法，适合复杂决策

##### 第4步：制定评分标准（推荐5分制）

| 分数 | 标准 |
|------|------|
| 5 = 优秀 | 满足所有要求，表现突出 |
| 4 = 良好 | 满足大部分要求，表现良好 |
| 3 = 一般 | 基本满足要求，表现一般 |
| 2 = 较差 | 有明显不足，部分不满足 |
| 1 = 很差 | 无法满足要求 |

##### 第5步：收集信息并评分

信息来源：官方文档、基准测试、社区反馈、PoC验证、专家意见

评分原则：基于事实、多人独立评分取平均、记录评分依据

##### 第6步：计算加权总分

公式：`加权总分 = Σ(维度评分 × 维度权重)`

##### 第7步：分析结果+敏感性分析

- 总分排名
- 各方案优劣势分析
- 敏感性分析：调整权重/评分，结果是否变化？

##### 第8步：做出决策

考虑：
1. 总分差距是否足够大？
2. 是否有"必须满足"的红线？
3. 团队偏好和实际情况
4. 长期战略对齐

#### 高级技巧

1. **红线机制**：设置"必须满足"的最低标准，任何维度低于标准直接淘汰
2. **敏感性分析**：测试权重和评分变化对结果的影响
3. **情景分析**：不同情景下（乐观/悲观/基准）的最优选择
4. **团队校准**：多人评分时取平均或讨论达成共识

#### 实战案例：前端框架选型

| 维度 | 权重 | React | Vue | Angular |
|------|------|-------|-----|---------|
| 性能 | 25% | 4 | 4 | 3 |
| 生态成熟度 | 20% | 5 | 4 | 4 |
| 学习曲线 | 20% | 3 | 5 | 2 |
| 长期维护性 | 20% | 4 | 4 | 5 |
| 社区活跃度 | 15% | 5 | 4 | 3 |
| **加权总分** | 100% | **4.15** | **4.20** | **3.40** |

**决策**：选择Vue。学习曲线优势对团队更重要，总分差距不大但关键维度表现更好。

### 5. RICE优先级评分框架

> RICE是一种简单但有效的优先级评分方法，帮助团队客观比较不同功能或项目的优先级。

#### 什么是RICE？

- **R**each（覆盖范围）：影响多少用户？
- **I**mpact（影响程度）：对用户的影响有多大？
- **C**onfidence（信心指数）：你对估算有多大把握？
- **E**ffort（投入成本）：需要多少工作量？

**公式**：`RICE Score = (Reach × Impact × Confidence) / Effort`

#### 详细说明

##### Reach（覆盖范围）
- **定义**：特定时间段内受影响的用户数
- **测量**：使用历史数据、明确时间范围（如每季度）

##### Impact（影响程度）
- **评分标准（3分制）**：
  - 3 = 巨大影响（解决核心痛点）
  - 2 = 高影响（显著改善体验）
  - 1 = 中等影响
  - 0.5 = 低影响（锦上添花）
  - 0.25 = 微小影响

##### Confidence（信心指数）
- **评分标准（百分比）**：
  - 100% = 高信心（有数据支持）
  - 80% = 中等信心（有部分数据）
  - 50% = 低信心（主要是假设）
  - 20% = 很低信心（完全猜测）

##### Effort（投入成本）
- **单位**：人天或人周
- **包含**：设计、开发、测试、部署、沟通协调
- **建议**：预留20-30% buffer

#### 实战案例：SaaS产品功能优先级

| 功能 | Reach | Impact | Confidence | Effort | RICE Score | 优先级 |
|------|-------|--------|------------|--------|------------|--------|
| 自动化工作流 | 15,000 | 3 | 80% | 25天 | **1,440** | P0 |
| 实时协作编辑 | 8,000 | 3 | 80% | 30天 | 640 | P1 |
| 移动端App | 20,000 | 2 | 70% | 60天 | 467 | P2 |
| 高级报表 | 5,000 | 1 | 90% | 15天 | 300 | P3 |
| API开放平台 | 2,000 | 2 | 60% | 45天 | 53 | P4 |

**计算示例（自动化工作流）**：
```
RICE = (15,000 × 3 × 0.8) / 25 = 36,000 / 25 = 1,440
```

#### 常见陷阱与避免方法

| 陷阱 | 解决方法 |
|------|----------|
| Reach虚高 | 使用实际数据，区分"可能使用"和"会使用" |
| Impact主观 | 建立统一标准，多人评分取平均，用A/B测试验证 |
| Confidence忽视 | 强制为信心找依据，低信心先做验证 |
| Effort低估 | 包含所有环节，考虑技术债务，预留buffer |

#### 进阶用法

1. **加权RICE**：根据战略目标调整各因素权重
2. **分层RICE**：按用户分层（新/老、免费/付费）分别计算
3. **时间维度RICE**：乘以时间敏感度因子（竞品即将发布×1.5，可等待×0.7）
