项目概述

ICE-DATA-WORK 是一个面向团队的数据智能工作台。核心工作方式是「用户 → 数字分身（Twin） → 工具 Agent」的三层协作：用户把目标交给自己的 Twin，Twin 负责编排合适的专业 Agent 接力干活，任何高风险动作（写文件、跑命令、发消息、付费调用、固化记忆、跨空间读取）都会回到用户面前请求确认。

🧑‍🤝‍🧑 协作现场

任务看板上，人和 Agent 都是一等的 assignee。同一个任务里，用户、Twin、多个工具 Agent 可以在一条对话流里流式接力。

🔒 安全先于智能

三轴权限（平台角色 × 成员角色 × Twin 等级）+ 高风险动作必须确认 + 用户贡献代码一律沙盒执行 + 全程审计 + 一键暂停。

🧠 越用越懂你

记忆分区隔离（默认私有、显式共享）+ 技能自演进（跑通的代码沉淀为可复用 Skill）+ Twin 成熟度成长。

本页涵盖的三大部分

① 前端：全局导航与布局 ① 前端：19 个页面的按钮与操作框 ① 前端：常驻交互组件（Twin Dock / ⌘K） ② 后端：架构与文件优先存储 ② 后端：对话运行时与多 Agent 编排 ② 后端：安全与治理机制 ③ Agent：11+ 内置 Agent 能力 ③ Agent：18 个 Skill 资产 ③ Agent：平台扩展与外部集成
01

系统架构与技术栈

前后端分离；后端 FastAPI，前端 React 18 + Vite + TypeScript + Zustand；存储遵循 G3「文件优先」不变量——文件系统是唯一真相源，SQLite 与 pgvector 都是可重建的派生索引。

技术栈

后端FastAPI · Python · JWT / 米盾(Aegis) 双认证 · 统一错误信封
前端React 18 · Vite · TypeScript · Zustand · React Router
存储G3 文件优先（JSON / JSONL / .memory.md）+ SQLite 派生索引 + 可选 Postgres+pgvector 语义检索
实时WebSocket 流式对话（逐 token 推送 + 工具卡 + 转交卡）
模型mify 网关（OpenAI 兼容）/ Anthropic 直连 / 无 key 时 mock 流降级

四层空间（G3 文件优先）

任务空间tasks/{tid}/ — 对话、文件、计划、过程记忆（不跨任务）
Agent 空间agents/{aid}/ — 手册、技能、经验（定义团队共享，情景记忆按用户分区）
用户空间users/{uid}/ — Twin 画像、长期记忆（仅本人）
团队/项目teams/{tid}/… — 共享文件 / 知识 / 产物（按团队或项目隔离）

请求主链路

前端页面 / ⌘K→ API 客户端（Bearer Token）→ FastAPI 路由（三轴权限校验）→ Service 层→ G3 文件读写 + 审计
对话输入→ WebSocket /ws/tasks/{id}→ Agent 运行时（组装上下文 → LLM 流式 → 工具 → 落盘）→ 逐事件回推前端

生产环境后端可单端口伺服前端 SPA（frontend/dist），并注入安全响应头（X-Content-Type-Options / X-Frame-Options / HSTS / CSP），生产不回显原始异常。

02

核心理念：User → Twin → Agents

这是贯穿整个产品的协作模型，也是理解每个页面与后端逻辑的钥匙。

👤 User（你）

目标的提出者与最终决策者。高风险动作的确认权永远在你手上，Twin 可以请求，但不能替你拍板。

🧠 Wisdom Twin（分身）

你的 AI 化身，代表你在平台中与 Agent 协作。它编排任务、仲裁 Agent 之间的转交、按计划推进、主动提议下一步。权限分 L0–L5 六级。

🤖 Agents（工具 Agent）

专业分工的执行者（数据分析、实验、灰度、知识库…）。像同事一样被指派到任务，可在 Twin 仲裁下互相接力（A2A）。

💡 落到产品上：你在「新建任务」里说目标 → Twin 生成计划预览并推荐 Agent 组合 → 你确认权限/风险/成本后创建 → 在「工作区」里三方流式协作 → 高风险动作进「审批」队列 → 沉淀记忆与技能，下次更聪明。

06

后端运行逻辑

后端 FastAPI，分层清晰：api/v1 路由（三轴权限校验）→ services/*_svc 服务层（业务逻辑）→ core/storage（G3 文件读写 + 原子事务）。所有文件 IO 走服务层，不在路由层直接读写。共 40+ 服务模块、23 个路由域。

G3「文件优先」不变量

  • 文件系统是唯一真相源；SQLite 与 pgvector 都是可重建派生索引
  • 启动时若索引缺失或与文件不一致，自动重建（lifespan 自愈）
  • 列表查询走索引加速候选定位，详情一律回文件读
  • 多文件写入走 file_transaction 事务（备份 + 回滚）；追加型用 .jsonl
  • 一切路径走 core/storage/paths.py 的 safe_child 防穿越，禁止内联拼接

统一约定

  • 错误信封：统一 {code, message, error_code, data}；服务层 raise APIError
  • 外部 CLI 降级：kyuubi / feishu 缺失返回 *_NOT_CONFIGURED，不阻塞启动
  • 治理：高风险动作必确认 + 审计；新机制用 IDW_*_ENABLED 灰度、默认关
  • 生产基线：拒绝默认密钥/演示账号、安全响应头、异常不回显、WS 短票据

四层空间与派生索引

tasks/{tid}agents/{aid}users/{uid}teams/{tid}/… ⟶ 派生 ⟶ .cache/index.db（SQLite）pgvector（可选语义检索）

派生索引仅加速列表/搜索候选定位，可随时重建（make reindex 或 admin「重建索引」）；语义检索 IDW_PGVECTOR_ENABLED 默认关，未开启走关键词检索。

6.5 · 代码沙盒 services/sandbox.py

用户贡献的代码一律视为不可信，先进沙盒执行。总开关 IDW_SELF_EVOLVE_ENABLED，未启用返回 SANDBOX_NOT_ENABLED（不阻塞启动）。

约束实现
无网络preamble monkeypatch socket 断网（始终注入，纵深防御）+ Linux network namespace（unshare -r -n）内核级真断网
netns 策略IDW_SANDBOX_NETNS：auto（可用则真断网否则回退）/ on（不可用则失败关闭 SANDBOX_NETNS_UNAVAILABLE，绝不降级）/ off（仅 monkeypatch）
限资源RLIMIT_CPU（默认 10s）/ RLIMIT_FSIZE（默认 50MB）/（可选）RLIMIT_AS 内存上限
无凭证清空环境变量，仅保留 PATH/LANG 等白名单，绝不传 *_KEY/TOKEN/SECRET
隔离目录cwd / HOME / TMPDIR 指向任务沙盒目录，禁止访问任务外
墙钟超时默认 15s，超时整组 kill（setsid 新进程组）；输出截断 20000 字符
诚实边界：进程级沙盒在 macOS/通用 POSIX 上无内核级隔离；Linux 生产开 netns 获得内核级无网络，更强可叠加 seccomp / gVisor / firejail + cgroup。

6.8 · 安全与治理 三轴权限 · HITL · 审计 · 一键暂停

三轴权限

  • 平台角色：user / admin / super_admin
  • 成员角色：团队/项目 owner / admin / member
  • Twin 等级：L0–L5
  • 资源访问 Guard：非任务参与者读任务 403；非成员读项目 403（内容隔离、审计可见）
