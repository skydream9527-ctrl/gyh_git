# 子Agent派单 (spawn_subagent) 协作规范

> 来源：ICE Data Workbench 多Agent协作设计
> 更新时间：2026-06-29

---

## 一、核心原则

把**有界、可独立完成、能用文字交付**的子任务派给更专业的 agent，调 `spawn_subagent(agent_id, prompt)`。子 agent 跑独立 ReAct、写文件直接落到本任务工作区，仅把 `final_text` 回灌给主 agent，节省上下文窗口。

---

## 二、通用约束

1. **子 agent 无对话通道**：prompt 必须自包含，写清：
   - 指标口径定义
   - 时间窗范围
   - 表名/数据源信息
   - 期望产物（文件名 + 结论格式要求）

2. **任务性质要求**：
   - ✅ 适合派单：跑数、写SQL、出图、整理一段事实、飞书文档检索、独立分析报告
   - ❌ 不要派单：澄清需求、分阶段确认、需要和用户多轮交互的决策性判断

3. **深度限制**：
   - 子 agent **不能再 spawn**（不能再派生子 agent）
   - 子 agent 不能进 plan mode
   - 避免无限递归

4. **上下文隔离**：
   - 子 agent 与主 agent **不共享对话历史**
   - 长期上下文（OKR、季度主轴、业务背景）需要在 prompt 里显式复述

5. **长任务处理**：
   - 预计 >2 min 的任务用 `run_background`（需开启 `ICE_BG_TASK_ENABLED`）
   - 不要硬挤进 `spawn_subagent` 阻塞父任务执行

6. **结果汇总责任**：
   - 收到子 agent 的 `final_text` 后，**父 agent 必须做最终判断与汇总**
   - ❌ 禁止无脑透传子 agent 的结果给用户
   - 需要整合、提炼、判断结论合理性

---

## 三、何时派单，何时自己干

| 场景 | 建议 | 理由 |
|------|------|------|
| 任务跨多个专业领域 | ✅ 派单 | 每个子任务由专家agent处理质量更高 |
| 子任务有专门的专家agent覆盖 | ✅ 派单 | 复用专业agent的领域知识和prompt |
| 需要调用的工具数远超本agent工具白名单 | ✅ 派单 | 避免主agent加载过多工具导致上下文膨胀 |
| 一两步工具调用内能完成 | ❌ 自己干 | 派单 overhead 不值得 |
| 需要和用户多轮澄清需求 | ❌ 自己干 | 子agent无法交互，需求容易走偏 |
| 需要做决策性判断（如要不要放量、结论是否可靠） | ❌ 自己干 | 判断责任在主agent，子agent只负责执行 |
| 子任务输出需要立即和用户确认 | ❌ 自己干 | 子agent无对话通道，无法确认 |

**具体可派单的agent列表**由 runtime 按本 agent 的 `spawn_targets` 字段动态注入。

---

## 四、并行派单 (spawn_parallel)

当任务拆解出**多个相互独立**的子任务时，用 `spawn_parallel(tasks=[{agent_id, prompt}, ...])` 一次性并行执行（最多 5 个），比逐个 `spawn_subagent` 快数倍。

### ✅ 适用场景
- T1~T5 个子任务互不依赖（比如各查不同维度的SQL）
- 同时让 data-analysis 跑数 + know 查文档 + 另一个 agent 整理公式
- 多个独立维度的下钻分析可以并行

### ❌ 不适用场景
- 子任务之间有依赖关系（T2 用到 T1 的输出）
- 只有 1 个子任务（直接用 `spawn_subagent`）
- 子任务总数超过 5 个（分批次，避免资源耗尽）

## 五、派单Prompt模板

```markdown
## 任务：[一句话说清子任务目标]

### 背景信息
[提供必要的上下文：业务背景、为什么做这个、相关决策历史]

### 明确要求
1. 指标口径：[清晰定义每个指标怎么算]
2. 时间范围：[YYYY-MM-DD 到 YYYY-MM-DD，同比/环比要求]
3. 数据来源：[表名、库名、数据源信息]
4. 输出要求：
   - 结论放在最前面
   - 关键数据用表格展示
   - 生成文件保存到：[明确文件路径]
   - 必须覆盖的维度：[列出需要下钻的维度]
5. 异常处理：[如果数据为空/异常怎么处理]

### 禁止事项
- 不要问我问题，假设合理的默认值继续执行
- 不要做决策性判断，只客观呈现数据和事实
- 最终输出控制在 [X] 字以内
```

## 七、派单Anti-Patterns（避坑指南）

| ❌ 错误做法 | ✅ 正确做法 |
|------------|------------|
| prompt只说"帮我分析下XX数据"，不给口径 | 必须写清指标定义、时间范围、表名 |
| 把需要用户确认的问题派给子agent | 需要确认的问题主agent自己问用户 |
| 子agent返回结果直接复制给用户 | 主agent必须审核、整合、提炼后再给用户 |
| 一个大任务拆成10个并行子任务 | 最多5个并行，分批次执行 |
| 子agent之间互相依赖还强行并行 | 有依赖的任务串行派单，等前一个完成再派下一个 |
| 什么任务都派给general agent | 优先派给专业agent，质量更高 |
