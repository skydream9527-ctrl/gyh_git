## 1. 结论：V3 如何优化 V2

### 1.1 总结论

V2 的“四账本、Landmark、右删失、贡献毛利、时间外验证”方向正确，但仍不宜直接开始正式建模。V3 的关键不是增加特征，而是把**财务预测目标、活跃频率、生存时间和因果增量彻底拆清**。

本版建议：

1. **财务主目标直接预测**：以未来 N 天贡献毛利为唯一财务 Ground Truth，不用 LT 模型表现代替财务 LTV 表现。
2. **LT × V 从常数乘法改为逐日条件期望**：避免 RMST、活跃天数和 V 变化被错误压成两个常数。
3. **RMST 不再叫“期望活跃天数”**：RMST 是长沉默事件发生前的日历生存时间；实际活跃天数另算。
4. **V 不用分群中位数进入财务公式**：中位数不可加总且可能因零膨胀变成 0；使用条件均值与层级收缩。
5. **MVP 收敛到一套主窗口**：建议 14 天特征 + 90 天结果，避免 7/14/30 × 90/180/365 同时择优。
6. **预测和因果分权**：预测模型只回答“未来可能值多少”；策略是否创造增量必须由实验回答。
7. **所有阈值重新降级**：无业务数据支持的 C-index、AUC、护栏、holdout、样本量和归因窗口均不得作为正式门槛。
8. **归因后置**：先跑通财务/预测/实验校准，入口和场景描述性归因不阻塞 MVP。

### 1.2 当前评审判断

**有条件通过 V3 方向，进入 Phase 0 定义冻结与数据可行性审计；暂不批准现有 Python/SQL 资产用于正式拟合或策略决策。**

原因：当前 `python/ltv_coefficient_fitting.py` 与 SQL 仍基于旧的 `Δ_score + 三模型异量纲加权` 架构，和 V3 的预测对象、样本、标签及验证方式不兼容。

## 2. 目标、边界与四账本

### 2.1 业务目标

在 Landmark 时点，仅使用当时已知的信息，预测用户群未来 N 天的财务贡献，并为长期实验提供早期信号：

- 识别短期财务下降但长期可能回正的策略；
- 识别短期增收但可能透支长期价值的策略；
- 为广告负载、Push 频次、内容体验等实验提供长期视角；
- 最终仍以长期实验贡献毛利增量作为上线依据。

### 2.2 四账本职责

| 账本 | 回答的问题 | 核心输出 | 能否单独决定上线 |
|---|---|---|---|
| A 财务事实账 | 实际发生了多少价值？ | `actual_CM_90/180/365` | 是最终价值尺度，但历史事实本身不等于策略因果 |
| B 长期预测账 | 基于当前信息，未来可能值多少？ | `pred_CM_N`、`pred_active_days_N`、`survival_RMST_N` | **不能**；只作排序、诊断和早期代理 |
| C 策略因果账 | 策略真正创造/损失了多少长期价值？ | `ΔCM_N`、`Δactive_days_N`、`ΔRMST_N`、证据等级 | **可以参与决策**，前提是实验与功效合格 |
| D 描述归因账 | 已有价值按什么规则分给入口/场景？ | 入口 credit、场景体验健康度、助攻 | **不能**替代因果；经营诊断使用 |

### 2.3 MVP 明确不做

- 不做人工行为系数加权的“综合分”作为上线指标；
- 不做单用户精确 LTV 承诺；
- 不把观察性预测系数解释为业务动作因果；
- 不用 90 天预测冒充 365 天真实结果；
- 不在首版同时推进复杂归因、个体 uplift、多状态回流和在线实时策略；
- 不用未经校准的固定 2%/5%、3%-5% holdout 或 0.2 助攻 credit 决策。

## 3. 预测对象、时间轴与术语

### 3.1 Landmark 时间轴

```text
[L-W, L)                         L                         [L, L+N)
历史特征窗口                     Landmark                  结果窗口
只能使用当时已可获得的信息       预测/随机分流时点           只用于标签与长期验证
```

- `L`：Landmark date；通用模型建议按周固定，实验模型为随机分流时点。
- `W`：特征窗口，MVP 建议 14 天。
- `N`：结果窗口，MVP 建议 90 天。
- 所有 join 必须遵循 as-of 原则：事件时间、处理时间、标签生成时间均不得晚于对应边界。

### 3.2 人群定义必须分开

| 人群 | 推荐定义 | 禁止事项 |
|---|---|---|
| 预测人群 | Landmark 前满足稳定 ID 且符合产品资格；若限定曝光用户，只能使用 L 前曝光 | 不能用 L 后曝光/活跃决定是否入样 |
| 实验 ITT 人群 | 随机前全部合格且已分配用户 | 不能只看实际收到 Push、点击或消费的人 |
| 模型训练人群 | 与目标应用场景一致，并保留第三方调起、自然启动等来源标记 | 不能只保留高活或标签完整的幸存者 |
| 报告人群 | 实验臂 × 预注册生命周期层 × Landmark cohort，满足最小精度要求 | 不能无限下钻小分群并只展示显著结果 |

### 3.3 四个容易混淆的 LT 指标

| 指标 | 严格定义 | 业务含义 |
|---|---|---|
| `active_days_N` | `Σ I(第t天有效活跃)` | N 天内实际活跃了多少天，允许沉默后回流 |
| `survival_time` | 从 L 到首次满足长沉默事件的日历时间 | 何时首次进入长沉默状态 |
| `survival_RMST_N` | `E[min(T,N)]` | N 天内尚未发生长沉默事件的日历时间期望；**不是活跃天数** |
| `active_on_day_N` | 第 N 天当天是否有效活跃 | 点时留存，不等同未流失 |

另需单独定义：

- `retained_in_window_N`：N 日附近预设窗口内是否有活跃；
- `not_long_silent_by_N`：截至 N 日是否尚未发生长沉默；
- `first_long_silence_onset`：首次连续 g 天沉默的起始日；
- `event_confirmed_date`：能够确认该事件的日期，即 onset + g。

## 4. 核心估计量与公式

### 4.1 财务 Ground Truth

设 `CM_{u,t}` 为用户 u 在 Landmark 后第 t 天的贡献毛利，`d_t` 为财务确认的折现因子：

```math
Y^{CM}_{u,N}=\sum_{t=1}^{N} d_t\cdot CM_{u,t}
```

预测目标：

```math
\mu_N(X_L)=E\left[Y^{CM}_{u,N}\mid X_L\right]
```

其中 `X_L` 仅包含 Landmark 当时已知特征。MVP 推荐 `N=90`，输出记为 `pred_CM_90`。

### 4.2 LT × V 的正确可估计分解

若价值主要在活跃日发生，定义：

```math
q_t(X_L)=P(A_t=1\mid X_L)
```

```math
v_t(X_L)=E(CM_t\mid A_t=1,X_L)
```

则：

```math
\widehat{LTV}_N=\sum_{t=1}^{N}d_t\cdot \hat q_t(X_L)\cdot \hat v_t(X_L)
```

这仍然保留会议提出的“LT/活跃机会 × V”解释，但不再假设整个周期只有一个固定 LT 和一个固定 V。

如果非活跃日仍存在 Push、补贴、CDN 等成本，应扩展为：

```math
\widehat{LTV}_N=\sum_{t=1}^{N}d_t\cdot[\hat q_t\hat v_{1,t}+(1-\hat q_t)\hat v_{0,t}]
```

### 4.3 为什么不能直接使用 `RMST × 分群中位数 V`

普通乘法只有在以下强条件同时满足时才成立：

1. “存活”与“当天产生价值”完全一致，存活期间没有不活跃空档；
2. 每个存活日的期望贡献毛利恒定，不随时间、生命周期、策略变化；
3. 没有回流、非活跃日成本或长尾价值；
4. 使用的是条件**均值**而非中位数；
5. LT 与 V 的相关性已被特征充分吸收；
6. 折现时使用 discounted RMST，而非普通 RMST。

信息流和 Push 回流场景通常不满足这些条件。因此 V3 将 RMST 保留为诊断指标，而不是直接与活跃日 V 相乘。

### 4.4 V 为什么必须用条件均值

分群中位数存在四个核心问题：

- 财务总盘是期望和，中位数不可加总；
- 若超过一半用户日收入为 0，中位数会直接为 0；
- 收入/毛利长尾用户会被系统性低估；
- 过去 90 天活跃样本可能产生幸存者偏差。

V3 推荐：

- 以 Landmark 前信息定义分群；
- 使用条件均值，并对小样本分群做层级收缩/回退；
- 明确分母是“用户日”“活跃日”还是“收入日”；
- 中位数仅作为分布诊断，不进入财务 LTV 公式；
- 通过 `Σ q_t × v_t` 能否重建实际群体贡献毛利总盘来验收。

### 4.5 因果增量不能由预测乘法替代

策略总效应必须直接估计：

```math
\Delta CM_N=E[Y_N(1)]-E[Y_N(0)]
```

即使做 LT 与 V 的因果分解，也存在交互项，不能只算 `ΔLT × V`：

```math
D_1V_1-D_0V_0=(D_1-D_0)V_0+D_0(V_1-V_0)+(D_1-D_0)(V_1-V_0)
```

因此上线决策优先使用实验中的 `ΔCM_N` 总效应。

| 字段组 | 字段示例 | 规则 |

### 5.4 数据质量门禁

在建模前必须完成：

1. **财务对账**：用户日贡献毛利汇总与财务总盘差异、未分配比例及原因；
2. **身份审计**：稳定用户 ID 覆盖、跨端合并、匿名/登录态切换；
3. **埋点审计**：新旧有效时长口径、重复计时、曝光漏数、session 切断；
4. **时间审计**：event time、ingestion time、可用时间和标签时间分开；
5. **实验审计**：SRM、串组、污染、分流稳定性、Push 反转异常；
6. **分群审计**：分群是否使用未来信息、版本是否可回放；
7. **异常审计**：真实零、缺失、负毛利、极端长尾分别处理。

所有涉及信息流产品形态或消费路径改动的方案，终审必须邀请算法同学评估埋点和算法特征影响；未通过的数据字段不得进入 LTV 正式模型。

## 6. 特征设计与解释边界

### 6.1 特征分层

| 层级 | 示例 | 用途 | 能否直接指导动作 |
|---|---|---|---|
| 历史状态特征 | 活跃天数、时长、session、历史毛利 | 提升预测准确率 | 否，不等于原因 |
| 体验信号 | 互动率、负反馈率、快速划走率、秒退率 | 预测与诊断 | 需实验验证 |
| 商业处理信号 | 广告密度、Push 频次、商业内容质量 | 预测与候选干预 | 只有实验效应可指导动作 |
| 环境/控制 | 设备、网络、版本、渠道、第三方调起、供给质量 | 控制分布差异和稳定性 | 否 |
| 数据质量特征 | 历史覆盖天数、字段缺失率、身份稳定性 | 缺失机制与降级 | 否 |

### 6.2 设计原则

- 次数与机会分母同时保留，如互动次数 + 互动率；
- 长尾变量根据数据分布选择 `log1p`、样条、单调约束或稳健损失；
- 过去收入可以用于预测，但收入本身不是“伤害用户”的原因；广告负载/打扰机制才是候选处理变量；
- 所有特征必须登记事件时间、数据可用延迟、来源表、负责人和版本；
- 特征选择以时间外提升、群体校准和稳定性为准，不以单次相关系数或显著性为准；
- 任何特征对业务动作的解释必须标记证据等级：预测相关、准实验、随机实验。

## 7. MVP 模型架构

### 7.1 模型梯队

| 编号 | 模型 | 目的 | 备注 |
|---|---|---|---|
| B0-1 | cohort/生命周期历史条件均值 | 最简单财务基线 | 必须 point-in-time 安全 |
| B0-2 | 近期贡献毛利 run-rate 外推 | 可解释业务基线 | 需处理新用户与季节性 |
| B1 | 直接预测 `CM_90` 的正则化/树模型 | 财务主模型 | 损失函数由零膨胀、负值和长尾审计决定 |
| C1 | 逐日 `q_t × v_t` 分解模型 | 解释活跃机会与活跃日价值 | 必须能重建群体财务总盘 |
| S0 | KM/分层 KM | 生存基线 | 不使用行为复杂特征 |
| S1 | Cox | 可解释生存 benchmark | 必须检验比例风险假设 |
| S2 | 离散时间 hazard | 非比例风险挑战者 | 可加入时间项与交互 |

### 7.2 模型选择原则

- 复杂模型必须在多个滚动时间外验证块中稳定优于简单基线；
- 财务模型主看群体总量、校准和业务损失，不以 AUC/C-index 代替；
- 生存模型的排序指标只用于 LT 诊断；
- 直接模型 B1 与分解模型 C1 同时保留：若两者偏差明显，优先排查 V/活跃分解假设；
- 首版不生成无唯一经济含义的 `relative_value_score`，直接输出财务和状态预测；
- 现有旧拟合脚本不得继续使用 40/30/30 异量纲合成，需要按 V3 重构后再运行。

## 8. 验证体系与准入门

### 8.1 G0：定义与数据可用

必须全部通过：

- P0 问题签字冻结；
- 财务贡献毛利可与总盘对账；
- Landmark、标签成熟度和 as-of join 审计通过；
- 核心埋点、用户 ID、session 可用；
- Push 异常实验关闭或隔离；
- 算法参与最终数据/特征评审。

### 8.2 G1：预测有效性（影子看板资格）

| 维度 | 必报指标 | 通过方式 |
|---|---|---|
| 财务总量 | 分 cohort/实验臂的预测总额与实际总额偏差 | 相对 B0/B1 稳定改善，门槛由业务损失冻结 |
| 群体校准 | 预测十分位、生命周期、来源、平台的 observed/predicted | 报告区间，不只看平均 |
| 个体误差 | MAE/RMSE/稳健损失 | 仅辅助，不以个体误差否定群体用途 |
| 生存 | 校准曲线、删失感知 Brier、RMST 误差、C-index | 和 KM/Cox 基线比较，不预设固定硬线 |
| 稳定性 | 多个 rolling OOT 时间块、策略外样本 | 改善方向一致且无关键分群失真 |
| 不确定性 | 用户聚类 bootstrap 区间 | 报告估计精度与不可判定区 |

未看到基线方差与业务损失前，V3 **不设** C-index≥0.65、AUC≥0.70、校准偏差<10% 等正式阈值。

### 8.3 G2：实验级代理有效性（早期代理资格）

验证单位必须是**独立实验的处理效应**，而不是用户级预测与结果的相关系数：

1. 冻结模型版本，不用待验证实验结果重新训练；
2. 对每个实验估计短期 `Δpred_CM_90` 与成熟后的 `Δactual_CM_90/180`；
3. 比较方向、排序、效应大小校准和置信区间；
4. 按广告、Push、推荐等干预类型分别报告可迁移性；
5. 使用实验级留一验证，避免同一实验同时训练和验收；
6. 实验数量不足时结论为“证据不足”，不强行升级代理权限。

### 8.4 Push 与干扰实验硬规则

- Push 类、存在组间干扰或历史两组不等质的实验必须使用 DID，禁止直接比较实验后绝对值或 TGI；
- DID 前必须检查实验前多期平行趋势；
- 报告 ITT 效应，不只分析实际收到/点击用户；
- 稳定随机 holdout 需检查 SRM、串组和污染；
- 若存在真实 spillover/机会挤占，DID 本身不能修复，应采用集群随机、曝光映射或重新设计随机单元；
- 可使用实验前指标做 CUPED/ANCOVA 方差缩减，但必须预注册且不改变 ITT 人群。

### 8.5 G3：决策资格

| 证据状态 | 允许用途 | 禁止用途 |
|---|---|---|
| 仅 G1 通过 | 影子看板、问题诊断、实验延长建议 | 不能据预测分直接全量 |
| G1+G2 通过 | 作为早期代理参与“继续/暂停观察” | 不能替代长期贡献毛利因果结果 |
| 长期随机/合格准实验成熟 | 联合短期预算与护栏做上线决策 | 不得忽略不可判定区 |
| 观察性归因 | 经营诊断、提出实验假设 | 不能做 ROI 或资源增量结论 |

正式决策门槛由以下四项共同决定：长期 `ΔCM_N` 的效应与区间、护栏风险预算、短期现金流预算、实验功效与完整性。具体数值在 Phase 0/实验预注册时冻结。

## 9. 归因账本的 V3 边界

### 9.1 入口归因必须守恒

V2 按每个 session 分配“用户整个未来 LTV”可能重复计算。V3 要求先选择分配单位：

- **方案 A：一次性获取/回流归因**——每个用户在一个明确价值周期仅归给一个主入口；
- **方案 B：非重叠价值窗口归因**——每个入口只分配其后不重叠窗口内的实际/预测价值。

无论选择哪种：

- 每个被分配价值单位的主 credit 合计必须等于 1；
- 助攻单列，不进入主 ROI 分母；
- “其他回流=0.5”不能留下未解释余额；
- 30 分钟/2 小时窗口与 0.2 credit 在校准前只作候选规则。

### 9.2 场景体验分不等于场景 LTV

- 场景内互动率、负反馈率、广告密度、时长可形成“场景体验健康度”；
- 健康度用于发现问题，不宣称创造了多少长期财务价值；
- 场景财务价值若需分摊，必须定义守恒规则；
- 重要场景的资源决策使用入口开关、频控或随机流量实验的 `ΔCM_N`。

### 9.3 Push 三视角

| 视角 | 正确用途 | 禁止事项 |
|---|---|---|
| Push 入口描述价值 | 描述由 Push 带回用户的后续价值 | 不当作 Push 因果增量 |
| Push 内流体验健康度 | 诊断内容、广告与负反馈 | 不与入口价值相加 |
| Push 长期因果增量 | 通过稳定实验 + DID/预注册方法估计 | 不用 TGI/点击用户子样本代替 ITT |

## 11. 主要风险与缓解

| 风险 | 典型后果 | 缓解措施 |
|---|---|---|
| RMST 与活跃天数混用 | 高估低频活跃/回流用户 | 标签分列、逐日活跃模型、禁止普通 RMST×V |
| 中位数 V | 总盘系统性低估或直接归零 | 条件均值、层级收缩、总盘重建验收 |
| 标签未成熟 | 把未知当 0/存活 | N 天完整窗 + g 天确认缓冲 |
| 实验后曝光筛选 | 随机性被破坏、方向可能反转 | 随机前资格 + ITT |
| 时间穿越 | 线下表现虚高 | as-of join、标签可见性留空、时间块验证 |
| 埋点重复/漏数 | 特征和标签污染 | 数据质量门禁、算法参与终审 |
| Push 组间干扰 | 直接 TGI 得出错误结论 | DID + 平行趋势；真实 spillover 重设随机单元 |
| 财务口径漂移 | 模型与财务盘无法对账 | 数据合同、版本化、reconciliation |
| 策略分布漂移 | 历史模型在新策略失效 | 策略外验证、按干预类型报告、漂移触发重训 |
| 小分群追显著 | 假阳性与不稳定结论 | 预注册层级、最小精度、层级回退、多重检验 |
| 预测被误当因果 | 优化分数但长期价值下降 | 证据等级、权限门禁、长期 holdout |
| 归因重复计量 | 渠道价值合计超过总盘 | 非重叠窗口/一次性归因、credit 守恒 |

## 12. V2 → V3 关键变更

| 维度 | V2 | V3 |
|---|---|---|
| 财务主目标 | 文档同时存在“直接贡献毛利”和“RMST×V”两种表述 | 唯一主目标为未来 N 天贡献毛利条件期望 |
| LT×V | RMST × 分群日毛利中位数 | 逐日 `P(active_t) × E(CM_t|active_t)`；普通乘法仅作受限近似 |
| RMST | 被解释为期望活跃天数 | 定义为长沉默事件前日历生存时间，与活跃天数分开 |
| V | 分群中位数 | 条件均值 + 层级收缩；中位数仅描述 |
| MVP 周期 | 同时跑多组窗口 | 建议 14 天特征 + 90 天结果；其他做敏感性 |
| 用户范围 | 信息流曝光用户 | 预测 L 前资格与实验 ITT 人群分开 |
| 标签成熟 | 只考虑 N 天 | 长沉默事件另加 g 天确认缓冲 |
| 模型 | Cox 优先 | 直接财务模型为主；Cox 仅生存 benchmark |
| 验证 | 固定 C-index/AUC/方向阈值 | 相对基线的 OOT 财务校准 + 实验级代理验证，阈值待业务损失冻结 |
| 输出 | relative_value_score + 多状态指标 | 直接输出 `pred_CM_N`、`pred_active_days_N`、`survival_RMST_N` |
| Push | 随机 holdout 为主 | 强制 DID/平行趋势规则 + ITT + 干扰诊断 |
| 归因 | 与预测并行推进 | 降为 P2；入口守恒、场景健康度与财务价值分开 |
| 上线权限 | 四要素但仍残留固定护栏 | G0-G3 分级授权；预测不替代长期因果 |
