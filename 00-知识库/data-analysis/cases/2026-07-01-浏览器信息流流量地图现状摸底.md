## 分析框架（可复用）

本次流量地图分析形成了**模块化分析框架**，适用于任何产品模块的流量摸底：

```
1. 模块大盘 → 一级模块UV/时长/PV占比，定全局盘子
2. 启动方式 → 用户从哪来（icon/push/第三方/内容中心等）
3. 交叉矩阵 → 模块×启动方式，看各入口对各模块的贡献
4. 页面细分 → 模块内page维度UV/时长，找核心页面
5. 频道细分 → 模块内feed_channel维度UV/时长，找核心频道
6. 双层下钻 → page×channel交叉，看各页面的频道构成
7. 有效消费 → 官方口径有效DAU/时长/留存，校准真实消费
8. 页面流转 → 进入页→离开页矩阵，看用户路径
9. 模块留存 → 各模块次日留存，看粘性
10. 频道留存 → 各频道次日留存，看频道粘性
11. 四象限分析 → 留存×时长/UV，给策略建议
12. 行动建议 → P0/P1/P2分级
```

## 关键数据发现（当前口径下）

> ⚠️ 注意：UV和时长口径后续会更新，以下结论基于当前口径（7天汇总÷7日均），待新口径后需更新

1. **推荐频道是绝对基本盘**：日均UV 1032万，日均时长2.29亿分钟（70%+），人均22.2分钟，频道留存41.25%（最高）
2. **热点留存≈推荐，但消费深度差5倍**：留存41.11% vs 41.25%，但人均时长4.86min vs 22.2min，详情页转化是关键
3. **push用户质量极高但召回弱**：人均21.2min（与推荐相当），频道留存仅29.1%
4. **文章详情页是时长核心场景**：日均UV仅247万，但人均21.2min（最高），日均时长5246万分钟
5. **热榜/小视频留存断崖**（15%/12%/8%），可整合收束
6. **沉浸式新闻埋点缺失**：日均UV 232万但PV=0、时长=0
7. **信息流有效DAU**：日均372万，人均时长34.71分钟/天，APP次日留存49.39%

## 经验沉淀

### 1. SQL批量执行方法
- 用Python脚本批量执行SQL文件（每个文件可能有多个查询，用`;`分割）
- 结果保存为JSON（columns + rows结构）
- 失败查询单独重试，避免全量重跑
- 执行记录保存到`_execution_summary.json`

### 2. 数据报告→飞书→HTML的产出流程
1. SQL查询 → JSON结果
2. Python脚本读取JSON → 生成Markdown汇总
3. 手工提炼结论 → 结构化Markdown文档
4. `feishu docx create/update` 发布到飞书
5. 参考编辑风格HTML模板 → 生成可视化HTML报告
