## 标题层级映射

金字塔层级 → 飞书标题级别：

| 金字塔层级 | 内容 | 飞书标题级别 | 说明 |
|-----------|------|------------|------|
| 顶层结论 | 核心结论/建议（1句话） | **加粗文本** 或文档首段 | 不用标题，直接正文突出 |
| 第一层论点 | 3-5个支撑论点 | `# 一级标题` | 每个论点一个大章节 |
| 第二层论点 | 细化支撑 | `## 二级标题` | 每个论点的子论证 |
| 第三层证据 | 数据/案例/事实 | `### 三级标题` 或正文 | 尽量用正文+列表，避免层级过深 |

**原则**：callout 只出现在两个固定位置，其他层级禁止使用。

| 层级 | callout | 说明 |
|------|---------|------|
| 文档顶部 | ✅ 必须 | 全文总结论，一句话 |
| 一级标题（`#`）下方 | ✅ 必须 | 该章节的分论点结论句 |
| 二级标题（`##`）下方 | ❌ 禁止 | 用**加粗第一句话**代替 |
| 三级及以下 | ❌ 禁止 | 纯正文，不需要结论句设计感 |

**二级标题格式**：用"[角色/性质]：[对象]"结构，让读者不读正文也能判断这一节的定位。

- ❌ `## Modena-5` → 只有名词，不知道这节在讲什么
- ❌ `## 极高散差车辆（>10%）` → 描述对象，但不说明角色
- ✅ `## 重点车型：Modena-5散差问题最集中` → 角色（重点车型）+ 对象 + 性质
- ✅ `## 优先处置：散差>10%的15辆已启动售后介入` → 角色（优先处置）+ 对象 + 状态

```markdown
# 成本分析

<callout emoji="bulb" background-color="light-blue">
自动化方案可节省 90% 人工成本，ROI 达 300%。
</callout>

## 主要来源：人工处理占当前总成本的80%

**当前每周人工处理耗时8小时，自动化后预计降至0.5小时。**

[正文细节...]
```

## 块类型选择指南

| 内容类型 | 推荐飞书块 | 说明 |
|---------|----------|------|
| 核心结论 | 📢 引用块（callout） | 视觉突出，读者第一眼看到 |
| 并列观点 | 📋 无序列表 | 3个以上并列内容 |
| 步骤流程 | 🔢 有序列表 | 有先后顺序的步骤 |
| 数据对比 | 📊 表格 | 多维度数据对比 |
| 重要提示 | ⚠️ 高亮块 | 需要读者特别注意的内容 |
| 代码/命令 | 💻 代码块 | 技术文档中的命令/代码 |
| 章节过渡 | 分割线 | 大章节之间视觉分隔 |
| 待办事项 | ✅ 任务列表 | 行动项、后续计划 |

**Callout 块推荐用于顶层结论**，示例：
```
💡 结论：建议在 Q2 启动 SOH 自动化项目，预计节省每周 8 小时人工，ROI 达 300%。
```

## 文档模板：汇报型

适用于：项目汇报、周报、决策申请、方案评审

```markdown
<!-- 顶层结论（callout块） -->
💡 [核心结论/建议，一句话]

<!-- SCQA 背景（可选，简短） -->
**背景**：[情境 S] [冲突 C，1-2句]

---

# [论点A：支撑结论的第一个原因]

[论点A的核心说明，2-3句]

## [子论点 A1]
[证据/数据]

## [子论点 A2]
[证据/数据]

---

# [论点B：支撑结论的第二个原因]

[论点B的核心说明]

...

---

# 行动建议

1. [最重要的行动项] — 负责人 / 截止日期
2. [次要行动项] — 负责人 / 截止日期
3. [后续跟进项]
```

## 排版规范

### 长度控制
- 每个章节正文不超过 150 字（超出则细分子章节）
- 列表项不超过 7 条（超出则分组）
- 表格列数不超过 6 列

### 视觉层次
- 一篇文档最多使用 3 级标题
- 同级内容用相同块类型（不要混用列表和标题）
- 相邻同类内容之间不加分割线（只在大章节间用）

### 数字与数据
- 数据出现时必须同时给出结论（数据说明了什么）
- 百分比保留整数（90%，不是 90.3%）
- 时间维度必须明确（"上周" vs "2026年第12周"）

## 通过飞书 CLI 写入

本 skill 的写作输出需配合飞书操作能力。执行写入时：

1. **生成 Markdown 内容**（按本规范的模板结构）
2. **调用飞书 skill 写入**：加载 `feishu` skill，使用飞书 CLI 工具创建/更新文档
3. **验证写入结果**：确认标题层级、callout 块、表格渲染正确

**注意**：飞书文档的 Markdown 映射规则：
- `# 标题` → 飞书一级标题块
- `> 引用` → 飞书引用块
- `- 列表` → 飞书无序列表
- `**加粗**` → 飞书加粗文本
- 表格 → 飞书表格块（需检查列宽）
